# Requirements: Degenerus Protocol — v35.0 BURNIE Near-Future Per-Pull Level Resample

**Defined:** 2026-05-09
**Milestone:** v35.0
**Goal:** Convert the near-future BURNIE coin jackpot from one-level-per-call distribution to per-pull-level sampling at both call sites in `DegenerusGameJackpotModule.sol`, validated against Phase 261's reusable chi-squared infrastructure.
**Audit baseline:** v34.0 source-tree HEAD `6b63f6d4daf346a53a1d463790f637308ea8d555`
**Core Value:** Every finding a C4A warden could submit is identified and either fixed or documented as known before the audit begins.
**READ-only audit posture:** LIFTED — audit-then-commit (or impl-then-audit) with per-commit user approval per `feedback_no_contract_commits.md` + `feedback_batch_contract_approval.md`.
**Seed source:** `.planning/notes/2026-05-08-burnie-near-future-per-pull-level.md` (locked decisions captured prior to milestone start).

## v35.0 Requirements

### PPL — Per-Pull Level Resample (Core Implementation)

- [ ] **PPL-01**: `payDailyCoinJackpot` (purchase phase, ~L1708 of `DegenerusGameJackpotModule.sol`) samples per-pull random level in `[minLevel, maxLevel]` via `keccak256(randomWord, COIN_LEVEL_TAG, i) % range` instead of picking one upfront `targetLevel`.
- [ ] **PPL-02**: `payDailyJackpotCoinAndTickets` (jackpot phase, ~L624 of `DegenerusGameJackpotModule.sol`) samples per-pull random level in `[lvl+1, lvl+4]` via the same `keccak256(randomWord, COIN_LEVEL_TAG, i) % range` scheme.
- [ ] **PPL-03**: Flat 50-pull loop replaces `_computeBucketCounts` for these two paths; `for i in 0..cap` with `trait_idx = i % 4` guarantees equal-share trait representation without upfront cap-distribution math.
- [ ] **PPL-04**: Per-winner amount stays `coinBudget / cap`; `coinBudget % cap` remainder spread by cursor (current scheme preserved byte-identical at the share-math layer).
- [ ] **PPL-05**: Empty-bucket silent skip — when sampled `(lvl', trait_i)` has `effectiveLen == 0` (no real holders AND no deity for that symbol), drop the pull silently. No re-roll, no fallback, no redistribution. Structural underspend accepted; no carry-forward, no top-up to remaining winners.
- [ ] **PPL-06**: Per-trait deity caching — `deityBySymbol[fullSymId]` read once per trait at loop entry into `address[4] memory deityCache`; reused across all pulls of that trait. Virtual-count math (`len/50`, min 2) recomputed per pull because `len` varies by sampled level.
- [ ] **PPL-07**: Salt scheme — within `_randTraitTicket` (or its inlined replacement), the holder-index keccak becomes `keccak256(randomWord, trait, lvl, i)`, adding `lvl` as discriminator so two pulls at the same `(trait, i)` but different levels don't collapse to the same index. The `salt` parameter is dropped from this code path; pull index `i` discriminates within a level.
- [ ] **PPL-08**: `JackpotBurnieWin(winner, lvl, traitId, amount, ticketIndex)` event signature byte-identical; `lvl` field semantically reflects the per-pull sampled level (off-chain indexers will see ~50 distinct `lvl` values per call instead of 1 — flag in AUDIT-06).

### STAT — Statistical Validation (Phase 261 infra reuse)

- [ ] **STAT-01**: Per-pull level distribution chi-squared test confirms uniform sampling over `[minLevel, maxLevel]` (p > 0.05 at α = 0.05) using N ≥ 10K aggregated samples.
- [ ] **STAT-02**: Per-trait share test confirms ~25% per trait across pulls (chi² < 7.815 at df=3 / α=0.05) under the `trait_idx = i % 4` rotation.
- [ ] **STAT-03**: Empty-bucket skip rate measurement vs analytical bound; cumulative monetary underspend bounded and disclosed in findings (per D-09 gating, expected to remain INFO unless rate exceeds disclosed threshold).
- [ ] **STAT-04**: Phase 261 chi-squared infrastructure reuse confirmed (or path forward documented if one-shot — decision captured during planning as D-NN-INFRA-01; default branch = reuse since seed-note assumption is reusable).

### SURF — Cross-Surface Verification

- [ ] **SURF-01**: `_randTraitTicket` other callers (event-only emission sites, equal-split tickets/coin sites at L598/599/L1687, lootbox flat-bucket at L683, far-future portion `_awardFarFutureCoinJackpot`) byte-identical OR proven non-regressing across full callsite sweep (full sweep matches v34.0 Phase 260 8-non-injection-site discipline).
- [ ] **SURF-02**: Far-future BURNIE coin jackpot path (`_awardFarFutureCoinJackpot`) byte-identical (already per-pull-level upstream; explicit out-of-scope).
- [ ] **SURF-03**: ETH daily jackpot paths byte-identical (out of scope; v34.0 `_pickSoloQuadrant` injection sites at L282/L349/L524/L1147 unchanged).
- [ ] **SURF-04**: Purchase-phase ticket distribution (`_distributeTicketJackpot`) byte-identical (out of scope).
- [ ] **SURF-05**: Gas regression — extra ~70K–110K per call confirmed within accepted budget (50 pulls × ~1.5–2.2K extra/pull, dominated by cold length SLOADs on fresh `(lvl', trait_i)` slots, warming after ~16 distinct slots); no path explosion in `advanceGame` ceiling (≥ 1.99× margin preserved). Test asserts paired-empty-wrapper analogous delta or per-call delta within disclosed envelope.

### AUDIT — Adversarial Audit + Findings Consolidation

- [ ] **AUDIT-01**: Delta-surface table covering all source-tree changes v34.0 HEAD `6b63f6d4` → v35.0 closure HEAD (modified functions, new helpers, removed dead code including `_computeBucketCounts` for this path, removed/inlined `_randTraitTicket` callers if touched).
- [ ] **AUDIT-02**: Adversarial surfaces enumerated and resolved (predictability / trait-stacking pre-call, level-salt collision between callers, deity-cache staleness across pulls, cross-caller `_randTraitTicket` salt collision now that `salt` parameter is dropped, off-chain indexer semantic-shift attack surface, gas-griefing via repeated cold SLOAD across 50 distinct `(lvl', trait_i)` slots).
- [ ] **AUDIT-03**: Conservation re-proof — `coinBudget` conservation across new loop (`Σ paid ≤ coinBudget` even with empty-bucket skips), solvency invariant (`claimablePool ≤ ETH balance + stETH balance`) preserved, BURNIE mint-supply conservation across all coin-jackpot paths.
- [ ] **AUDIT-04**: Zero-new-state scan — zero new storage slots, zero new public/external mutation entry points, zero new admin functions, zero new upgrade hooks, zero new modifiers that escalate authority.
- [ ] **AUDIT-05**: `audit/FINDINGS-v35.0.md` published as milestone deliverable; FINAL READ-only at v35.0 closure HEAD; closure signal `MILESTONE_V35_AT_HEAD_<sha>` emitted; KNOWN-ISSUES.md UNMODIFIED unless D-09 gating promotes a finding.
- [ ] **AUDIT-06**: Off-chain indexer documentation — `JackpotBurnieWin.lvl` AND `DailyWinningTraits.bonusTargetLevel` semantic shifts surfaced in §3 of FINDINGS-v35.0.md and KNOWN-ISSUES.md per D-09 gating decision (default INFO unless gated upward). `JackpotBurnieWin.lvl` shifts from shared-call-level to per-pull-sampled-level (~50 distinct values per call instead of 1). `DailyWinningTraits.bonusTargetLevel` shifts from authoritative single-level anchor to advisory non-authoritative pre-announcement (the `lvl + 1 + uint24(coinEntropy % 4)` formula at L519/L537 keeps emitting byte-identically; downstream `payDailyJackpotCoinAndTickets` no longer targets that single level under per-pull resample).
