### Stale `getDailyHeroWinner` Public View Disposition
- **D-293-STALE-VIEW-01:** **Defer to v43+ explicit cleanup phase.** The leftover v41 deterministic-leader public view at `contracts/DegenerusGame.sol:2545` (`getDailyHeroWinner(uint32 day) returns (uint8 winQuadrant, uint8 winSymbol, uint256 winAmount)`) — body unchanged from the deleted `_topHeroSymbol` algorithm (most-wagered slot wins); `@notice` reading "Get the winning hero symbol for a given day (most wagered across all quadrants)" — is **NOT touched at Phase 293**. Zero Phase 293 source-tree mutation; zero Phase 296 SWEEP adversarial inclusion; zero Phase 297 §3.A delta-surface inclusion; zero Phase 297 §3.B finding-block candidate. Pure forward-cite into the v43+ roadmap backlog. **Why:** v42.0 milestone scope is HRROLL + MINTCLN + DPNERF mechanic changes + their regression coverage; the stale view is an indexer-helper surface that downstream off-chain consumers query but doesn't affect on-chain mechanic correctness. Modifying the view at Phase 293 (NatSpec-only or full removal) would (i) re-open the Phase 292 audit-subject boundary, (ii) violate the HRROLL-07 public-ABI byte-identity lock if the view body were rewritten to take an entropy parameter, (iii) cost a second USER-APPROVED contract commit at a phase that was supposed to be test-only. v43+ has the bandwidth to either rewrite the view body, remove it cleanly behind indexer-team migration, or formally accept the v41-leftover-semantic disposition via a F-43-NN finding block. Mirrors v41 FINDINGS-v41.0.md row (F) `getDailyHeroWager` semantic-disposition pattern for a future milestone. **How to apply:** Plan-phase MUST NOT touch `contracts/DegenerusGame.sol:2520-2563` (the `getDailyHeroWager` + `getDailyHeroWinner` view block). Plan-phase MUST NOT seed `getDailyHeroWinner` as a TST-HRROLL assertion vehicle (it returns v41-deterministic-leader output, NOT `_rollHeroSymbol`'s weighted-random output). TST-HRROLL test file JSDoc header MAY note this fact as a path-of-investigation clarification (so downstream readers of the test file understand why TST-HRROLL doesn't use it). 293-CONTEXT.md `<deferred>` carries the forward-cite into the Phase 297 D-42N-FCITE-01 "Deferred to Future Milestones" register pattern for v43+ inclusion.
