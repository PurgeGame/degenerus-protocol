**Finding**
**[LOW] First-Claim Window Still Forfeits Older Coinflip Seed/Win Value**

Location: [BurnieCoinflip.sol:151](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:151), [BurnieCoinflip.sol:483](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:483), [BurnieCoinflip.sol:1123](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:1123)

Invariant: first-claim window change should strand no seed/winning value.

Code: `COIN_CLAIM_FIRST_DAYS = 180`, but first-time claimers with `lastClaim == 0` still clamp `start` to `latest - 180`. Any winning stake before that clamped start is skipped and later made unreachable once `lastClaim` advances. Same clamp exists in the preview path.

Concrete effect: a first claimant after the 180-day lookback can lose old coinflip wins. This especially matters for VAULT seed stakes; sDGNRS is auto-settled during daily coinflip processing, but VAULT is not. This is not an ETH solvency drift, but it diverges from “no seed/winning value stranded.”
