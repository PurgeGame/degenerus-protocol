**1. EV-neutrality in code: Verified**

- **Ticket Budget:** Per-ticket budget is accurately scaled by 11/9. The code uses `LOOTBOX_TICKET_ROLL_BPS = 19_678` (which is `16_100 * 11 / 9`).
- **Far/Near Budgeting:** The EV-neutral formula is perfectly maintained. Far rolls get 1.5x (`LOOTBOX_TICKET_FAR_BUDGET_BPS = 15_000`) and near rolls get 0.875x (`LOOTBOX_TICKET_NEAR_BUDGET_BPS = 8_750`). Math checks out: `(0.8 * 0.875) + (0.2 * 1.5) = 1.0`.
- **Variance Tiers:** The tier midpoint values rigorously preserve the ~0.786 overall variance EV. The `LOOTBOX_TICKET_VARIANCE_TIER` constants define perfectly symmetric continuous bands centered precisely on the old fixed multipliers (e.g., Tier 1: 3.20x-6.00x = 4.6x mean).

**2. Money pump: Refuted**
- The new 10% recycle bonus is gated correctly at `totalClaimableUsed >= priceWei * 3` in `DegenerusGameMintModule.sol`.
- There is no closed positive-EV loop. A recycled claimable spend returns 5% via affiliate + 10% in flip-credit (which has a 50% survival rate, making it ~5% EV). The remaining box EV is bounded strictly by the 10-ETH per-player/per-level EV-cap logic (`LOOTBOX_EV_BENEFIT_CAP`). The total marginal return from a recycle can never exceed 100%.

**3. Degenerette spins freeze-safety + one-shot: Verified**
- **Freeze-Safety:** Spin seeds are derived via `EntropyLib.hash2(seed, BOX_*_SPIN_TAG)`. The root `seed` is strictly anchored to the `rngWord` frozen at commitment. No live `day` or player-controlled `block` variables are included, preventing open-timing manipulation.
- **Bounded Recirc:** The ETH-spin cascade is safely bounded to depth 1. `resolveEthSpinFromBox` recirculates the box but enforces `allowEthSpin=false` for the new roll. In `_resolveLootboxRoll`, an `allowEthSpin=false` box overrides the 5% ETH-spin outcome and securely awards tickets instead.
- **One-Shot:** The Degenerette modules emit their own `BoxSpin` events and instantly mint/allocate value without touching memory pool accumulators or leaving intermediate state vulnerable to replay.
