- **3a: Who Can Read lastLootboxRngWord and When** -- Document that lastLootboxRngWord is an internal Solidity variable but stored in contract storage, readable by anyone via `eth_getStorageAt`. It is updated in two places: (1) mid-day drain path at AdvanceModule:166 (`lastLootboxRngWord = word`), (2) piggyback path at AdvanceModule:789 via `_finalizeLootboxRng`. After either write, the value is observable on-chain before the next `processTicketBatch` call.

- **3b: What Information It Reveals** -- Explain that knowing lastLootboxRngWord reveals the EXACT trait assignment for any given queue composition. The LCG seed `(baseKey + groupIdx) ^ entropyWord` uses all-knowable components: level (public), queueIdx (derivable from ticketQueue array), player address (public). So an observer CAN predict which traits each ticket will receive BEFORE processTicketBatch executes. This is deterministic BY DESIGN (enables verification).

- **3c: Why the Read Buffer Prevents Exploitation** -- The critical defense: even though traits are predictable given the word and queue, the queue composition in the READ buffer is FROZEN at swap time. An attacker who observes lastLootboxRngWord cannot: (1) add tickets to the read buffer (writes go to write buffer), (2) remove tickets from the read buffer (no removal function exists), (3) reorder tickets in the read buffer (array is append-only, frozen). The only "attack" would be to NOT purchase future tickets based on predicted traits -- but this is a personal choice, not a protocol exploit.

- **3d: SAFE/EXPLOITABLE Verdict** -- State verdict explicitly. Expected: SAFE. Reasoning: observability does not create a manipulation vector because the read buffer is immutable after swap. The double-buffer mechanism serves as a structural commit-reveal: tickets are committed (write buffer) before the entropy that determines their traits is known (VRF callback), and the entropy cannot change what it indexes into (frozen read buffer). Note the VRF_MIDDAY_CONFIRMATIONS = 3 asymmetry (vs 10 for daily) as a design tradeoff observation, not a vulnerability (VRF proof verified on-chain regardless).
