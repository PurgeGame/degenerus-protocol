Create `audit/v1.2-delta-rng-impact-assessment.md` with the following structure:

**Header:** Title, scope (RNG impact assessment of v1.0 contract changes), classification definitions:
- **NO IMPACT**: Change does not touch, influence, or gate any RNG state variable or RNG-consuming function
- **NEW SURFACE**: Change introduces a new path that reads, writes, or gates RNG state
- **MODIFIED SURFACE**: Change alters an existing RNG-touching path (guard added/removed, flow changed, variable renamed)

**Methodology:** Read `v1.0-contract-diffs.patch` (1410 lines, 11 files). For each file's diff hunks, assess whether the changed lines touch any variable or function from the Phase 12 RNG inventory (`audit/v1.2-rng-storage-variables.md`, `audit/v1.2-rng-functions.md`).

**Process each of the 11 changed files:**

1. **DegenerusGame.sol** -- `error E()` removal, `recordMint` freeze branching, prize pool accessor renames (`nextPrizePoolView`/`futurePrizePoolView` to legacy getters)
2. **AdvanceModule.sol** -- rngLockedFlag guard changes, `requestLootboxRng` guard re-add, drain gate, `ticketsFullyProcessed`, stale word routing, mid-day swap threshold
3. **DecimatorModule.sol** -- FIX-1 freeze guard, legacy accessor renames
4. **DegeneretteModule.sol** -- `jackpotResolutionActive` removal, freeze-aware payout routing, legacy accessor renames
5. **EndgameModule.sol** -- Legacy accessor renames
6. **GameOverModule.sol** -- Legacy accessor renames
7. **JackpotModule.sol** -- Legacy accessor renames, pool operation changes
8. **LootboxModule.sol** -- rngLockedFlag guard removals (LOCK-03, LOCK-04), legacy accessor renames
9. **MintModule.sol** -- rngLockedFlag guard removal (LOCK-01, LOCK-02), lootbox queue changes, legacy accessor renames
10. **WhaleModule.sol** -- rngLockedFlag guard re-add for deity pass, freeze-aware pool routing, legacy accessor renames
11. **DegenerusGameStorage.sol** -- New storage variables, helper functions, error centralization

**For each file, use this format:**
```
### [FileName].sol

| Hunk/Lines | Change Description | RNG Variable/Function Touched | Impact | Reasoning |

| @@ -X,Y +A,B @@ | [brief] | [var/fn name or "none"] | NO IMPACT / NEW SURFACE / MODIFIED SURFACE | [1-2 sentences] |
```

Group trivial renames (legacy accessor renames that are pure refactors with no functional change) into a single row per file to avoid noise, e.g.:
```
| multiple | Legacy _get/_set* renames (N sites) | none | NO IMPACT | Pure rename, no behavioral change |
```

**Focus attention on:**
- Any line that adds/removes/modifies a check of `rngLockedFlag`, `prizePoolFrozen`, `rngRequestTime`

- Any new storage variables that could influence RNG outcome selection

**End with a summary table:**

| ... | ... | ... | ... | ... |
| **Total** | **N** | **N** | **N** | **N** |

And a consolidated list of all NEW SURFACE and MODIFIED SURFACE findings for downstream use by Plan 03.
