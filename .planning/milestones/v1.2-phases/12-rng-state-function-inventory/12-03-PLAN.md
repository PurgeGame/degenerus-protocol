Create `audit/v1.2-rng-data-flow.md` with:

**Section 1: Daily RNG Data Flow**

Text-based flow diagram tracing:
1. Chainlink VRF callback arrives at `rawFulfillRandomWords()`
2. Word stored in `rngWordCurrent`
3. `advanceGame()` reads `rngWordCurrent` via `rngGate()` or `_applyDailyRng()`
4. Word stored in `rngWordByDay[day]`
5. Word consumed by each downstream function (jackpot draws, trait assignment, etc.)

For each consumption point, document:
- The exact function and line where the word (or derived value) is read
- How entropy is derived (direct use, keccak256 with other inputs, modular arithmetic)
- What outcome the entropy determines (winner selection, trait assignment, etc.)
- What other state participates in the outcome (bucket sizes, ticket counts, etc.)

Use ASCII art flow diagram format similar to the v1.0 audit's rngLockedFlag state machine.

**Section 2: Lootbox RNG Data Flow**

Same treatment for the lootbox path:
1. `requestLootboxRng()` sends VRF request
2. Callback stores in `lootboxRngWordByIndex[index]`
3. `_finalizeLootboxRng()` processes the word
4. `lastLootboxRngWord` updated
5. Downstream consumers: `openLootBox`, `openBurnieLootBox`, `processTicketBatch`, etc.

Trace each consumer with same detail as daily path.

**Section 3: Mid-Day Ticket RNG Flow**

Specific trace of:
1. How `midDayTicketRngPending` gets set
2. How lootbox RNG word flows to ticket processing
3. The `processTicketBatch` consumption path
4. How this interacts with the daily RNG path

Include a combined diagram showing both daily and lootbox paths with their interaction points (if any).

Add Section 4 to `audit/v1.2-rng-data-flow.md`:

**Section 4: Entry Point Call Graphs**

For every external/public entry point identified in v1.2-rng-functions.md (Section 2), build a call graph showing the path from external call to each RNG state mutation.

Format for each entry point:
```
entryPoint() [Contract:line]
  -> internalHelper() [Module:line]
    -> deeperHelper() [Module:line]
      -> WRITES rngVariable [line]
      -> READS otherRngVariable [line]
```

Group by RNG impact category:
- **RNG Producers** (functions that create or store new entropy): advanceGame, rawFulfillRandomWords, requestLootboxRng
- **RNG Consumers** (functions that read entropy to determine outcomes): openLootBox, openBurnieLootBox, jackpot draw functions, processTicketBatch
- **RNG Influencers** (functions that modify state consumed alongside entropy): ticket purchases, deity pass purchases, reverseFlip/nudge
- **RNG Guards** (admin/emergency functions): updateVrfCoordinatorAndSub, any owner-only RNG state setters

For each call graph, annotate:
- Guard conditions checked along the path (rngLockedFlag, prizePoolFrozen, etc.)
- The point at which the function becomes "committed" (past all guards, will definitely mutate state)
- Whether the path is reachable during locked/frozen states

**Section 5: Cross-Reference Summary**

Create a matrix table:
```
| Entry Point | RNG Vars Written | RNG Vars Read | Guards Checked | Callable When Locked | Callable When Frozen |
```

This provides a single-glance view for the manipulation window analysis in Phase 14.
