    **CP-07 fix: Split claimRedemption so ETH is claimable even when coinflip is unresolved.**

    The current `claimRedemption()` (StakedDegenerusStonk.sol:575-613) reverts with `FlipNotResolved()` if the coinflip for the period's flipDay hasn't been processed yet. This blocks the ENTIRE claim (including ETH) when the coinflip is stuck.

    Modify `claimRedemption()` to pay ETH regardless, and only pay BURNIE when the coinflip IS resolved:

    In `claimRedemption()` at StakedDegenerusStonk.sol, starting around line 575:

    1. Keep the existing `NoClaim()` and `NotResolved()` checks (lines 578-581).
    2. Replace the `FlipNotResolved()` revert (line 585) with conditional BURNIE handling:

    **Before (lines 583-613):**
    ```solidity
    // Check coinflip resolved
    (uint16 rewardPercent, bool flipWon) = coinflip.getCoinflipDayResult(period.flipDay);
    if (rewardPercent == 0 && !flipWon) revert FlipNotResolved();

    uint16 roll = period.roll;

    // ETH payout: roll only
    uint256 ethPayout = (claim.ethValueOwed * roll) / 100;

    // BURNIE payout: 0 on loss, roll * flip multiplier on win
    uint256 burniePayout;
    if (flipWon) {
        burniePayout = (claim.burnieOwed * roll * (100 + rewardPercent)) / 10000;
    }

    // Release ETH segregation
    pendingRedemptionEthValue -= ethPayout;

    // Clear claim
    delete pendingRedemptions[player];

    // Pay ETH
    _payEth(player, ethPayout);

    // Pay BURNIE
    if (burniePayout != 0) {
        _payBurnie(player, burniePayout);
    }

    emit RedemptionClaimed(player, roll, flipWon, ethPayout, burniePayout);
    ```

    **After:**
    ```solidity
    uint16 roll = period.roll;

    // ETH payout: roll only (always claimable once period resolved)
    uint256 ethPayout = (claim.ethValueOwed * roll) / 100;

    // BURNIE payout: depends on coinflip resolution
    uint256 burniePayout;
    bool flipResolved;
    {
        (uint16 rewardPercent, bool flipWon) = coinflip.getCoinflipDayResult(period.flipDay);
        flipResolved = (rewardPercent != 0 || flipWon);
        if (flipResolved && flipWon) {
            burniePayout = (claim.burnieOwed * roll * (100 + rewardPercent)) / 10000;
        }
    }

    // Release ETH segregation
    pendingRedemptionEthValue -= ethPayout;

    if (flipResolved) {
        // Full claim: clear entirely
        delete pendingRedemptions[player];
    } else {
        // Partial claim: clear ETH portion, keep BURNIE for later
        claim.ethValueOwed = 0;
    }

    // Pay ETH (always)
    _payEth(player, ethPayout);

    // Pay BURNIE (only if flip resolved)
    if (burniePayout != 0) {
        _payBurnie(player, burniePayout);
    }

    emit RedemptionClaimed(player, roll, flipResolved, ethPayout, burniePayout);
    ```

    This ensures:
    - ETH is always paid when the period is resolved (roll != 0)
    - BURNIE is paid only when the coinflip is also resolved
    - Partial claims clear ethValueOwed to prevent double-claim of ETH
    - The `flipWon` parameter in the event is replaced with `flipResolved` to convey semantics

    Also update the `FlipNotResolved` error: it may still be useful for other contexts, so keep the declaration but it's no longer used in `claimRedemption`. Optionally remove it if unused elsewhere (grep for other uses first).

    Verify `forge build` still passes after this change.
