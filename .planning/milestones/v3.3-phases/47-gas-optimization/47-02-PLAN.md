Create `test/fuzz/RedemptionGas.t.sol` -- a Foundry test contract that measures gas for all redemption functions. The test must:

1. Inherit `DeployProtocol` and call `_deployProtocol()` in `setUp()`
2. Set up a player with sDGNRS tokens (use `vm.prank` + pool transfer or direct balance manipulation via `vm.store`)

4. Exercise each function in isolation for clean gas measurement

**Critical implementation details:**

The test needs to handle the full protocol deployment. Use `_deployProtocol()` from DeployProtocol.sol which deploys all 28 contracts (5 mocks + 23 protocol). After deployment:

- The sDGNRS creator address holds 20% of supply (200B tokens)
- The game is active (not gameOver)
- `currentDayView()` returns >= 1 (since vm.warp(86400) is set in _deployProtocol)

To get a player with sDGNRS tokens, use `vm.store` to directly set `balanceOf[player]` and adjust `totalSupply`. The `balanceOf` mapping is at slot 1 in StakedDegenerusStonk (slot 0 = totalSupply, slot 1 = balanceOf mapping). Computing the storage slot for a mapping key:

```solidity
bytes32 playerBalSlot = keccak256(abi.encode(player, uint256(1)));
```

However, a simpler approach: use `vm.prank(ContractAddresses.SDGNRS)` to call `transferFromPool` -- but that requires the caller to be authorized. Instead, the cleanest approach is:

- Use `vm.store` to set `balanceOf[player]` to some test amount (e.g., 1_000_000e18)
- Use `vm.store` to adjust `totalSupply` (slot 0) to account for the tokens
- Alternatively, if sDGNRS has a `transferFromPool` function callable by the game, use `vm.prank(game)` to transfer from a pool

Look at how existing tests set up sDGNRS balances. The `transferFromPool` function is restricted to authorized callers (game contract). Use `vm.prank(address(game))` to call `sdgnrs.transferFromPool(Pool.Reward, player, amount)`.

For the `resolveRedemptionPeriod` test, the function is restricted to `ContractAddresses.GAME`. Use `vm.prank(address(game))`.

For `claimRedemption` test, the full lifecycle must be set up:
1. Player burns sDGNRS (submits gambling claim)
2. Game resolves period (via `vm.prank(address(game))` calling `resolveRedemptionPeriod`)
3. Coinflip day must be resolved (mock or `vm.store` the result)
4. Player calls `claimRedemption()`

For coinflip resolution, use `vm.store` to set the coinflip day result in BurnieCoinflip's storage, or use `vm.mockCall` to mock `getCoinflipDayResult` to return resolved values.
