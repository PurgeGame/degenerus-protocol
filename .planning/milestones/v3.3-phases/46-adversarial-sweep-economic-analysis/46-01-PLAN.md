**Sweep the 4 core gambling burn contracts using all 3 warden personas:**

1. **Contract Auditor perspective** -- For each of the 4 contracts, check:
   - Storage layout correctness (slot collisions, struct packing, proxy compatibility)
   - Reentrancy: CEI pattern at every external call site
   - Access control: every `msg.sender` check, every modifier, every `if (msg.sender != ...)` guard
   - State machine correctness: all state transitions, illegal state prevention
   - Integer overflow/underflow in `unchecked` blocks
   - Event emission completeness and ordering

2. **Zero-Day Hunter perspective** -- For each of the 4 contracts, check:
   - EVM-level exploits: selfdestruct force-send, create2 address prediction, returndata bomb
   - `unchecked` arithmetic: every `unchecked` block, verify no overflow/underflow possible
   - Assembly correctness: any inline assembly blocks
   - Temporal edge cases: block.timestamp manipulation, multi-block attack sequences
   - The CP-07 split claim partial state (`ethValueOwed=0, burnieOwed>0`): verify no double-claim, no stuck state
   - `_payEth` fallback-to-stETH logic: verify ETH transfer failure handling
   - `rngGate` vs `_gameOverEntropy` parallel paths: verify no path skips redemption resolution

3. **Economic Analyst perspective** -- For each of the 4 contracts, check:
   - MEV extraction: sandwich attacks on burn/claim transactions
   - Flash loan vectors: any path where borrowed funds enable profit
   - Pricing manipulation: any on-chain price the contract reads that can be manipulated
   - Solvency: reserved funds vs available balance at every mutation point
   - Game theory: timing advantages, information asymmetry
