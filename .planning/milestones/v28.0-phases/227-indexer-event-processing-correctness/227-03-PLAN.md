### TIER DECISION RULE (inherits D-225-04 / D-227-03)

- **Tier A** — comment describes a specific invariant (idempotency target, reorg depth, refresh trigger) AND the code EITHER (a) does not implement it, OR (b) implements a DIFFERENT invariant. Example: comment says "idempotent on (player, day)" but `.onConflictDoUpdate` target is `[player]`. Enumerate + finding.
- **Tier B** — comment is partially correct / ambiguous / omits a branch. Example: comment says "upserts streak" but code has conditional branch that SETS streak only under certain contractName values. Enumerate + finding.
- **Tier C** — missing comment entirely. Counted, not enumerated.
- **Tier D** — typo / stale method name. Skipped.
