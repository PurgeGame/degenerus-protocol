## Rationality rubric (D-226-05)

For each DDL statement in migration N:
- **JUSTIFIED** — there is a corresponding change in `src/db/schema/*.ts` that would produce this DDL if drizzle-kit were regenerated (e.g. `ADD COLUMN "foo" text NOT NULL` in SQL ↔ `foo: text().notNull()` present in the matching `pgTable` in TS).
- **DRIFT — schema side missing** — DDL exists but TS has no corresponding declaration. Emit finding with resolution RESOLVED-CODE (patch schema).
- **DRIFT — migration side missing** — TS has a declaration with no DDL across any migration. Emit finding with resolution RESOLVED-CODE (next migration should add the missing DDL) OR INFO-ACCEPTED if the TS-declared thing is runtime-only (e.g. views created by `view-refresh.ts` — per 226-RESEARCH.md Open Q1, default INFO until Phase 228 dispositions).

## Meta cross-check rubric (D-226-01 secondary)

- `tables.public.<t>.columns` in snapshot N must match the SQL accumulator state after applying migrations 0..N.
- Disagreement = finding (drizzle-kit internal record vs actual SQL, direction `schema<->migration`).
