Tier definitions (verbatim from D-225-04):

**Tier A (always flag): Outright wrong.** Comment makes a factual claim code contradicts. Examples:
- Comment: "returns users sorted by score desc" — code: sorts asc
- Comment: "paginates by cursor" — code: has no cursor logic
- Comment: "always returns 200" — code: returns 404 when player not found
- Comment: documents a parameter the code doesn't read

**Tier B (flag when material): Stale/incomplete.** Comment documents some behavior but omits a meaningful side effect, branch, or post-condition. "Material" = a caller behaving on the comment alone would make a wrong decision. Examples:
- Comment: "returns player stats" — code also writes to a cache; cache stampedes on cold path
- Comment: "fetches jackpot history" — code filters to the caller's day silently
- Comment: lists two error codes — code throws a third

**Tier C (DO NOT flag per-handler; count as context only):** Handler has NO JSDoc at all.
**Tier D (DO NOT flag anywhere):** Cosmetic / typos / stale parameter names after rename.

Severity scheme (carried from D-224-04 via 225-CONTEXT.md "Severity scheme"):
- Default: **INFO**
- Promote to **LOW** only when caller-breaking:
  - Handler returns a field type Zod says is `string` but actually is `number` (runtime serialization mismatch)
  - Required request parameter documented optional in openapi.yaml (callers omit it, get 400)
  - Response field documented in openapi.yaml that the handler never emits (callers expecting it crash)
- Do NOT promote beyond LOW — documentation drift is not a vulnerability class.

Finding-ID format per D-225-06: `F-28-225-NN` — sequential allocation across all three plans in Phase 225 (this plan, 225-02, 225-03) share the same numeric pool. Allocate during audit execution, in order of discovery.

Scope exclusion per D-225-01: Do NOT audit any file in `/home/zak/Dev/PurgeGame/database/src/handlers/*.ts`. Those are indexer event handlers (dispatched by `src/indexer/event-processor.ts`, consume `ctx.args`) and are deferred to Phase 227 (requirement IDX-03).
