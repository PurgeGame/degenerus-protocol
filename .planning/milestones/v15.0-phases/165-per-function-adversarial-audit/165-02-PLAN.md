   - (a) `purchaseLevel = cachedJpFlag ? cachedLevel : cachedLevel + 1` -- verify phase-correctness (purchase phase: level+1, jackpot phase: level). Verify cachedJpFlag is read from jackpotPhaseFlag storage.
   - (b) `PriceLookupLib.priceForLevel(purchaseLevel)` -- verify purchaseLevel (not level) is the argument.
   - (c) Lootbox base level = `cachedLevel + 1` -- verify this is always level+1 regardless of phase (was `level + 2` before, which was a bug fixed in 160.1).
   - (d) Single `quests.handlePurchase()` call -- verify all 6 arguments match what the old 3 separate calls provided. Check reentrancy: is handlePurchase called before or after state updates? If before, can the callee re-enter?

   - (f) Batched `coinflip.creditFlip` -- verify total credit amount is sum of all individual credits (ETH mint reward + BURNIE quest reward + lootbox reward + level quest reward if any).
   - (g) `claimableWinnings[buyer]` read once as `initialClaimable` -- verify no write to claimableWinnings occurs between the read and its use in shortfall calculation.
   - (h) Ticket queuing moved from _callTicketPurchase to _purchaseFor -- verify tickets are still queued AFTER payment validation.

4. `_callTicketPurchase()` (line ~864, v14.0): Returns `(bonusCredit, adjustedQty32, targetLevel, ethMintUnits, burnieMintUnits)`. Check: x00 century bonus logic removed (moved to caller) -- verify caller (_purchaseFor) correctly implements the bonus.
