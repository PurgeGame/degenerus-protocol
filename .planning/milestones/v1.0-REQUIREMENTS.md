# Requirements: Degenerus Protocol Security Audit

**Defined:** 2026-02-28
**Core Value:** Every ETH that enters the protocol must be accounted for, every RNG outcome must be unmanipulable, and no actor can extract value beyond what the game mechanics intend.

## v1 Requirements

Requirements for the security audit. Each maps to roadmap phases.

### Storage Foundation

- [x] **STOR-01**: All 10 delegatecall modules have identical storage layout to DegenerusGame — no module declares instance storage variables
  *Outcome: PASS. Zero non-constant state variables in all 10 modules. Two independent grep patterns confirmed.*
- [ ] **STOR-02**: Storage slot ordering in DegenerusGameStorage matches `forge inspect` output for all module contracts
  *Outcome: Slot map produced (135 variables, slots 0–108) but formal requirement closure not documented. Left open.*
- [ ] **STOR-03**: ContractAddresses compile-time constants are correctly mapped — no address(0) values remain in deployed bytecode
  *Outcome: PASS conditional — 29 constants enumerated, patch pipeline regex coverage verified, but relies on deploy script being used correctly. Not formally closed.*
- [x] **STOR-04**: No testnet configuration (`TESTNET_ETH_DIVISOR`) bleeds into mainnet contract logic
  *Outcome: PASS. Testnet isolation via physical filesystem separation (hardhat.config.js source path switching). Zero leakage confirmed.*

### RNG State Machine

- [x] **RNG-01**: `rngLockedFlag` remains set continuously from VRF request through word consumption — no nudge manipulation window
  *Outcome: PASS. 22 total references traced: 1 SET, 2 CLEAR, 10 guard, 3 branch/read, 3 view. Lock held continuously.*
- [x] **RNG-02**: `rawFulfillRandomWords` cannot revert under any condition
  *Outcome: PASS. Callback cannot revert under normal operation. Coordinator rotation edge case fully traced.*
- [x] **RNG-03**: `rawFulfillRandomWords` gas cost stays under 200,000 gas
  *Outcome: PASS. Worst-case ~45k gas, 85% headroom against 300k limit.*
- [x] **RNG-04**: `requestId` matching is correct — no mismatch can cause wrong VRF word applied to wrong game state
  *Outcome: PASS. requestId lifecycle traced for daily and lootbox paths.*
- [x] **RNG-05**: Concurrent VRF requests cannot create requestId ordering conflicts
  *Outcome: PASS. Concurrent request impossibility proven via three mutual exclusion guards.*
- [x] **RNG-06**: RNG lock cannot be bypassed or stuck permanently
  *Outcome: PASS. All stuck states recoverable via 18h retry, 3-day emergency, 3-day fallback, 30-day sweep, liveness timeout.*
- [x] **RNG-07**: 18-hour VRF retry timeout cannot be abused
  *Outcome: PASS. 18h retry timeout abuse resistance analyzed.*
- [x] **RNG-08**: `reverseFlip()` nudge mechanism cannot be exploited by a block proposer
  *Outcome: PASS. Block proposer cannot front-run VRF fulfillment with reverseFlip nudges.*
- [x] **RNG-09**: `EntropyLib.entropyStep()` XOR-shift derivation does not introduce exploitable patterns
  *Outcome: PASS. xorshift (7,9,8) on 256-bit not exploitable with VRF seed.*
- [x] **RNG-10**: No code path uses `block.timestamp` or `blockhash` as a randomness source
  *Outcome: PASS. 27 block.timestamp usages confirmed all timing/gating only. VRF is sole entropy source.*

### Token and Pricing Math

- [x] **MATH-01**: Ticket price escalation formula (PriceLookupLib) is monotonically increasing and does not overflow
  *Outcome: PASS. Intra-cycle monotonicity confirmed; saw-tooth by design.*
- [x] **MATH-02**: Deity pass pricing T(n) = n*(n+1)/2 + 24 ETH does not overflow at n=100, n=1000
  *Outcome: PASS. Max intermediate value 53 orders of magnitude below uint256 max. Checked arithmetic context.*
- [ ] **MATH-03**: Whale bundle pricing (2.4 ETH levels 0-3, 4 ETH x49/x99) is correctly enforced across all purchase paths
  *Outcome: F01 HIGH FOUND — whale bundle lacks level eligibility guard. Requirement not satisfied.*
- [x] **MATH-04**: Lazy pass pricing (sum-of-10-level-prices at level 3+) correctly sums the price curve
  *Outcome: PASS. Verified with 15-level reference table.*
- [x] **MATH-05**: Lootbox EV multiplier formula — activity score cannot create guaranteed positive EV extraction
  *Outcome: PASS. Cap tracks raw input (conservative design). No guaranteed positive-EV extraction.*
- [x] **MATH-06**: Degenerette bet resolution pays correctly — no bet timing creates advantaged positions
  *Outcome: PASS. Commit-reveal pattern sound. futurePrizePool geometric decay converges to 1 wei.*
- [x] **MATH-07**: Coinflip 50-150% bonus range is correctly bounded
  *Outcome: PASS conditional. Base range [50,150] verified; presale +6 can push to 156/159 but is intentional promotional feature (Informational).*
- [x] **MATH-08**: BitPackingLib 24-bit field packing/unpacking is correct — no field overflow or bleed
  *Outcome: PASS. All 32 setPacked and 34 read sites verified. Complete mintPacked_ field layout map produced.*

### Cross-Contract Interactions

- [x] **XCON-01**: All delegatecall return values are checked
  *Outcome: PASS (plan 07-01 complete). All 30 sites use uniform (bool ok, bytes memory data) + _revertDelegate(data). XCON-F01: deityBoonSlots staticcall reads module storage (MEDIUM, view-only).*
- [x] **XCON-02**: stETH.submit() and stETH.transfer() return values are checked
  *Outcome: PASS (plan 07-02 complete). 12/12 state-changing stETH calls checked. 2 submit() uses try/catch with documented intent. Zero unchecked stETH calls.*
- [x] **XCON-03**: LINK.transferAndCall() return value is checked and cannot create reentrancy
  *Outcome: PASS (plans 07-02 + 07-03 complete). Return value checked in try/catch. VRF coordinator does not call back to Admin. No circular reentrancy path.*
- [x] **XCON-04**: BurnieCoin.burnCoin() behavior on insufficient balance is safe
  *Outcome: PASS (plan 07-02 complete). burnCoin() reverts on insufficient balance; revert propagates through delegatecall. No free nudges or bets.*
- [x] **XCON-05**: Cross-function reentrancy — ETH callback from claimWinnings cannot reenter purchase
  *Outcome: PASS (plan 07-03 complete). All 48 state-changing entry points verified safe during mid-claim callback. Sentinel pattern + CEI confirmed complete.*
- [x] **XCON-06**: stETH rebasing cannot create reentrancy vector via Lido internal callbacks
  *Outcome: PASS (plan 07-03 complete). stETH is standard ERC-20; not ERC-677/ERC-777. No recipient callbacks on transfer/submit. Rebasing is passive oracle-driven.*
- [x] **XCON-07**: Constructor-time cross-contract calls execute in correct order
  *Outcome: PASS. All 22 constructors verified safe. No constructor calls a higher-nonce contract.*
