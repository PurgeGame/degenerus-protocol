Classification rules:
- **KNOWN**: The finding matches an existing entry in FINAL-FINDINGS-REPORT.md or KNOWN-ISSUES.md by contract, mechanism, and severity class. Map to the specific prior finding ID (M-02, DELTA-L-01, I-xx, DELTA-I-xx).

- **EXTENDS**: The finding is related to a known finding but adds meaningful new detail (different attack path, expanded impact analysis, new code reference).
- **DUPLICATE**: Multiple wardens independently found the same issue. List all warden IDs. This VALIDATES coverage -- if multiple agents independently find M-02, it confirms the audit process is working.
