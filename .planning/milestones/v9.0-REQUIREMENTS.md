**Defined:** 2026-03-28
**Core Value:** Every finding a C4A warden could submit is identified and either fixed or documented as known before the audit begins.

## Fresh-Eyes Wardens

- [ ] **WARD-01**: RNG/VRF warden produces PoC exploit or SAFE proof for every VRF commitment window, request-to-fulfillment path, and RNG consumer — tracing cross-contract through all modules
- [ ] **WARD-02**: Gas ceiling warden produces PoC exploit or SAFE proof for every advanceGame execution path under adversarial state construction — tracing cross-contract through all delegatecall modules
- [ ] **WARD-03**: Money correctness warden produces PoC exploit or SAFE proof for every ETH/token flow, BPS rounding chain, and cross-token interaction (sDGNRS/DGNRS/BURNIE/GNRUS/wXRP) across all 24 contracts
- [ ] **WARD-04**: Admin resistance warden produces PoC exploit or SAFE proof for every admin-accessible path, distinguishing bootstrap phase (admin-dominant DGNRS, can unwrap to sDGNRS) from post-distribution (governance effective), with special attention to the two Chainlink-death-gated governance paths (VRF coordinator swap, price feed swap)
- [ ] **WARD-05**: Composition warden produces PoC exploit or SAFE proof for cross-domain attack sequences (RNG+money, admin+gas, etc.) and delegatecall module seam interactions
- [ ] **WARD-06**: Every warden receives ONLY contract source + C4A README + KNOWN-ISSUES.md — zero prior audit findings or SAFE verdicts
- [ ] **WARD-07**: Every finding includes coded PoC with concrete calldata, or explicit SAFE proof with cross-contract trace — no INFO-level hand-waving
