# Plan 118-01: Cross-Contract Interaction Map + Shared Storage Inventory

## Objective

Build the complete cross-contract interaction map and shared storage inventory for all 29 contracts in the protocol. This is the Taskmaster-equivalent step for the integration sweep -- establishing what needs to be verified before the attack analysis begins.

## Context

- @118-CONTEXT.md -- Phase scope and decisions
- @118-RESEARCH.md -- Aggregated findings and call patterns from all 15 units
- @.planning/ULTIMATE-AUDIT-DESIGN.md -- Unit 16 scope definition

## Tasks

### Task 1: Build Cross-Contract Interaction Map
type="auto"

**Action:**
1. Read all 15 unit findings reports and attack reports for cross-contract call sites
2. Compile the COMPLETE cross-contract call graph organized by:
   - Module -> Standalone calls (delegatecall context)
   - Standalone -> Standalone calls
   - Callback / re-entry chains
   - Nested delegatecall chains (LootboxModule -> BoonModule, DegeneretteModule -> LootboxModule)
3. For each call edge, document: source function, target function, data flow (what parameters/state cross the boundary), whether the call is in delegatecall context

**Output:** `audit/unit-16/INTEGRATION-MAP.md` -- Section 1: Cross-Contract Call Graph

### Task 2: Build Shared Storage Write Inventory
type="auto"

**Action:**
1. From unit findings and attack reports, compile every storage variable that is written by more than one module (within the delegatecall context)
2. For each shared variable: which modules write it, which modules read it, what access pattern (fresh read vs cached local)
3. Flag any variable where two modules could write within the same top-level transaction

**Output:** `audit/unit-16/INTEGRATION-MAP.md` -- Section 2: Shared Storage Write Map

### Task 3: Build Access Control Matrix
type="auto"

**Action:**
1. For every external/public function across all 29 contracts, document what access control gate prevents arbitrary EOA calls
2. Categorize: onlyGame (compile-time), onlyOwner (DGVE majority), onlyCoin (compile-time), permissionless (anyone), conditional (gameOver, rngLocked, etc.)
3. Flag any function that lacks access control but modifies state

**Output:** `audit/unit-16/INTEGRATION-MAP.md` -- Section 3: Access Control Matrix

## Verification

- [ ] Every cross-contract call site from all 15 unit reports is represented in the map
- [ ] Every shared storage variable has its writer set documented
- [ ] Every external/public state-changing function has an access control entry
- [ ] The map is self-consistent (no orphaned edges, no missing contracts)

## Done Criteria

INTEGRATION-MAP.md exists in audit/unit-16/ with all three sections populated and cross-referenced against unit findings.
