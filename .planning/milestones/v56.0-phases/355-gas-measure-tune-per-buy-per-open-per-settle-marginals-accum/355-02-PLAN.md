<objective>
Run the gas-scavenger over the v56 afking STAGE/accrue/settle/open surface and the gas-skeptic to validate every candidate, producing the candidate ledger that 355-03 consumes. This is where the GAS-04 mode/SLOAD collapse candidates and the GAS-02 accumulator-packing finalization fall out, each gated by the skeptic against the security-over-gas floor.

Purpose: the security floor OVERRIDES gas. The scavenger proposes; the skeptic disposes. Any "win" that silently widens an attack surface (unmanipulability of the afking buy/open/settle, the double-credit guards, RNG-freeze, the byte-frozen SOLVENCY-01 debit) is REJECTED WITH REASONING here so 355-03 only ever lands skeptic-APPROVED changes — or records Outcome-A no-diff if everything is rejected. A SECOND gate also applies: because the per-tx-batch sizing is now a DUAL BOUND (every batched chunk worst-case must target < 10M and provably never exceed 16.7M at the cap), no candidate may RAISE the binding settle-day per-sub marginal — a candidate that increases per-sub settle-day cost shrinks the safe SUB_STAGE_BATCH and is flagged even if otherwise safe.

Output: `355-02-GAS-AUDIT.md` — the candidate list, the per-candidate skeptic verdicts, the APPROVED set (file:line + implementation note) and the REJECTED set (the invariant each would have traded). Doc only; agent-committable (no contract gate).
</objective>
