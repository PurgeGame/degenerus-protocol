**Step E — Strip CBOR metadata from each bytecode dump:**

The Solidity compiler appends a CBOR-encoded metadata block to each contract's bytecode containing the compiler version + IPFS hash of the source metadata. This metadata changes between builds even with identical source. The CBOR block starts with the marker `a165627a7a72` (`bzzr0` legacy) or `a264697066735822` (current `ipfs` form) — depending on solc version.

Implementation: use a Python or shell helper to truncate the hex string at the CBOR marker. Recommended one-liner pattern (POSIX-portable):

```
strip_metadata() {
  local infile="$1"
  local outfile="$2"
  # Strip the CBOR metadata block — anything after the metadata marker AND the trailing 4-byte length suffix
  # The metadata-length suffix is the LAST 4 hex chars (2 bytes) — strip those first, then strip everything after the marker
  python3 -c "
import sys, re
with open('$infile') as f:
    data = f.read().strip()
# Strip 0x prefix if present
if data.startswith('0x'):
    data = data[2:]
# CBOR metadata marker patterns (covers solc >=0.5.x): a165627a7a72 (bzzr0) and a264697066735822 (ipfs)
m = re.search(r'(a165627a7a72|a264697066735822)', data)
if m:
    body = data[:m.start()]
else:
    body = data  # No marker found — assume already stripped or unknown form
print(body)
" > "$outfile"
}

strip_metadata /tmp/v31-244-qst05/quests-baseline.bytecode  /tmp/v31-244-qst05/quests-baseline-stripped.bytecode

strip_metadata /tmp/v31-244-qst05/mintmodule-baseline.bytecode  /tmp/v31-244-qst05/mintmodule-baseline-stripped.bytecode
```
