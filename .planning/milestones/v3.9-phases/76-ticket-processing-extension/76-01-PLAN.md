<objective>
Extend processFutureTicketBatch to drain both the read-side queue and far-future (FF) key queue, with cursor state encoding that distinguishes processing phases, and fix _prepareFutureTickets to correctly resume FF-encoded ticketLevel.

Purpose: After Phase 75 routes far-future tickets to the FF key space, those tickets sit permanently unprocessed because processFutureTicketBatch only reads from _tqReadKey(lvl). This plan makes the function drain the FF key after the read-side queue, using the existing uint24 ticketLevel with bit 22 to signal which phase is active. Without this fix, all far-future tickets are stranded and never minted.

Output: Modified processFutureTicketBatch (MintModule), modified _prepareFutureTickets (AdvanceModule), Foundry test file proving all three PROC requirements.
</objective>
