  <action>
Parse the 4naly3er report and create a structured triage document. 4naly3er produces findings in categories (Gas Optimizations, Non-Critical, Low, Medium). For EVERY finding:

1. Read the finding description and the code instances listed
2. Read the actual contract code at each flagged location to understand context
3. Classify as one of:
   - **FIX** — Real issue that should be addressed in Phase 134
   - **DOCUMENT** — Real behavior but intentional/acceptable, add to known-issues triage
   - **FALSE-POSITIVE** — Detector is wrong about this code
4. Write reasoning for the classification (1-3 sentences explaining WHY)

Per D-05: Default disposition is DOCUMENT, not FIX. Do not touch contract code.
Per D-06: Batch all findings -- do not escalate individual findings.
Per D-07: Use judgment on grouping vs itemizing. 4naly3er groups by category already -- maintain that grouping but add per-instance analysis where instances differ in context.

Document structure:
```markdown
# 4naly3er Triage — Phase 130 Bot Race

**Tool:** 4naly3er (Code4rena bot tool)
**Date:** {date}
**Scope:** 17 top-level contracts + 5 libraries (mocks excluded)
**Total finding categories:** {N}
**Total instances:** {N}

## Summary

| Severity | Categories | Instances | FIX | DOCUMENT | FP |
|----------|------------|-----------|-----|----------|----|
| Medium | {n} | {n} | {n} | {n} | {n} |
| Low | {n} | {n} | {n} | {n} | {n} |
| Non-Critical | {n} | {n} | {n} | {n} | {n} |
| Gas | {n} | {n} | {n} | {n} | {n} |

## Findings Requiring Action (FIX)

### [{severity}-{N}] {finding title}
- **Severity:** {Medium/Low/NC/Gas}
- **Instances:** {N}
- **Locations:** {contract:line for each instance}
- **Description:** {what 4naly3er found}
- **Reasoning:** {why this needs fixing}

## Findings to Document (DOCUMENT)

### [{severity}-{N}] {finding title}
- **Severity:** {level}
- **Instances:** {N}
- **Locations:** {list}
- **Description:** {what 4naly3er found}
- **Reasoning:** {why this is intentional/acceptable}

## False Positives

### [{severity}-{N}] {finding title}
- **Instances:** {N}
- **Locations:** {list}
- **Reasoning:** {why these are FPs}
```

Cross-reference against KNOWN-ISSUES.md -- if a finding maps to an already-documented known issue, note "Already in KNOWN-ISSUES.md" in reasoning.
