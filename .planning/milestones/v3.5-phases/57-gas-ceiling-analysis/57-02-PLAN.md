**Section 4: Purchase Analysis (CEIL-03, CEIL-04)**

Profile each purchase path with the same rigor as advanceGame stages:

**Path 1: Ticket Purchase (ETH) -- purchase() with ticketQuantity > 0, lootBoxAmount = 0**
- Entry: DegenerusGame.sol purchase() -> delegatecall MintModule._purchaseFor()
- _resolvePlayer: ~5 SLOADs (~10,500 cold) + potential SSTORE if new player
- _callTicketPurchase: consumePurchaseBoost self-call (~5,000) + recordMint self-call
- recordMint{value}: NOT a delegatecall -- this is an actual CALL to self with ETH. Cost: ~2,600 (cold address, though self) + internal execution
  - _processMintPayment: prize pool split writes (~5 SSTOREs = ~25,000-110,000), affiliate fee calculation
  - _recordMintDataModule: delegatecall MintModule (~2,600) + ticket counting storage
  - _awardEarlybirdDgnrs: conditional sDGNRS transfer (~10,000 if triggered, 0 if past earlybird window)
- affiliate.payAffiliate() external call: ~15,000 (cold address + affiliate storage writes)
- coin.creditFlip() external call: ~5,000 (BURNIE balance SSTORE)
- _queueTicketsScaled: O(1) append per call -- 1 SSTORE for queue slot (~5,000), 1 SSTORE for queue length (~5,000)
- Total per single ticket purchase: ~150,000-250,000 gas (depending on cold/warm state and earlybird)
- Batch scaling: purchase() is called once per transaction, not looped. Each call processes one ticketQuantity parameter. The _queueTicketsScaled is O(1) regardless of quantity -- it stores (buyer, quantity, level) as a single queue entry
- Therefore: ticket batch size does NOT scale gas linearly. One purchase() call handles any ticketQuantity in O(1)
- Worst case: first-ever purchase by new player with all cold storage = ~250,000 gas
- Max batch size under 14M: effectively unlimited (O(1) per call), batch size is bounded by msg.value / ticket price, not gas

**Path 2: Lootbox Purchase (ETH) -- purchase() with ticketQuantity = 0, lootBoxAmount > 0**

CEIL-04 section: "Maximum Ticket Batch Size Under 14M"
- Key finding: purchase() queues tickets in O(1) via _queueTicketsScaled, so batch size is NOT gas-limited
