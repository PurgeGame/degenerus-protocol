Append to `audit/v3.5-gas-standalone-and-dead-code.md` a new section: "## Part 2: Dead Code and Redundant Check Analysis (GAS-02)"

Perform the following systematic searches across ALL files in `contracts/` (excluding `contracts/mocks/` and `contracts/test/`):

**2a. Orphaned Errors**
Run: `rg 'error \w+' contracts/ --type sol -o` to find all error declarations.
For each error found, run: `rg '<ErrorName>' contracts/ --type sol --count` to see if it is used (reverted with) anywhere.
If an error is declared but never used in a revert: flag as DEAD ERROR.
The `E()` error in DegenerusGameStorage is the universal revert -- confirm it is used.
v3.2 already removed `TakeProfitZero` from BurnieCoinflip -- verify it is gone.

**2b. Orphaned Events**
Run: `rg 'event \w+' contracts/ --type sol -o` to find all event declarations.
For each event, run: `rg 'emit <EventName>' contracts/ --type sol --count`.
If an event is declared but never emitted: flag as DEAD EVENT.

**2c. Unreachable Branches**
Search for patterns that indicate unreachable code:
- `if (false)` or conditions that are always true/false based on type constraints
- `else` branches after `revert` in the `if` branch (the else is the normal path, but check for dead else-after-always-true)
- Functions that are `internal` but never called from any other function

For each `internal` function in DegenerusGameStorage.sol helper functions, verify at least one call site exists.

**2d. Redundant Guard Checks**
Search for patterns where the same condition is checked multiple times in a call chain:
- `gameOver` checks: `rg 'gameOver' contracts/ --type sol` -- identify if any function checks `gameOver` that is already guaranteed by a modifier or caller
- `msg.sender` checks that duplicate access control already enforced by caller

**2e. Unused Internal Functions**
For each `internal` or `private` function declaration in contracts/:
- Run `rg 'functionName' contracts/ --type sol --count`
- If only 1 match (the declaration itself): flag as DEAD FUNCTION

Format findings with these subsections:

"### Dead Errors" with a table: Error, Contract, Line, Status
"### Dead Events" with a table: Event, Contract, Line, Status
"### Unreachable Branches" (None found / list any found)
"### Redundant Guards" (None found / list any found with estimated gas impact)
"### Unused Internal Functions" (None found / list any found)
"### Dead Code Summary" table: Category, Checked, Dead Found
