Create `audit/v3.5-gas-findings.md` with the following structure:

Title: "# v3.5 Gas Optimization Findings"

Then "## Storage Packing Opportunities (GAS-03)" section.

For DegenerusGameStorage, document KNOWN packing opportunities from the research:

**Known wasted-byte slots (from research):**
- Slot 1: 5 bytes padding (after prizePoolFrozen)
- Slot 17: 23 bytes padding (ticketCursor + ticketLevel + dailyEthWinnerCursor = 9 bytes)
- Slot 19: 30 bytes wasted (dailyCarryoverWinnerCap is uint16 in its own slot)
- Slot 24: 24 bytes padding (gameOverTime + gameOverFinalJackpotPaid + finalSwept = 8 bytes)
- Slot 43: 19 bytes padding (lastDailyJackpotWinningTraits + lastDailyJackpotLevel + lastDailyJackpotDay = 13 bytes)
- Slot 58: 12 bytes wasted (vrfCoordinator is address, 20 bytes)
- Slot 61: 26 bytes wasted (lootboxRngIndex is uint48, 6 bytes)
- Slot 74: 31 bytes wasted (midDayTicketRngPending is bool, 1 byte)
- Slot 104: 29 bytes wasted (centuryBonusLevel is uint24, 3 bytes)
- Slot 106: 26 bytes wasted (lastVrfProcessedTimestamp is uint48, 6 bytes)

For each, determine if packing is POSSIBLE:
- Can adjacent scalars be moved next to each other?
- Are the variables co-accessed (read/written in same function)?
- What is the estimated gas saving per transaction?

Use these gas costs:
- Packing saves 1 cold SLOAD per co-access: 2,100 gas
- Packing saves 1 cold SSTORE if co-written: up to 20,000 gas (zero to nonzero) or 2,900 gas (nonzero to nonzero)

**Boon mapping packing pattern:**
Document the pattern ONCE with `lootboxBoon5Active` + `lootboxBoon5Day` as the concrete example:
- Current: 2 mappings, 2 slots per player lookup
- Proposed: 1 mapping(address => uint256) with bool + uint48 packed
- Savings: 2,100 gas per boon check (1 cold SLOAD)
- Applies to: enumerate all similar pairs (lootboxBoon5, lootboxBoon15, lootboxBoon25, whaleBoon, coinflipBoon, activityBoon, purchaseBoost, decimatorBoost, coinflipBoonBps, lazyPassBoon)
- Note: requires contract redeployment

Also run `forge inspect` for standalone contracts with wasted bytes:
- `forge inspect BurnieCoinflip storageLayout`
- `forge inspect StakedDegenerusStonk storageLayout`
- `forge inspect DegenerusAdmin storageLayout`

Format each packing opportunity as a subsection "### PACK-NN: [Description]" with a table containing: Contract, Current slots, Proposed, Savings, Co-access (Yes/No + which functions), Requires redeployment (Yes), Severity (GAS-INFO or GAS-LOW).
