## Methodology

RNG-02's question: "Can a permissionless attacker, who sees the VRF request tx, change a state variable before the fulfillment tx lands in a way that influences a consumer's output?" Five categories of new RNG consumer are audited:

1. **Earlybird bonus-trait roll (`20a951df`)** — `_runEarlyBirdLootboxJackpot` invokes `_rollWinningTraits(rngWord, true)` at `DegenerusGameJackpotModule:677` with the `BONUS_TRAITS_TAG` compile-time salt. 2. **BAF sentinel emission (`104b5d42`)** — `runBafJackpot` uses `traitId = 420` as a sentinel to distinguish BAF winners from main-trait winners; no new player-controllable input added.

### Phase 235 D-11 Invariant Annotation

Per CONTEXT.md D-11 (locked by user): "During the rngLocked window (VRF request → fulfillment), across the newly-packed housekeeping step: (a) NO far-future ticket queue write may occur, AND (b) NO write may land in the active (read-side) buffer. Writes to the write-side buffer at the current level ARE PERMITTED — they drain next round with the next VRF word. rngLocked is NOT a blanket ticket-queueing block."

**Invariant (a) — no far-future queue writes:** Enforced structurally by the `isFarFuture && rngLockedFlag && !rngBypass` guard at `contracts/storage/DegenerusGameStorage.sol:570 / 602 / 658`. Any call to `_queueTickets` / `_queueTicketsScaled` / `_queueTicketRange` with `targetLevel > level + 5` reverts with `RngLocked()` while `rngLockedFlag == true`.

**Invariant (b) — no writes to the read-side buffer:** Enforced by the `_swapTicketSlot` mechanic at `contracts/storage/DegenerusGameStorage.sol:758-763`. The write path at `_tqWriteKey(lvl) = ticketWriteSlot ? lvl | TICKET_SLOT_BIT : lvl` and read path at `_tqReadKey(lvl) = !ticketWriteSlot ? lvl | TICKET_SLOT_BIT : lvl` are ALWAYS disjoint by construction (one bit flipped between them). Purchase paths write via `_tqWriteKey`; the consumer drains via `_tqReadKey`. An attacker cannot write into the active read-side buffer — the bit toggle makes the buffers disjoint.

**Writes to write-side buffer are PERMITTED:** Current-level ticket purchases during the window write into the write buffer (via `_tqWriteKey`). These tickets drain next round with the next VRF word, not this round. The buffer SWAP fires at RNG REQUEST TIME per D-12 (`_swapAndFreeze(purchaseLevel)` at `AdvanceModule:292`; also invoked at the mid-day lootbox path at `AdvanceModule:1082` and the advanceGame transition at `AdvanceModule:595`). After the swap, the write buffer is empty and the read buffer contains the tickets that will be drained with the in-flight VRF word.
