Per-path algebraic proof shape: for each execution-path endpoint, write `sum-before + ingress = sum-after + egress` where:

- `sum-before = currentPrizePool + nextPrizePool + futurePrizePool + claimablePool` at entry.
- `ingress` = net `msg.value` ingress at the payable entry point minus any `msg.value` forwarded out in the same tx.
- `egress` = net ETH transferred OUT of the game contract during the tx (currently zero for all in-scope paths — all ETH payouts flow to `claimableWinnings` pull-pattern storage, not direct transfers).
- `sum-after` = pool totals after the function returns.

For paths where the ETH stays inside the game but moves between pools (earlybird `futurePool → nextPool`, decimator consolidated block), the proof reduces to `Δ(nextPool) + Δ(futurePool) + Δ(currentPool) + Δ(claimablePool) = 0`.

- **(INFO-B) `claimablePool` is an aggregate liability, not an ETH pool.** Per the invariant `claimablePool >= sum(claimableWinnings[*])` at DegenerusGameStorage.sol:342, `claimablePool` is a uint128 accumulator tracking total ETH owed to players via pull-pattern. The CONS-01 conservation proof uses `claimablePool` as a sink: when `futurePrizePool -= spend` fires in the consolidate block, `claimablePool += spend` fires in the same transaction, closing the ETH conservation locally. Actual ETH transfer to the player occurs later via `withdrawClaimable` (pull pattern), which is out-of-scope for CONS-01 (no v29.0 delta touches `withdrawClaimable`).
- **(INFO-C) `decimatorPool` terminology is a plan-level abstraction.** The plan frontmatter references `decimatorPool` as a distinct pool. In the code, there is no single `decimatorPool` slot — the decimator pool is synthesized per resolution level via `decClaimRounds[lvl].poolWei` (DecimatorModule:258) and settled by crediting `claimablePool` via `_consolidatePoolsAndRewardJackpots` batched SSTORE (AdvanceModule:892). This is a naming clarification for downstream readers; no code change implied.

### Path A — Earlybird Purchase (f20a2b5e)

**Entry point:** `DegenerusGameMintModule._purchaseFor(buyer, ticketQuantity, lootBoxAmount, affiliateCode, payKind)` at `contracts/modules/DegenerusGameMintModule.sol:913`.

**Ingress:** `msg.value` (paid by user at external entry point). Let `V = msg.value`; `P = costWei = ticketCost + lootBoxAmount = (priceWei * ticketQuantity) / (4 * TICKET_SCALE) + lootBoxAmount`.

**Per-branch breakdown:**

For the ticket-side prize contribution (routed via `recordMint` self-call at MintModule:1284): let `C_ticket = costWei - 0 if DirectEth else (claimableUsed portion)`. The split is `futureShare = (C_ticket * PURCHASE_TO_FUTURE_BPS) / 10_000`, `nextShare = C_ticket - futureShare`. Both halves land in `prizePoolsPacked` at DegenerusGame.sol:374-380.

For the lootbox-side prize contribution (routed via MintModule _purchaseFor:1062-1078): let `C_lb = lootBoxAmount`. Distress/presale/normal branches produce `futureShare` + `nextShare` + `vaultShare` where `futureShare + nextShare + vaultShare = C_lb`. `vaultShare` is EGRESSED via `payable(ContractAddresses.VAULT).call{value: vaultShare}("")` at MintModule:1080-1083 — this is external egress outside the pool system.

**Conservation equation for combined purchase (DirectEth payment):**
