## Deviations from Plan

### Scope expansion — Plan 01 Revision 2 corrections absorbed mid-flight

Plan 02 as planned assumed Plan 01's fix was correct. During Task 1, three issues surfaced:
1. **Liveness bug:** Revision 1 reverted `NotTimeYet()` on fresh deploy (LR_INDEX=0 underflow).
2. **Raw-vs-nudged inconsistency:** Revision 1 wrote RAW word; game-over path writes NUDGED word. Player-influenceable via game-over trigger.
3. **Stylistic inconsistency:** Revision 1 used custom `emit + creditFlip + return` on partial drain; other stages use `stage = STAGE_*; break;`.

User approved a Revision 2 contract correction mid-Plan-02 (commit `d09e93ec`). The FIX.md was updated with a §Revision 2 supplement. This adds one contract commit outside Plan 02's scope but was necessary for Plan 02's tests to be meaningful (the binding test's RED-GREEN witness on pre-fix `df6674f0` vs post-fix `d09e93ec` is only interpretable with a correct post-fix contract).

### AC-6 narrowed interpretation

SPEC AC-6 originally read "existing forge test suite passes with zero new failures." Plan 01's fix surfaced ~86 pre-existing tests that implicitly relied on the `entropy=0` bug (they called `advanceGame` without VRF setup; pre-fix silently consumed entropy=0; post-fix correctly reverts `NotTimeYet()`). User directed AC-6 be narrowed to Plan 02's new tests only. Legacy test modernization tracked for follow-up. Documented in TESTS.md § Existing Suite Regression Check.

### Slot-constant bug in test helper

Initial binding test used slot 38 for LR_INDEX (following VRFPathHandler precedent). `forge inspect DegenerusGame storage-layout` revealed LR_INDEX lives in `lootboxRngPacked` at slot 37, and `lootboxRngWordByIndex` mapping at slot 38. Fixed in the binding test + handler. Pre-existing VRFPathHandler/LootboxRngLifecycle retain the old slot numbers — tracked as separate tech debt, not modified in this plan.

## Issues Encountered

- **Plan 01 Revision 1 liveness bug** — discovered via failing binding test on fresh deploy. Addressed in Revision 2.
- **Player-influenceable raw-vs-nudged ordering** — discovered when user asked about the NUDGE handling. Addressed in Revision 2.
- **Storage slot mismatch in test helpers** — inherited slot numbers from existing handlers were wrong (post-schema-migration drift). Fixed locally; other handlers retain the bug as tracked tech debt.
- **Game-over transition requires multi-tx progression** — first game-over advance requests VRF; subsequent advance after fulfillment sets `gameOver = true`. Test loops up to 20 times to drive the transition. Single-advance assertion would fail.
- **`_currentMintDay` / `dailyIdx` / `day` semantics** — `dailyIdx` is set to `currentDay` in constructor, not 0. After fresh-deploy warp +1 day, `day=2, dailyIdx=1`. Mid-day path conditioning on `day == dailyIdx` was correctly skipped in the binding test setup. Initial diagnostic output was misleading because the first slot-reading attempt used wrong offsets (`>> 168` instead of `>> 208` for `ticketsFullyProcessed`).
