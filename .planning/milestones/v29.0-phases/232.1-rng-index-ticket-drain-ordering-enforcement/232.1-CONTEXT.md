<decisions>
## Implementation Decisions

### Structural Fix — Lazy Pre-Finalize at L242 Entry

- **D-01: Fix mechanism.** Insert a lazy slot-population gate at the very top of the L242-262 daily-drain block in `DegenerusGameAdvanceModule.advanceGame`. Concrete shape:
  ```solidity
  if (!ticketsFullyProcessed) {
      uint48 idx = uint48(_lrRead(LR_INDEX_SHIFT, LR_INDEX_MASK)) - 1;
      if (lootboxRngWordByIndex[idx] == 0) {
          uint256 cw = rngWordCurrent;
          if (cw == 0) revert NotTimeYet();      // VRF still pending — caller retries
          _finalizeLootboxRng(cw);                // VRF arrived; populate slot before drain reads it
      }
      // ... existing L243-262 drain code unchanged ...
  }
  ```
  This matches the user's articulated invariant verbatim: "skip this if the lootboxrngwordbyindex is not set (and revert if it needs to revert), but then run it before doing anything else if the rng came in and needs to be set."

- **D-02: Selection rationale.** Path enumeration showed two reachable read sites for `processTicketBatch:690` could see zero entropy:
  1. **Daily-flow primary bug** — end-of-day VRF requested, LR_INDEX bumped, callback writes `rngWordCurrent` only, lootbox slot deferred to inside rngGate's `_finalizeLootboxRng` at L1108, but L242-262 drain runs BEFORE rngGate. Race window writes entropy=0 into traits and early-returns at L249-258 before rngGate would have reverted RngNotReady.
  2. **Mid-day → cross-day edge** — mid-day request crossing a calendar boundary without VRF fulfillment; `LR_MID_DAY=1`, slot=0, falls through to L242-262 (same daily-branch code path).

  The lazy pre-finalize at L242 covers both. Mid-day fulfillment writes the lootbox slot atomically at the callback (L1632), so when `rngWordCurrent==0` for mid-day, the L242 guard correctly reverts because slot is also zero.

- **D-03: Backfill interaction accepted.** With this fix, the in-flight slot `[LR_INDEX-1]` gets the RAW VRF word (via `_finalizeLootboxRng(rngWordCurrent)` at L242) instead of the DERIVED `keccak(currentWord, idx)` that `_backfillOrphanedLootboxIndices` would have written first when invoked from inside rngGate during a multi-day stall. Verified harmless: in any reachable stall state, LR_INDEX is bumped exactly once (rngLockedFlag prevents further daily requests; LR_MID_DAY blocks further mid-day requests), so backfill's "loop" only ever touches the in-flight slot, and that slot now gets raw entropy from our pre-finalize. No ticket reads `lootboxRngWordByIndex[Y]` for `Y < LR_INDEX-1`, so any backfill no-ops on earlier orphaned slots have no game-correctness consequence.

### Entropy Sourcing — No Change at L690

- **D-04: `processTicketBatch:690` storage read kept.** The structural fix at L242 makes `lootboxRngWordByIndex[LR_INDEX-1] == 0` at L690 unreachable by construction in the daily-drain path. No signature change to `processTicketBatch`. No interface delta on `IDegenerusGameModules`. No defensive revert guard at L690 (matches `feedback_no_dead_guards.md` and SPEC.md §Constraints). Caller of `_runProcessTicketBatch` is responsible for ensuring the slot is populated; the callee trusts the call-site invariant.

### Game-Over Path — Audit-Only, Vacuously Safe

- **D-05: Game-over path verified isolated; no code change in GameOverModule.** Three independent isolation proofs:
  1. `_handleGameOverPath` at AdvanceModule.sol:178-181 returns early with `STAGE_GAMEOVER` emit before reaching L242 drain or any subsequent ticket-processing code.
  2. `handleGameOverDrain` (GameOverModule.sol:79-...) reads `rngWordByDay[day]` at L97-98 (defense-in-depth re-check), NOT `lootboxRngWordByIndex[]`. Different RNG storage namespace.
  3. Grep across `GameOverModule.sol` shows zero matches for `_swapAndFreeze` / `_swapTicketSlot` / `_runProcessTicketBatch` / `processTicketBatch` / `_raritySymbolBatch`. The terminal-decimator path (`runTerminalDecimatorJackpot`, `claimTerminalDecimatorJackpot`, audited in Phase 232 DCM-03) likewise does not reach lootbox-RNG-indexed reads.

  **Acceptance Criterion 3 satisfied vacuously by construction** — the forge test for game-over → zero `entropyWord == 0` would assert a property that is structurally impossible to violate. Plan should still produce a forge test that confirms the path-isolation claim (e.g., assert that simulating game-over while tickets are queued does not produce any `_raritySymbolBatch` invocation in event logs).

### Mid-Day Guard L190-194 — Leave Alone

- **D-06: L190-194 unchanged.** Existing same-day mid-day guard `if (LR_MID_DAY != 0 && lootboxRngWordByIndex[LR_INDEX-1] == 0) revert NotTimeYet()` is locally correct because mid-day fulfillment writes the slot atomically at the callback (L1632). No refactor to a shared `_ensureLootboxRngReady()` helper; minimum-diff principle. Two slightly different guard shapes coexist (mid-day and daily), each correct for its branch.

### Out of 232.1 Scope (Deferred Decisions)

- **D-07: `rngWordCurrent` removal deferred.** The user observed that `rngWordCurrent` could potentially be eliminated by having `rawFulfillRandomWords` always write directly to `lootboxRngWordByIndex[]` (mirroring the mid-day path) and routing daily-word reads through `rngWordByDay[day]`. Trace revealed `rngWordCurrent` has TWO roles:
  - **Role 1 (in-flight buffer):** between callback and rngGate processing — would be eliminated by the proposal.
  - **Role 2 (post-nudge persistent storage):** `_applyDailyRng` at L1711 overwrites with `finalWord = rawWord + nudges`; consumed cross-tx by `DegenerusGame.sol:850` (jackpot fallback), `DegenerusGame.sol:2175` (public view), and referenced by `DegenerusGameJackpotModule.sol:575-578` for multi-tx jackpot RNG threading.

  Role 2 is load-bearing and removing it requires multi-module changes beyond `DegenerusGameAdvanceModule.sol`. Out of scope per SPEC.md §In Scope. Flagged as a candidate for a future v30+ milestone or Phase 232.2.

## Existing Code Insights

### Reusable Assets
- **`_finalizeLootboxRng(uint256 rngWord)`** at AdvanceModule.sol:1127-1132 — the exact function to call from the new L242 pre-finalize block. Already idempotent via `if (lootboxRngWordByIndex[index] != 0) return;` short-circuit, so a redundant call from rngGate later in the same tx is a no-op.
- **`NotTimeYet`** custom revert — already used at L194 (mid-day guard) and L219 (mid-day fall-through). Reuse for the new L242 revert (don't introduce a new error type; symmetry helps tooling and indexers).
- **`rngWordCurrent`** state variable (storage at `DegenerusGameStorage.sol:368`) — read directly via inherited storage; no accessor needed.
- **`_lrRead(LR_INDEX_SHIFT, LR_INDEX_MASK)`** packed accessor — same pattern used at L1128, L191, L243; one cold SLOAD shared across the function via Solidity caching.

### Established Patterns
- **Mid-day vs daily VRF flows are intentionally asymmetric** — mid-day writes lootbox slot atomically at fulfillment (callback); daily defers slot-write to inside rngGate. This fix narrows the asymmetry in effect (daily slot can be lazy-finalized at drain time too) without unifying the underlying state machine. The two flows remain code-distinct.
- **Idempotent slot writes** — `_finalizeLootboxRng` and `_backfillOrphanedLootboxIndices` both no-op when slot already non-zero. Calling either redundantly is safe.
- **Stage signal pattern** — advanceGame uses STAGE_* constants (`STAGE_TICKETS_WORKING`, `STAGE_RNG_REQUESTED`, `STAGE_GAMEOVER`, etc.) for caller observability via the `Advance` event. The `revert NotTimeYet()` path produces no Advance event (consistent with existing L194 / L219 reverts) — caller observes the revert string.
- **Caller-retry liveness** — escalating bounty multiplier at L222-239 incentivizes callers to retry stalled drain paths. The new L242 revert path does not pay bounty (consistent with L194 / L219 behavior — no progress, no payment).

### Integration Points
- **`processTicketBatch:690`** — the consumer the fix protects. No code change here; the call-site invariant ("entropy is non-zero when reached") is now guaranteed by the L242 gate plus the existing L194 mid-day gate.
- **`rawFulfillRandomWords`** — unchanged. Daily branch still writes `rngWordCurrent`; mid-day branch still writes lootbox slot atomically. The L242 fix bridges the daily-flow gap without touching the callback.
- **`_handleGameOverPath` short-circuit at advanceGame:178-181** — guarantees game-over path skips L242 drain entirely. No change needed; documented as path-isolation proof in D-05.
- **Forge test infrastructure** — `test/{fuzz,invariant,unit,integration,edge,halmos}` directories all exist. Planner picks the appropriate location(s); no new framework setup required.
