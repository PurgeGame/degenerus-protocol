### Salt-Space Isolation (Bonus vs Main)

`_rollWinningTraits` at lines 1865-1875:

```solidity
function _rollWinningTraits(
    uint256 randWord,
    bool isBonus
) private view returns (uint32 packed) {
    uint256 r = isBonus
        ? uint256(keccak256(abi.encodePacked(randWord, BONUS_TRAITS_TAG)))
        : randWord;
    uint8[4] memory traits = JackpotBucketLib.getRandomTraits(r);
    _applyHeroOverride(traits, r);
    packed = JackpotBucketLib.packWinningTraits(traits);
}
```

With `BONUS_TRAITS_TAG = keccak256("BONUS_TRAITS")` at line 171. Two observations:

1. **Disjoint input spaces.** For any fixed `randWord`, `r_bonus = keccak256(abi.encodePacked(randWord, BONUS_TRAITS_TAG))` and `r_main = randWord` are equal only if an attacker found a preimage of `randWord` through a single-block keccak — cost 2^256. Treating keccak256 as a random oracle, `r_bonus` and `r_main` are independent samples of a uniform 256-bit distribution.

2. **Trait derivation is injective on the low 24 bits.** `JackpotBucketLib.getRandomTraits` (lines 281-286 of the library) reads the low 24 bits of `r` into four 6-bit quadrants. Two different `r` values can produce the same 4-trait output only if they happen to agree on the low 24 bits (2^-24 per pair) and the deity override (`_applyHeroOverride`, which reads the full `r`) coincides. Across independent keccak samples this probability is negligible.

Salt-space isolation is therefore sound: the `bonus=true` branch and the `bonus=false` branch cannot produce the same 4-trait output for the same `randWord` modulo 2^-24 × 2^-extra collision events well below any C4A severity threshold.

### Queue-Level Fix (`lvl+1`)

**Pre-fix (`20a951df^`, `contracts/modules/DegenerusGameJackpotModule.sol`, lines ~670-735 of parent commit):**

```solidity
uint256 entropy = rngWord;
uint24 baseLevel = lvl;
uint256[5] memory levelPrices;
for (uint8 l; l < 5; ) {
    levelPrices[l] = PriceLookupLib.priceForLevel(baseLevel + uint24(l));
    unchecked { ++l; }
}

// Process each winner (uniform trait selection; no weighting by ticket counts)
for (uint256 i; i < maxWinners; ) {
    entropy = EntropyLib.entropyStep(entropy);
    uint8 traitId = uint8(entropy);
    (
        address[] memory winners,
        uint256[] memory ticketIndexes
    ) = _randTraitTicket(
            traitBurnTicket[lvl],
            entropy,
            traitId,
            1,
            uint8(i)
        );
    address winner = winners.length != 0 ? winners[0] : address(0);
    if (winner != address(0)) {
        // Roll for level offset 0-4 (20% chance each)
        entropy = EntropyLib.entropyStep(entropy);
        uint24 levelOffset = uint24(entropy % 5);           // pre-fix spread
        uint256 ticketPrice = levelPrices[levelOffset];
        if (ticketPrice != 0) {
            uint32 ticketCount = uint32(perWinnerEth / ticketPrice);
            if (ticketCount != 0) {
                _queueTickets(
                    winner,
                    baseLevel + levelOffset,               // pre-fix queue key
                    ticketCount,
                    true
                );
                emit JackpotTicketWin(
                    winner,
                    baseLevel + levelOffset,               // pre-fix emit key
                    traitId,
                    ticketCount,
                    baseLevel,
                    ticketIndexes[0]
                );
            }
        }
    }
    unchecked { ++i; }
}
```

Pre-fix winners were queued at `baseLevel + levelOffset` with `levelOffset ∈ [0,4]` — winners were spread across 5 consecutive levels: `baseLevel`, `baseLevel+1`, `baseLevel+2`, `baseLevel+3`, `baseLevel+4`.

**Post-fix (HEAD, lines 691-698):**

```solidity
_queueTickets(winner, lvl, ticketCount, true);
emit JackpotTicketWin(
    winner,
    lvl,
    traitId,
    ticketCount,
    lvl,
    ticketIndexes[i]
);
```

Post-fix queues all winners at the single argument `lvl` — the resolution level.

So the argument `lvl` to `_runEarlyBirdLootboxJackpot` is `outer game level + 1` — the resolution level (the level that tickets issued at this boundary are destined to resolve at).

**Consistency with DCM-01 at §1.8.** `BurnieCoin.decimatorBurn` at HEAD writes burns using `uint24 lvl = degenerusGame.level() + 1;` — keying by resolution level. The earlybird `lvl+1` pattern matches this convention: both write into storage at the level where the jackpot/burn resolves, not the level currently live at the time of the call. The §1.2 delta-map row assertion ("All winners queued at `lvl` (= outer caller level + 1) instead of spreading across `baseLevel..baseLevel+4`") is fully verified.
