```solidity
/// @notice Thrown by vote() when the slot fails a state-based pre-condition. Reason codes:
///         0 = REJECT_EMPTY_SLOT     — currentSlate[slot] == address(0)
///         1 = REJECT_ALREADY_VOTED  — hasVoted[level][voter][slot] == true
///         2 = REJECT_ZERO_WEIGHT    — sdgnrs.balanceOf(voter) / 1e18 == 0
error VoteRejected(uint8 reason);

/// @notice Thrown by pickCharity() when the level argument fails a state-based pre-condition. Reason codes:
///         0 = REJECT_LEVEL_NOT_ACTIVE       — level != currentLevel
///         1 = REJECT_LEVEL_ALREADY_RESOLVED — levelResolved[level] == true
error PickCharityRejected(uint8 reason);
```

Reason-code constants (placed in CONSTANTS block per CONTEXT.md `<specifics>`):

```solidity
/// @notice VoteRejected reason: slot is empty in current slate
uint8 private constant REJECT_EMPTY_SLOT = 0;

/// @notice VoteRejected reason: voter has already voted on this (level, slot)
uint8 private constant REJECT_ALREADY_VOTED = 1;

/// @notice VoteRejected reason: voter has zero whole-token sDGNRS balance
uint8 private constant REJECT_ZERO_WEIGHT = 2;

/// @notice PickCharityRejected reason: level argument does not match currentLevel
uint8 private constant REJECT_LEVEL_NOT_ACTIVE = 0;

/// @notice PickCharityRejected reason: level has already been resolved
uint8 private constant REJECT_LEVEL_ALREADY_RESOLVED = 1;
```

REASON CODE SHARING NOTE: The two reason namespaces deliberately do NOT share — `REJECT_EMPTY_SLOT == 0` and `REJECT_LEVEL_NOT_ACTIVE == 0` are independent semantically; they happen to share the integer value because each error type starts its enumeration at 0. Tests assert via `revertedWithCustomError(contract, "VoteRejected").withArgs(0)` (or `"PickCharityRejected"`), so the type discriminator carries the namespace. Per CONTEXT.md — "the failure domains are distinct (vote vs. resolve) and shared codes would be confusing".
