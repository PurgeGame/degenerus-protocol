**Document structure:**

1. **Overview** -- Deity passes are premium NFT-like positions (max 32, one per symbol). Owners can issue boons to other players. Pass ownership boosts activity score. Pricing escalates quadratically. Deity pass owners also receive virtual jackpot entries.

2. **Deity Pass Pricing Curve (DEIT-01)** -- Quadratic escalation:
   - Base price: DEITY_PASS_BASE = 24 ETH

   - Maximum 32 passes (symbolId 0-31)

3. **Deity Pass Transfer (DEIT-01 continued)** -- Transfer cost and side effects:
   - Burns 5 ETH worth of BURNIE from sender: burnAmount = (5 ETH * 1000 ether) / price
   - Nukes sender's mint stats (levelCount, levelStreak, lastLevel, mintStreakLastCompleted all zeroed)
   - Resets sender's quest streak via resetQuestStreak
   - Not available at level 0

5. **Boon Expiry Rules (DEIT-02 continued)** -- Two source types with different lifetimes:
   - Deity-sourced: expire at end of current day (deityDay != currentDay triggers clear)
   - Lootbox-sourced: expire after N days from stamp day
   - Per-category expiry days: Coinflip 2d, Lootbox boost 2d, Purchase boost 4d, Deity pass 4d (lootbox) / current-day (deity), Activity 2d, Whale/lazy pass 4d

   - Include exact Solidity from BoonModule for expiry checks

   - 3 slots per deity per day (DEITY_DAILY_BOON_COUNT = 3)

   - Cannot issue to self (deity == recipient reverts)
   - One boon per recipient per day (deityBoonRecipientDay tracking)
   - Requires valid RNG word for the day
   - Include exact Solidity

7. **Deity Pass Activity Score Impact (DEIT-03)** -- Score modification:
   - DEITY_PASS_ACTIVITY_BONUS_BPS = 8000 (+80% activity score)
   - Replaces streak/count with floor values: streak=50 (max 50%), count=25 (max 25%)
   - Quest streak and affiliate bonus still calculated normally on top
   - Maximum with deity: 50% + 25% + 100% + 50% + 80% = 305% (vs 265% without)
   - DGNRS and VAULT get deityPassCount=1 in constructor (virtual deity-equivalent boost)
   - Cross-reference audit/v1.1-endgame-and-activity.md for base activity score formula

8. **Deity Pass Jackpot Virtual Entries (DEIT-03)** -- Jackpot entry multiplier for deity pass owners:
   - When a jackpot bucket draw occurs, the deity who owns the symbol matching the trait's fullSymId receives virtual entries
   - Formula: `virtualCount = floor(bucketTicketCount / 50)` (i.e., floor 2% of bucket tickets)
   - Minimum 2 virtual entries if a deity exists for that symbol (if virtualCount < 2, set to 2)
   - Virtual entries are appended to the effective ticket count: `effectiveLen = len + virtualCount`
   - Winner selection: if random index falls in `[0, len)` range, actual ticket holder wins; if index falls in `[len, effectiveLen)` range, the deity address wins
   - Symbol mapping: `fullSymId = (trait >> 6) * 8 + (trait & 0x07)`, deity looked up via `deityBySymbol[fullSymId]`
   - Only applies when fullSymId < 32 and a deity exists for that symbol (deityBySymbol[fullSymId] != address(0))
   - Include exact Solidity from DegenerusGameJackpotModule.sol lines 2274-2306
   - Worked example: "Bucket has 200 tickets, deity exists for this symbol: virtualCount = floor(200/50) = 4, effectiveLen = 204, deity win probability = 4/204 = 1.96%"
   - Worked example: "Bucket has 50 tickets: virtualCount = floor(50/50) = 1, but minimum is 2, effectiveLen = 52, deity win probability = 2/52 = 3.85%"

   > **PITFALL: Virtual entries scale with bucket size.** The floor(2%) formula means deity advantage is roughly constant at ~2% for large buckets, but the minimum-2 floor creates a disproportionate advantage in small buckets (e.g., 10 tickets = 2/12 = 16.7% deity win rate).
