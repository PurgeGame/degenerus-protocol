**Section 5: Freeze/Unfreeze Mechanics** -- POOL-03
5a. Freeze activation:
- Function: `_swapAndFreeze(purchaseLevel)` in DegenerusGameStorage
- Trigger: advanceGame requests daily RNG (day boundary crossed)
- Action: sets prizePoolFrozen=true, zeros prizePoolPendingPacked (first freeze only)
- If already frozen (jackpot multi-day): pending accumulators keep growing

5b. Purchase behavior during freeze:
- All purchase functions check prizePoolFrozen
- Frozen: writes to prizePoolPendingPacked
- Unfrozen: writes to prizePoolsPacked
- Include the exact conditional code pattern

5c. Unfreeze triggers:
- After daily RNG resolves (purchase phase, non-jackpot)
- After phase transition completes (jackpot -> purchase)
- After jackpot phase ends (day 5 complete)
- Function: `_unfreezePool()` -- merges pending into live, zeros pending, sets frozen=false

5d. During 5-day jackpot phase:
- Pool stays frozen entire 5 days
- Pending accumulators grow across all days
- Unfreeze only when jackpot phase ends

5e. Key detail: freeze applies to packed storage only. currentPrizePool (separate uint256) is NOT frozen.

**Section 6: Purchase Target and Level Advancement** -- POOL-04
6a. Ratchet system:
- Target for level N+1: nextPrizePool must reach levelPrizePool[N]
- levelPrizePool[0] = BOOTSTRAP_PRIZE_POOL = 50 ether (set in constructor)
- levelPrizePool[N] = snapshot of nextPrizePool at level N transition
- Special: x00 levels set target to futurePrizePool / 3

6b. Target check:
- In advanceGame: `if (_getNextPrizePool() >= levelPrizePool[purchaseLevel - 1])` -> lastPurchaseDay = true
- Compressed jackpot flag: `(day - purchaseStartDay <= 2)` -- level reached target in 2 days or fewer

6c. Time-based future take (pre-consolidation):
- Runs when lastPurchaseDay triggers, before jackpot phase
- Skims portion of nextPrizePool into futurePrizePool
- Ensures future pool stays funded even during fast level advancement

**Section 7: Distress Mode Pool Behavior** -- POOL-01 (supplementary)
- Condition: within DISTRESS_MODE_HOURS(6) of liveness guard expiry
- Level 0: 365-day deploy timeout. Level > 0: 120-day liveness guard
- Effect: lootbox ETH goes 100% to nextPool (helps meet purchase target)
- Document the exact _isDistressMode() conditional
