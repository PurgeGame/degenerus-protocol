### _processAutoRebuy (JM:959-999)
```solidity
function _processAutoRebuy(
    address player,
    uint256 newAmount,
    uint256 entropy,
    AutoRebuyState memory state
) private returns (uint256 claimableDelta)
```
- Calls `_calcAutoRebuy` to produce `AutoRebuyCalc` with fields: `hasTickets`, `toFuture`, `targetLevel`, `ticketCount`, `ethSpent`, `reserved`
- If `!calc.hasTickets`: calls `_creditClaimable`, returns — NO pool writes
- If `calc.hasTickets && calc.toFuture`: calls `_setFuturePrizePool(_getFuturePrizePool() + calc.ethSpent)` — 1 read-modify-write on prizePoolsPacked
- If `calc.hasTickets && !calc.toFuture`: calls `_setNextPrizePool(_getNextPrizePool() + calc.ethSpent)` — 1 read-modify-write on prizePoolsPacked
- Each `_setFuturePrizePool` or `_setNextPrizePool` call reads the full packed slot then writes the full packed slot (both components together)

### _setFuturePrizePool / _setNextPrizePool (Storage.sol:740-755)
```solidity
function _setNextPrizePool(uint256 val) internal {
    (, uint128 future) = _getPrizePools();  // SLOAD prizePoolsPacked
    _setPrizePools(uint128(val), future);   // SSTORE prizePoolsPacked
}
function _setFuturePrizePool(uint256 val) internal {
    (uint128 next, ) = _getPrizePools();   // SLOAD prizePoolsPacked
    _setPrizePools(next, uint128(val));    // SSTORE prizePoolsPacked
}
function _setPrizePools(uint128 next, uint128 future) internal {
    prizePoolsPacked = uint256(future) << 128 | uint256(next);  // SSTORE
}
```
Both helpers perform a full read-modify-write on slot `prizePoolsPacked`. After the first write in a tx, subsequent writes to the same slot cost the warm SSTORE rate (100 gas) not 5,000.
