    Act as an adversarial C4A warden specializing in money correctness and economic exploits. You have ZERO knowledge of any prior audit — approach the codebase with completely fresh eyes. Read the C4A README and KNOWN-ISSUES first.

    Systematically audit every money-related attack surface:

    **1. ETH Flow Tracing**
    For EVERY function that receives ETH (payable functions, receive/fallback):
    - Trace where the ETH goes (vault deposit, prize pool, affiliate cut, protocol fee)
    - Verify the sum of all outflows equals the inflow (no ETH stranded or leaked)
    - Check for reentrancy on ETH transfers (external calls before state updates)
    For EVERY function that sends ETH (payouts, withdrawals, refunds):
    - Verify the sender has sufficient balance
    - Verify access control (only entitled recipients can claim)
    - Check for double-claim or replay attacks

    **2. BPS Rounding Chain Analysis**
    For every basis-point split (affiliate %, protocol fee %, prize pool %):
    - Trace the full chain of BPS calculations
    - Verify rounding direction (should favor protocol solvency)
    - Calculate worst-case cumulative rounding loss over maximum game duration
    - Check if rounding can be exploited via many small transactions (dust attacks)

    **3. Token Accounting**
    For each token (DGNRS, sDGNRS, BURNIE, GNRUS, wXRP, VaultShare):
    - Trace mint/burn/transfer paths
    - Verify totalSupply matches sum of all balances at all times
    - Check for unauthorized mint paths
    - Verify burn paths cannot extract more value than burned tokens represent

    **4. Cross-Token Interactions**
    - DGNRS <-> sDGNRS wrap/unwrap: verify exchange rate integrity, check for sandwich attacks
    - BURNIE burn-to-play in BurnieCoinflip: verify wager accounting, check for manipulation
    - GNRUS burn redemption: verify proportional ETH/stETH calculation
    - wXRP: verify deposit/withdraw accounting
    - VaultShare: verify share-to-asset ratio

    **5. Prize Pool and Jackpot Accounting**
    - Trace prize pool accumulation (mint fees -> pool)
    - Trace jackpot distribution (daily, weekly, special)
    - Verify winner selection cannot be manipulated to steal funds
    - Check for prize pool draining attacks

    **6. Affiliate and Fee Extraction**
    - Trace affiliate referral fee flows
    - Verify affiliate cannot claim more than entitled
    - Check for affiliate fee manipulation via self-referral

    **7. Cross-Domain Findings**
    Report any RNG, gas, admin, or composition issues found during money analysis.

    - Executive Summary

    - ETH Flow Map (table: entry point -> destinations -> BPS split)
    - Token Accounting Summary (table: token -> mint paths -> burn paths -> invariants)
    - Findings (each with: title, severity, description, affected contracts, PoC with value calculations)
    - SAFE Proofs (each with: attack surface, arithmetic trace, conclusion)
    - Cross-Domain Findings (if any)
    - Attack Surface Inventory (every money surface checked with disposition)
