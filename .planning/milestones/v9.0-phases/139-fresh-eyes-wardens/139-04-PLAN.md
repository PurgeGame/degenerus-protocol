    Systematically audit every admin-related attack surface:

    **1. Admin Function Inventory**
    For EVERY contract, find all functions restricted by access control (onlyOwner, onlyAdmin, require(msg.sender == admin), etc.):
    - List the function, its power level (what damage it can do), and who can call it
    - Classify: configuration (low risk), fund-moving (high risk), upgrade/migration (critical risk)
    - Check if any admin function can be called during active game state to grief players

    **2. Bootstrap vs Post-Distribution Analysis**
    - Identify the DGNRS vesting schedule (50B initial + 5B/level, fully vested at level 30)
    - During bootstrap: admin holds majority DGNRS, can unwrap to sDGNRS for governance votes
    - Post-distribution: DGNRS widely distributed, governance reflects community
    - For EACH admin function: assess impact differently in bootstrap vs post-distribution
    - Check if admin can prevent transition from bootstrap to post-distribution

    **3. Chainlink Death Clock Governance Paths**
    Two governance paths are gated behind Chainlink service death:
    - VRF coordinator swap: what happens if Chainlink VRF stops responding?
    - Price feed swap: what happens if Chainlink price feed goes stale?
    - For each: trace the governance mechanism, identify who can trigger it, what power it grants
    - Check for time-lock manipulation, proposal stuffing, or flash-loan vote attacks

    **4. Ownership and Privilege Escalation**
    - Check for ownership transfer functions (transferOwnership, renounceOwnership)
    - Check for proxy/upgrade patterns that could change contract logic
    - Check for backdoor admin functions (hidden modifiers, assembly-level checks)
    - Verify no path allows a non-admin to gain admin privileges

    **5. Admin Griefing During Gameplay**
    - Can admin pause the game and strand player funds?
    - Can admin change game parameters mid-round to disadvantage players?
    - Can admin front-run player transactions with parameter changes?
    - What are the time-lock protections (if any)?
