## v59.0 Requirements

### SALV — F-01 salvage uint32-truncation (CRITICAL)
> `sellFarFutureTickets` values the full `uint256 quantities[i]` in the quote (`DegenerusGameMintStreakUtils.sol:173-181`) but the execution debit truncates to `uint32(quantities[i]) * 4` (`DegenerusGameMintModule.sol:1013-1015`); the holdings guard `_removeFarFutureTickets` (`:1059`) reads a `uint32 owed` so it constrains only the truncated quantity. Exploit: `n = 2^32 + k` → debit `k*4` entries (passes), quote values 4.29B+ phantom tickets → drains sDGNRS claimable + BURNIE reserves. RNG-clean / solvency-positive fix.
- [ ] **SALV-01** (IMPL): Widen the existing per-line zero-check in the quote loop so the valued quantity equals the debited `uint32` quantity — `if (n == 0) revert E();` → `if (n == 0 || n > type(uint32).max) revert E();` (house custom-error idiom, no new line, beside the existing `if (d < 6 || d > 100) revert E();`). The quote runs before the debit → every reachable swap now values exactly the entries it removes; the `n = 2^32 + k` exploit reverts.

### AFAFF — F-02 afking `affiliateBase` unit slip (HIGH)
> `_deliverAfkingBuy` stores the 18-decimal `_ethToBurnie(...)` output straight into the whole-BURNIE `uint32 affiliateBase` field (`GameAfkingModule.sol:879`) MISSING `/ 1 ether` — the clamp saturates to 100M on the first buy, `DegenerusAffiliate.claim` re-scales `aShare * 1 ether` (`:693`) → ~100M whole BURNIE minted to the affiliate tree vs the legitimate ~70 (a `recordAmount=0` unbacked `creditFlip`). BURNIE-supply, not ETH.
- [ ] **AFAFF-01** (IMPL): Add the missing divide at `GameAfkingModule.sol:879` — `base = ((_ethToBurnie(amount, mp) * 7) / 100) / 1 ether;` — mirroring the two sibling accruals in the same block (`bonusWhole = bonusBase / 1 ether` `:826`, `QUEST_SLOT0_REWARD / 1 ether` `:887`). `affiliateBase` is then stored in whole-BURNIE units: no clamp saturation, no unbacked emission.

### SOLV — F-03 + F-04 whale-pass remainder solvency-identity corrections (MEDIUM)
> Two `claimablePool` vs backing mismatches that live inside the jackpot/decimator whale-pass *remainder* sub-paths (the depth the 366 ETHFLOW edge-walk did not reach). Both restore the documented identity `claimablePool == Σ claimableWinnings + Σ afkingFunding`.
