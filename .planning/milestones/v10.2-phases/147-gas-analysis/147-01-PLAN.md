    **Section 3 — Adversarial Ticket Distribution Analysis (CAP-03):**
    Model the worst-case scenario: attacker buys hundreds of 1-ticket purchases from unique addresses.

    Adversarial factors that maximize per-ticket gas:
    1. Every `ticketsOwedPacked[rk][player]` SLOAD is cold (2100 gas) — unique addresses mean no warm cache
    2. Every player has owed=1, so baseOv=4 (the `processed == 0 && owed <= 2` branch)
    3. Every `_raritySymbolBatch` call is for a single ticket — full memory setup (8192 gas) amortized over just 1 trait
    4. Each trait write is to a new storage slot (22100 gas for SSTORE to new slot)
    5. `_finalizeTicketEntry` writes new packed value (5000 gas for non-zero to non-zero, or 22100 for zero-to-nonzero depending on prior state)
    6. Queue array SLOADs: each `queue[idx]` is a cold SLOAD (2100 gas) for the first access

    - Actually: _raritySymbolBatch allocates 256*32*2 = 16384 bytes of memory arrays. But Solidity memory doesn't shrink, so subsequent calls to _raritySymbolBatch don't pay expansion cost — they reuse the high watermark. Verify this by checking if the function is called in a loop.
    - LCG computation for 1 trait: ~50 gas

    - TraitsGenerated event: ~631 gas

    Realistic adversarial per-entry: ~30,000-35,000 gas (some warm from same-trait hits across entries)

    **Section 4 — Derive Maximum Cap Value (CAP-04):**

    Gas budget equation:
    ```
    total_gas = overhead_fixed + (entries * per_entry_gas) <= 14,000,000
    ```

    ```
    available = 14,000,000 - 100,000 = 13,900,000
    cap = floor(available / worst_case_per_entry_gas)
    ```
