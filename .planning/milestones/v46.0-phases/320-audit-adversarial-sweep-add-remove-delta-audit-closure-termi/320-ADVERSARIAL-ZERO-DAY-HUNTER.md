### FINDING_CANDIDATE — full attack narrative: H-CANCEL-SWAP-MISS

**Title:** External `setDailyQuantity(0)` cancel swap-pops the tail subscriber behind the persistent mid-day cursor, causing a one-day auto-buy miss — violating the locked SUB-07 "tombstone-on-cancel moves nothing" requirement.

**Novelty / why prior review missed it.** The IMPL deliberately reuses the in-sweep swap-pop helper (`_removeFromSet`) for the external cancel path, contradicting the SPEC's explicit "moves nothing" lock. The one test that exercises cancel-then-sweep (`testCancelSwapPopOccupantStillProcessed`) runs the cancel at cursor=0 (before any sweep that day), where the swap-pop relocates the tail BACKWARD into a slot the subsequent full sweep still covers — so the relocation is benign in that fixture. The dangerous case requires the cursor to be ALREADY ADVANCED mid-day (a second sweep tx in the same day) when the cancel fires — a state the test suite never constructs. 318-04-SUMMARY:108 explicitly routed "deeper griefing review" to this Phase 320 audit.

**Attack / bug trace.**
- Set state mid-day: `_subscribers = [A, B, C, D]` (indices 0..3). A first `sweep(2)` processes A, B → `_sweepCursor = 2`, all four still active and well-funded.
- Subscriber A (index 0, BEHIND cursor 2, already swept today) calls `setDailyQuantity(0)`. `_removeFromSet(A)`: `idx=0`, `last=3`, `mover = D`. D is written into index 0; tail popped. Array becomes `[D, B, C]`, `_sweepCursor` still 2.
- A later `sweep(...)` the same day resumes at `cursor = 2`, reads `_subscribers[2] = C`, processes C, then `cursor = 3 == length` → loop exits.
- **D — active, funded, unprocessed today — was relocated from index 3 to index 0 (behind cursor 2) and is NEVER reached this day.** D's `lastSweptDay` is unchanged (the `:767` stamp only fires on success), so the idempotency backstop offers no help (it prevents double-buys, not misses). D misses one day's scheduled subscription buy.
- Next day: `_sweepDay != today` at `:580` → cursor resets to 0 → D (now at index 0) is processed normally. Self-healing.

**Who is the actor.** Any subscriber `A` who (i) is positioned behind the current persistent cursor and (ii) cancels at a moment when the unprocessed tail element is a target. `A` cannot freely choose the victim (the relocated element is whatever is currently the array tail) and `A` forfeits its own subscription (and any unpaid window) by cancelling. A determined griefer with a behind-cursor slot can relocate "the current tail" once per cancel, but cannot reliably aim it at a pre-chosen address without controlling insertion order + prior swap-pops + the live cursor — a fragile, transient alignment.

**Impact (C4A lens).** LOW. One subscriber misses AT MOST one day's auto-buy (the scheduled ticket/lootbox spend they configured). No ETH or BURNIE is lost — the victim's `_poolOf` and `claimableWinnings` are untouched and fully `withdraw`-able; the missed buy simply does not happen that day. No double-buy (day-stamp backstop). Self-healing on the next-day cursor reset. The miss sits inside the system's accepted liveness model, which is bounty-incentivized reachability rather than a hard every-entry-every-day guarantee — a subscriber can already miss a day if cranker coverage is thin. The realistic damage is bounded by one day of one victim's self-configured spend (which would simply roll to the next day), with the attacker forfeiting their own paid sub to do it.

**Recommendation (for orchestrator adjudication, NOT applied — SOURCE-TREE FROZEN).** Two candidate closes, both for the §5 RE-PASS envelope only if the orchestrator/user elevates:
1. **Honor the locked SUB-07 "no move" design** — make the external cancel a logical tombstone (`dailyQuantity = 0` + clear the windowPaid-reclaim, but do NOT call `_removeFromSet`); let the in-sweep reclaim swap-pop the tombstone when the cursor reaches it (the SPEC's stated model). This eliminates the relocate-behind-cursor primitive entirely. Trade-off: tombstone slots linger until the next sweep reaches them (the SPEC accepted this; the IMPL chose swap-pop to avoid dead-slot buildup — see `testNoDeadSlotBuildupAcrossCancels`).
2. **Defense-in-depth (does NOT pass the strict structural arm, lower preference)** — on an external cancel, if the canceller's index `< _sweepCursor` (when `_sweepDay == today`), decrement `_sweepCursor` so the relocated mover is re-covered. Adds a conditional cursor write to the cancel path.
Per `feedback_design_intent_before_deletion.md` the original SUB-07 intent ("moves nothing, to avoid the one miss case") is precisely what close #1 restores; close #1 is the structural-elimination preference. This is a Tier-1 single-skill FINDING_CANDIDATE → AskUserQuestion user-pause per the charge §5 routing; the user adjudicates {elevate-to-fix / SAFE_BY_DESIGN-accept-the-one-day-miss / NEGATIVE-VERIFIED-on-reconsideration}.
