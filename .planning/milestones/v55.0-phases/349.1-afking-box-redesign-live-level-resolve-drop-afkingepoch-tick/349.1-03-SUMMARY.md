# Phase 349.1 Plan 03: Box Open (Live-Level Core) Summary

**`resolveAfkingBox` collapses to the LIVE-level twin of `resolveLootboxDirect` (live `level+1` roll, sole-live EV-cap RMW) with exactly two deviations — a caller-passed `rngWordByDay[stampDay]` and the frozen stamped `day` in the seed — and the GameAfkingModule open-leg is fully day-keyed (`lastOpenedDay` marker, `rngWordByDay[day]` word). All `_afkingEpoch` / index / baseLevel plumbing is gone.**

## Accomplishments
- **LootboxModule.resolveAfkingBox:** new signature `(address player, uint256 amount, uint32 day, uint256 rngWord, uint16 activityScore)` — dropped `index` + `baseLevel`, added `rngWord`. Body mirrors `resolveLootboxDirect`: `currentLevel = level + 1` (LIVE), `targetLevel = _rollTargetLevel(currentLevel, seed)`, single `_applyEvMultiplierWithCap`, `_resolveLootboxCommon(..., true, true, 0, 0)`. Seed `keccak256(abi.encode(rngWord, player, day, amount))` uses the PASSED frozen `day`. Removed the `lootboxRngWordByIndex[index]` read + the `RngNotReady` guard (caller pre-gates). Doc-comment rewritten to the live-level + two-deviation model.
- **GameAfkingModule._afkingBoxReady:** `sub.lastOpenedDay < sub.lastAutoBoughtDay && rngWordByDay[sub.lastAutoBoughtDay] != 0`.
- **GameAfkingModule._openAfkingBox:** effects-first `sub.lastOpenedDay = day`, then delegatecall `resolveAfkingBox(player, uint256(sub.amount), day, rngWordByDay[day], uint16(sub.scorePlus1) - 1)`. Kept `GAME_LOOTBOX_MODULE` target + `_revertDelegate(data)` tail.
- Fixed the stale `lastOpenedIndex` mention in the `_autoOpen` doc-comment → `lastOpenedDay`.
