<objective>
Land the STORAGE-layer foundation of the 349.1 AfKing box redesign: rename the
Sub open-marker from a lootbox-index (`lastOpenedIndex`, uint48) to a process-day
(`lastOpenedDay`, uint32), and DELETE the entire `_afkingEpoch` per-day epoch
machinery (the map + its pack/unpack helpers + the shift constant). Under the
redesign the box no longer stores a frozen `(index, baseLevelPlus1)` epoch — it
resolves at the LIVE level and sources its RNG word from `rngWordByDay[day]` (an
existing map) — so the epoch storage is dead.

Purpose: this is the producer every consumer keys on. The no-orphan guard
(`sub.lastOpenedDay < sub.lastAutoBoughtDay`, plan 02) and the open marker
(plan 03) both compare these two day fields; the epoch deletion lets the open-leg
(plan 03) stop unpacking a stored epoch. Storage must change FIRST so the later
plans edit against the final layout.

Output: a revised `DegenerusGameStorage.sol` — Sub packs into one slot with a
`lastOpenedDay` marker, and the `_afkingEpoch` block is gone.

NOTE — this is a REVISION of a HELD diff, not greenfield. The whole Phase-349
batched contract diff is already applied to the working tree (contracts-only
`forge build` clean). The line anchors below are guidance; `read_first` the actual
file and locate the identifiers by name. Pre-launch redeploy-fresh — a storage
layout break is fine, no migration.
