**Step 2: Capture the canonical layout from DegenerusGameStorage**

```
forge inspect DegenerusGameStorage storage-layout --json > /tmp/layout-canonical.json
```

**Step 3: Compare every inheriting contract's layout against canonical**

Run forge inspect for EVERY contract that inherits DegenerusGameStorage. Compare the storage-layout output — slot numbers, offsets, and types must be IDENTICAL for every shared variable.

Contracts to inspect (all modules + main contract):
```
for contract in DegenerusGame \
    DegenerusGameAdvanceModule \
    DegenerusGameJackpotModule \
    DegenerusGameMintModule \
    DegenerusGameEndgameModule \
    DegenerusGameLootboxModule \
    DegenerusGameWhaleModule \
    DegenerusGameBoonModule \
    DegenerusGameDecimatorModule \
    DegenerusGameDegeneretteModule \
    DegenerusGameGameOverModule; do
  echo "=== $contract ==="
  forge inspect "$contract" storage-layout --json > "/tmp/layout-${contract}.json" 2>/dev/null
  if [ $? -ne 0 ]; then
    echo "WARN: $contract not found or failed to inspect"
  fi
done
```

**Step 4: Verify slot identity**

For each module, compare the first N entries (shared storage variables) against DegenerusGameStorage. The slot number, offset, and type must match exactly. Modules may have no additional variables (they should not), but the shared prefix must be identical.

Check specifically:
- Slot 0: all 15 fields present, offsets match
- Slot 1: purchaseStartDay, ticketWriteSlot, prizePoolFrozen, currentPrizePool — offsets match, currentPrizePool is t_uint128
- Slot 2: prizePoolsPacked (was slot 3)
- No contract has slot 2 = currentPrizePool (old layout)

**Step 5: Assembly audit**

Verify no production contract uses inline assembly to access slots 0, 1, or the former slot 2:

```
grep -rn 'sload\|sstore' contracts/ | grep -v '\.slot\|test/\|node_modules\|delegatecall\|_revertDelegate'
```

Any hit that references slot 0, 1, or a numeric slot must be verified against the new layout. Per the STACK.md research, no production assembly accesses slots 0-2, so this should return zero relevant hits.

**Step 6: Verify no direct currentPrizePool access remains in production code**

```
grep -rn 'currentPrizePool' contracts/ | grep -v '_getCurrentPrizePool\|_setCurrentPrizePool\|///\|@dev\|@notice\|@param\|@return\|NatSpec\|currentPrizePoolView\|uint128 internal currentPrizePool'
```

This should return zero lines of actual variable access — only the declaration, helpers, view function name, and comments.
