## Refuted / by-design section

Each verifier verdict below was independently spot-checked by NET-2 against the frozen `4921a428` tree; the cited mechanics were confirmed present and correct.

### D01 — storage-layout alignment — REFUTED (HIGH)
- **Claim refuted:** a module could corrupt a field another module owns via misaligned base layout or a divergent packed mask.
- **NET-2 confirmation:** `forge inspect storageLayout` on Mint / Lootbox / Afking modules each yields an **identical 87-entry layout** (first entries `purchaseStartDay@0:0`, `dailyIdx@0:3`, `rngRequestTime@0:6` byte-for-byte). Modules declare no contract-level state (inherit only `DegenerusGameStorage`), so base layout is identical by construction and unshifted. `grep` confirms **zero** redeclarations of any `BP_`/`LR_PENDING_ETH_*` constant inside `contracts/modules/` — per-module mask divergence is impossible. All packed constants live once in `BitPackingLib`/`DegenerusGameStorage`.

### D02 — nested msg.value / msg.sender — REFUTED (HIGH)
- **Claim refuted:** a nested payable delegatecall observes a stale or double-counted `msg.value`, enabling double-spend or strand.
- **NET-2 confirmation:** the two redemption entries read `msg.value` at single DC depth and guard `if (msg.value > amount) revert E()` **before** the `unchecked { stethPortion = amount - msg.value; }` (LootboxModule.sol:931-934 and 1006-1009 read verbatim) — underflow is impossible and the ETH leg can never exceed the owed amount. The Mint purchase chain reads `msg.value` once, caps fresh at cost, credits overpay to afking, and threads the partitioned amount down as an explicit `uint256` param so nested bodies never re-read it. Nested bodies (Boon `consumePurchaseBoost`, Degenerette `resolveEthSpinFromBox`) are declared payable solely for nested-DC-frame compatibility and are value-blind. `msg.sender` is preserved end-to-end by delegatecall semantics; consent guards live on explicit player/sender params.

### D03 — raw delegatecall(msg.data) dispatch — REFUTED (HIGH)
- **Claim refuted:** a raw-msg.data dispatch could be hijacked to an attacker-chosen target/selector or escape via fallback.
- **NET-2 confirmation:** the `consumeCoinflipBoon` HUB stub (DegenerusGame.sol:827) gates `msg.sender != COIN && != COINFLIP -> revert E()` **before** `GAME_BOON_MODULE.delegatecall(msg.data)`; delegatecall preserves msg.sender so the gate cannot be spoofed. All afking router stubs (subscribe/mintFlip/claimAfkingFlip/drainAffiliateBase/decurse/smite/recordAfkingSecondary) follow `delegatecall(msg.data); if (!ok) _revertDelegate(data)`, targeting the immutable `GAME_AFKING_MODULE` constant. No target is storage/param/attacker-influenceable. No fallback/receive in the modules — a non-matching selector reverts in the module dispatcher and bubbles.

### D04 — revert bubbling — REFUTED (HIGH)
- **Claim refuted:** a swallowed delegatecall failure leaves partial committed state while the Game assumes a successful transition.
- **NET-2 confirmation:** `_revertDelegate` (DegenerusGame.sol:948) re-reverts the original reason via memory-safe assembly (or `revert E()` on empty), so every module revert aborts the host tx atomically. The sole deliberate swallow, `_handleGameOverPath` (AdvanceModule.sol:662-689, read verbatim), swallows only `dOk=false` from `processTicketBatch(lvl+1)` and falls through to `handleGameOverDrain`; the reverting delegatecall is atomic so no partial ticket-drain state leaks, and the drain path reads no ticket-queue/cursor state (gates on `GO_JACKPOT_PAID`, computes from balance + steth + claimablePool) — undrained tickets merely forfeit trait-bucket eligibility while terminal fund release proceeds, exactly as the inline comment promises.

### D05 — module wiring — REFUTED (HIGH)
- **Claim refuted:** a column could be repointed at a zero/unset/wrong module, or a module is harmfully reachable as a direct external call, or an access guard breaks under delegatecall.
- **NET-2 confirmation:** every DC target is a `ContractAddresses` internal constant (no storage var, no admin setter to mutate at runtime). Codeless-target fail-safe verified: value-returning stubs `if (data.length == 0) revert E()` (confirmed at DegenerusGame.sol:407,979,1001,1023,1078,1109). Deployed modules have empty storage so a direct call runs inert (Game never reads module storage), and Game-trusted callees gate on `msg.sender == GAME` which a direct module call (`address(this)==module`) cannot satisfy. Privileged guards (VRF coordinator, ADMIN, COIN, AFFILIATE, QUESTS, onlySelf) check the real external caller, preserved by delegatecall.

### critic — depth≥3 recirc + packed multi-module slot RMW — REFUTED (HIGH)
- **Claim refuted:** a depth≥3 recirculation cascade re-reads value, or a multi-module packed-slot RMW drops a mask and corrupts a co-resident field.
- **NET-2 confirmation:** the recirc entry `resolveLootboxDirect` passes `allowEthSpin=false` as the final positional arg to `_resolveLootboxCommon` (LootboxModule.sol:906, signature confirmed at 1247-1261); the `if (allowEthSpin)` gate at 2031 only fires `_callEthSpin` when true, so recirc boxes award tickets instead and depth terminates at 3 (no 4th-level ETH-spin). No recirc frame re-reads `msg.value`/`address(this).balance`. Slot-7 `balancesPacked` has **zero** module-direct writes — all writers route through the inherited `_creditClaimable/_debitClaimable/_creditAfking/_debitAfking` accessors in `DegenerusGameStorage.sol:934-976` with explicit low-half borrow guards. The four raw slot-34 `lootboxRngPacked` RMWs (Mint:1611, Whale:916, Afking:1043, Advance:1665) each use the correct `(word & ~(MASK<<SHIFT)) | ((val & MASK)<<SHIFT)` clear-then-OR with the shared `LR_PENDING_ETH_SHIFT/MASK`, caching the full word with no intervening slot-34 mutation — co-resident fields preserved.
