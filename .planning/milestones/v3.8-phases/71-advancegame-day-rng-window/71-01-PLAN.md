**Part 1: Timeline Diagram**

Create an ASCII timeline showing the three distinct periods:

```
Period A: COMMITMENT WINDOW
  advanceGame() call N                     rawFulfillRandomWords callback
  |                                        |
  rngGate -> _requestRng -> return 1       VRF coordinator delivers word
  advanceGame -> _swapAndFreeze            rngWordCurrent = word
  rngLockedFlag = true                     (word stored, not yet consumed)
  prizePoolFrozen = true
  lootboxRngIndex incremented
  |<---------- VRF IN FLIGHT ------------>|

Period B: STORED-BUT-UNPROCESSED SUB-WINDOW
  rawFulfillRandomWords return             next advanceGame() call
  |                                        |
  rngWordCurrent != 0                      rngGate finds word ready
  rngLockedFlag still true                 _applyDailyRng runs
  All guards still active                  consumers fire
  |<---------- WORD WAITING -------------->|

Period C: PROCESSING + UNLOCK
  rngGate processes word                   _unlockRng(day)
  |                                        |
  _applyDailyRng (nudges)                  dailyIdx = day
  9 consumers fire sequentially            rngLockedFlag = false
  All within single tx                     rngWordCurrent = 0
  |<---------- SINGLE TX --------------->|
```

Explain why Period B is safe: rngLockedFlag remains true, so all guards active during Period A remain active during Period B. The word is committed (VRF delivered it), so no on-chain state can change the word itself.
