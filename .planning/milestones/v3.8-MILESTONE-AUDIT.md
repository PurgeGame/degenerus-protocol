## Vulnerability Found

**TQ-01 (MEDIUM):** `_awardFarFutureCoinJackpot` at JackpotModule:2544 reads `ticketQueue[_tqWriteKey(candidate)]` — the active write buffer, not the frozen read buffer. After VRF fulfillment, an attacker can precompute winning levels and add tickets via `purchase()` before jackpot processing.

**Recommended Fix:** Change `_tqWriteKey` to `_tqReadKey` at JackpotModule:2544 (one-line change). Global swap proof confirms correctness.
