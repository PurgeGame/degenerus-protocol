## v59.0 Requirements

### SALV ‚Äî F-01 salvage uint32-truncation (CRITICAL)
> `sellFarFutureTickets` values the full `uint256 quantities[i]` in the quote (`DegenerusGameMintStreakUtils.sol:173-181`) but the execution debit truncates to `uint32(quantities[i]) * 4` (`DegenerusGameMintModule.sol:1013-1015`); the holdings guard `_removeFarFutureTickets` (`:1059`) reads a `uint32 owed` so it constrains only the truncated quantity. Exploit: `n = 2^32 + k` ‚Üí debit `k*4` entries (passes), quote values 4.29B+ phantom tickets ‚Üí drains sDGNRS claimable + BURNIE reserves. RNG-clean / solvency-positive fix.
- [ ] **SALV-01** (IMPL): Widen the existing per-line zero-check in the quote loop so the valued quantity equals the debited `uint32` quantity ‚Äî `if (n == 0) revert E();` ‚Üí `if (n == 0 || n > type(uint32).max) revert E();` (house custom-error idiom, no new line, beside the existing `if (d < 6 || d > 100) revert E();`). The quote runs before the debit ‚Üí every reachable swap now values exactly the entries it removes; the `n = 2^32 + k` exploit reverts.

BURNIE-supply, not ETH.
- [ ] **AFAFF-01** (IMPL): Add the missing divide at `GameAfkingModule.sol:879` ‚Äî `base = ((_ethToBurnie(amount, mp) * 7) / 100) / 1 ether;` ‚Äî mirroring the two sibling accruals in the same block (`bonusWhole = bonusBase / 1 ether` `:826`, `QUEST_SLOT0_REWARD / 1 ether` `:887`). `affiliateBase` is then stored in whole-BURNIE units: no clamp saturation, no unbacked emission.
