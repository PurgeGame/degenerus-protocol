### LBX-EVT — New `LootboxTicketRoll` Event (Phase 274 Wave 1 + Wave 2; purely additive — NO existing event breaking changes)

- [x] **LBX-EVT-01**: NO breaking change to `LootBoxOpened.futureTickets`, `BurnieLootOpen.tickets`, `TicketsQueuedScaled`, or any other existing event. `LootBoxOpened.futureTickets` continues emitting the post-distress scaled value on BOTH manual and auto-resolve paths (identical to v38.0 behavior; matches `LootboxTicketRoll.preRollTickets` on the manual path). The new `LootboxTicketRoll` event is purely additive — UI / indexer consumers can opt in for remainder visibility without rebasing existing event reads.
- [x] **LBX-EVT-02**: Manual paths emit `TicketsQueued(buyer, level, qty)` (qty = whole) via the `_queueTickets` helper. Auto-resolve paths continue emitting `TicketsQueuedScaled(buyer, level, qty)` (qty = scaled) via the `_queueTicketsScaled` helper. Mint-boost call sites at `DegenerusGameMintModule.sol` line 1142 also continue emitting `TicketsQueuedScaled`. Origin tracking by event type: `TicketsQueued` = manual lootbox open; `TicketsQueuedScaled` = mint-boost OR auto-resolve lootbox.
- [x] **LBX-EVT-03**: NEW event `LootboxTicketRoll` declared on `IDegenerusGameLootboxModule` (interfaces) + at the LootboxModule contract level near the existing event block (~line 60-110). Signature: `event LootboxTicketRoll(address indexed player, uint48 indexed lootboxIndex, uint32 preRollTickets, bool roundedUp);`. Both `player` and `lootboxIndex` indexed for filter efficiency. Emitted exactly once inside `_resolveLootboxCommon`'s manual branch (`index != type(uint48).max`) at the end of the `if (futureTickets != 0)` block — after the Bernoulli collapse and after the if/else split that either queues whole tickets or pays consolation. Fires if-and-only-if (a) manual path AND (b) ticket-path produced a non-zero pre-Bernoulli scaled value. Does NOT fire on auto-resolve paths. Does NOT fire when ticket-path was not selected. Does NOT fire when `_lootboxTicketCount` math truncated scaled to 0 from the start.
- [x] **LBX-EVT-04**: `LootboxTicketRoll` field semantics: `lootboxIndex` is the per-lootbox storage index from `openLootBox(player, index)` / `openBurnieLootBox(player, index)`. Auto-resolve paths NEVER emit this event, so the `type(uint48).max` sentinel value never appears as an emitted `lootboxIndex`. `preRollTickets` is the post-distress-bonus, pre-collapse scaled value (i.e. the value snapshotted in LBX-WT-02 as `scaledPre`); equals same-tx `LootBoxOpened.futureTickets` on manual paths. Divide by `TICKET_SCALE=100` for the floor-tickets, modulo `100` for the round-up Bernoulli weight (e.g. `247` → floor `2`, frac `47`). `roundedUp` is `true` only when the Bernoulli condition `(uint8(seed >> 152) % 100) < uint8(frac)` evaluated true AND `frac != 0`; `false` when `frac == 0` (no roll needed; whole-only scaled) OR `frac != 0` but Bernoulli failed. Consumer derives whole-tickets-awarded as `(preRollTickets / 100) + (roundedUp ? 1 : 0)` and infers consolation as `whole == 0 && preRollTickets > 0` (corroborated by same-tx `LootBoxWwxrpReward`).
- [x] **LBX-EVT-05**: Threading the index + behavior gating: `_resolveLootboxCommon` signature gains one new parameter `uint48 index` (placed alongside `day` for parameter-order coherence). All 4 callers update: `openLootBox` passes its `index` arg (real index); `openBurnieLootBox` passes its `index` arg (real index); `resolveLootboxDirect` passes `type(uint48).max` (sentinel → auto-resolve path); `resolveRedemptionLootbox` passes `type(uint48).max` (sentinel → auto-resolve path). Index parameter is dual-purpose: (a) identifies the lootbox in the `LootboxTicketRoll` emit (manual paths only); (b) gates the behavioral split between manual (Bernoulli + `_queueTickets` + emit) and auto-resolve (`_queueTicketsScaled`, status quo).
- [x] **LBX-EVT-06**: Event-emission ordering inside the manual-path `if (futureTickets != 0)` block (single tx, ticket-path win): `TicketsQueued(player, level, whole)` (or `LootBoxWwxrpReward(...)` on consolation) → `LootboxTicketRoll(player, index, preRollTickets, roundedUp)` → outer `LootBoxOpened(player, day, amount, level, scaledPreFutureTickets, burnie, bonus)` (existing; emitted by the wrapping function at line ~999, scaled value unchanged). Consumers group by `tx` + `player` + `lootboxIndex` to correlate. Auto-resolve paths emit `TicketsQueuedScaled` + `LootBoxOpened` (no `LootboxTicketRoll`) — same ordering as v38.0.
