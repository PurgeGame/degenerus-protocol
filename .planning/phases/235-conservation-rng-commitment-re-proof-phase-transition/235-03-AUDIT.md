## Methodology

Per D-07 + D-09 treatment rules, RNG-01 enumerates five categories of new RNG consumer introduced by the v29.0 delta and its two post-Phase-230 addendum commits:

1. **Category 1 — Earlybird bonus-trait roll (20a951df):** `_runEarlyBirdLootboxJackpot` now rolls the same 4 bonus traits the coin jackpot uses via `_rollWinningTraits(rngWord, true)`. The VRF `rngWord` flows through a new on-chain consumption path whose input commitment must be proven.
2. **Category 2 — BAF `traitId=420` sentinel emission (104b5d42):** `runBafJackpot` emits `JackpotEthWin` / `JackpotTicketWin` with `BAF_TRAIT_SENTINEL = 420` (compile-time constant) in place of literal `0`. Sentinel is NOT a new entropy derivation — it is a compile-time label. However, `runBafJackpot`'s pre-existing `rngWord` consumption (via `_addClaimableEth` / `_awardJackpotTickets` / `_queueWhalePassClaimCore`) is audited for completeness as the entropy chain is unchanged.
3. **Category 3 — Entropy passthrough to `processFutureTicketBatch` (52242a10):** The MintModule body-top `uint256 entropy = rngWordCurrent;` SLOAD was removed; `entropy` now arrives via calldata from AdvanceModule's `_processFutureTicketBatch` helper. Three new AdvanceModule call sites (IM-10 / IM-11 / IM-12) thread `rngWord` from the `rngGate` return into the delegatecall payload.
4. **Category 4 — c2e5e0a9 entropy-mixing sites (17 per-site per D-07):** XOR+xorshift mixing replaced with keccak formulations (`EntropyLib.hash2` for 2-input sites; `keccak256(abi.encode(a, b, c))` for 3-input sites). Each site is a distinct consumer per D-07. No equivalence-class shortcuts.
5. **Category 5 — `_raritySymbolBatch` keccak-seed diffusion (314443af per D-09):** LCG seed derivation replaced from `(baseKey + groupIdx) ^ entropyWord` to `uint256(keccak256(abi.encode(baseKey, entropyWord, groupIdx)))`. Phase 235 RNG-01 owns DIFFUSION (does keccak diffuse all 256 bits properly); 232.1-03-PFTB-AUDIT owns AVAILABILITY (is entropyWord != 0 at all 4 reachable `_processFutureTicketBatch` call sites) — clean split per D-09.

For every row, the backward-trace chain ends at the VRF coordinator callback at `AdvanceModule.rawFulfillRandomWords:1702` (daily RNG branch — `rngWordCurrent = word;` SSTORE) OR at the `rngWordByDay[day]` early-return short-circuit at `rngGate:1141`. Both anchors ultimately depend on the VRF-coordinator-provided random word that is cryptographically infeasible to predict prior to the callback tx (Chainlink VRF v2 v1.7+ security model: block-hash-derived VRF proof; keyhash + future-block-hash preimage resistance).

Per D-06 the mandatory `## 232.1 Ticket-Processing Impact` sub-section walks each 232.1 fix series change against RNG-01. Per D-04 the cross-cite table re-verifies 233-02-AUDIT JKP-02, 231-02-AUDIT EBD-02, and 232.1-03-PFTB-AUDIT at HEAD 1646d5af.
