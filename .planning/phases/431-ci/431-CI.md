## CI-01 (✅ — strong guarantees made durable)

Added a `deep-guarantees` job gated on `github.event_name == 'schedule' || 'workflow_dispatch'` (weekly `cron: '0 6 * * 1'` + manual), `timeout-minutes: 330`:
1. Install Halmos (`pip install halmos`), run the 31 symbolic proofs.
2. Run the deep-profile invariant sweep (the `[profile.deep.invariant]` already in `foundry.toml`).

A proof or deep invariant that regresses now fails a scheduled run rather than silently shipping — the meta-defect class the survey flagged ("a proof exists but nothing enforces it").

**Mutation in CI — honest trade:** the mutation campaign is deliberately NOT wired into CI. Its harness lives in the gitignored `audit/` tree (no committed `.DONE` state for CI to resume from) and a full three-module run far exceeds the 6h GitHub Actions job ceiling. It is run locally/overnight (`audit/mutation/run-campaign-v68.sh`, resumable) and its measured kill-rate is committed to the audit pack as point-in-time evidence. Claiming it as a per-run CI gate would be dishonest engineering; documenting the trade is the correct call.

## CI-02 (✅ — fast PR gates)

The per-PR `foundry` job now runs the **MECH-02 storage-layout oracle** (new step before `forge test`) alongside the pre-existing EIP-170 ceiling guard and full `forge test`. So a slot move or a near-ceiling overflow can no longer merge. `RngReuseJackpotStraddle` (the RNGPROOF follow-on's "promote to a CI gate" item) is confirmed already in the per-PR `forge test` set — no skip — so it is gated.

## Validation

- `ci.yml` parses (yaml.safe_load): jobs = `foundry`, `slither`, `aderyn`, `deep-guarantees`.
- No untrusted `github.event.*` interpolation in any `run:` (no workflow-injection surface — every step runs static scripts).
- The layout oracle runs green locally on the frozen tree; the build cache is warm from `forge build --sizes` so the 24 `forge inspect` calls are cheap.

## Verdict

CI-01 ✅, CI-02 ✅. The strong guarantees (Halmos + deep invariants) are now enforced on a schedule; the fast gates (layout oracle + EIP-170 + full suite) run per-PR; the mutation trade is documented honestly.
