   - If no changes: confirm the proof still holds

3. Enumerate ALL paths that modify sDGNRS supply:
   - `_mint()` -- constructor only (one-time, fixed supply)
   - `burn()` -- proportional redemption (reduces totalSupply and balanceOf simultaneously)
   - `transferFromPool()` -- moves between addresses (no supply change)
   - `transfer()` -- if it exists or was removed (soulbound enforcement)
   - Any other function that modifies `_balances` mapping

   - `totalSupply_after - totalSupply_before == sum(balanceOf_after) - sum(balanceOf_before)`

   - For transfers: totalSupply unchanged, sender decreases, receiver increases by same amount
