# Plan 15-01: GameHandler and Base ETH Solvency Invariant Scaffold

## Goal

Create GameHandler with purchase/advanceGame/claimWinnings functions and an EthSolvency invariant test that asserts the primary solvency property across randomized call sequences.

## Tasks

<tasks>
- Create `test/fuzz/handlers/GameHandler.sol` with actor management, purchase(), advanceGame(), claimWinnings(), and ghost variables (ghost_totalDeposited, ghost_totalClaimed)
- Create `test/fuzz/invariant/EthSolvency.inv.t.sol` that inherits DeployProtocol, sets up GameHandler + VRFHandler as targetContracts, and asserts `address(game).balance >= currentPrizePool + nextPrizePool + claimablePool + futurePrizePool`
- Run `make invariant-test` and verify the invariant holds with zero violations
</tasks>

## Verification

- [ ] `make invariant-test` passes with zero assertion failures
- [ ] GameHandler purchase() succeeds for at least some calls (non-zero ghost_totalDeposited)
- [ ] EthSolvency invariant asserts the primary solvency property

## Must Haves

- GameHandler with bounded inputs using `bound()` for ticket quantity and lootbox amount
- Multi-actor support (10 actors, each funded with 100 ETH)
- Ghost variable tracking for ETH deposited and claimed
- Graceful early return on impossible states (game over, insufficient balance)
- VRFHandler from Phase 14 reused as second targetContract
- Invariant test uses `targetContract()` to register both handlers
