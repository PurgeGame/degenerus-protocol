_threeDayRngGap (DegenerusGameAdvanceModule.sol line ~1240):
```solidity
function _threeDayRngGap(uint48 day) private view returns (bool) {
    if (rngWordByDay[day] != 0) return false;
    if (rngWordByDay[day - 1] != 0) return false;
    if (day < 2 || rngWordByDay[day - 2] != 0) return false;
    return true;
}
```
Condition: no VRF word recorded for current game day, day-1, or day-2 (three consecutive game days with no RNG fulfillment).

18h retry logic (DegenerusGameAdvanceModule.sol line ~647-654):
```solidity
if (rngRequestTime != 0) {
    uint48 elapsed = ts - rngRequestTime;
    if (elapsed >= 18 hours) {
        _requestRng(isTicketJackpotDay, lvl);
        return 1;
    }
    revert RngNotReady();
}
```
rngLockedFlag is set when a VRF request is pending. advanceGame() reverts with RngNotReady during the first 18h after a request. After 18h it retries automatically on the next advanceGame() call.

Stall trigger conditions from RESEARCH.md:
1. LINK subscription drain — VRF charges LINK; empty subscription → requests fail → no words → 3+ days → stall
2. Chainlink coordinator unavailable — external failure; requests fail similarly
3. Deliberate admin non-funding — admin never funds subscription after deployment
4. Admin deploys reverting coordinator via wireVrf — requests always fail (hard revert path)
5. Admin-induced stall loop — call emergencyRecover with reverting coordinator → requests fail → 3 days → emergencyRecover callable again with real coordinator (griefing cycle)

rngLockedFlag blocks only (BLOCKED during 18h lock):
- advanceGame() — reverts RngNotReady
- decimatorBurn() when lastPurchaseDay && rngLockedFlag (decWindowOpen gating)
- reverseFlip() — reverts RngLocked
- requestLootboxRng() — separate check

NOT blocked during 18h lock (PERMITTED):
- purchase() — ticket purchases continue; MintModule has no rng lock check
- purchaseBurnieLootbox() — continues
- purchaseDeityPass() — continues
- openLootBox() — continues if lootbox RNG already fulfilled
- claimWinnings() — withdrawals continue
- claimDecimatorJackpot() — continues
- setOperatorApproval() — continues
- setAutoRebuy() and related — continue
- resolveDegeneretteBets() — continues
- adminStakeEthForStEth() — continues (not gated)

Key insight on front-running risk during 18h window:
- Ticket purchases during pending RNG: tickets are recorded but resolved in advanceGame() using the pending word. A player who knows the pending word could purchase tickets specifically timed to benefit — BUT the pending word is the Chainlink VRF output, which is only known after fulfillment. The 18h window is a retry period for failed fulfillment; during normal operation, advanceGame() + VRF fulfillment happen within the same game day.
- openLootBox() during pending RNG: uses lootboxRngWordByIndex, NOT the daily game word. No conflict.
- claimWinnings() during pending RNG: uses claimableWinnings[player] — already finalized values. No conflict.
