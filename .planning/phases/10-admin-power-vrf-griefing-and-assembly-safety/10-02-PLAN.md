onlyOwner in DegenerusAdmin.sol (lines ~368-374):
```solidity
modifier onlyOwner() {
    if (
        msg.sender != ContractAddresses.CREATOR &&
        !vault.isVaultOwner(msg.sender)
    ) revert NotOwner();
    _;
}
```
Two auth paths: (1) CREATOR EOA, (2) any holder of >30% of DGVE tokens via isVaultOwner.

wireVrf in DegenerusGameAdvanceModule.sol (lines ~298-310):
```solidity
function wireVrf(
    address coordinator_,
    uint256 subId,
    bytes32 keyHash_
) external {
    if (msg.sender != ContractAddresses.ADMIN) revert E();
    address current = address(vrfCoordinator);
    vrfCoordinator = IVRFCoordinator(coordinator_);  // NO idempotency guard
    vrfSubscriptionId = subId;
    vrfKeyHash = keyHash_;
    emit VrfCoordinatorUpdated(current, coordinator_);
}
```

updateVrfCoordinatorAndSub in DegenerusGameAdvanceModule.sol (lines ~1115-1135):
```solidity
function updateVrfCoordinatorAndSub(...) external {
    if (msg.sender != ContractAddresses.ADMIN) revert E();
    if (!_threeDayRngGap(_simulatedDayIndex())) revert VrfUpdateNotReady();  // STALL GATE
    // ... resets: rngLockedFlag, vrfRequestId, rngRequestTime, rngWordCurrent
}
```

if (msg.sender != address(vrfCoordinator)) revert OnlyVRFCoordinator();

Admin function inventory from RESEARCH.md:

DegenerusAdmin.sol CREATOR/VaultOwner-gated:
1. setLinkEthPriceFeed(address) — replace LINK/ETH oracle (only if feed unhealthy)
   Worst-case: malicious feed → suppress BURNIE rewards or trigger unbounded BURNIE via near-zero LINK price. Bounded by FeedHealthy guard while current feed is live.
2. swapGameEthForStEth() — swap admin ETH for GAME-held stETH 1:1
   Worst-case: net ETH-neutral swap; cannot touch claimablePool. LOW.
3. stakeGameEthToStEth(amount) — stake GAME ETH into Lido
   Worst-case: guarded by claimablePool reserve check (ACCT-09 PASS). Cannot stake player reserves. LOW.
4. setLootboxRngThreshold(newThreshold) — set minimum pending ETH for lootbox RNG
   Worst-case: set to uint256.max → lootbox RNG never triggers → all lootbox buyers permanently frozen. HIGH consequence; admin-key-required → MEDIUM by C4 methodology.
5. emergencyRecover(newCoordinator, newKeyHash) — migrate to new VRF coordinator after 3-day stall
   Worst-case: substitute attacker coordinator → full RNG manipulation. Gated by 3-day stall. HIGH consequence; admin-key-required + stall-gated → MEDIUM by C4 methodology.
6. shutdownAndRefund(target) — cancel subscription, sweep LINK after gameOver
   Worst-case: only callable after gameOver=true. Sweeps LINK to target. Cannot harm players. INFO.

DegenerusGame.sol ADMIN-only:
1. wireVrf(coordinator, subId, keyHash) — wire VRF coordinator into game
   Worst-case: NO idempotency guard → repeatable mid-game without 3-day stall. Attacker coordinator controls rawFulfillRandomWords → full RNG manipulation. HIGH consequence; admin-key-required → MEDIUM by C4. But note: ungated vs. gated is a distinguishing factor from emergencyRecover.
2. updateVrfCoordinatorAndSub(newCoord, newSubId, newKeyHash) — emergency coordinator rotation
   Worst-case: identical RNG manipulation risk as wireVrf but gated by 3-day stall. MEDIUM by C4.
3. adminSwapEthForStEth(recipient, amount) — swap admin ETH for GAME stETH
   Worst-case: value-neutral swap; claimablePool not affected. LOW.
4. adminStakeEthForStEth(amount) — stake GAME ETH into Lido
   Worst-case: guarded by claimablePool check. LOW.
5. setLootboxRngThreshold(newThreshold) — relay from DegenerusAdmin
   Worst-case: same as Admin version — lootbox freeze. MEDIUM.

GAS-07-I1 forwarded finding: CREATOR key-management risk.
- CREATOR is a single EOA with no multi-sig, no time-lock, no key rotation path.
- If CREATOR key is compromised, attacker has full admin power instantly via both auth paths.
- Key rotation requires a new deployment (ContractAddresses uses compile-time constants).
- Classification: INFO/QA — known centralization risk, documented for Code4rena judges.

Code4rena severity methodology for admin-key-required findings:
- Impact CRITICAL + admin key required = MEDIUM (centralization risk, not exploitable by third party)
- Unless the finding is also exploitable without key compromise (then it's a separate classification)
- wireVrf severity: MEDIUM (admin-key-required, HIGH impact — full RNG control)
