<objective>
Close the verifier oracle holes and intake the seven surface-maps as a tracked finding-candidate
ledger (FND-04). Two independent deliverables: (1) an ORACLE-HOLE audit — for each invariant / proof
test that targets a post-v62 CHANGED surface (the redemption rework, the CEI/yield-surplus payout,
the RNG-window freeze over the repacked slots, the EV-cap two-window eviction, the BURNIE emission
seeds, the decimator uint32 claim seed), confirm the test ACTUALLY exercises its target code — a test
that passes because its target branch never executes is a false oracle that would let the sweeps
miss a real finding; (2) a single consolidated FINDING-CANDIDATES ledger that intakes EVERY lead from
all seven `.planning/v63-surface-map/*.md` maps, each routed to its owning sweep phase (389-394) with
its source citation, severity hint, and a one-line restatement, so no lead is lost between FOUNDATION
and the sweeps.

Purpose: the surface-map foundation found 0 HIGH on inspection but surfaced ~30 MED/LOW/INFO leads
across seven dimensions. Without a tracked intake, leads scatter; the sweeps (389-394) must each
receive their routed subset as the prime targets. And without the oracle-hole check, a green baseline
(Plan 03) could be green because a target path is never reached, not because the invariant holds.

Output: 388-02-ORACLE-HOLES.md (the exercised-target audit) + 388-02-FINDING-CANDIDATES.md (the
routed intake ledger). No contract changes; no test edits expected (this is an audit + intake plan —
if an oracle hole requires a NEW assertion the owning sweep phase builds it, recorded here as a
routed candidate, NOT fixed in this plan).
</objective>
