**PAY-14: sDGNRS burn() Proportional Redemption** (StakedDegenerusStonk:373-462)

This is a critical audit target due to CP-04 (pending claims inflation risk).

The audit MUST cover:

1. **Claim-then-compute ordering**: Verify sDGNRS.burn() calls `game.claimWinnings()` to pull pending ETH winnings BEFORE computing the proportional share. This is the key defense against CP-04 (pending claims inflating totalMoney). If claimWinnings is NOT called first, totalMoney would include claimable ETH that hasn't been withdrawn, inflating the burn payout.

2. **totalMoney calculation**: Verify `totalMoney = ethBal + stethBal + claimableEth`. Trace each component:
   - `ethBal`: verify this is `address(this).balance` or equivalent
   - `stethBal`: verify this is the stETH balance held by the contract
   - `claimableEth`: verify this is `_claimableWinnings()` from the game contract AFTER claimWinnings has been called (should be 0 or 1 sentinel)

3. **Proportional formula**: Verify `payout = (totalMoney * amount) / supplyBefore` where `amount` is the sDGNRS being burned and `supplyBefore` is the total supply BEFORE the burn. Verify rounding direction favors the protocol (truncation).

4. **ETH-preferred ordering**: Verify the payout prioritizes ETH first, then stETH for remainder. This means: send ETH up to min(payout, ethBal), then send stETH for the rest. Verify no path where user receives more than their proportional share.

5. **BURNIE component**: Verify sDGNRS burn also redeems proportional BURNIE. Trace the BURNIE transfer path. Verify the BURNIE proportional calculation uses the same formula.

6. **Supply conservation**: Verify sDGNRS supply decreases by exactly `amount` after burn. Verify no path creates or destroys more tokens than intended.

7. **Multiple sequential burns**: Verify that if multiple users burn sequentially, each receives correct proportional share. After first burn, `supplyBefore` decreases and `totalMoney` decreases, so second burn uses updated values.

8. **CEI ordering**: Verify burn follows CEI: (1) check amount > 0, (2) burn tokens (effect), (3) transfer ETH/stETH/BURNIE (interaction).

**PAY-15: DGNRS Wrapper burn()** (DegenerusStonk:164-223)

The audit MUST cover:

1. **Delegation to sDGNRS**: Verify DGNRS.burn(amount) calls sDGNRS.burn(amount). Verify the DGNRS wrapper correctly unwraps and delegates the burn.

2. **Forward mechanism**: Verify DGNRS.burn() forwards the ETH/stETH/BURNIE received from sDGNRS.burn() to the caller. Verify no value is retained by the wrapper.

3. **unwrapTo**: Verify `unwrapTo` converts DGNRS (transferable) to sDGNRS (soulbound). Verify this is a one-way conversion with no value transfer.

4. **Access control**: Verify only the token holder can burn their own tokens. Verify no path for unauthorized burns.
