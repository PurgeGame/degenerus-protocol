**Path structure to trace (all paths start from _consolidatePoolsAndRewardJackpots with memFuture loaded from storage):**

**FLOW-01: Non-auto-rebuy winner (top 5 large, small even-index)**
OLD path in caller:
  memFuture -= bafPoolWei
  (netSpend, claimed, lootboxToFuture) = runBafJackpot(bafPoolWei, ...)
  // Inside runBafJackpot: claimableDelta += _addClaimableEth(winner, amount, rngWord)
  //   _addClaimableEth returns weiAmount when no auto-rebuy
  // netSpend = poolWei - refund; lootboxToFuture = lootboxTotal
  memFuture += (bafPoolWei - netSpend)     // add back refund
  memFuture += lootboxToFuture             // add back lootbox portion
  claimableDelta += claimed

NEW path in caller:
  claimed = runBafJackpot(bafPoolWei, ...)
  memFuture -= claimed
  claimableDelta += claimed

Prove: For a non-auto-rebuy winner who receives ETH (not lootbox), the old memFuture change equals:
  -bafPoolWei + (bafPoolWei - netSpend) + lootboxToFuture = -(netSpend - lootboxToFuture)
And netSpend = poolWei - refund, so the old net deduction from memFuture is:
  -(poolWei - refund - lootboxToFuture) = -(claimed) because claimed = poolWei - refund - lootboxToFuture
Wait -- this is NOT the identity. claimableDelta (the return from runBafJackpot) is only the ETH that goes to claimable balances. The key insight: poolWei = refund + netSpend, and netSpend = claimableDelta + lootboxToFuture. So:
  Old memFuture change: -bafPoolWei + (bafPoolWei - netSpend) + lootboxToFuture = -netSpend + lootboxToFuture = -(claimableDelta + lootboxToFuture) + lootboxToFuture = -claimableDelta
  New memFuture change: -claimed = -claimableDelta
  EQUIVALENT.
