   - advanceGame() calls rngGate() which calls _applyDailyRng()

   - Nudges accumulated AFTER _requestRng (during VRF wait) are blocked by rngLockedFlag check in reverseFlip()
   Confirm this is the actual behavior.

4. Verdict for RNG-01: Is the rngLockedFlag continuously set from VRF request through word consumption? Document any gap, however brief.

**Analysis B: Block Proposer Attack (RNG-08)**

1. A block proposer can see the VRF fulfillment transaction in the mempool before including it in a block. Can they:
   a. Front-run with reverseFlip() calls? (Should be blocked — rngLockedFlag is already true)
   b. Reorder the fulfillment transaction relative to other transactions? (Fulfillment stores rngWordCurrent; nudges are applied later in advanceGame)
   c. Delay the fulfillment transaction? (Only delays the game; 18h timeout eventually triggers re-request)

2. Verdict for RNG-08: Can a block proposer who sees the fulfilled VRF word exploit reverseFlip()?

4. Verdict for RNG-06: Can the RNG lock be permanently stuck?
