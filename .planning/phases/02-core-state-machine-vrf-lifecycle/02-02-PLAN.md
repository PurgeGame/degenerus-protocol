<objective>
Measure the gas cost of rawFulfillRandomWords at worst-case state and verify it cannot revert under any condition. The VRF coordinator does NOT retry failed callbacks — a revert permanently loses the VRF request, requiring the 18h timeout recovery path. The callback must also stay well under the 300,000 gas VRF_CALLBACK_GAS_LIMIT to prevent out-of-gas reverts.

Purpose: RNG-02 (no revert) and RNG-03 (gas under 200k) are safety-critical — a reverting or gas-exhausting callback silently breaks the VRF lifecycle. This plan provides definitive measurements rather than estimates.

Output: `02-02-FINDINGS-vrf-callback-gas.md` containing gas measurements, revert path analysis, and RNG-02/RNG-03 verdicts.

VRF callback entry point (DegenerusGame.sol):
  rawFulfillRandomWords(uint256 requestId, uint256[] calldata randomWords)
  This is the external entry point called by the VRF coordinator.
  It delegates to the AdvanceModule via delegatecall.

VRF callback implementation (AdvanceModule.sol lines ~1199-1220):
  function rawFulfillRandomWords(uint256 requestId, uint256[] calldata randomWords) external {
      if (msg.sender != address(vrfCoordinator)) revert E();
      if (requestId != vrfRequestId || rngWordCurrent != 0) return;
      uint256 word = randomWords[0];
      if (word == 0) word = 1;
      if (rngLockedFlag) {
          rngWordCurrent = word;          // Daily: 1 SSTORE
      } else {
          // Lootbox: SLOAD + 2 SSTORE + event
          uint48 index = lootboxRngRequestIndexById[requestId];
          lootboxRngWordByIndex[index] = word;
          emit LootboxRngApplied(index, word, requestId);
          vrfRequestId = 0;
          rngRequestTime = 0;
      }
  }

VRF_CALLBACK_GAS_LIMIT: 300,000 (constant in AdvanceModule)

Research gas estimates:
  Daily branch (rngLockedFlag=true): ~28,600 gas
  Lootbox branch (rngLockedFlag=false): ~34,075 gas
  With delegatecall overhead: ~40,000-45,000 gas total
  These are ESTIMATES — this plan produces actual measurements.

Existing gas test file: test/gas/AdvanceGameGas.test.js (860 lines)
Test helper: test/helpers/deployFixture.js
Test framework: Hardhat + ethers v6 (ESM)

Potential revert paths in rawFulfillRandomWords:
  1. msg.sender != vrfCoordinator → revert E() — only reachable by unauthorized callers
  2. requestId mismatch → silent return (NOT a revert)
  3. rngWordCurrent != 0 → silent return (NOT a revert)
  4. word == 0 → mapped to 1 (NOT a revert)
  5. All SSTOREs → cannot fail (storage writes don't revert)
  6. Event emission → cannot fail (LOG opcodes don't revert)
  7. Delegatecall from DegenerusGame → check if return value is checked

IMPORTANT: The coordinator check `revert E()` is the ONLY revert. But the coordinator
is always the authorized caller, so this should never trigger in practice.
The concern is: could updateVrfCoordinatorAndSub() change vrfCoordinator mid-flight,
causing a legitimate fulfillment to be rejected? This is the research open question #2.
