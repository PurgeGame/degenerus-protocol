## High-Risk Patterns Analyzed

### Unified Award Call Per Purchase (IM-01..IM-04)

All 4 purchase entry points now issue a single unified 2-arg `_awardEarlybirdDgnrs(buyer, wei)` call at the tail of the caller:

- **`DegenerusGameMintModule._purchaseFor` (IM-01)** — at line 1165, with `wei = ticketFreshEth + lootboxFreshEth`. Unconditional (NOT inside any `if (ticketCost != 0)` or `if (lootBoxAmount != 0)` branch). Pure claimable-payment path produces `wei = 0` and hits the early-return at Storage line 1005 — INTENDED zero-award (no fresh ETH, no DGNRS credit). All other paths deliver a non-zero award.
- **`DegenerusGameWhaleModule._purchaseWhaleBundle` (IM-02)** — at line 263, `wei = totalPrice`. The quadratic-telescoping property ensures the single call advances `earlybirdEthIn` by `totalPrice` exactly as two separate calls of `a + b = totalPrice` would have (integration is additive across the partition).
- **`DegenerusGameWhaleModule._purchaseLazyPass` (IM-03)** — at line 476, `wei = benefitValue`. Note: award basis is the UNDISCOUNTED benefit value, not the discounted `totalPrice`. The prior 3-arg signature used the same `benefitValue` basis; no semantic change.
- **`DegenerusGameWhaleModule._purchaseDeityPass` (IM-04)** — at line 587, `wei = totalPrice`. Paired with `rngLockedFlag` early-revert at line 543, so no level transition is concurrent with the award.

Cross-caller exactly-once verification: no caller issues the award inside a conditional branch that could be skipped on a valid purchase-completion path. Each caller reaches the award line exactly once per successful purchase (and never on a reverting path, since Solidity 0.8.34 rolls back all state on revert).

### Level-Transition Sentinel Flip (IM-05)

`_finalizeEarlybird` (contracts/modules/DegenerusGameAdvanceModule.sol:1582-1595) enforces one-shot semantics via the `earlybirdDgnrsPoolStart == type(uint256).max` sentinel:

1. **Hook entry gate** — called from `_finalizeRngRequest` line 1518-1520 under the guard `if (lvl == EARLYBIRD_END_LEVEL)`. Nested inside `if (isTicketJackpotDay && !isRetry)` (line 1510) so the hook cannot fire on VRF retries nor on non-jackpot-day `advanceGame` calls. The enclosing `_finalizeRngRequest` itself is callable only from `_requestRng` / `_tryRequestRng` (lines 1457, 1476) — both private and invoked by `advanceGame` / `requestLootboxRng` per the AdvanceModule flow.
2. **Internal sentinel gate** — line 1583 short-circuits if already finalized. Defense-in-depth against even pathological re-entry (which the call graph doesn't actually permit).
3. **Purchase-side no-op after finalization** — once flipped, every call to `_awardEarlybirdDgnrs` in Storage (line 1011) also no-ops. The post-finalization purchase flow therefore awards ZERO DGNRS, which is correct — earlybird window is over.

The hook cannot re-fire on subsequent level transitions because `lvl == EARLYBIRD_END_LEVEL` (= 3) is a ONE-SHOT equality, not a `>=` check. Once `level` advances past 3, the condition at line 1518 is `false` forever. Combined with the Storage sentinel, finalization is idempotent across any adversarial sequencing of level transitions and purchases.

Reachability proof: `EARLYBIRD_END_LEVEL = 3` (contracts/storage/DegenerusGameStorage.sol:175). Level monotonically increments by 1 per jackpot-day cycle (`level = lvl` at AdvanceModule:1514 where `lvl` is `level + 1` per the `purchaseLevel` semantics). The sequence 0→1→2→3 is therefore deterministic under any admin-free game progression, and the `lvl == 3` check fires on the 2→3 transition — exactly the "end of earlybird window" intent.

### Signature Contraction (3-arg → 2-arg)

The dropped `currentLevel` / `passLevel` / `startLevel` third arg was threaded from each of the 4 purchase callers into the OLD `_awardEarlybirdDgnrs` body's finalization branch (old body: `if (currentLevel >= EARLYBIRD_END_LEVEL) { ... dump-remaining-pool-into-Lootbox ... return; }`). Removing the argument required:

1. **Finalization relocation** — the entire dump-pool-into-Lootbox block moved to `_finalizeEarlybird` in AdvanceModule. The driving state is now `level` (real game state), not caller-supplied `currentLevel` (caller-controllable and, for whale bundles, could be `passLevel > level` — a prior bug class per the commit message).
2. **Body-side invariance** — the new Storage body at contracts/storage/DegenerusGameStorage.sol:1001-1044 references `currentLevel` NOWHERE. A fresh read from HEAD confirms zero `level()` calls, zero `storageGetter().level()` calls, and zero state reads that could substitute the dropped argument. The quadratic-curve math (lines 1021-1034) is purely a function of the pool snapshot and cumulative `earlybirdEthIn` — level-independent.
3. **Caller-side invariance** — the 4 call sites (IM-01 through IM-04) drop the level arg cleanly. No caller relied on the callee reading the level for correctness (prior semantics were only for gating finalization, which is now gated externally).

The contraction is therefore correct AND eliminates a class of bug (caller-poisoned finalization).
