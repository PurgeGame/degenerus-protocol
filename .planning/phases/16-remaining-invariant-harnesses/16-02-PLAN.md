# 16-02: Game FSM Invariant (FUZZ-03)

## Goal
Assert game FSM transitions are valid: level monotonic, gameOver terminal, phase transitions valid.

## Invariants
1. Level only increases (never decreases)
2. gameOver is terminal (once true, stays true)
3. Valid state: exactly one of {PURCHASE, JACKPOT, GAMEOVER}

## Implementation
- New handler: `test/fuzz/handlers/FSMHandler.sol` -- wraps GameHandler calls with pre/post state snapshots
- New file: `test/fuzz/invariant/GameFSM.inv.t.sol`
- FSMHandler snapshots `level` and `gameOver` before each call; invariant checks monotonicity

## Acceptance
- `make invariant-test` passes with no violations
- All 3 invariants hold across 256 runs x 128 depth
