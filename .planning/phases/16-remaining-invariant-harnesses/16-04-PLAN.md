# 16-04: Ticket Queue Invariant (FUZZ-05)

## Goal
Assert no player appears twice at the same level in the ticket queue.

## Invariants
1. For any level, each address in ticketQueue[level] is unique
2. Checked for current level and level+1 (the two actively queued levels)

## Implementation
- New file: `test/fuzz/invariant/TicketQueue.inv.t.sol`
- Reuses existing handlers
- Invariant scans queue entries at current and next level

## Acceptance
- `make invariant-test` passes with no violations
