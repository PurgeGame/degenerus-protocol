Read ALL source contracts and ALL prior audit reports (especially the "Cross-Contract Call Graph" and "Callers/Callees" sections from each audit). Build a comprehensive protocol-wide call graph.

Create the report file with the following structure:

```markdown
# Protocol Call Graph and State Mutation Matrix

**Scope:** 22 deployable contracts + 10 delegatecall modules + 5 libraries
**Date:** [current date]
**Source:** Phases 50-56 audit data + direct source verification

## Part 1: Cross-Contract Call Graph (XREF-01)
```

The call graph must contain THREE sections:

**Section A: Delegatecall Dispatch Map**
For DegenerusGame.sol, enumerate every delegatecall dispatch path:
- Which external/public function triggers the delegatecall
- Which module address constant is the target (from ContractAddresses.sol)
- Which function selector is called on the module
- Example row: `purchase() -> GAME_MINT_MODULE.delegatecall(purchaseTickets.selector, ...)`

Read DegenerusGame.sol source directly to extract every `.delegatecall(` pattern and every `ContractAddresses.GAME_*_MODULE` reference. Cross-reference with the module audit reports from Phases 50-52 to verify completeness.

Format as a table:
| # | Game Function | Target Module | Module Function | Call Type |
|---|--------------|---------------|-----------------|-----------|

**Section B: Cross-Contract External Calls**
For each of the 22 non-module contracts, list every external call TO another protocol contract. Extract from the "Cross-Contract Call Graph" sections in each audit report (Phases 54-56 audits all have this section). Verify against source using grep for ContractAddresses.* references and interface calls.

Format as a table grouped by source contract:
| Source Contract | Function | Target Contract | Method Called | Call Type |
|----------------|----------|----------------|--------------|-----------|

Categories for Call Type: `external`, `delegatecall`, `staticcall`, `library`

**Section C: Inbound Call Summary**
For each contract, list which OTHER contracts call INTO it. This is the reverse of Section B. Useful for understanding each contract's attack surface.

Format:
| Target Contract | Called By | Via Function | Method |
|----------------|-----------|-------------|--------|

Ensure no call is missed. If the audit data and source code disagree, note the discrepancy. Count total unique cross-contract call edges at the end.
