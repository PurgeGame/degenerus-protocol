<objective>
Enumerate every activity score inflation vector and compute the cost-per-EV-unit for each. The activity score controls lootbox EV multiplier (80-135%), so cheap inflation means cheap access to above-neutral lootbox returns. For each vector (quest streaks, affiliate self-referral, coordinated chains, whale bundles, deity pass), compute the minimum ETH cost to reach maximum component contribution and compare against the EV benefit unlocked.

Purpose: ECON-02 requires proving that activity score cannot be cheaply inflated to unlock high-EV lootboxes. "Cheaply" means the cost of inflation is less than the EV benefit gained. This plan quantifies both sides of that inequality for every inflation vector.

Output: `05-02-FINDINGS-activity-score-inflation.md` containing vector enumeration, cost analysis, and ECON-02 verdict.
</objective>

**1. Activity Score Component Map**

Read `_playerActivityScoreInternal` in DegeneretteModule (lines ~1020-1093) or `_playerActivityScore` in DegenerusGame.sol (lines 2430-2506). Extract each component with exact BPS values and caps.

**2. Quest Streak Inflation (max 10,000 BPS)**

a. How is questStreakRaw incremented? Read `awardQuestStreakBonus` in DegenerusGame.sol and trace the storage update. What constitutes a "quest" completion? Is there a minimum ETH requirement per quest?
b. Minimum cost per day: What is the cheapest transaction that qualifies as quest participation? If it requires a ticket purchase, compute the minimum purchase cost at the lowest ticket price (level 0-4 = 0.01 ETH per full ticket, minimum quantity).
c. Total cost for 100-day streak: 100 * daily_min_cost.
d. EV benefit of quest streak component: Going from 0 BPS to 10,000 BPS in quest streak, what is the marginal EV multiplier improvement? Map 10,000 BPS through the piecewise linear EV formula.
e. Cost-per-EV-unit: total_cost / EV_benefit_per_level. Compare against the 3.5 ETH max benefit at cap.

**3. Affiliate Self-Referral (BLOCKED)**

a. Confirm the self-referral prevention code in DegenerusAffiliate.sol. Find the exact code that detects `sender == code.owner` and locks to VAULT.
b. Document the code path with line numbers.
c. Can a player create two wallets and cross-refer? Wallet A refers Wallet B, Wallet B refers Wallet A. Is this detectable or preventable on-chain?
d. If cross-referral is possible: what is the net cost? Each wallet must deposit ETH, and affiliate rewards are BURNIE (not ETH). The BURNIE earned does not offset the ETH spent.

**4. Coordinated Affiliate Chains (max 5,000 BPS)**

a. Read `affiliateBonusPointsBest` in DegenerusAffiliate.sol. How are affiliate bonus points computed?
b. What referral volume is needed per level to maximize the 50-point affiliate bonus? (Research says: 1 ETH per point per level across 5 levels = 50+ ETH total referred volume.)
c. Cost to manufacture: if the Sybil group self-refers, each member deposits ETH. The affiliate rewards are BURNIE, not ETH. The ETH goes into the prize pool (which the group partially recovers through game mechanics). Net cost = ETH deposited - expected ETH return from game + BURNIE value.
d. Is the BURNIE earned enough to offset the ETH cost of generating affiliate volume?

**5. Whale Bundle Boost (1,000 or 4,000 BPS)**

a. Standard whale bundle: +1,000 BPS (10%). Cost: 2.4 ETH (levels 0-3) or 4 ETH (other levels).
b. Premium whale bundle (100-level): +4,000 BPS (40%). Cost: same price * 100 quantity? Or is the premium distinction based on bundle type?
c. Read WhaleModule to confirm how bundleType is determined and what triggers the 4,000 BPS vs 1,000 BPS.
d. Cost-per-BPS: 4 ETH / 4,000 BPS = 0.001 ETH per BPS for premium. Convert to EV benefit.

**6. Deity Pass (15,500 BPS effective)**

a. Deity pass grants: 8,000 BPS bonus + 5,000 BPS streak floor + 2,500 BPS mint count floor = 15,500 BPS.
b. Cost: 24 ETH minimum (k=0), up to 520 ETH (k=31).
c. Cost-per-BPS: 24 ETH / 15,500 BPS = 0.00155 ETH per BPS.
d. But deity pass also has game value beyond score (boon issuance, refund at game over). Document total value.

**7. Mint Count Bonus (max 2,500 BPS)**

a. How is _mintCountBonusPoints computed? Is it proportional to levels minted?
b. Cost: buying minimum tickets at each level. At early levels this is cheap (0.01 ETH). At high levels more expensive.

**8. Streak Bonus (max 5,000 BPS)**

a. How is streak computed? Consecutive level mints without missing a level?
b. Cost: minimum ticket purchase at each consecutive level.

**9. Composite Cost-to-Benefit Analysis**

For each activity score tier (0%, 60%, 100%, 200%, 305%), compute:
- Total minimum ETH investment to reach that tier
- Lootbox EV multiplier at that tier
- Maximum lootbox benefit per level at that tier
- Cost-to-benefit ratio: levels needed to recoup activity score investment through enhanced lootbox EV

**10. Lootbox EV Score Lock**

Document that `lootboxEvScorePacked` stores activity score at purchase time. This means:
- A player cannot inflate score AFTER buying lootboxes to get retroactive EV boost
- A player CAN buy lootboxes at peak score and open them later
- This is by design (locks score at purchase), not an exploit

**11. ECON-02 Verdict**

For each inflation vector: Is cost-to-inflate < EV-benefit-unlocked?
If YES for any vector: FAIL with specific numeric example.
If NO for all vectors: PASS with cost-to-benefit ratios.
