# Phase 221 Plan 01 — Raw Selector & Calldata Audit

**Date:** 2026-04-12
**Scope:** Every non-interface-bound calldata construction site in `contracts/` that matches one of four patterns:
1. `bytes4(0x...)` hex literal (CSI-04)
2. `bytes4(keccak256("..."))` string-derived selector (CSI-05)
3. `abi.encodeWithSignature(...)` / `abi.encodeCall(...)` (CSI-06)
4. `abi.encode*(...)` whose output is the payload of a low-level call (`.call` / `.delegatecall` / `.staticcall` / `transferAndCall`) (CSI-06)

Patterns already covered by Phase 220 (`abi.encodeWithSelector(IFACE.fn.selector, ...)`) and Phase 222 (direct interface-cast external calls `IFace(addr).fn(...)`) are OUT OF SCOPE per D-05.

**Method:** Four targeted greps (one per pattern) plus a fifth multi-line awk scan for Pattern 4 (abi.encode-feeding-low-level-call). Enumeration runs against `contracts/` excluding `contracts/mocks/` and `contracts/interfaces/` for verdict classification; a separate pass enumerates mocks sites for catalog completeness per D-09. Baseline was re-verified at execution time against the Plan 221-01 expected counts — no divergence.

**Enforcement:** `scripts/check-raw-selectors.sh` (installed by Plan 221-01 Task 2) runs the same four+one pattern scan as a regression gate, wired as a sibling prerequisite of `make test-foundry` and `make test-hardhat` alongside `check-interfaces` and `check-delegatecall`. Any future raw-selector or hand-rolled encoder introduction fails `make test` unless silenced by an inline `// raw-selectors: justified — <reason>` marker or added to the `JUSTIFIED_FEEDERS` allowlist. Both override paths are visible-diff PR-reviewed (per T-221-02 mitigation from the Plan 221-01 threat model).

## Findings

### INFO-221-01-01 — MockVRFCoordinator.sol:88 (Chainlink VRF v2 callback mimic)

Site: `contracts/mocks/MockVRFCoordinator.sol:88`. Construct: `abi.encodeWithSignature("rawFulfillRandomWords(uint256,uint256[])", requestId, words)`. Verdict: JUSTIFIED. Severity: INFO. Rationale: replicates the exact wire format Chainlink's production VRF v2 coordinator uses to invoke `rawFulfillRandomWords` on a `VRFConsumerBaseV2` subscriber. No repo-local interface exists for the coordinator-side caller because the coordinator is external. Test-only; excluded from gate via `EXCLUDE_PATHS=contracts/mocks`.

### INFO-221-01-02 — MockVRFCoordinator.sol:111 (Chainlink VRF v2 raw helper)

Site: `contracts/mocks/MockVRFCoordinator.sol:111`, function `fulfillRandomWordsRaw`. Construct and rationale identical to INFO-221-01-01 — same external standard, second helper entry point for multi-day test sequences. Verdict: JUSTIFIED. Severity: INFO.

### INFO-221-01-03 — MockLinkToken.sol:51 (ERC-677 onTokenTransfer mimic)

Site: `contracts/mocks/MockLinkToken.sol:51`, function `transferAndCall`. Construct: `abi.encodeWithSignature("onTokenTransfer(address,uint256,bytes)", msg.sender, value, data)`. Verdict: JUSTIFIED. Severity: INFO. Rationale: replicates the exact wire format the production Chainlink LINK token uses to invoke ERC-677 `onTokenTransfer` on receivers. The ERC-677 standard defines the selector; no Degenerus-owned interface can replace it.

### INFO-221-01-04 — DegenerusAdmin.sol:911→914 (ERC-677 transferAndCall, VRF swap path)

Site: `contracts/DegenerusAdmin.sol:911` (opener) / `:914` (payload), function `_executeSwap(proposalId)`. Construct: `linkToken.transferAndCall(newCoordinator, bal, abi.encode(newSubId))`. Verdict: JUSTIFIED. Severity: INFO. Rationale: Chainlink LINK's ERC-677 `transferAndCall` forwards an `abi.encode(subId)` payload to the receiver's `onTokenTransfer`, which is how LINK tops up VRF subscriptions. The `transferAndCall` payload format is fixed by Chainlink; no local interface binding is possible. Gate-silenced via `JUSTIFIED_FEEDERS=("DegenerusAdmin.sol:transferAndCall")`.

### INFO-221-01-05 — DegenerusAdmin.sol:997 (ERC-677 transferAndCall, subscription funding path)

Site: `contracts/DegenerusAdmin.sol:997`, function `onTokenTransfer(from, amount, ...)`. Construct: `linkToken.transferAndCall(coord, amount, abi.encode(subId))`. Verdict: JUSTIFIED. Severity: INFO. Rationale: same Chainlink ERC-677 wire format as INFO-221-01-04; different caller context — users deposit LINK to this contract via `transferAndCall`, and `onTokenTransfer` forwards the LINK onward to the VRF coordinator. Gate-silenced via the same `JUSTIFIED_FEEDERS` entry.

### INFO-221-01-06 — Regex-gate coverage limits (Known Limit T-221-01)

The `check-raw-selectors.sh` gate is a pattern-matcher. It cannot detect indirection — e.g., `bytes4 s = bytes4(uint32(0x12345678)); addr.call(abi.encodeWithSelector(s))` slips past Pattern A because the hex literal is wrapped in `uint32(` before `bytes4(`. This is an accepted residual risk tracked under T-221-01. Mitigation layers: (a) Phase 222's external-coverage audit exercises all entry points at runtime so unexercised paths cannot hide behind a masked selector; (b) `check-interfaces` / `check-delegatecall` gates already bound the surface where such masking would matter. Severity: INFO (residual risk only; no current finding).

**No HIGH findings in Phase 221.** The `mintPackedFor`-class runtime mismatch that motivated v27.0 has zero raw-selector analogue in the current tree.

## Known Limits

### T-221-01 — Regex-gate false negatives (ACCEPTED)

`check-raw-selectors.sh` is a pattern-matcher; it cannot detect a raw-selector bypass expressed via variable indirection, intermediate casts, or dynamic construction. Example: `bytes4 s = bytes4(uint32(0x12345678)); addr.call(abi.encodeWithSelector(s))` passes the gate because Pattern A anchors on `bytes4(` immediately followed by `0x`, and the hex literal is wrapped in `uint32(` first. Mitigation: Phase 222's coverage-gap audit exercises all external entry points at runtime, surfacing any unexercised path that could hide a masked raw selector. The gate is a first line of defense, not the only line. Severity-equivalent: INFO (tracked as INFO-221-01-06).

### T-221-04 — Catalog staleness (MITIGATED BY THE GATE)

If a new `abi.encode*` feeder is added to `contracts/` post-audit, this catalog becomes stale. Mitigation: `scripts/check-raw-selectors.sh` fails `make test` on the new site unless the author adds the inline `// raw-selectors: justified — <reason>` marker or extends the `JUSTIFIED_FEEDERS` allowlist. Both override paths are visible-diff PR-reviewed (per T-221-02 mitigation from Plan 221-01 threat model). Any PR adding an override MUST also update this catalog — enforcement is by reviewer discipline, but the gate guarantees no silent addition can merge and that the override bears a visible rationale.

### Constructs intentionally out of scope (per D-05)

The gate and this catalog do NOT flag:
- `abi.encode*(...)` whose output feeds `keccak256(...)` (hash-salt construction — dozens of sites, all legitimate; not calldata)
- `abi.encode*(...)` whose output feeds `string(...)` or is returned as `bytes memory` for NFT metadata / JSON token-URI construction (DegenerusDeityPass, DegenerusAffiliate `tokenURI` paths; not calldata)
- `abi.encodeWithSelector(IFace.fn.selector, ...)` — interface-bound, Phase 220-audited
- Direct interface-cast external calls `IFace(addr).fn(...)` — Phase 222-audited

These exclusions are encoded in the gate via Pattern E's low-level-call anchoring (Pattern E fires only when an `abi.encode*` call is the payload of `.call` / `.delegatecall` / `.staticcall` / `transferAndCall`) and via string-prefix stripping that rejects `abi.encodeWithSelector(` / `abi.encodeWithSignature(` / `abi.encodeCall(` before evaluating the bare `abi.encode*(...)` match.
