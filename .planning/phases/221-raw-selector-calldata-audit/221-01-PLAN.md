### Call-receiver patterns the gate must recognize

Solidity call syntax variants for the "payload passed to a low-level call" check:
```

addr.call{value: x}(abi.encode*(...))
addr.call{gas: g, value: v}(abi.encode*(...))
addr.delegatecall(abi.encode*(...))

The existing Phase 220 `abi.encodeWithSelector(IFace.fn.selector, ...)` pattern is intentionally OUT OF SCOPE of this gate — it is interface-bound calldata, already audited. ```bash
# Returns 0 (true) if the source line OR either of the two preceding lines

is_justified() {
  local file="$1" lineno="$2"
  local start=$(( lineno > 2 ? lineno - 2 : 1 ))
  awk -v s="$start" -v e="$lineno" 'NR >= s && NR <= e' "$file" \
    | grep -q 'raw-selectors: justified'
}
```

**Implementation — two-pass awk per file to avoid `getline` NR-advancement skipping back-to-back openers:**

The subtle bug with `getline` inside the main `{...}` action: it advances NR by however many lines it peeks ahead. If opener X is at NR=N and opener Y is at NR=N+2, after X's action `getline`s 3 lines forward, NR becomes N+3 and Y at N+2 is never re-evaluated. Two-pass approach: slurp the whole file into an array first, then scan the array without `getline`.

```bash
# Pattern E — abi.encode*(...) feeding low-level call. Two-pass awk per file
# so back-to-back openers within 4 lines are both evaluated. Outputs file:line
# tuples for sites that need a verdict (allowlist + justified-marker checked
# in the bash post-processing loop).
find "$CONTRACTS_DIR" -name '*.sol' -type f 2>/dev/null | while read -r file; do
  # Skip excluded paths (match if $file begins with any EXCLUDE_PATHS entry).
  skip=0
  for excl in "${EXCLUDE_PATHS[@]}"; do
    if [[ "$file" == "$excl"/* ]]; then skip=1; break; fi
  done
  (( skip )) && continue

  awk -v file="$file" '
    # Pass 1: slurp every line into lines[NR].
    { lines[NR] = $0 }
    END {
      # Pass 2: for each line in the file, test if it is a low-level call
      # opener. If yes, peek the 4-line window starting at opener (inclusive)
      # and test for a bare abi.encode*( that is NOT already handled by
      # Patterns C/D (encodeWithSignature / encodeCall / encodeWithSelector).
      for (n = 1; n <= NR; n++) {
        if (lines[n] !~ /\.(call|delegatecall|staticcall|transferAndCall)[[:space:]]*\(/) continue

        # Assemble 4-line window: [opener .. min(opener+3, NR)].
        window = lines[n]
        for (j = 1; j <= 3 && (n + j) <= NR; j++) {
          window = window "\n" lines[n + j]
        }

        # Strip the three already-handled encode-forms from consideration.
        # Replace each occurrence with placeholder text so the remaining
        # pattern scan sees only unhandled abi.encode*(...) forms.
        stripped = window
        gsub(/abi\.encodeWithSelector[[:space:]]*\(/, "HANDLED_EWS(", stripped)
        gsub(/abi\.encodeWithSignature[[:space:]]*\(/, "HANDLED_EWSIG(", stripped)
        gsub(/abi\.encodeCall[[:space:]]*\(/, "HANDLED_ECALL(", stripped)

        # After stripping, any remaining `abi.encode(` or `abi.encodePacked(`
        # is a Pattern E violation candidate. Emit the opener-line tuple.
        if (stripped ~ /abi\.encode(Packed)?[[:space:]]*\(/) {
          print file ":" n
        }
      }
    }
  ' "$file"
done | sort -u | while IFS=: read -r file lineno; do
  [[ -z "${file:-}" ]] && continue

  # Extract the opener-line content for the content-based allowlist check.
  opener_content=$(awk -v n="$lineno" 'NR == n' "$file")

  if is_justified "$file" "$lineno"; then
    printf "%bJUST%b %s:%s  abi.encode*(...) payload of low-level call — justified by marker\n" \
      "$YELLOW" "$NC" "$file" "$lineno"
    justified_total=$((justified_total + 1))
  elif is_justified_feeder "$file" "$opener_content"; then
    printf "%bJUST%b %s:%s  abi.encode*(...) payload of low-level call — justified by allowlist\n" \
      "$YELLOW" "$NC" "$file" "$lineno"
    justified_total=$((justified_total + 1))
  else
    printf "%bFAIL%b %s:%s  abi.encode*(...) payload of low-level call — CSI-06 violation (interface-bound form required)\n" \
      "$RED" "$NC" "$file" "$lineno"
    fail_total=$((fail_total + 1))
  fi
done
```

The `gsub`-based strip is the critical correctness mechanism: by replacing each already-handled `abi.encodeWithSelector(` / `abi.encodeWithSignature(` / `abi.encodeCall(` with a placeholder token BEFORE the Pattern E match test, we avoid double-FAILs when a window contains both a handled form and a raw `abi.encode(` (unlikely in practice but still correct).

Two-pass awk (slurp into `lines[]`, then scan in END) has zero `getline` NR side-effects, so every line is re-evaluated as a potential opener regardless of how many preceding windows peeked ahead.

# Entries are "<file_basename>:<call_receiver_or_method>" pairs — if Pattern E
