## Known Limits

### T-221-01 — Regex-gate false negatives (ACCEPTED)

The `check-raw-selectors.sh` gate is a pattern-matcher; it cannot detect a raw-selector bypass expressed via indirection — e.g., `bytes4 s = bytes4(uint32(0x12345678)); addr.call(abi.encodeWithSelector(s))` passes the gate because the hex literal is wrapped in `uint32(` before reaching `bytes4(`. Mitigation: Phase 222's coverage-gap audit exercises all external entry points at runtime, surfacing any unexercised path that a cleverly-hidden raw selector could hide behind. The gate is a first line of defense, not the only line.

### T-221-04 — Catalog staleness (MITIGATED BY THE GATE)

If a new `abi.encode*` call-feeder is added to `contracts/` post-audit, this catalog becomes stale. Mitigation: the gate in `scripts/check-raw-selectors.sh` fails `make test` on the new site unless it carries the inline `// raw-selectors: justified — <reason>` marker or an entry is added to `JUSTIFIED_FEEDERS`. Both override paths are visible-diff PR-reviewed (per T-221-02 mitigation from Plan 221-01 threat model). Any PR adding an override MUST also update this catalog — enforcement is by reviewer discipline, but the gate ensures no silent additions can merge.

### Constructs intentionally out of scope (per D-05)

The gate and this catalog do NOT flag:
- `abi.encode*(...)` whose output feeds `keccak256(...)` (hash-salt construction — 40+ sites, all legitimate)
- `abi.encode*(...)` whose output feeds `string(...)` or is returned as `bytes` memory for NFT metadata / JSON-token-URI construction (DegenerusDeityPass, DegenerusAffiliate tokenURI paths)
- `abi.encodeWithSelector(IFace.fn.selector, ...)` — interface-bound, Phase 220-audited
- Direct interface-cast external calls `IFace(addr).fn(...)` — Phase 222-audited

These exclusions are encoded in the gate via Pattern E's low-level-call anchoring (which only triggers when an `abi.encode*` is the payload of `.call` / `.delegatecall` / `.staticcall` / `transferAndCall`) and via string-prefix matching that rejects `abi.encodeWithSelector` before evaluating Pattern E.
