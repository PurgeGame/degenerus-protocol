**Section 2a: Daily VRF Path Timeline**

Trace the complete adversarial timeline across blocks:

1. Block N: `advanceGame()` calls `_requestRng()`, sets `rngLockedFlag=true`. VRF request sent.
   - Block builder capabilities: Can delay advanceGame call but cannot prevent it indefinitely. Cannot predict VRF word.
   - User capabilities: Purchases go to write buffer. 12 entry points blocked by rngLockedFlag. List what IS still callable.

2. Block N+K: `rawFulfillRandomWords()` stores `rngWordCurrent`. rngLockedFlag still true.
   - Block builder capabilities: Can read VRF callback payload from mempool. Can order user txs before/after callback within the block. BUT rngLockedFlag is still true so no RNG-influencing entry points are callable.
   - Critical analysis: Even though block builder knows the word, they cannot use it because all RNG-influencing actions are blocked.

3. Block N+K+M: `advanceGame()` consumes word in `rngGate`. All 9 daily consumption points execute atomically.
   - Block builder capabilities: Can sandwich advanceGame with other txs, but consumption is atomic within the call. No user tx can interleave with the 9 consumption steps.

**Section 2b: Mid-Day Lootbox VRF Path Timeline**

Trace for lootbox VRF:

1. Block N: `requestLootboxRng()` sends VRF request. rngLockedFlag=false, rngRequestTime!=0.
   - User capabilities: Full — purchases, lootbox opens (for prior words), coinflip, etc. But these operate on PRIOR lootbox words, not the pending one.

2. Block N+K: `rawFulfillRandomWords()` finalizes word to `lootboxRngWordByIndex[index]`.
   - Block builder capabilities: Can read word from mempool. Can order user txs before/after callback.
   - Critical analysis: Per-player entropy `keccak256(word, player, day, amount)` means knowing the word doesn't let the builder predict another player's outcome. The builder CAN predict their OWN outcome if they also control a player address. Analyze whether self-front-running yields an advantage.

3. Block N+K+M: Users call `openLootBox`/`resolveBets`/etc.
   - Standard MEV: Block builder could theoretically observe outcome and decide inclusion/exclusion, but outcomes are per-player deterministic — including or excluding the tx doesn't change the outcome for future txs.
