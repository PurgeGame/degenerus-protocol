**Section 3a: 5-Day Jackpot Phase State Machine**

Trace the exact state of rngLockedFlag and prizePoolFrozen across each of the 5 jackpot days by reading `advanceGame` in AdvanceModule. For each day:

- When rngLockedFlag goes false (_unlockRng)

- jackpotPhaseFlag state

Document the inter-block gap between each day: the period after _unlockRng clears rngLockedFlag and before the next day's advanceGame call sets it again.

**Section 3b: Available Actions During Inter-Block Gaps**

Using Phase 12 entry point callability matrix and guard analysis, enumerate every entry point callable when rngLockedFlag=false AND prizePoolFrozen=true. For each callable entry point:
- Does it modify any co-state used by NEXT day's jackpot draws?
- If yes, can the modification influence outcomes?

**Resolve Open Question 1 (deity pass purchase):**
Read the `purchaseDeityPass` code path in DegenerusGameMintModule.sol. Trace every guard condition. Determine if purchaseDeityPass is blocked by ANY condition during jackpot phase (jackpotPhaseFlag, prizePoolFrozen, or other). If callable:
- deityPassOwners and deityBySymbol are co-state for jackpot winner selection
- Analyze: Can purchasing a deity pass between Day N and Day N+1 change who wins Day N+1's jackpot draw?
- Consider: The word for Day N+1 is not yet known (fresh VRF), so the attacker cannot predict which deity symbol wins

**Resolve Open Question 2 (reverseFlip / nudge during jackpot):**
Read `reverseFlip` code path. Confirm it increments `totalFlipReversals`. During jackpot inter-block gap, this nudge affects Day N+1's finalized word via `_applyDailyRng`.
- Economic analysis: Each reverseFlip costs 100+ BURNIE (compounding). The nudge adds an integer offset to a uint256 VRF word. The probability of shifting a modular outcome (e.g., `word % bucketCount`) by a controlled amount is approximately `nudge_count / 2^256` for a specific target — astronomically small.
- Calculate: Even 1 million nudges (cost: astronomical BURNIE) shifts probability by ~10^-72. This is SAFE BY DESIGN (economic infeasibility + cryptographic margin).

**Resolve Open Question 3 (processTicketBatch entropy):**
Read `_swapTicketSlot` and `processTicketBatch` within `advanceGame` for jackpot-phase days. Trace which `lastLootboxRngWord` value is used:
- Is it the piggybacked daily word (from _finalizeLootboxRng in rngGate)?
- Or the most recent independent lootbox VRF word?
- Does it matter for manipulation? (Likely no — trait assignment is independent of winner selection)

**Section 3c: Cross-Day State Mutation Analysis**

For each type of state change possible during inter-block gaps, assess whether it creates a manipulation path:
- New ticket purchases (write buffer) — processed with unknown future VRF
- Lootbox opens — use prior committed words, per-player entropy
- Coinflip deposits — BURNIE burns, no pool/RNG interaction (Phase 13 confirmed SAFE)
- Deity pass purchases — changes co-state but next word unknown
- Nudges — economically infeasible probability shift
