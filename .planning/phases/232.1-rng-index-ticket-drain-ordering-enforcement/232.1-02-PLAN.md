The handler must be capable of producing the SPEC §Background bug window: "VRF fulfilled but read slot undrained". This is achieved by:
1. Queue tickets at level N (handler.queueTickets(N, count))
2. Call advance() which triggers _swapAndFreeze + LR_INDEX bump from X to X+1 + VRF request
3. Mock VRF fulfillment lands rngWordCurrent (handler.mockVrfFulfill())
4. Call advance() AGAIN -- this is where the bug manifested pre-fix (drain at L242-262 reads lootboxRngWordByIndex[X+1-1]=lootboxRngWordByIndex[X])

The slot at index X was never written by the daily callback (only mid-day callbacks write directly per L1632); rngGate would write it later in the SAME tx, AFTER the L242-262 drain. The fix populates it lazily at the gate.
