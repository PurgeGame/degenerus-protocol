## Theoretical Worst-Case Gas

Derived analytically BEFORE empirical measurement, per `feedback_gas_worst_case.md`.

| Path | Gates added | Storage ops | Other | Total overhead |
|---|---|---|---|---|
| Hot (slot already populated) | 1 SLOAD + 1 JUMPI | 0 SSTORE | — | ~2100 (cold) / ~100 (warm) |
| Cold (slot zero, rngWordCurrent non-zero) | 2 SLOAD + 1 SSTORE + 1 LOG3 | 1 SSTORE 0→nonzero (22100) | LOG3 ~2700 | net ≈ SLOAD only (SSTORE + LOG subsume rngGate's later write) |
| Stall (slot zero, rngWordCurrent zero) | 1 SLOAD + revert | 0 | revert | gas consumed up to revert; bounty not paid |

**Derivation:**

- The storage slot for `LR_INDEX` (packed in the `_lrRead(LR_INDEX_SHIFT, LR_INDEX_MASK)` SLOAD) is already touched earlier in the function at L191 (mid-day guard). On the daily-drain flow (`LR_MID_DAY != 0` check miss), that SLOAD did NOT execute, so LR_INDEX is cold on first post-fix access. EIP-2929 prices a cold SLOAD at 2100 gas; a warm SLOAD at 100.
- `lootboxRngWordByIndex[idx]` is a mapping slot (separate access-list entry from any packed state). On the daily-drain path, this is the first read of this slot in the tx — cold SLOAD, 2100 gas.
- `rngWordCurrent` is a plain state variable. Cold on first access in the cold-path branch. SLOAD ~2100 gas.
- `_finalizeLootboxRng(cw)` writes `lootboxRngWordByIndex[index] = rngWord`. The slot is zero pre-write; EIP-2200 SSTORE 0→nonzero costs 22100 gas. It then emits `LootboxRngApplied(index, rngWord, vrfRequestId)` — LOG3 (3 topics + data) ≈ base 1500 + 3*375 topics + data ≈ ~2700.
- **Cold-path net cost vs pre-fix:** In pre-fix code the SSTORE + LOG3 were paid later in the same tx inside rngGate's `_finalizeLootboxRng` call. The fix moves that work earlier in the tx without duplicating it (the idempotent short-circuit `if (lootboxRngWordByIndex[index] != 0) return;` at L1129 makes rngGate's later call a no-op). Net new cost is the 2 SLOADs only.
- **Stall-path:** revert consumes gas up to the revert opcode; caller is charged. Identical liveness profile to the existing L194 / L219 reverts (per CONTEXT.md D-02 and §Code Context "Caller-retry liveness"). Bounty is NOT paid.
- **Hot-path:** slot is already non-zero (the common post-VRF case once rngGate has run at least once in the tx's sequence). Only 1 new SLOAD + 1 JUMPI. Cold SLOAD ≈ 2100 gas; warm ≈ 100 gas.

**+5% budget derivation per SPEC §Constraints:** The happy-path advanceGame call cost varies by test scenario. From the pre-fix VRFCore.t.sol gas report (8638 calls across 22 tests exercising VRF fulfillment): min=16,684, avg=931,139, median=522,921 on the module-level advanceGame entry. A +2100 gas overhead on the happy path is +0.4% of avg and +12.6% of min. On any realistic production call (deploy + daily drain + RNG request, ≥ ~150k gas), +2100 is comfortably within +5%.
