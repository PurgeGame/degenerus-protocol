## Entropy Non-Zero Proof Chain (structural summary)

For every call site above, the non-zero-entropy guarantee reduces to the following structural chain:

1. **VRF callback zero-guards the raw word.** `rawFulfillRandomWords` at AdvanceModule L1690-1711:
   ```solidity
   uint256 word = randomWords[0];
   if (word == 0) word = 1;                       // L1698 â€” structural non-zero guard
   if (rngLockedFlag) {
       rngWordCurrent = word;                     // L1702 â€” daily-RNG branch
   } else {
       // Mid-day branch â€” stores directly to lootboxRngWordByIndex[index]; also non-zero by the L1698 guard
       lootboxRngWordByIndex[index] = word;
   }
   ```
   The line `if (word == 0) word = 1;` at L1698 is the root of the non-zero guarantee for every downstream consumer.

2. **`rngWordCurrent` is non-zero when non-empty.** After L1702, `rngWordCurrent` holds a non-zero value until cleared by `_applyDailyRng` at L1785 (which first applies nudges and re-stores the result â€” still non-zero by construction; see step 4) or by `_unlockRng` / end-of-day housekeeping (which zero it, but only AFTER `_applyDailyRng` has consumed and emitted the value; see call chain in `rngGate`).

3. **`rngGate` returns one of three values â€” only non-zero words reach the do-while body.**
   - Path A: `rngWordByDay[day] != 0` early return at L1141 â€” returns a previously-recorded daily word (non-zero by construction; see step 4).
   - Path B: `currentWord != 0 && rngRequestTime != 0` at L1146 â€” returns `_applyDailyRng(day, currentWord)` at L1164 (non-zero; see step 4).
   - Path C: waiting for VRF at L1186 â€” either returns sentinel `(1, 0)` on timeout retry at L1191, sentinel `(1, 0)` on fresh request at L1198, or reverts `RngNotReady()` at L1193.
   The do-while body at L291 then tests `if (rngWord == 1) { ... break; }`, which exits the loop before any `_processFutureTicketBatch` call site. Every `rngWord` flowing past L291 is neither `0` (never returned) nor `1` (trapped by break), therefore strictly `> 1`, therefore non-zero.

4. **`_applyDailyRng` preserves non-zero.** AdvanceModule L1777-1789:
   ```solidity
   uint256 nudges = totalFlipReversals;
   finalWord = rawWord;                   // non-zero on entry (from rngWordCurrent which was guarded at L1698)
   if (nudges != 0) {
       unchecked { finalWord += nudges; } // 2^256-arithmetic; still non-zero
       totalFlipReversals = 0;
   }
   rngWordCurrent = finalWord;            // store the finalized value
   rngWordByDay[day] = finalWord;         // memoize for same-day re-read at L1141
   ```
   Non-zero input â†’ non-zero output. The `unchecked` addition of `totalFlipReversals` preserves non-zero except under a 2^256 collision which requires inverting the VRF word â€” cryptographically infeasible.

5. **Plan 01 pre-drain gate closes the last structural hole.** Before Plan 01, the daily-drain path at the former L242-262 could read `lootboxRngWordByIndex[LR_INDEX-1]` while still zero (VRF request issued but not yet fulfilled), and pass 0 through `processTicketBatch` â†’ `_raritySymbolBatch`. Plan 01 Revision 2 added the pre-drain gate at AdvanceModule L257-279 inside the do-while loop; Plan 01 Revision 4 aligned the gate's revert selector to `RngNotReady()` at L263 when `rngWordCurrent == 0`. When `rngWordCurrent` is zero (VRF pending), the gate now reverts and the drain does not proceed; this means any `rngWord` local returned by `rngGate` at L283 and used at L315 / L407 is sourced from a `rngWordCurrent` that was already non-zero at the entry to the do-while iteration (or from `rngWordByDay[day]` if the day was already processed â€” non-zero by step 3/4). The `processFutureTicketBatch` call sites never receive an `entropy == 0` from any reachable path.
