### Peak Gas & Safety Margin

- **Peak gas:** 6,275,799 gas (Phase Transition, stage=2)
- **Safety margin:** 30,000,000 / 6,275,799 = **4.78x** (threshold: >= 1.5x)
- **Prior baseline (v15.0):** 7,023,530 gas / 1.99x margin (measured via forge gas-report on advanceGame worst-case, Phase 167)
- **Delta from prior:** -747,731 gas (-10.6%) -- peak gas DECREASED
- **Verdict:** **SAFE**

The peak gas of 6,275,799 provides a 4.78x safety margin against the 30M block gas limit. This exceeds the 1.5x threshold by more than 3x.

Note on baseline comparison: The prior v15.0 baseline of 7,023,530 gas was measured via Foundry's `forge test --gas-report` on a static worst-case scenario. The current measurement uses the Hardhat gas benchmark harness (test/gas/AdvanceGameGas.test.js) which drives advanceGame through every stage via dynamic game state progression. The measurement methodologies differ but both represent worst-case per-call gas. The current Hardhat harness provides more granular per-stage visibility.

### New Jackpot Path Coverage

The v23.0 changes from commits 93c05869 and 520249a2 introduced these new code paths in JackpotModule:

| New Code Path | Benchmark Coverage | Notes |
|---|---|---|
| Specialized events (JackpotDailyNormalWinner, JackpotDailySoloWinner, etc.) | Covered | These replace the old JackpotTicketWinner event. Every jackpot payout path in the benchmark (stages 4, 7, 9, 10) exercises these events. Gas impact is neutral -- same log data, different event signature. |
| Whale pass daily-only path restriction | Covered | The daily jackpot benchmark (stage=4 "Jackpot Daily ETH") exercises payDailyJackpot which contains the whale pass path. The restriction REMOVES code (no whale pass BAF payout), reducing gas. |
| DGNRS solo reward fold into daily ETH | Covered | The DgnrsSoloBucketReward.test.js confirms the fold path works. In the gas benchmark, the daily jackpot path (stage=4) exercises this code. The fold consolidates two payouts into one, reducing gas. |

All three new code paths are exercised by the existing benchmark. None increase worst-case gas:
- Specialized events: Same calldata, different selector (neutral)
- Whale pass daily-only: Removes a code path (reduces gas)
- DGNRS fold: Consolidates payouts (reduces gas)

The peak gas (6,275,799 on Phase Transition) occurs in `_consolidatePoolsAndRewardJackpots` which calls `runBafJackpot`. This path was measured directly and is well within limits.
