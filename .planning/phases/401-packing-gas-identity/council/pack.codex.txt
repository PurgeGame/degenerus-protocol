**Verdict**

Not clean. Storage layout and delegatecall/getter identity mostly hold, but I found real narrowing/RMW divergences.

I could not run `forge inspect` directly because this checkout is read-only and Foundry tried to write artifacts. I used the existing `DegenerusGame` storage-layout artifact and `solc --storage-layout` stdout for the other contracts.

**Findings**

1. [contracts/BurnieCoinflip.sol:1184](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:1184) — `coinflipStakePacked` lane write is not lossless for over-wide stake values.
   Invariant: each day stake lane is 128 bits and must preserve the sibling day.
   Code: `_setFlipStake` clears one 128-bit lane, then ORs raw `(weiAmount << shift)` without masking or rejecting `weiAmount > type(uint128).max`.
   Writers: `_addDailyFlip` sums `prevStake + coinflipDeposit` at [674-678](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:674); `creditFlip` accepts `uint256 amount` at [970-975](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:970); direct deposits add quest/recycle/boon credit at [310-323](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:310) and [660-668](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:660).
   Effect: for even-day lane, high bits spill into the sibling day; for odd-day lane, high bits are truncated out of the slot. This is both truncation and co-resident clobber.

2. [contracts/BurnieCoinflip.sol:758](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:758), [761](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:761), [796](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:796) — `autoRebuyStop` silently truncates user input.
   Invariant: `uint128 autoRebuyStop` must be at least the maximum reachable `takeProfit`.
   Code: external `takeProfit` is `uint256` and is cast directly to `uint128`.
   Effect: a caller can set `takeProfit > 2^128-1`; storage wraps while the event emits the original value, so later auto-rebuy settlement uses a different threshold than the caller configured.

3. [contracts/BurnieCoinflip.sol:632](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:632) — `autoRebuyCarry` is not bounded to `uint128`.
   Invariant: `uint128 autoRebuyCarry` must cover the maximum rolling carry.
   Code: claim settlement compounds `carry` into `stake`, then `payout = stake + stake * rewardPercent / 100`, with reward up to 156% at [861-883](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:861), then stores `uint128(carry)`.
   Effect: a run of winning resolved days can grow carry above `uint128.max`; the stored carry truncates high bits. Related `claimableStored` additions at [279](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:279), [828](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:828), and [1011](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/BurnieCoinflip.sol:1011) have the same unguarded narrowing pattern for accumulated mintable value.

4. [contracts/modules/GameAfkingModule.sol:905](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/modules/GameAfkingModule.sol:905) — `Sub.amount` can truncate the afking lootbox stamp.
   Invariant: `uint24 amount` stores milli-ETH, so max is `16,777.215 ETH`.
   Code: `_resolveBuy` explicitly allows uncapped reinvest from claimable at [716-722](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/modules/GameAfkingModule.sol:716), derives cost/amount at [724-735](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/modules/GameAfkingModule.sol:724), then writes `uint24(_packEthToMilliEth(amount))`. `_packEthToMilliEth` itself returns `uint64` at [contracts/storage/DegenerusGameStorage.sol:1575](/home/zak/Dev/PurgeGame/degenerus-audit/contracts/storage/DegenerusGameStorage.sol:1575).
   Effect: if a claimable-funded lootbox auto-buy exceeds `16,777.215 ETH`, the actual debit remains full wei value, but the later box stamp records a wrapped milli-ETH amount.
