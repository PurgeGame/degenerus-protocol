**useSimulation.ts hook:**
- State: `config` (Partial&lt;SimConfig&gt;), `result` (SimResult | null), `snapshots` (DaySnapshot[]), `running` (boolean), `error` (string | null)
- `updateConfig(patch)`: merges partial config
- `runSimulation()`: creates engine via `createEngine(config)`, runs step-by-step collecting DaySnapshot after each `step()` call (capture global state + player states, converting bigint pools/balances to ETH-as-number). Runs in a microtask loop using `requestAnimationFrame` or `setTimeout(0)` batches of ~50 steps to avoid blocking UI. Sets `running=true` during execution, `running=false` + populates `result` and `snapshots` on completion.
- The step-by-step snapshot collection is KEY -- this is what powers the time-series charts. Each snapshot captures the full state at that day.
- Default config from `DEFAULT_CONFIG` but with `levelCount: 5` (fast for interactive use).

**ParameterPanel.tsx:**
- Renders controls for: seed (number input), playerCount (slider 10-500, default 100), levelCount (slider 1-20, default 5), archetype sliders (degen/evMax/whale/hybrid as 4 linked sliders that sum to 1.0 -- when one changes, normalize the rest proportionally), affiliateRate (slider 0-1), stEthDailyBps (number input, default 10), initialEthBalance (number input in ETH, converted to wei bigint).
- "Run Simulation" button (disabled while running, shows spinner during run).
- Compact layout: labels above inputs, sliders with value display, grouped in a sidebar-friendly column.
- Calls `updateConfig` on changes and `runSimulation` on button click.

**DashboardShell.tsx:**
- Responsive CSS Grid layout: sidebar (280px fixed) + main area (1fr).
- Sidebar: ParameterPanel at top, simulation status below (days simulated, levels completed, total events).
- Main area: 2x2 grid of chart slots (children passed as props or via named slots). Each slot has a header and a placeholder "Chart: {name}" div.
- Below the 2x2 grid: a full-width slot for the drill-down panel.
- Mobile: stack sidebar above main area (single column).
- Use CSS modules or inline styles (no Tailwind -- keep dependencies minimal for embeddability).

**App.tsx:**
- Instantiate useSimulation hook.
- Render DashboardShell with ParameterPanel in sidebar.
- Pass snapshots/result to chart slot placeholders (charts will be implemented in plan 41-02).
- Show "Run a simulation to see results" message when result is null.
