**WealthChart.tsx** (VIZ-02):
- Props: `{ snapshots: DaySnapshot[], onPlayerClick?: (playerId: number) => void }`
- Multi-line time-series chart (x = day, y = ETH).
- Default view: aggregate wealth by archetype (4 lines, one per archetype, each is sum of ethBalance + claimable for all players of that archetype).
- Toggle: "Per-player" mode shows individual thin lines colored by archetype (may have 100+ lines -- use low opacity).
- Each line is clickable -- calls `onPlayerClick(id)` for per-player mode.
- Y-axis: ETH. X-axis: Day. Level transition markers via `addLevelMarkers`.
- Tooltip on hover: shows day, level, and value.

**PoolChart.tsx** (VIZ-03):
- Props: `{ snapshots: DaySnapshot[] }`
- Stacked area chart (x = day, y = ETH) showing the 4 pools.
- Use `d3.stack()` with pool keys. Colors from POOL_COLORS.
- Legend below chart showing pool names with colored squares.
- Tooltip on hover: shows all 4 pool values at that day.
- Level transition markers.

**BurnieChart.tsx** (VIZ-04):
- Props: `{ snapshots: DaySnapshot[], events: SimEvent[] }`
- Dual-axis chart: left axis = BURNIE supply (line), right axis = coinflip volume (bar or area).
- BURNIE supply: continuous line from `snapshot.burnieSupply`.
- Coinflip volume: aggregate from events of type `coinflip_result` grouped by day. If no coinflip events exist yet (Phase 38 not done), show supply-only with a note "Coinflip data requires Phase 38".
- Implied price: if ticket price and BURNIE cost are known, compute ratio. For now, show supply only as the primary metric -- implied price can be derived later when BURNIE ticket purchasing is implemented.

**ActivityChart.tsx** (VIZ-05):
- Props: `{ snapshots: DaySnapshot[], selectedDay?: number }`
- Histogram of activity scores at a selected day (default: last day).
- Bins: 0-5000 BPS in 500-BPS increments (about 60 bins covering 0-30500 BPS range, or adapt to actual data range).
- Bars colored by archetype composition (stacked bars showing how many of each archetype fall in each bin).
- Day slider below the chart to scrub through time.
- X-axis: Activity score (BPS). Y-axis: Player count.
