## LANDMINE — VTST-04 assertion shape (LOCKED here per planning brief, per 312-01-SUMMARY Deviations §1)

Phase 312 DELIBERATELY OMITTED the `wireVrf` init-only runtime lock that the 311 SPEC originally mandated (user-approved deviation). There is NO `if (address(vrfCoordinator) != address(0)) revert E();` guard in the contract. A test asserting that specific init-lock revert WILL FAIL. The reason the omission is sound: `wireVrf` is reachable ONLY from the DegenerusAdmin constructor (`:458`), gated by the impl's `:503` `msg.sender != ContractAddresses.ADMIN` check; DegenerusAdmin exposes no post-construction forwarder, so a second `wireVrf` with `msg.sender == ADMIN` is TOPOLOGICALLY UNREACHABLE.

Therefore VTST-04 proves the one-shot property in the form that ACTUALLY EXISTS:

1. **Access-guard revert (the "second wire reverts" provable shape):** a `wireVrf` call from any NON-ADMIN caller reverts via the `:503` ADMIN guard. Assert both the direct `DegenerusGame.wireVrf` entry and the routed reach revert for a non-ADMIN sender. (This is the Foundry analog of `CoverageGap222.t.sol:test_gap_game_vrf_admin_paths`, which already proves the non-admin revert for both `wireVrf` and `updateVrfCoordinatorAndSub`.)
2. **Routed-reach companion (VRF-05):** `updateVrfCoordinatorAndSub` from a non-ADMIN sender also reverts (the same `:1693` ADMIN guard at the delegatecall target), proving the routed admin dispatch is covered.
3. **Structural one-shot attestation:** a source-level grep confirms `DegenerusAdmin.sol` reaches `wireVrf` only from its constructor (`gameAdmin.wireVrf` appears once, inside the constructor near `:458`) and exposes NO post-construction forwarder/multicall/delegatecall that could re-emit it — so the ADMIN/constructor path is one-shot by construction. This attests that "a second wire" cannot happen even from the ADMIN sender.

Do NOT assert an init-lock exists. Do NOT use `vm.expectRevert` against a specific init-lock selector. Use the access-guard form + the structural attestation.
