<objective>
Rewrite the AFKING BOX OPEN to the 349.1 live-level / day-keyed-word shape — the
freeze-critical core. Two coupled sides (caller in GameAfkingModule, callee in
LootboxModule):

**LootboxModule.resolveAfkingBox (§1, §2, §5).** Collapse it to ≈
`resolveLootboxDirect` (the auto-resolve, LIVE-level twin) with exactly TWO
deviations from that function:
1. the RNG word is a CALLER-PASSED param (the GameAfkingModule passes
   `rngWordByDay[lastAutoBoughtDay]`), NOT read from `lootboxRngWordByIndex[index]`
   — so the `index` param is DROPPED;
2. the box seed `keccak256(rngWord, player, day, amount)` uses the FROZEN stamped
   `day` (a passed param), NOT the live `_simulatedDayIndex()` — the day MUST stay
   frozen in the seed or a self-keepering player could grind the seed by
   open-timing (§1).
Everything else is identical to `resolveLootboxDirect`: roll `targetLevel =
_rollTargetLevel(level + 1, seed)` from the LIVE level, the SINGLE
`_applyEvMultiplierWithCap(player, level+1, amount, …)` RMW at open (live, the
accepted sole live-read), and `_resolveLootboxCommon(...)` with the SAME tail
flags (`emitLootboxEvent = true`, `payColdBustConsolation = true`).

**GameAfkingModule open-leg (§5).** `_afkingBoxReady(sub)` becomes
`sub.lastOpenedDay < sub.lastAutoBoughtDay && rngWordByDay[sub.lastAutoBoughtDay]
!= 0`. `_openAfkingBox` is effects-first: set `sub.lastOpenedDay =
sub.lastAutoBoughtDay`, then call `resolveAfkingBox(player, amount,
/*day=*/lastAutoBoughtDay, rngWordByDay[lastAutoBoughtDay], scorePlus1 - 1)`. Both
drop all `_unpackAfkingEpoch` / `lastOpenedIndex` / `baseLevel` plumbing (the live
roll needs no stored baseLevel floor).

Purpose: this is why the redesign exists — auto-opened boxes don't need the
level/day freezes a human `openLootBox` needs (the player can't time the open;
the permissionless bounty opens it). Live level + frozen-day word is the §2
structurally-safe model. Do NOT re-litigate the freeze proof (§2) or the
skip-vs-force-open decision (§3) — they are LOCKED.

Output: revised `resolveAfkingBox` (LootboxModule) + `_openAfkingBox` /
`_afkingBoxReady` (GameAfkingModule).

NOTE — REVISION of a HELD diff. `read_first` the actual file; anchors are
guidance. This plan depends on plan 02 (the STAGE) and plan 01 (the
`lastOpenedDay` field). Built once at the end (plan 05).

    if (amount == 0) return;
    uint32 day = _simulatedDayIndex();              // afking box uses the PASSED frozen day instead
