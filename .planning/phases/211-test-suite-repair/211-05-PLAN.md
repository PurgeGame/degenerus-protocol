**Root cause analysis (verified via debug testing):**

The _driveToLevel helper fails because:
1. It calls advanceGame in a tight inner loop that breaks on ANY revert (including NotTimeYet after daily processing completes on the same day).
2. It only calls _fulfillVrfIfPending ONCE per inner iteration but the game may generate NEW VRF requests during phase transitions (rngGate requests VRF when rngWordCurrent is consumed).
3. The phaseTransitionActive state requires additional VRF-advance cycles across MULTIPLE days to complete.
4. After VRF fulfillment, the game needs multiple advanceGame calls to: process tickets, pay daily jackpot, check target, consolidate pools, enter jackpot phase, process jackpot days (5 days x VRF each), then complete phase transition.

The game loop for one complete level cycle:
- Purchase phase: daily VRF + ticket processing + daily jackpot. When target met, lastPurchaseDay=true.
- Next day with lastPurchaseDay=true: VRF + ticket processing + pool consolidation -> jackpotPhaseFlag=true.
- Jackpot phase: 5 days of daily VRF + daily jackpot draws. jackpotCounter increments each day.
- After 5 jackpots: phaseTransitionActive=true, rngLocked cleared.
- Phase transition: next day VRF + FF ticket drain -> level increments, purchaseStartDay updated.

**Fix for _driveToLevel (TicketLifecycle.t.sol):**

Replace the _driveToLevel function (around line 2283) with this implementation:

```solidity
function _driveToLevel(uint256 targetLevel) internal {
    uint256 simTime = block.timestamp;

    for (uint256 day = 0; day < 500; day++) {
        if (game.level() >= targetLevel) break;
        if (game.gameOver()) break;

        simTime += 1 days + 1;
        vm.warp(simTime);

        // Seed prize pool for fast transitions
        _seedNextPrizePool(49.9 ether);

        // Buy tickets to push over prize pool target and populate queues
        _buyTickets(buyer1, 4000);
        _buyTickets(buyer2, 2000);

        // Drive advanceGame + VRF until nothing more to do today
        for (uint256 j = 0; j < 80; j++) {
            // Fulfill any pending VRF BEFORE calling advanceGame
            _fulfillVrfIfPending();

            (bool ok, ) = address(game).call(
                abi.encodeWithSignature("advanceGame()")
            );
            if (!ok) {
                // Fulfill VRF one more time in case the failed call generated a request
                _fulfillVrfIfPending();
                // Retry once — the fulfillment may have unblocked progress
                (ok, ) = address(game).call(
                    abi.encodeWithSignature("advanceGame()")
                );
                if (!ok) break;
            }
        }
    }
}
```

Key changes from original:
- Calls `_fulfillVrfIfPending()` BEFORE `advanceGame()` in the inner loop (not after), so that VRF words are available when advanceGame processes the daily RNG.
- On failure, fulfills VRF once more and retries. This handles the case where advanceGame requests VRF (returns (1,0) from rngGate) and the NEXT call needs the word.
- Same outer structure (500 day loop, seed + buy + advance pattern).

**Fix for VRFLifecycle.t.sol:**

In test_vrfLifecycle_levelAdvancement (around line 85), the test needs to handle the full multi-day game loop. Replace the loop (lines 115-138) with:

```solidity
uint24 initialLevel = game.level();
uint256 ts = block.timestamp;
uint256 lastFulfilledId;
for (uint256 day = 0; day < 30; day++) {
    ts += 1 days;
    vm.warp(ts);

    // Fulfill pending VRF BEFORE advancing
    uint256 reqId = mockVRF.lastRequestId();
    if (reqId > lastFulfilledId && reqId > 0) {
        (, , bool fulfilled) = mockVRF.pendingRequests(reqId);
        if (!fulfilled) {
            try mockVRF.fulfillRandomWords(reqId, uint256(keccak256(abi.encode(day)))) {
                lastFulfilledId = reqId;
            } catch {}
        }
    }

    // Try advanceGame -- may revert if not ready
    try game.advanceGame() {} catch { continue; }

    // Fulfill VRF if a new request was fired
    reqId = mockVRF.lastRequestId();
    if (reqId > lastFulfilledId && reqId > 0) {
        (, , bool fulfilled) = mockVRF.pendingRequests(reqId);
        if (!fulfilled) {
            try mockVRF.fulfillRandomWords(reqId, uint256(keccak256(abi.encode(day, uint256(1))))) {
                lastFulfilledId = reqId;
            } catch {}
        }
    }

    // Process until unlocked (may take multiple calls as tickets are batched)
    for (uint256 j = 0; j < 50; j++) {
        // Fulfill any new VRF requests mid-loop
        reqId = mockVRF.lastRequestId();
        if (reqId > lastFulfilledId && reqId > 0) {
            (, , bool fulfilled) = mockVRF.pendingRequests(reqId);
            if (!fulfilled) {
                try mockVRF.fulfillRandomWords(reqId, uint256(keccak256(abi.encode(day, j + 2)))) {
                    lastFulfilledId = reqId;
                } catch {}
            }
        }
        (bool ok, ) = address(game).call(abi.encodeWithSignature("advanceGame()"));
        if (!ok) break;
    }

    // Check if level changed
    if (game.level() > initialLevel) break;
}
```

Key changes from original:
- Fulfills VRF BEFORE first advanceGame call each day (word from previous day's request).
- Fulfills VRF AFTER first advanceGame (for new requests fired by advanceGame).
- Fulfills VRF INSIDE the inner processing loop (for requests fired during ticket processing).
- Uses distinct entropy for each fulfillment (abi.encode(day, j+2) etc.) to ensure unique random words.
