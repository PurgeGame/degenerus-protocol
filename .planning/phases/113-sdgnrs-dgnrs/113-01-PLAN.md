## Tasks

### Task 1: Build Coverage Checklist
type="auto"

Read both contracts line-by-line. For every state-changing function (external, public, internal, private):
1. Record function name, line range, visibility, access control, primary storage writes, external calls
2. Classify into Category B (external state-changing), C (internal/private state-changing), or D (view/pure)
3. Tag risk tier for Category B: Tier 1 (complex state + external calls), Tier 2 (moderate), Tier 3 (simple)
4. Mark MULTI-PARENT for Category C functions called from multiple parents
5. Note subsystem grouping

Output: `audit/unit-11/COVERAGE-CHECKLIST.md`

Format per category table:
| # | Function | Lines | Access Control | Primary Storage Writes | External Calls | Risk Tier | Subsystem | Analyzed? | Call Tree? | Storage Map? | Cache Check? |

Include:
- Category B: all external/public state-changing functions in both contracts
- Category C: all internal/private state-changing helpers
- Category D: all view/pure functions (security note review only)
- Total function count with category breakdown
- Constructor analysis notes (one-time, but verify supply math)

## Verification
- Every function from both contracts is listed
- No function skipped with "standard" or "simple"
- Categories correctly assigned
- MULTI-PARENT tags present where applicable
