**System Card 1: GO-01 — Terminal Jackpot (handleGameOverDrain)**

Info table:
- Trigger: `handleGameOverDrain called after GAMEOVER triggered (death clock expiry)`
- Source Pool: `90% of remaining funds after deity refunds and decimator allocation (+ decRefund if no decimator burns)`
- Calculation: `budget = totalFunds - claimablePool; remaining = budget - totalRefunded; decPool = remaining / 10; terminalJackpotPool = remaining - decPool + decRefund`
- Recipients: `VRF-selected winners via runTerminalJackpot (same mechanism as normal jackpot)`
- Claim Mechanism: `claimWinnings() during 30-day window; liabilityDelta added to claimablePool`
- Currency: `ETH`

Formula block:
```
// GameOverModule.sol:68-164 — handleGameOverDrain
totalFunds = address(this).balance + steth.balanceOf(address(this));
budget = totalFunds - claimablePool;

// [1] Deity refunds: 20 ETH per pass, FIFO, budget-capped
// GameOverModule.sol:68-107
// totalRefunded = sum of min(20 ETH, remaining budget) per deity pass

// [2] Remaining for distribution
remaining = totalFunds - claimablePool - totalRefunded;

// [3] Terminal decimator allocation
decPool = remaining / 10;  // 10% to terminal decimator
// GameOverModule.sol:108-142

// [4] Terminal jackpot
terminalJackpotPool = remaining - decPool + decRefund;
// decRefund = full decPool if totalWinnerBurn == 0

// [5] Terminal jackpot distribution
// JackpotModule.sol:288-324, 1537-1576
// runTerminalJackpot -> _distributeJackpotEth
// claimablePool += ctx.liabilityDelta

// [6] Undistributed sweep
// GameOverModule.sol:155-164
// _sendToVault(undistributed) — vault gets remainder
```

File references: `GameOverModule.sol:68-164`, `JackpotModule.sol:288-324`, `JackpotModule.sol:1537-1576`

**System Card 3: GO-07 — Deity Pass Refunds**

Info table:
- Trigger: `GAMEOVER at levels 0-9 (early termination); processed during handleGameOverDrain`
- Source Pool: `budget = totalFunds - claimablePool`
- Calculation: `20 ETH per deity pass, FIFO ordering, budget-capped (cannot exceed remaining budget)`
- Recipients: `Deity pass holders (FIFO: first minted, first refunded)`
- Claim Mechanism: `Refund added to claimableWinnings[player]; claimed via claimWinnings() during 30-day window`
- Currency: `ETH`

Formula block:
```
// GameOverModule.sol:68-107
budget = totalFunds - claimablePool;

// FIFO refund loop:
for each deity pass (in mint order):
    refund = min(20 ETH, budget - totalRefunded);
    if (refund == 0) break;  // budget exhausted
    _creditClaimable(holder, refund);
    totalRefunded += refund;

// claimablePool += totalRefunded  (GameOverModule.sol:105)
// Safe unchecked arithmetic: totalRefunded <= budget by construction
```

File references: `GameOverModule.sol:68-107`, `DegenerusGameStorage.sol:1025-1034`

SVG flow diagram (viewBox="0 0 700 250"):
- budget (pool, left) --"20 ETH each"--> Deity Pass Refund Loop (contract, center) --"FIFO"--> claimablePool (pool, right) ---> Deity Holders (recipient)
- Annotation: "Budget-capped: stops when budget exhausted"

Edge cases:
- Budget exhaustion: If many deity passes exist, later passes may get partial or zero refund
- Level 0 GAMEOVER: All deity refunds consume from same budget; may leave zero for decimator/jackpot
- FIFO ordering: First-minted deity passes get priority refund
- Unchecked arithmetic provably safe: `totalRefunded` bounded by `budget` by loop construction
