- Recipients: `futurePrizePool (indirect — feeds back into jackpot/scatter/decimator systems)`
- Claim Mechanism: `Automatic at x00 milestone; remaining 50% kept for next milestone`
- Currency: `ETH (pool accounting)`

Formula block:
```
// PAY-12 — JackpotModule.sol:928-958

accumulatorShare = (yieldPool * 4600) / 10_000;  // 46% to yieldAccumulator
```

- Trigger: `Holder calls burn(amount) on StakedDegenerusStonk`
- Source Pool: `Contract ETH + stETH + pending claimable (proportional share)`
- Calculation: `totalMoney = ethBal + stethBal + claimableEth; share = (amount * totalMoney) / supplyBefore`
- Recipients: `Burning holder (receives proportional share of all assets)`
- Claim Mechanism: `Direct transfer — ETH preferred, then stETH, with lazy-claim for pending amounts`
- Currency: `ETH + stETH + BURNIE`

Info table (PAY-15 DGNRS Wrapper Burn):
- Trigger: `Holder calls burn(amount) on DegenerusStonk wrapper`
- Source Pool: `Delegated to sDGNRS (PAY-14 path)`
- Calculation: `Burns DGNRS, then calls sDGNRS.wrapperBurnTo() which executes the PAY-14 formula`
- Recipients: `Burning holder`
- Claim Mechanism: `_burn DGNRS then delegate; unwrapTo restricted to creator-only with VRF stall guard`
- Currency: `ETH + stETH + BURNIE (forwarded from sDGNRS)`

Formula block:
```
// PAY-14 — StakedDegenerusStonk.sol:373-435
totalMoney = ethBal + stethBal + claimableEth;
// claimableEth includes pending claimable amounts (lazy-claim CP-04 defense)
share = (amount * totalMoney) / supplyBefore;
// ETH-preferred payout: pays from ETH first, then stETH, then claims lazily

// supplyBefore ensures sequential burns are correctly proportional
// Each burn reduces supply, increasing per-token share for remaining holders

// PAY-15 — DegenerusStonk.sol:164-181
_burn(msg.sender, amount);                    // burn DGNRS wrapper token
dgnrs.wrapperBurnTo(msg.sender, amount);      // delegate to sDGNRS.burn()
// Forwards ETH + stETH + BURNIE to caller
// unwrapTo: creator-only with VRF stall guard (prevents unwrap during pending VRF)
```

File references: `StakedDegenerusStonk.sol:373-435`, `DegenerusStonk.sol:164-181`

SVG flow diagram (viewBox="0 0 800 400"):
- Two entry points at left:
  - sDGNRS holder --"burn(amount)"--> StakedDegenerusStonk.burn (contract)
  - DGNRS holder --"burn(amount)"--> DegenerusStonk.burn (contract) --"delegate"--> StakedDegenerusStonk.burn
- StakedDegenerusStonk.burn --> Calculate share: "(amount * totalMoney) / supplyBefore"
- Share distributes to 3 outputs:
  - ETH (preferred, first) --> Holder
  - stETH (if ETH insufficient) --> Holder

- Annotation: "Lazy-claim: claimable included in totalMoney, claimed on-demand"

Edge cases:
- Single sDGNRS holder burning 100%: Gets proportional share of totalMoney
- Lazy-claim CP-04 defense: `claimableEth` included in `totalMoney`, claimed on-demand when ETH balance insufficient

- DGNRS `unwrapTo` restricted to creator-only with VRF stall guard (prevents unwrap during pending VRF resolution)
- ETH-preferred payout order: ETH first, stETH only if ETH balance insufficient
