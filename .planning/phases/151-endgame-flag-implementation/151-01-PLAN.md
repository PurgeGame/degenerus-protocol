   /// @dev WAD representation of 0.9925 (1 - 0.0075 conservative drip rate).
   ///      Projection uses 0.75% daily decay (conservative vs actual 1% drip)

3. **Add `_wadPow` private pure function after the constants section (before the Errors section, around line 182)**:
   ```solidity
   /// @dev Compute base^exp in WAD scale via repeated squaring.
   ///      base is WAD-scaled (e.g., 0.9925e18), exp is a plain integer.
   ///      Max 7 iterations for exp <= 120. ~700 gas.
   function _wadPow(uint256 base, uint256 exp) internal pure returns (uint256) {
       uint256 result = WAD;
       while (exp > 0) {
           if (exp & 1 == 1) {
               result = (result * base) / WAD;
           }
           base = (base * base) / WAD;
           exp >>= 1;
       }
       return result;
   }
   ```
   Per D-02. Uses `internal` visibility so AdvanceModule can call it.

4. **Add `_projectedDrip` internal pure function immediately after `_wadPow`**:
   ```solidity
   /// @dev Projected total drip from futurePool over n remaining days.
   ///      Closed-form geometric series: futurePool * (1 - 0.9925^n).
   ///      Returns 0 when daysRemaining == 0 (no drip possible).
   function _projectedDrip(uint256 futurePool, uint256 daysRemaining) internal pure returns (uint256) {
       if (daysRemaining == 0) return 0;
       uint256 decayN = _wadPow(DECAY_RATE_WAD, daysRemaining);
       return (futurePool * (WAD - decayN)) / WAD;
   }
   ```
   Per D-02 and DRIP-01.

Do NOT add `internal` to the WAD/DECAY_RATE_WAD constants — keep them `private` since only the functions in this file use them directly. The functions themselves are `internal` so subcontracts (AdvanceModule) inherit them.
