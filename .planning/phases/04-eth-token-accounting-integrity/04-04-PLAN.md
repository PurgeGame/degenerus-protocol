function _claimWinningsInternal(address player, bool stethFirst) private {
    uint256 amount = claimableWinnings[player];       // CHECK
    if (amount <= 1) revert E();
    unchecked {
        claimableWinnings[player] = 1;                // EFFECT (sentinel)
        payout = amount - 1;
    }
    claimablePool -= payout;                           // EFFECT (aggregate)
    emit WinningsClaimed(player, msg.sender, payout);
    // INTERACTION:
    _payoutWithEthFallback / _payoutWithStethFallback  // External call
}

**Part B: Enumerate ALL functions reachable from ETH callback**

When _payoutWithStethFallback or _payoutWithEthFallback sends ETH via call{value:}(""), the recipient's receive() function executes. From receive(), the attacker can call ANY external function on DegenerusGame.

Enumerate ALL external/public functions on DegenerusGame and for each, determine if it can be exploited during mid-claim state. Key state during mid-claim:
- claimableWinnings[attacker] = 1 (sentinel)
- claimablePool has been decremented by payout
- ETH is being transferred (contract balance reduced by payout)

For each reachable function, analyze:

1. **claimWinnings()** (re-claim): Will see amount=1 and revert. SAFE.
2. **purchase()** using claimable: _processMintPayment checks claimableWinnings[player] which is 1. Can the attacker spend other players' claimable? NO -- it checks msg.sender/player. VERIFY.
3. **purchase()** with fresh ETH: Can the attacker buy tickets with new msg.value during mid-claim? Check if rngLockedFlag or other guards block this. Even if allowed, does this break invariant? (msg.value goes to pools, claimablePool already decremented)
4. **placeFullTicketBets()**: Check rngLockedFlag guard. Even if callable, msg.value goes to futurePrizePool.
5. **purchaseWhaleBundle()**: Check guards.
6. **purchaseDeityPass()**: Check guards.
7. **refundDeityPass()**: Can the attacker trigger a refund during mid-claim? Check guards.
8. **advanceGame()**: Can the attacker advance during mid-claim? Check rngLockedFlag.
9. **receive()**: futurePrizePool += msg.value. No state conflict.

For each function, write: Function | Guards | Mid-Claim State Conflict | Verdict (SAFE/UNSAFE)

**Part C: Analyze _payoutWithStethFallback and _payoutWithEthFallback**
