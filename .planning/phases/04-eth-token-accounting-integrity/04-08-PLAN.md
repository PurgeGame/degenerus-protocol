**Part A: DGVE (ETH+stETH) share redemption**

1. Read DegenerusVault.sol burnEth() function
2. Trace the redemption formula: `claimValue = (reserve * amount) / totalSupply`
3. Verify: `reserve` = total ETH + stETH held by vault? Or just ETH?
4. Verify: rounding direction -- integer division floors, so claimer gets slightly less (vault-favorable). Confirm this.
5. Verify: after claim, remaining shares have proportionally more backing (no extraction beyond fair share)
6. Edge cases:
   - Last share burned: does the claimer get the full remaining reserve?
   - First share burned: does the formula work correctly at max reserve?
   - Division by zero: what happens if totalSupply = 0? Is this guarded?
7. stETH yield impact: between deposits, stETH rebases upward, increasing `reserve`. Verify this passively increases DGVE value without any explicit mint/action.

**Part B: DGVB (BURNIE) share redemption**

1. Read burnCoin() function
2. Trace the formula: `coinOut = (coinBal * amount) / totalSupply`
3. Verify: `coinBal` = BurnieCoin.balanceOf(vault) + claimable coinflips?
4. Verify: rounding direction (vault-favorable)
5. Verify: vaultMintTo path -- when vault mints from allowance, does it increase coinBal correctly?
6. Edge cases: same as Part A (last share, first share, division by zero)

**Part C: Vault refill mechanism**

1. Find the refill logic: when all shares are burned, 1T new shares are minted
2. Verify: no residual ETH/stETH/BURNIE remains unclaimed after all shares burned
3. Verify: new 1T shares are minted BEFORE any new deposits (no deposit-before-shares race)
4. Verify: the refill cannot be triggered while shares still exist
5. Check: can an attacker burn all but 1 share, deposit a large amount, then claim disproportionate value?

**Part D: Cross-contract interactions**

1. Vault receives ETH from handleGameOverDrain/_sendToVault and handleFinalSweep
2. Verify: the ETH arrives correctly and increases DGVE backing
3. Vault receives BURNIE from BurnieCoin.vaultMintTo()
4. Verify: vaultAllowance is correctly decremented, totalSupply correctly incremented
