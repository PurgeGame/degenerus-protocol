## Cross-Cutting Patterns

The following recurring themes appear across multiple contracts and sweep phases.

### Pattern 1: DGNRS vs sDGNRS Recipient Confusion

Findings across three contracts (G05-01 in GameOverModule, G03-01 in GNRUS, 175-02-005 in JackpotModule) share the same root: comments say "DGNRS" (DegenerusStonk) where the actual ETH/stETH recipient is "sDGNRS" (StakedDegenerusStonk / `ContractAddresses.SDGNRS`). These are distinct contracts with distinct addresses. This is the highest-risk pattern because an auditor following the comment would examine the wrong contract when tracing gameover fund flows.

### Pattern 2: Stale Caller Attributions / Access Control Understatements

Multiple findings (BCF-IMPL-01, BCF-IFACE-01, BCF-02, BCF-03, QST-02, 177-02-001, 177-02-002, 177-02-006, DGM-IFACE-01, QST-01) document access control NatSpec that is either too narrow (omitting QUESTS or AFFILIATE), too broad (including COIN where not authorized), or wrong (listing COINFLIP as caller when the real caller is AFFILIATE). The pattern appears in both implementation NatSpec and interface layer NatSpec. At scale, this class of error causes auditors to either miss attack surfaces or waste time tracing ghost callers.

### Pattern 3: Orphaned / Dangling Comment Blocks

Findings B-01, D-01, LB-01, BC-01, BC-02, 175-02-007 all describe NatSpec blocks that have been separated from the code they documented ‚Äî either by function deletion, refactoring, or copy-paste error. These orphaned blocks actively misdirect readers who assume every NatSpec is followed by the function it describes.

### Pattern 4: Stale Line Number Self-References

Findings ADV-CMT-01 and ADV-CMT-06 document inline comments that cite specific line numbers within the same file to cross-reference related code. Both are stale by 50-100+ lines due to code additions and reorganizations. Self-referential line numbers have an inherently short shelf life and should be replaced with function name references.

### Pattern 5: Pre-v17.1 State Retained in Comments After v17.0 Cache and v17.1 Tiered Rate Changes

Findings MINT-CMT-01, DGM-02, DGST-01, AFF-01, and BCF-IFACE-02 all trace to comments that predate the v17.0 affiliate bonus cache and v17.1 tiered affiliate bonus rate changes. The cache and rate changes touched multiple layers (storage, game, interface, MintModule, MintStreakUtils) but comment updates lagged behind. A reader consulting any single contract in isolation would form an incorrect model of how affiliate bonus data flows.
