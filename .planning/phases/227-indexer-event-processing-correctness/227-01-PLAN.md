`database/src/handlers/index.ts:227` — `export const HANDLER_REGISTRY: Record<string, EventHandler> = { EventName: handleFn, ... }`. This is an object-map, NOT a switch. Classification grep MUST target the registry object, not `case 'X':`.

Shared-name events (Transfer, Approval, Burn, Claim, Deposit, QuestCompleted) are dispatched through ROUTER handlers (e.g., `handleTransferRouter`) that branch on `ctx.contractAddress` via `ADDRESS_TO_CONTRACT`. Coverage matrix key is `(contractFile, eventName)`; a registry entry may cover some contracts and silently drop others.

`raw_events` table receives every decoded event unconditionally at `event-processor.ts:118` via `.onConflictDoNothing()`. UNHANDLED means "no domain-table write", NOT "no write at all".

Four classification buckets (D-227-08 + RESEARCH.md §Classification Rules):
- PROCESSED — event name is a direct key in HANDLER_REGISTRY; handler body inline-writes a domain table via `.insert(...)` or `.update(...)`.
- DELEGATED — registry key points at a *Router handler that dispatches by `ctx.contractAddress` through `ADDRESS_TO_CONTRACT`.
- INTENTIONALLY-SKIPPED — event name appears in a comment INSIDE the `HANDLER_REGISTRY` object literal justifying non-registration (example: `index.ts:66-68` admin-events block). Comments elsewhere do NOT count.
- UNHANDLED — in contract-event universe but not in any of the above. Finding candidate.
