<decisions>
## Implementation Decisions

### Carryover Redesign
- **D-01:** Replace the Phase 1 ETH carryover distribution loop with direct ticket purchases. Current: complex Phase 1 state machine distributes ETH to trait winners across multiple advanceGame calls via dailyEthPhase/dailyCarryoverEthPool/dailyCarryoverWinnerCap. New: single-pass ticket distribution in the existing Phase 0 flow (no multi-call resumption needed).
- **D-02:** Budget changes from 1% (`DAILY_REWARD_JACKPOT_LOOTBOX_BPS = 100`) to 0.5% of futurePrizePool. The 0.5% budget goes to nextPrizePool (not distributed as ETH).
- **D-03:** Single random source level — current code uses `_pickCarryoverSourceOffset` to select among eligible levels. New: use a single source level derived from entropy (same randomness pattern but simpler — no multi-level eligibility scan needed since tickets go to a single level).
- **D-04:** Delete storage vars: `dailyCarryoverEthPool`, `dailyCarryoverWinnerCap`, `dailyEthPhase`. These are only used by the Phase 1 state machine.
- **D-05:** The existing `_budgetToTicketUnits` and `_mintTicketsForTraitWinner` patterns in `payDailyJackpotCoinAndTickets` are the ticket distribution mechanism. Carryover tickets fold into the same flow — no new ticket purchase function needed.
- **D-06:** `DailyCarryoverStarted` event — remove or repurpose. The Phase 1 state machine emitted this when starting ETH distribution. With tickets-only, this event is no longer meaningful.

### BurnieCoin Quest Routing Cleanup
- **D-07:** Remove `BurnieCoin.rollDailyQuest()` (line 592-606) — this is dead code with zero callers. JackpotModule already calls `quests.rollDailyQuest()` directly at lines 644 and 743.
- **D-08:** Remove `DailyQuestRolled` event from BurnieCoin.sol (line 81). DegenerusQuests has its own quest events. No event migration needed — the JackpotModule call sites don't emit this event.
- **D-09:** Remove `rollDailyQuest` from `IDegenerusCoin.sol` interface.
- **D-10:** Check if `BurnieCoin.questModule` storage var / import is still needed after removing rollDailyQuest. If `decimatorBurn` still uses it, keep. If not, remove.

### recordMintQuestStreak Access Control
- **D-11:** `DegenerusGame.recordMintQuestStreak` (line 432) — remove COIN from access check, keep GAME only. Phase 158 expanded access to COIN + GAME; now that no calls go through COIN, tighten back to GAME-only.
- **D-12:** Update `IDegenerusGame.sol` NatSpec to reflect GAME-only access.

## Specific Ideas

- The Phase 1 ETH distribution loop is the most complex part of the carryover system (~100 lines). Removing it and replacing with ticket-only distribution dramatically simplifies `payDailyJackpotEth`.

- The `dailyEthPhase` state machine exists because ETH distribution could exceed gas limits and needed to resume. Ticket distribution is cheaper — single-pass is feasible.
