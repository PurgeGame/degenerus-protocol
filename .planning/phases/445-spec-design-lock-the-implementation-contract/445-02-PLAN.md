<threat_model>
## Trust Boundaries (design basis for the storage section)

| Boundary | Description |
|----------|-------------|
| player → `foilRecord` slot | a player's own re-buy is the ONLY writer of their slot; another player or `advanceGame` never touches it |
| claimant → `foilMatchClaimed` | the keccak key isolates callers; a forged tuple cannot replay another player's claim |

## STRIDE Threat Register (storage-layout slice)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-445-D1 | Tampering | `foilRecord` slot collision with an existing slot | mitigate | append-only after `boxPlayers` (`:2393`); two new base mapping slots; no existing slot moved/retyped (SEC-04 layout golden re-pass at 448) |
| T-445-D2 | Repudiation | double-claim of the same (player,level,day,drawKind,ticketIndex) | mitigate | sparse `foilMatchClaimed[key]` set BEFORE payout (CEI); five distinct `abi.encode` fields → collision-free |
| T-445-D3 | Denial of Service | a fast `level++` strands an unclaimed match (grief) | accept (by-design, §5) | record persists per-level via the stamp; overwritten ONLY by the SAME player's next buy; single-slot loss edge surfaced for USER sign-off (PIN 1) |
</threat_model>
