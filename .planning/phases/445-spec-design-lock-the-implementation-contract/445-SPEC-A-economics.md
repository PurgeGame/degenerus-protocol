### A.1.1 The two new sibling producers (RARE-01 вҖ” frozen producers untouched)

The foil tickets are generated by a **new sibling** of the Degenerette producer, cloned
structurally from `packedTraitsDegenerette` / `_degTrait` (`DegenerusTraitUtils.sol:201-223`):

- `traitFromWordFoil(uint64 rnd, uint256 multBps) internal pure returns (uint8)` вҖ”
  the per-quadrant 6-bit `[CCC][SSS]` producer: **color** from the tapered ladder of В§A.1.4
  (the multiplier `multBps` enters here, `10000 = 1Г—`), **symbol** uniform `1/8` from
  `rnd >> 32 & 7` (byte-identical to `_degTrait`'s symbol stage). Quadrant bits added by the caller.
- `packedTraitsFoil(uint256 rand, uint256 multBps) internal pure returns (uint32)` вҖ”
  the 4-quadrant packer: calls `traitFromWordFoil` on each of the four 64-bit lanes
  (`uint64(rand)`, `>>64`, `>>128`, `>>192`), OR's in the quadrant identifiers
  (`| 64`, `| 128`, `| 192`), and packs four bytes into a `uint32` in the
  **identical `[QQ][CCC][SSS]`-per-byte layout** as `packedTraitsFromSeed` /
  `packedTraitsDegenerette` (`return uint32(traitA) | (traitB << 8) | (traitC << 16) | (traitD << 24)`).

**ONLY the color stage changes** vs the Degenerette sibling. The symbol stage and the pack
layout are copied verbatim. **RARE-01: the frozen producers `weightedColorBucket`,
`traitFromWord`, and `packedTraitsFromSeed` (`DegenerusTraitUtils.sol:115`, `:143`, `:169`)
are NOT edited, retyped, or moved** вҖ” the foil path is purely additive (a new sibling),
identical in spirit to how `packedTraitsDegenerette` already coexists beside the frozen
heavy-tail producer.

`multBps` is the `foilBoostBps` output (В§A.2), **frozen at buy** from `cachedScore` and passed
in at resolve вҖ” never live-read (RARE-03; see В§A.2.4).

### A.1.3 The taper rule (RARE-04 вҖ” tapered, NOT flat Г—M)

The boost is **linear in the rare-rank**: the rarest tier (gold) takes the full multiplier `M`,
each less-rare tier tapers toward `1Г—`. With `M = multBps / 10000`:

```
boost(c) = 1 + (M вҲ’ 1) В· w_c      with rare-rank weights w_c вҲҲ [0,1]
  w_7 = 5/5   (gold = full Г—M)
  w_6 = 4/5
  w_5 = 3/5
  w_4 = 2/5
  w_3 = 1/5   (least-rare boosted tier; still > 1Г— for any M > 1)
  w_0 = w_1 = w_2 = 0   (the three 25% commons вҖ” the SOLE funding sink; never boosted)
```

Per-tier probability:

```
p_c       = (base[c] / 256) В· (1 + (MвҲ’1)В·w_c)        for c вҲҲ {3,4,5,6,7}
p_0 = p_1 = p_2 = (192/256 вҲ’ О”) / 3                  where О” = ОЈ_{cвҲҲ{3..7}} p_c вҲ’ ОЈ_{cвҲҲ{3..7}} base[c]/256
```

This satisfies every D-03 acceptance constraint (all re-verified exactly in V1):

- **`p_gold = (2/256)В·M` exactly** вҖ” the LOCKED invariant. `w_7 = 5/5 = 1 вҮ’ p_7 = (2/256)В·M`.
  At `M = 6 вҮ’ 12/256 = 4.6875% вүҲ 4.7%` (the spec's gold target). **FOIL-05.**
- **Monotone-decreasing boost down the tail** вҖ” `w_7 > w_6 > w_5 > w_4 > w_3 > 0`, so
  `boost(7) > вҖҰ > boost(3) > 1` for all `M > 1` (factors `[6,5,4,3,2]` at M=6,
  `[2,1.8,1.6,1.4,1.2]` at M=2). **RARE-04.**
- **Commons 0/1/2 the sole funding source** вҖ” only the three 25% commons shrink; tiers 3вҖ“7 only grow.
- **Symbol uniform 1/8 вҖ” out of scope, unchanged** (the boost is color-tier ONLY; symbol stage
  copied verbatim from `_degTrait`). **RARE-02 / D-03.**

### A.1.4 Resolve mechanism вҖ” the `/15360` integer cutoff ladder (build-ready, exact)

256-resolution is too coarse for the taper (at M=2 the commons are `55.867/256` and tiers 3вҖ“6 are
`38.4 / 22.4 / 12.8 / 10.8`, none integral). The build-ready resolve computes cutoffs in a
`/15360` super-ladder (`15360 = 256 Г— 60`; the `Г—60` clears both the `/5` taper denominator and
the 3-way common split), then compares a scaled draw against the running sum вҖ” style-identical to
`weightedColorBucket`'s `if (scaled < вҖҰ) return вҖҰ` chain.

Per rare tier `c вҲҲ {3,4,5,6,7}` with `w5 = {3:1, 4:2, 5:3, 6:4, 7:5}` (the `w_c В· 5` integers):

```
width15360[c] = base[c]В·60 В· (50000 + (multBps вҲ’ 10000)В·w5[c]) / 50000
rare          = ОЈ_{cвҲҲ{3,4,5,6,7}} width15360[c]
rem           = 15360 вҲ’ rare
commons[0..2] = rem / 3, with the (rem mod 3) residual handed to the FIRST common (color 0)
```

Gold lands exactly on `width15360[7] = base[7]В·60В·M = 2В·60В·M = 120В·M`
(i.e. exactly `baselineГ—M`, the `p_gold` invariant on the ladder).

**The IMPL ladder, in directive form:** compute the five `width15360[c]` for c=3..7 by the formula
above; sum to `rare`; set `rem = 15360 вҲ’ rare`; the three commons each get `rem / 3`, and color 0
additionally absorbs `rem mod 3`. A draw scaled into `[0, 15360)` walks the running cutoff chain
in the order `[0, 1, 2, 3, 4, 5, 6, 7]` (or any fixed order whose widths sum to 15360) and returns
the first tier whose cumulative cutoff it falls under вҖ” identical control flow to `weightedColorBucket`.

## A.2 Locked Knobs вҖ” `foilBoostBps(score)` activity curve (RARE-02 / RARE-03 / FOIL-05)

### A.2.2 The 4 segment closed forms (integer floor division, matching the existing curves)

With explicit endpoint guard branches so there is **no interpolation rounding at the endpoints**:

- **`score == 0` вҮ’ `20000`** (explicit guard branch).
- **Seg A** `0 < score вүӨ 300`: `20000 + 100В·score`.
- **Seg B** `300 < score вүӨ 500`: `50000 + 25В·(score вҲ’ 300)`.
- **Seg C** `500 < score < 30000`: `55000 + (score вҲ’ 500)В·5000/29500`.
- **`score вүҘ 30000` вҮ’ `60000`** (saturation, explicit guard branch).

The interpolation form is identical to `decMultBps`: `V_lo + (score вҲ’ knee_lo)В·(V_hi вҲ’ V_lo)/(knee_hi вҲ’ knee_lo)`
per segment, with the two endpoint guards (`score == 0` and `score >= ACTIVITY_EFFECTIVE_CAP_POINTS`)
short-circuiting before any interpolation.

### A.2.4 Frozen-at-buy / apply-at-resolve / never-live-read (RARE-03)

freeze-at-buy / apply-at-resolve link to `frozen multBps stored in foilRecord` (the `/15360` ladder
of В§A.1.4 consumes exactly this frozen value). **RARE-03.**
