**Section 15: Carryover Edge Cases and Loss Path (DETH-04)**
Document the edge cases identified in the research:

1. **Unfilled buckets:** When `_processDailyEthChunk` finds empty trait pools (no tickets for winning trait at target level), the bucket is skipped. For Phase 0: ETH stays in currentPrizePool (only paidEth is deducted). For Phase 1 carryover: the pool was pre-deducted from futurePrizePool, so unpaid ETH stays in contract balance but is NOT attributed to any named pool. Flag this as a finding area.

2. **Integer division dust:** `perWinner = share / totalCount` integer division. For Phase 0: dust stays in currentPrizePool (same mechanism as unfilled buckets). For Phase 1: dust becomes untracked. Flag and assess severity.

3. **Carryover pre-deduction loss (key finding area):** When Phase 0 uses all 321 winners (or the cap), `dailyCarryoverWinnerCap` becomes 0 at JM:531 area. The 1% futurePrizePool slice was already deducted at JM:430. With cap=0, Phase 1 is skipped. The pre-deducted ETH stays in the contract's ETH balance but is not attributed to currentPrizePool, futurePrizePool, or nextPrizePool.
   - Trace the exact code path: JM:430 deducts -> JM:531 computes cap=0 -> JM:547 or equivalent condition skips Phase 1 -> ETH is orphaned
   - Assess: is this a bug or intentional solvency buffer? Estimate frequency (requires 321+ winners in Phase 0, which requires a large pool with max scaling)
   - Assign severity: likely INFO (small amount, improves contract solvency) or LOW (if amounts are significant)

4. **Day-to-day state reset:** `_clearDailyEthState` at JM:2785-2793 (verify exact lines) -- enumerate every variable zeroed. Confirm there is NO explicit rollover of unpaid ETH. The next day simply calculates a fresh budget from the (now-larger) remaining currentPrizePool. This is a natural retention mechanism, not a bug.

5. **Winner cap derivation edge:** `max(DAILY_ETH_MAX_WINNERS - totalDailyWinners, DAILY_CARRYOVER_MIN_WINNERS)` -- what happens when totalDailyWinners is between 301 and 321? The cap would be between 20 and 40. Confirm the min(20) floor from DAILY_CARRYOVER_MIN_WINNERS.
