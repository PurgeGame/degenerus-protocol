# Plan 157-03: Gap Closure

## Objective

Fix the MINT_BURNIE=0 sentinel collision that silently kills ~38% of level quests, pack level quest global state into a single slot to save ~2,600 gas per handler call, thread `questLevel` through eligibility to eliminate a redundant cross-contract call, and update REQUIREMENTS.md to reflect the direct roll pattern (no BurnieCoin hop).

## Context

**Gap 1 (Blocker):** `QUEST_TYPE_MINT_BURNIE = 0` collides with Solidity's default mapping value. `_handleLevelQuestProgress` uses `if (lqType == 0) return;` to detect unrolled quests, which also blocks all legitimately rolled MINT_BURNIE quests.

**Gap 2 (Gas):** `_handleLevelQuestProgress` calls `questGame.level()` cross-contract (~2,600 gas) on every invocation just to look up `levelQuestType[currentLevel]`. This can be replaced by a single local SLOAD packing both level and type.

**Gap 3 (Docs):** ROLL-01 requires BurnieCoin.rollLevelQuest routing. Design decision D-12 deliberately skipped this. REQUIREMENTS.md needs updating.

## must_haves

- [ ] QUEST_TYPE_MINT_BURNIE is no longer 0; value 0 in levelQuestType unambiguously means "unrolled"
- [ ] Level quest handler fast path (type mismatch) costs 1 local SLOAD, not a cross-contract call + mapping SLOAD
- [ ] `_isLevelQuestEligible` does not call `questGame.level()` — receives level as parameter
- [ ] REQUIREMENTS.md ROLL-01 marked superseded, ROLL-02 updated to reflect direct call pattern
- [ ] All 62 Solidity files compile cleanly

    uint256 delta,
    uint256 mintPrice
) internal {

    uint24 currentLevel = uint24(lqGlobal);
    uint8 lqType = uint8(lqGlobal >> 24);

    if (lqType == 0) return;

### F. Update getPlayerLevelQuestView (~line 1737)

function getPlayerLevelQuestView(address player)
    external
    view
    override
    returns (uint8 questType, uint128 progress, uint256 target, bool completed, bool eligible)
{
    uint256 lqGlobal = levelQuestGlobal;
    uint24 currentLevel = uint24(lqGlobal);
