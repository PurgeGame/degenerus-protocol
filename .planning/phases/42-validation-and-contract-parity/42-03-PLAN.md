<task type="auto">
  <name>Task 2: Hardhat cross-validation test scenario</name>
  <files>test/validation/SimContractParity.test.js</files>
  <action>
Create `test/validation/SimContractParity.test.js` that runs a concrete game scenario through both the sim bridge and real contracts, comparing results.

Use the existing `deployFullProtocol` fixture. The test scenario:

**Test 1: Price parity across levels.**
After deploy, call `game.purchaseInfo()` to get the initial `priceWei`. Compare against `simBridge.priceForLevel(0)`. They must match exactly.

**Test 2: Ticket purchase pool routing parity.**
1. Deploy full protocol via `loadFixture(deployFullProtocol)`.
2. Read initial pool state: `game.nextPrizePool()`, `game.futurePrizePool()`.
3. Calculate the purchase cost: `costWei = (priceWei * 400n) / 400n` (1 full ticket = qty 400).
4. Execute `game.connect(alice).purchase(alice.address, 400, 0, ethers.ZeroHash, 0, { value: costWei })` for 1 full ticket.
5. Read post-purchase pool state.
6. Compute expected split with sim bridge: `simBridge.routeTicketSplit(costWei)`.
7. Compare: contract nextPrizePool delta should match sim's nextPool. Contract futurePrizePool delta should match sim's futurePool.
Note: There may be small differences due to affiliate/DGNRS skims in the contract that the simple sim split doesn't account for. Account for this by checking the TOTAL pool increase equals the purchase amount (minus any affiliate/DGNRS siphons), and verifying the RATIO of next/future matches 90/10.

**Test 3: Whale bundle pricing parity.**
1. Read contract whale bundle price: call the game's whale purchase function with value matching `simBridge.calculateWhaleBundlePrice(0, 1, 0)`.
2. Verify the transaction succeeds (price accepted by contract means sim price >= contract price).
3. If the contract exposes a price view function, compare directly.

**Test 4: Multiple purchases accumulate correctly.**
1. Make 3 ticket purchases at level 0.
2. After each, compare the pool deltas to sim-predicted splits.
3. Verify cumulative pool balances track correctly.

Use `describe('Sim vs Contract Parity')` with `loadFixture(deployFullProtocol)` for each test. Import from `../../test/helpers/deployFixture.js` and `./simBridge.js`.

Important patterns from CLAUDE.md / project memory:
- `purchase(buyer, ticketQuantity, lootBoxAmount, affiliateCode, payKind)` -- qty scaled by 100, so 1 full ticket = 400
- Cost formula: `costWei = (priceWei * ticketQuantity) / 400`
- `purchaseInfo().priceWei` for current price
- MintPaymentKind: DirectEth = 0
- Game level starts at 0; purchaseInfo().lvl = level + 1 during purchase phase
