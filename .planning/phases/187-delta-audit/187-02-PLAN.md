**Section 5: Daily Jackpot Final-Day Change**

The diff shows a new branch in JackpotModule's daily jackpot payout (lines 447-460):
- Old: `_setCurrentPrizePool(_getCurrentPrizePool() - paidDailyEth)` (always)
- New: On `isFinalPhysicalDay_`, deducts full `dailyEthBudget` from current and routes `unpaidDailyEth = dailyEthBudget - paidDailyEth` to future pool
- On non-final days: unchanged behavior (`- paidDailyEth`)

Analyze:
- Is this a bugfix or behavioral change? (On the final physical day, there may be unpaid ETH if not all daily budget was distributed. Old code only removed paidEth, leaving unpaid ETH stranded in currentPool. New code moves it to futurePool where it participates in future game economics.)
- Pool accounting: `currentPool -= dailyEthBudget` is the same as `currentPool -= paidDailyEth - unpaidDailyEth` since `dailyEthBudget = paidDailyEth + unpaidDailyEth`. But futurePool gets `+unpaidDailyEth`. Net change to total obligations: same (currentPool goes down by dailyEthBudget, futurePool goes up by unpaidDailyEth, so total down by paidDailyEth = the actual ETH paid out). Verify this reasoning.
- State PASS or FINDING

**Section 7: distributeYieldSurplus Visibility Change**

- Old: `_distributeYieldSurplus(uint256 rngWord) private` inside JackpotModule
- New: `distributeYieldSurplus(uint256 rngWord) external` in JackpotModule
- No self-call guard on distributeYieldSurplus (unlike runBafJackpot). Verify whether this is safe: it is called via delegatecall from AdvanceModule (not via self-call). In the delegatecall context, msg.sender is the original transaction sender. An external caller directly calling distributeYieldSurplus on the JackpotModule implementation contract would execute against the implementation's storage (not the proxy's), which is harmless. But someone calling it on the proxy? That would require the proxy to have a fallback that delegates to JackpotModule -- check if DegenerusGame.sol has such a fallback for JackpotModule selectors.
- If the proxy does NOT route unknown selectors to JackpotModule, then external calls to distributeYieldSurplus on the proxy would revert (no matching selector in Game.sol). PASS in that case.
- If it DOES have a catch-all fallback, then anyone could trigger yield surplus distribution at will. State as FINDING.
