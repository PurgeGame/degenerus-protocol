3e. **Rebuy delta reconciliation:** Verify the formula `memFuture += _getFuturePrizePool() - storageBaseFuture` (line 759) correctly captures auto-rebuy writes. 3f. **x00 keep roll timing:** OLD: keep roll ran inside consolidatePrizePools, BEFORE runRewardJackpots (jackpots drew from already-rolled-down future pool). The old order was: keep roll reduces futurePool -> jackpots draw from reduced pool. New order: jackpots draw from full post-yield-dump pool -> keep roll reduces what remains. This means jackpots in the new code draw from a LARGER future pool. Document this as an intentional behavioral change per D-01 ("the audit must prove the new order is sound, not that outputs are byte-identical").

3g. **Yield surplus ordering:** OLD: yield surplus ran INSIDE consolidatePrizePools (after merge, after keep roll, after coinflip). NEW: yield surplus runs BEFORE _consolidatePoolsAndRewardJackpots (line 378). The obligations check in distributeYieldSurplus reads `_getCurrentPrizePool() + _getFuturePrizePool() + _getNextPrizePool() + claimablePool`. Since no pool values have changed yet (only levelPrizePool[purchaseLevel] was set), the obligations sum is the same as it would be at any point during consolidation (ETH moves between pools but total is conserved). 3h. **Time-based future take: purchaseLevel vs lvl:** Old `_applyTimeBasedFutureTake(ts, purchaseLevel, rngWord)` used `lvl` parameter (which was actually `purchaseLevel` passed in).
