**Attacker model (per D-06):**
An attacker sees the VRF request transaction in the mempool or on-chain. They have N blocks (typically 3+ for Chainlink VRF v2) before the coordinator calls rawFulfillRandomWords. Question: "What state that affects RNG outcomes can I change in this window?"

**Analysis structure:**

**Section 1: Daily VRF Window**

Window definition:
- OPENS: _requestRng() sets rngLockedFlag = true (AdvanceModule line 1442) and calls coordinator.requestRandomWords()
- CLOSES: rawFulfillRandomWords() stores word and sets rngLockedFlag = false (AdvanceModule lines 1492/1515)

For each player-callable function, determine:
1. Is it blocked by rngLockedFlag during the window? (check DegenerusGameStorage lines 566, 596, 650; DegenerusGame lines 1480, 1495, 1542, 1882; WhaleModule line 543; MintModule line 1231; AdvanceModule line 908)
2. If NOT blocked: what state can it change? Does that state affect any RNG outcome?
3. If blocked: is the block enforceable? (check the revert path — can it be bypassed?)

Enumerate every external/public function on DegenerusGame.sol and categorize:
- BLOCKED by rngLockedFlag (list which guard site blocks it)
- NOT BLOCKED but does not affect RNG outcomes (safe)
- NOT BLOCKED and could affect RNG outcomes (CONCERN — requires deeper analysis)

**Section 2: Lootbox VRF Window**

Window definition:
- OPENS: requestLootboxRng() calls coordinator (AdvanceModule line ~950)
- CLOSES: rawFulfillRandomWords() calls _finalizeLootboxRng() (AdvanceModule line ~1550)

Key question: Can a player open a lootbox, purchase a lootbox, or change lootbox-affecting state between request and fulfillment?

Trace the lootbox packed state (_lrRead/_lrWrite) to determine:
- What index is being fulfilled?
- Can new lootboxes be purchased that would be affected by this word?
- Does midDayTicketRngPending interact with the lootbox window?

**Section 3: Between-Day Window (advanceGame to next advanceGame)**

Window: rngGate() delivers word at start of advanceGame -> _requestRng() requests next word at end of advanceGame
- The word for "today" is known on-chain after rngGate returns
- What state gets resolved by this word during payDailyJackpot/payDailyJackpotCoinAndTickets?
- Was all that state committed before rngGate delivered the word?
- Specifically: ticket holders, trait selections, bucket assignments, bet amounts

**Section 4: Gameover Window**

_gameOverEntropy() uses block.prevrandao:
- prevrandao is known to the block proposer
- What state gets resolved by prevrandao in handleGameOverDrain?
- Is this window exploitable by a validator/proposer?

**For each window, produce a table:**

| Function | Blocked by rngLockedFlag? | Guard Site | State Changed | Affects RNG Outcome? | Risk |
|----------|--------------------------|------------|---------------|---------------------|------|

End with an overall risk matrix:
| Window | Duration | Controllable State | Risk Level | Rationale |
