# Phase 482 / Plan 01: Degenerette Dead-Mode Repack (gated contract diff #4 — BEHAVIOR-adjacent, encoding-only)

## Objective

Remove the dead fractional-bet "Full Ticket" mode from the Degenerette packed bet. Strip the dead
`mode` / `isRandom` / `hasCustom` bits (and the dead hero `reserved` bit), re-pack `degeneretteBets`
into the freed space, rename `FT_*_SHIFT` → `DEGEN_*_SHIFT` (shift VALUES change as the dead bits are
removed), and rename `_packFullTicketBet` → `_packDegeneretteBet`. The player-visible outcome
distribution (EV / resolution) MUST be **byte-identical** — only the internal bit ENCODING changes.

## HARD GATE — Dead-mode proof (executed before any strip)

The packed bet is WRITTEN at exactly one site (`_packDegeneretteBet`, formerly `_packFullTicketBet`,
called from `_placeDegeneretteBetCore`). It is READ at:
- `_resolveBet` (the full decode → resolution/score/payout/EV)
- `DegenerusGame.degeneretteResolve` `betPacked >> 42 & 0x3` (currency, reward-gate) + `== 0` sentinel
- `DegenerusGame.getDegeneretteBet` raw getter (off-chain only)

Bits NEVER read behaviorally: `mode` [0], `isRandom` [1], `hasCustom` [236], hero `reserved` [237].
- `mode` (MODE_FULL_TICKET): written always-1, never read. The only mode; one bet path.
- `isRandom`: never written (always 0) and never read — a pure comment relic.
- `hasCustom`: written always-1, never read.
- hero `reserved` bit: written always-1, never read (unpack reads `>> (FT_HERO_SHIFT + 1)`).

**Sentinel-safety invariant (Rule-2 correctness, must preserve):** `packed == 0` means "no bet".
Historically guaranteed non-zero by `mode`[0]=1. After the strip, non-zero is guaranteed independently
by `spinCount` (validated `≥ 1`, occupying `DEGEN_COUNT_SHIFT`). A live bet stays non-zero. PRESERVED.

Verdict: the mode is **provably dead** → strip is safe. (If any bit had been live → STOP+FLAG, no strip.)

## New bit layout (compact; LSB → MSB)

| field          | bits      | width | NEW const             | NEW val | OLD const            | OLD val |
|----------------|-----------|-------|-----------------------|---------|----------------------|---------|
| customTraits   | [0..31]   | 32    | DEGEN_TRAITS_SHIFT    | 0       | FT_TICKET_SHIFT      | 2       |
| spinCount      | [32..39]  | 8     | DEGEN_COUNT_SHIFT     | 32      | FT_COUNT_SHIFT       | 34      |
| currency       | [40..41]  | 2     | DEGEN_CURRENCY_SHIFT  | 40      | FT_CURRENCY_SHIFT    | 42      |
| amountPerSpin  | [42..169] | 128   | DEGEN_AMOUNT_SHIFT    | 42      | FT_AMOUNT_SHIFT      | 44      |
| index          | [170..201]| 32    | DEGEN_INDEX_SHIFT     | 170     | FT_INDEX_SHIFT       | 172     |
| activityScore  | [202..217]| 16    | DEGEN_ACTIVITY_SHIFT  | 202     | FT_ACTIVITY_SHIFT    | 220     |
| heroQuadrant   | [218..219]| 2     | DEGEN_HERO_SHIFT      | 218     | FT_HERO_SHIFT (+1)   | 237     |

Removed: `MODE_FULL_TICKET` (bit 0), `isRandom` (bit 1, comment-only), `FT_HAS_CUSTOM_SHIFT` (bit 236),
hero `reserved` (bit 237). Field WIDTHS preserved exactly (index stays 32-bit as today: `uint32(index)`
packed, `MASK_32` read — the 48-bit doc reservation collapses to the actual 32-bit field). 220 bits used
(was 240). No storage SLOT moves — `degeneretteBets` stays the same `mapping(... => uint256)` at slot 38.
