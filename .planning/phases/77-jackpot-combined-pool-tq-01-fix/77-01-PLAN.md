<objective>
Modify _awardFarFutureCoinJackpot to select jackpot winners from a combined pool spanning both the frozen read buffer (_tqReadKey) and the far-future key space (_tqFarFutureKey), replacing the vulnerable _tqWriteKey read at JackpotModule:2544.

Purpose: After Phase 75's routing change, far-future tickets land in the FF key space instead of the write buffer. If _awardFarFutureCoinJackpot only reads the write buffer, it (a) misses all FF-routed tickets (JACK-01), and (b) reads a mutable buffer vulnerable to frontrunning during the VRF commitment window (TQ-01/EDGE-03). The combined pool reads both the frozen read buffer and the FF key, covering all eligible tickets with commitment-window safety.

Output: Modified _awardFarFutureCoinJackpot in JackpotModule, Foundry test file proving all three requirements.
</objective>
