### 2.3 Sentinel-Index Correctness Verification

The guard at AdvanceModule:1174 reads `rngWordByDay[idx + 1]` where `idx = dailyIdx`. For the guard to correctly detect "backfill not yet run for this lock window," the sentinel index must satisfy:

2. **The sentinel key is the FIRST gap day written by `_backfillGapDays`:** loop at L1762 starts at `gapDay = startDay = idx + 1` and the first iteration writes `rngWordByDay[idx + 1] = derivedWord` at L1766. ✓ — first iteration always lands the sentinel write.
3. **No off-by-one against `idx`:** sentinel reads `idx + 1` (NOT `idx`). `dailyIdx == idx` already has `rngWordByDay[idx]` set (from prior wall-clock day's `_unlockRng` + `_applyDailyRng` cycle); if the guard read `rngWordByDay[idx]` it would always evaluate FALSE (already-set) and skip backfill on the very first lock-window entry. ✓ — `idx + 1` is correct.
4. **No off-by-one against `day`:** sentinel reads `idx + 1` (NOT `day - 1`). `day - 1` is the LAST gap day, not the first; reading `rngWordByDay[day - 1]` would incorrectly evaluate FALSE on multi-day re-entries where the loop got partway through but didn't finish (impossible under current single-call semantics, but defensive-correctness check). ✓ — `idx + 1` is the safer choice.
