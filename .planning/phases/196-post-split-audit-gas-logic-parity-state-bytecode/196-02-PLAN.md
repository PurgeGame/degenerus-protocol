**Section 1: Two-Call Split Parity Analysis**

For _processDailyEth, prove that call 1 + call 2 produces the same total paidEth as a hypothetical single call iterating all 4 buckets. Trace these specific properties:

A. **Share computation equivalence:** Both calls use the same ethPool value (call 2 reads the snapshot from resumeEthPool, which call 1 stored from the original ethPool param). Both calls pass the same shareBps and bucketCounts to JackpotBucketLib.bucketShares(). Therefore `shares[i]` is identical in both calls for all 4 buckets. The same `unit` and `remainderIdx` are computed from the same inputs. **Verdict:** shares are identical.

B. **Bucket partitioning completeness:** call1Bucket mask assigns exactly 2 buckets to call 1 (order[0] + remainderIdx, or order[0] + order[1] if order[0]==remainderIdx). Call 2 processes the remaining 2 (line 1219 skips when `isResume == call1Bucket[traitIdx]`). Together they cover all 4 buckets exactly once. **Verdict:** no bucket is processed twice or skipped.

C. **Per-winner payout equivalence:** For each bucket, the payout is `share / totalCount` where both `share` and `totalCount` are identical across calls (same shares array, same bucketCounts array). Winner selection via `_randTraitTicket` uses `entropyState` which DOES diverge between split and single-call execution (because the single call would XOR all 4 bucket indices sequentially, while split calls XOR only their assigned buckets). This means **different winners may be selected**, but the same number of winners per bucket, the same per-winner amount, and the same total payout. **Verdict:** different RNG chain is acceptable per SC-3; economic totals match.

D. **Pool accounting equivalence:** Call 1 returns paidEth for its 2 buckets. The caller (payDailyJackpot fresh daily flow) deducts paidEth from currentPool or futurePool. Call 2 returns paidEth for its 2 buckets. The caller (_resumeDailyEth lines 1147-1153) deducts paidEth2 from futurePool or currentPool. Total deduction = paidEth + paidEth2 = what a single call would return. **Verify:** the caller reads the pool AFTER each deduction, so there's no TOCTOU — call 1's deduction is committed before call 2 runs (separate EVM calls).

E. **claimablePool accounting:** Each call adds its own `liabilityDelta` to `claimablePool` (line 1286-1287). A single call would add the combined delta. Since `claimablePool += liabilityDelta` is additive, `a + b == a + b`. **Verdict:** equivalent.

F. **Edge case: ethPool == 0.** If pool is 0, both calls return 0 immediately (line 1183-1184).
