Naming-convention function (per D-03):
```
interface IDegenerusGameBoonModule       →  GAME_BOON_MODULE
       strip leading `I`                  →  DegenerusGameBoonModule
       replace `DegenerusGame` with ``    →  BoonModule
       CamelCase → UPPER_SNAKE            →  BOON_MODULE
       append `_MODULE` if missing        →  BOON_MODULE  (already has it)
       prepend `GAME_`                    →  GAME_BOON_MODULE
```
Reference implementation in bash (awk-based):
```bash
# Known naming exceptions where the ContractAddresses constant does NOT follow
# pure CamelCase → UPPER_SNAKE of the interface suffix. The canonical example:
# IDegenerusGameGameOverModule ↔ GAME_GAMEOVER_MODULE (not GAME_GAME_OVER_MODULE).
# These are keyed by the stripped suffix (post-`I` and post-`DegenerusGame`) and
# map directly to the post-`GAME_` portion of the constant. One row today; add
# more if future interfaces deviate from convention.
declare -A NAMING_EXCEPTIONS=(
  [GameOverModule]=GAMEOVER_MODULE
)

iface_to_constant() {
  local iface="$1"                          # e.g., IDegenerusGameBoonModule
  local stripped="${iface#I}"               # DegenerusGameBoonModule
  stripped="${stripped#DegenerusGame}"      # BoonModule
  # Check exception map FIRST — exception-handled names skip the sed transform.
  if [[ -n "${NAMING_EXCEPTIONS[$stripped]:-}" ]]; then
    printf 'GAME_%s\n' "${NAMING_EXCEPTIONS[$stripped]}"
    return
  fi
  # CamelCase → UPPER_SNAKE: insert `_` before each uppercase after position 1, then uppercase
  local snake
  snake=$(printf '%s' "$stripped" | sed -E 's/([a-z0-9])([A-Z])/\1_\2/g' | tr '[:lower:]' '[:upper:]')
  # snake == "BOON_MODULE"
  [[ "$snake" == *_MODULE ]] || snake="${snake}_MODULE"
  printf 'GAME_%s\n' "$snake"
}
```

**Self-test coverage:** When Task 2 runs its inline self-test against the 9 live interfaces, all 8 convention-conforming cases pass via the sed transform and `GameOverModule` passes via the exception map. 220-02 Task 2 defines the reverse-direction `constant_to_iface()` using the same exception map; both directions stay consistent.

Implementation approach:

1. **Enumerate sites** by anchoring on selector references, then extracting target constant from the preceding lines:
```bash
# Anchor: every line matching `IDegenerusGame.+Module.<fn>.selector` outside interfaces/ and mocks/
sites=$(grep -rn --include='*.sol' -E 'IDegenerusGame[A-Za-z]+Module\.[a-zA-Z_]+\.selector' contracts/ \
  | grep -v '^contracts/interfaces/' \
  | grep -v '^contracts/mocks/')
```

2. **For each anchor match, walk back up to 10 lines** looking for `.GAME_[A-Z_]+_MODULE.delegatecall(` on the same or preceding line. Use awk with `file:line` lookups or a small shell loop reading each file's lines into an array. Reference pattern (shell only, no node/python):
```bash
while IFS=: read -r file line_no match; do
  # read file once, slice lines [line_no-10 .. line_no]
  window=$(awk -v n="$line_no" 'NR >= n-10 && NR <= n' "$file")
  target=$(printf '%s\n' "$window" | grep -oE '\.GAME_[A-Z_]+_MODULE\.delegatecall\(' | tail -1 | sed -E 's/^\.([A-Z_]+)\.delegatecall\(.*$/\1/')
  iface=$(printf '%s' "$match" | grep -oE 'IDegenerusGame[A-Za-z]+Module')
  # apply naming-convention transform (see iface_to_constant in interfaces block above)
  expected=$(iface_to_constant "$iface")
  if [[ "$target" == "$expected" ]]; then
    printf "%bOK%b   %s:%d  %s → %s\n" "$GREEN" "$NC" "$file" "$line_no" "$iface" "$target"
  else
    printf "%bFAIL%b %s:%d  %s expects %s but targets %s\n" "$RED" "$NC" "$file" "$line_no" "$iface" "$expected" "$target"
    fail_total=$((fail_total + 1))
  fi
done <<< "$sites"
```

3. **Edge cases** the script must handle correctly:
- No `.GAME_*_MODULE.delegatecall(` found in the 10-line window → print `WARN` line `"orphan selector reference at $file:$line (no delegatecall target found within 10-line window)"` and increment warn_total. This catches cases where someone uses `.selector` outside the canonical delegatecall pattern.
- Derived constant name contains characters outside `[A-Z0-9_]` (shouldn't happen, but defensive) → print `WARN` `"non-conventional interface name $iface"`.
- Allowlist support: if a line within the 10-line window contains the literal comment `// delegatecall-alignment: justified` (D-07), treat the mismatch as `JUSTIFIED` and do NOT count it as a failure, but still print `WARN` with the justification string for visibility.

4. **Naming-convention helper** — implement `iface_to_constant()` exactly as shown in the `<interfaces>` block of this plan. Test it inline during script execution on the 9 known interfaces and abort with a clear error if any transformation fails to produce a constant that exists in the output of `grep -E 'GAME_[A-Z_]+_MODULE' contracts/ContractAddresses.sol` (addresses D-05 MAPPING_ERROR case).

5. **Exit code policy** (match check-interface-coverage.sh):
- exit 0 if fail_total == 0 AND warn_total == 0
- exit 1 if fail_total > 0
- exit 1 if warn_total > 0 (orphan selector references or mapping errors should block — they indicate either dead code or a convention break)

6. **Keep under 200 lines** (D discretion: bash, not node/python, matches the sister script's stack).
