## High-Risk Patterns Analyzed

### Shared-Helper Determinism (`_rollWinningTraits`)

`_rollWinningTraits(uint256 randWord, bool isBonus) private view returns (uint32 packed)` at line 1865-1875. Declared `private view` — reads state but does not modify. Function body:
1. **Line 1869-1871:** `uint256 r = isBonus ? uint256(keccak256(abi.encodePacked(randWord, BONUS_TRAITS_TAG))) : randWord;` — with `isBonus=true` (the only case relevant to JKP-03), `r` is the keccak of `(randWord || BONUS_TRAITS_TAG)`. `BONUS_TRAITS_TAG` is a compile-time constant (line 171), so the keccak depends only on `randWord`.
2. **Line 1872:** `uint8[4] memory traits = JackpotBucketLib.getRandomTraits(r);` — pure library function, deterministic bit-sample of `r`.
3. **Line 1873:** `_applyHeroOverride(traits, r);` — IMPURITY CANDIDATE (reads state); see next sub-section.
4. **Line 1874:** `packed = JackpotBucketLib.packWinningTraits(traits);` — pure library function, deterministic bit-packing.

Cross-path agreement reduces to: given the same `randWord` input AND identical `_applyHeroOverride` state reads, does `_rollWinningTraits` produce byte-identical `packed` output at all five consumer sites? Answer: YES, subject to frozen `_applyHeroOverride` state (classified in next sub-section).

### Bonus vs Main Salt-Space Separation (Prior-Art Citation)

Phase 231 EBD-02 audit (`.planning/phases/231-earlybird-jackpot-audit/231-02-AUDIT.md`, committed 2026-04-17 per STATE.md) previously established that `BONUS_TRAITS_TAG = keccak256("BONUS_TRAITS")` at line 171 provides preimage-resistant domain separation between `isBonus=true` and `isBonus=false` output spaces for the same `randWord`. `keccak256(abi.encodePacked(randWord, 32-byte-tag))` at line 1870 hashes 64 bytes (32-byte randWord + 32-byte tag); `randWord` alone (the `else` branch at line 1871) is 32 bytes; different preimage lengths guarantee disjoint outputs by keccak-256 collision resistance even before domain separation by tag content.

JKP-03 cites this as prior art per D-09 non-overlap. JKP-03's own focus is cross-path agreement (same flag, same input → same output across all five consumer sites), not salt-space separation (different flag → disjoint output spaces).

### Cross-Path Invocation-Shape Agreement

- **Identical second argument:** the literal `true` constant passed at all five call sites. Verified by grep.
- **First argument from same VRF source:** consumer #1 (521) + consumer #2 (507) + consumer #3 (593) + consumer #4 (1679) all receive `randWord` from the delegatecall calldata of `payDailyJackpot(isJackpotPhase, lvl, randWord)` / `payDailyJackpotCoinAndTickets(randWord)` — both dispatched by AdvanceModule's `payDailyJackpot(...)` wrapper which sources `randWord` from `rngGate` return at AdvanceModule:283 (cross-referenced to JKP-02 entropy-passthrough audit). Cross-path invocation shape holds by construction (Phase 231 EBD-02 prior art confirms this specifically for consumer #5 vs consumers #4 + line 1705; JKP-03 extends to all five sites).

### `_applyHeroOverride` State-Reading Classification

`_applyHeroOverride(uint8[4] memory w, uint256 randomWord) private view` at line 1539-1566. State reads:
1. **`_topHeroSymbol(_simulatedDayIndex())` at line 1547** — computes `day = _simulatedDayIndex()` (which delegates to `GameTimeLib.currentDayIndex()` at `storage.sol:1213`; returns `block.timestamp / DAY_SECONDS` — tx-fixed value), then reads `dailyHeroWagers[day][q]` for `q = 0..3` at `JackpotModule:1579`.
2. No other state reads inside `_applyHeroOverride` body — the subsequent lines 1550-1565 are pure computations on `randomWord` bits and `heroQuadrant` / `heroSymbol` / `heroColor` locals.

**`dailyHeroWagers` writer surface:** grep confirms the ONLY writer is `DegenerusGameDegeneretteModule._placeBet` at line 477 (`dailyHeroWagers[day][heroQuadrant] = wPacked;`). `_placeBet` requires `msg.sender == player` — a player-originated transaction.
