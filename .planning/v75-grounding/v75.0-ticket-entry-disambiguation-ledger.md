## 1. Canonical unit glossary (ground truth, verified 3 ways)

- **ticket** = a WHOLE unit = `priceForLevel(level)` = **exactly 4 entries**. As a code token, "ticket" denotes ONLY the whole unit (or the whole-ticket *purchase leg* / reward path / queue *mechanism* — never a sub-unit count).
- **entry** = `price/4` = **1 owed unit**. The `ticketsOwedPacked`→`entriesOwedPacked` accumulation basis. As a code token, "entry"/"entries" must mean ONLY this, protocol-wide.
- **TICKET_SCALE = 100** (→ **renamed `QTY_SCALE`** per §10.7, owner 2026-06-29; unit-neutral scale, not a unit name), **AFKING_TICKET_SCALE = 400** — SCALE FACTORS, not unit names. AFKING_TICKET_SCALE KEEP. (One whole ticket in afking purchase units = `4 entries × 100 = 400`.)
- **decimator BET** — the side-game per-player burn record (`DecEntry`/`TerminalDecEntry`). NOT a ticket and NOT an entry. The decimator has NO "tickets" and NO "entries"; it has **bets**.
- **degenerette SPIN / bet** — the side-game wager. One bet path only (the fractional "Full Ticket" mode is removed/dead). A degenerette "ticket" is a relic: either a **spin** (bet unit) or a **4-trait reel/card** (`traits`). The degenerette has NO "tickets".
- Unit proof (3 ways): (a) 1-whole-ticket buy costs `priceWei·ticketQuantity/(4·TICKET_SCALE)` with `ticketQuantity=400` → `_queueTicketsScaled(400)` → `owed += 4`; (b) `_budgetToTicketUnits=(budget<<2)/price=4·budget/price` = entries; (c) `FOIL_PACK_ENTRIES=16="4 tickets × 4 quadrants"` and `FOIL_PACK_TICKETS=10="ten ticket prices"` together pin `1 ticket = 4 entries`.
