### [High] BAF Whale-Pass Remainder Asset Deficiency
**File: contracts/modules/DegenerusGameJackpotModule.sol:1949, 2001**
**File: contracts/modules/DegenerusGamePayoutUtils.sol:58**
**File: contracts/modules/DegenerusGameAdvanceModule.sol:902**

When a large BAF win (or an odd-indexed small win) is processed, the `lootboxPortion` is handled by `_queueWhalePassClaimCore`. This function calculates a `remainder` (rounding dust below the 2.25 ETH whale-pass price) and credits it to the winner's `claimableWinnings` while incrementing the global `claimablePool` (PayoutUtils:58). However, the `runBafJackpot` function only returns the "immediate ETH" portion (`claimableDelta`) to the caller. The `AdvanceModule` then debits only this returned `claimableDelta` from `memFuture` (`futurePrizePool`) at line 902. 

Because the `remainder` is added to the `claimablePool` liability but never debited from the `futurePrizePool` asset bucket, the same ETH is counted twice in the protocol's obligations. This creates a cumulative solvency leak where `address(this).balance + stETH_held < Σ liabilities`.

### [High] Decimator Lootbox Remainder Missing Liability
**File: contracts/modules/DegenerusGameDecimatorModule.sol:596**
**File: contracts/modules/DegenerusGameDecimatorModule.sol:390**

During a Decimator jackpot claim, `_creditDecJackpotClaimCore` debits the full `lootboxPortion` from `claimablePool` (line 390). If the payout is large enough to trigger whale passes, `_awardDecimatorLootbox` calculates a `remainder` (dust after awarding whole passes) and credits it back to the player's winnings via `_creditClaimable` (line 596). 

However, `_creditClaimable` only updates the player's individual `claimableWinnings` and does NOT increment the aggregate `claimablePool`. Since the full `lootboxPortion` (including the remainder) was already debited from `claimablePool`, this "returned" ETH liability is now missing from the aggregate accounting. This breaks the core invariant `claimablePool == Σ claimableWinnings + Σ afkingFunding`, leaving the protocol under-reporting its true liability.

### [Medium] Yield Over-distribution via Under-reported Liability
**File: contracts/modules/DegenerusGameJackpotModule.sol:718, 693**

The `distributeYieldSurplus` function calculates the distributable yield surplus by subtracting protocol `obligations` from the total assets (line 709). These obligations include the `claimablePool` (line 693). Due to the Decimator remainder omission (see Finding above), the `claimablePool` artificially under-reports the protocol's true liability. 

Consequently, `distributeYieldSurplus` will over-estimate the available yield and mint new `claimablePool` liability (line 718) that is not backed by actual stETH appreciation. This compounds the solvency error and further depletes the reserve backing player winnings.
