The Degenerus Protocol peripheral contracts and cross-contract call seams were audited at commit `c4d48008`. The audit focused on solvency, RNG integrity, and selector correctness across delegatecall boundaries.

### **Summary of Verified Peripherals**
- **`DegenerusAdmin`**: VRF coordinator-swap governance is robust. The 48-hour delay for threshold reduction (`2500` BPS) is correctly implemented.
- **`DegenerusDeityPass`**: Successfully soulbound; all transfer paths are blocked. The 32-token cap is strictly enforced via symbol-id gating.
- **`BurnieCoin` / `BurnieCoinflip`**: Mint/burn authority is correctly gated to the Game/Vault. The `creditFlip` bounty system honors the trust boundary.

### **Findings**

#### **1. Predictable RNG in sDGNRS Redemption Lootboxes (CHOICE-OF-ENTROPY)**
**PROPERTY:** Deterministic lootbox outcomes if claimed on the day of resolution.  
**CALL SEQUENCE:** `StakedDegenerusStonk.claimRedemption(day)`  
**CONTRACT:** `StakedDegenerusStonk.sol:412`  
**SEVERITY:** **HIGH**  
**WHY PROTECTIONS FAIL:** `claimRedemption` is reachable as soon as `redemptionPeriods[day]` is set (during the `advanceGame` of day `day`). It keys entropy to `game.rngWordForDay(day + 1)`. However, on day `day`, the word for `day + 1` is not yet available and returns `0`. A player can (1) predict the `0`-word outcome and only claim if itâ€™s a win, or (2) wait until `day + 1` to get a different outcome from the real VRF word, effectively choosing between two seeds.

#### **2. Affiliate Credit Theft via Heterogeneous Sub-Arrays**
**PROPERTY:** Permissionless redirection of referral credits to an arbitrary upline.  
**CALL SEQUENCE:** `DegenerusAffiliate.claim([victimSub, attackerSub])`  
**CONTRACT:** `DegenerusAffiliate.sol:112`  
**SEVERITY:** **HIGH**  
**WHY PROTECTIONS FAIL:** The `claim(address[] calldata subs)` function resolves the affiliate `a` (and uplines `u1/u2`) ONLY for `subs[0]`. It then iterates the entire array, draining `affiliateBase` from all subscribers and summing them into `sumB`. The total `sumB` is then credited to the `subs[0]` upline. An attacker can bundle their own sub at index 0 with a list of other active subs to steal their accrued whole-BURNIE credits.

#### **3. Deity Pass Ticket Queue Gap (Levels 49/50 Missed)**
**PROPERTY:** Under-delivery of promised whale-equivalent tickets.  
**CALL SEQUENCE:** `DegenerusGameWhaleModule.purchaseDeityPass`  
**CONTRACT:** `DegenerusGameWhaleModule.sol:349`  
**SEVERITY:** **LOW**  
**WHY PROTECTIONS FAIL:** The `ticketStartLevel` calculation `((passLevel + 1) / 50) * 50 + 1` creates a gap if a pass is purchased on level 48 (moving to 49). `passLevel = 49`, `(50/50)*50 + 1 = 51`. The queue starts at 51, but the pass is active for 49 and 50. The user misses two levels of ticket entries they paid for.

#### **4. Missing/Renamed Interface Stubs in `IDegenerusGame`**
**PROPERTY:** Mismatch between peripheral interface and contract implementation.  
**CONTRACT:** `interfaces/IDegenerusGame.sol` vs `DegenerusGame.sol`  
**SEVERITY:** **LOW**  
**WHY PROTECTIONS FAIL:** While the `DegenerusGame.sol` implementation correctly contains the `delegatecall` stubs (fixing the `claimAfkingBurnie` bug class), the `IDegenerusGame.sol` interface is missing several public-facing entry points (e.g., `subscribe`, `mintBurnie`, `claimAfkingBurnie`, `decurse`, `smite`). This prevents other contracts from calling these functions via the standard interface without using raw `low-level calls`.

#### **5. stETH-Strand Fix Divergence in Vault**
**PROPERTY:** Potential redemption blockage for contract-based DGVE holders.  
**CONTRACT:** `DegenerusVault.sol:368`  
**SEVERITY:** **INFORMATIONAL**  
**WHY PROTECTIONS FAIL:** `StakedDegenerusStonk` contains the C-2 fix (`_payEth` attempts to send ETH then falls back to stETH if the user rejects ETH). However, `DegenerusVault.sol` uses a different `burnEth` flow that calculates `ethOut` and `stEthOut` but calls `_payEth` (which reverts on failure) regardless. If a contract holder cannot receive ETH, they cannot redeem their stETH share either, although funds are not "lost" as they retain their shares.
