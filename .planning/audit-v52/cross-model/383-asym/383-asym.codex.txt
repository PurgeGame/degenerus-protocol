**Finding: ASYM-03 Deity Pass Queues The Wrong 100-Level Ticket Window**

PROPERTY: Deity pass “whale-equivalent” queued tickets diverge from the whale bundle sibling at half-century boundaries.

Reachable call sequence:
1. Game is live, not RNG-locked, at `level = 48`; therefore `passLevel = level + 1 = 49`.
2. Player calls `purchaseDeityPass(buyer, symbolId)`.
3. `_purchaseDeityPass` accepts payment and records the deity pass.
4. It computes:
   `ticketStartLevel = ((passLevel + 1) / 50) * 50 + 1 = 51`
5. It queues deity whale-equivalent tickets for levels `51..150`, skipping levels `49` and `50`.
6. Same issue occurs at `passLevel = 50`, and repeats around `99/100`, `149/150`, etc.

STATE VAR + file:line:
- Wrong deity ticket anchor: `contracts/modules/DegenerusGameWhaleModule.sol:639-641`
- Deity pass state: `mintPacked_`, `deityBySymbol`: `contracts/modules/DegenerusGameWhaleModule.sol:611-618`
- Queue mutation: `ticketQueue`, `ticketsOwedPacked`: `contracts/storage/DegenerusGameStorage.sol:670-695`
- Symmetric sibling comparison:
  - Whale bundle anchors at `passLevel`: `contracts/modules/DegenerusGameWhaleModule.sol:190`, `:222`, `:323-327`
  - Lazy pass anchors at current next level: `contracts/modules/DegenerusGameWhaleModule.sol:446`, `:498`

SEVERITY: LOW, paid-pass reward/odds correctness. No solvency break and no RNG freeze break.

Why protections do not stop it:
- The guards only check game liveness, RNG lock, symbol uniqueness, and one-deity-pass ownership.
- `_queueTicketRange` only prevents far-future queueing during RNG lock; it does not validate that the chosen start level matches the paid pass window.
- Deity’s special virtual jackpot handling does not repair the missing queued tickets, because this code separately queues “whale-equivalent” tickets.

If deity is meant to mirror whale exactly, the start should be `passLevel`. If it is meant to align to the current 50-level epoch, the formula should use `(passLevel - 1) / 50`, not `(passLevel + 1) / 50`.
