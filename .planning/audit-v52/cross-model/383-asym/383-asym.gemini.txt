### ASYM-03 — Pass types: Lazy Pass price-basis asymmetry
PROPERTY · `_purchaseLazyPass` credits presale boxes and lootbox-10% rewards using the undiscounted `benefitValue` (e.g. 0.24 ETH) even when the buyer pays a discounted `totalPrice` via a Boon. Sibling paths (`WhaleBundle`, `DeityPass`) correctly use the discounted `totalPrice` (ETH paid). · `presaleBoxCredit` + `_recordLootboxEntry` at `WhaleModule.sol:493/498` · **LOW** · Boon-users get "free" box credit and a larger lootbox relative to their actual ETH committed.

### ASYM-03 — Pass types: Whale Pass `LAST_LEVEL` regression
PROPERTY · `_applyWhalePassStats` (used by `claimWhalePass` and Decimator claims) sets `LAST_LEVEL` directly to the `newFrozenLevel`. If a player's `LAST_LEVEL` was already higher (e.g. level 500) than the pass's horizon (e.g. level 200), the stat regresses. This allows the player to "re-mint" level 500, causing `LEVEL_COUNT` to double-dip in `MintModule.recordMintData` (which only skips `total++` if `lvl == prevLevel`). · `mintPacked_[player]` (LAST_LEVEL field) at `storage/DegenerusGameStorage.sol:1316-1321` · **LOW** · Inflates Activity Score via `levelCount` double-counting.

### ASYM-05 — RNG-consume: `resolveLootboxDirect` seed collision
PROPERTY · `resolveLootboxDirect` (used by Decimator and Degenerette) generates its `seed` using `keccak256(abi.encode(rngWord, player, amount))`. Unlike the standard `openLootBox` path, it omits the `index` (or `betId`). A player resolving multiple boxes (e.g. from multiple Degenerette bets) at the same index with the same amount will receive identical rewards (same boons/passes). · `seed` at `modules/DegenerusGameLootboxModule.sol:762` · **LOW** · Lack of domain separation for multi-box resolutions in a single window.

### ASYM-06 — Pool/credit updates: SYMMETRIC
FAMILY VERIFIED SYMMETRIC: Pool mutations for `claimablePool`, `futurePrizePool`, and `nextPrizePool` are correctly paired across all modules. `DegeneretteModule` correctly debits the `runningFuture` local before flushing to `futurePrizePool` while simultaneously crediting `claimablePool`. `JackpotModule` distributions (including solo-bucket 75/25 conversion) conserve the solvency spine.
