### New Findings

### CMT-V32-001: decClaimable @return still references "expired" after expiry removal

- **What:** The `@return amountWei` NatSpec for `decClaimable` at line 437 states `Claimable amount (0 if not winner, already claimed, or expired).` Since commit 19f5bc60 removed the decimator claim expiry, claims are now persistent and never expire. The "or expired" clause is stale.
- **Where:** DegenerusGameDecimatorModule.sol:437
- **Why:** A warden reading this NatSpec would expect a time-based or round-based expiry mechanism for decimator claims. They would search for the expiry logic, not find it, and potentially file a finding about missing expiry enforcement -- or conversely, wonder if claims should expire but the check was accidentally removed. The function returns 0 for exactly two reasons: not a winner, or already claimed. The "expired" case no longer exists.
- **Suggestion:** Change to: `@return amountWei Claimable amount (0 if not winner or already claimed).`
- **Category:** comment-inaccuracy
- **Severity:** INFO
