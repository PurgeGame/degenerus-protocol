### CMT-V32-002: processFutureTicketBatch @return writesUsed describes queue entries but returns write-budget units

- **What:** The `@return writesUsed` NatSpec at line 301 states `Number of queue entries processed in this batch.` The actual return value at line 421 (`writesUsed = used`) is the accumulated write-budget cost, not a count of queue entries. A single queue entry with many owed tickets may consume tens or hundreds of budget units (line 391: `writesThis = (take <= 256) ? (take * 2) : (take + 256)`), and a skipped/zero entry still costs 1 unit (line 353). The value returned is therefore always >= the number of entries processed but often much larger.
- **Where:** DegenerusGameMintModule.sol:301
- **Why:** A caller or warden reading this NatSpec would interpret `writesUsed` as a simple entry count and use it to estimate how many more calls are needed to drain the queue. If `writesUsed` returns 500 for 10 actual entries processed, the caller would grossly overestimate progress and underestimate remaining work. The advanceGame caller uses this return value for gas budget decisions, so correct interpretation is important.
- **Suggestion:** Change to: `@return writesUsed Write-budget units consumed in this batch (not a simple entry count; each entry costs 2+ units depending on ticket count).`
- **Category:** comment-inaccuracy
- **Severity:** INFO
