Pool Architecture Overview

All ETH entering the protocol flows through a 4-pool lifecycle. futurePrizePool accumulates incoming ETH, transitions to currentPrizePool at each level, which is drawn down by jackpot distributions into claimablePool. Players claim from claimablePool via claimWinnings(). The yieldAccumulator collects stETH rebasing surplus and releases 50% to futurePrizePool at x00 milestones.

2. Scatter / Decimator

Pool Source Distinction: BAF scatter and x00 decimator use baseFuturePool (a snapshot taken at the start of level processing). Normal decimator uses futurePoolLocal (the running total after BAF deduction). This means the normal decimator's 10% is calculated AFTER the BAF scatter has already deducted its share from the pool.

BAF Normal Scatter (PAY-03)

Fires at levels 10, 20, 30, etc. (every lvl % 10 == 0). Uses 10% of baseFuturePool, except at level 50 where the percentage is elevated to 20%. The pool allocation is distributed across 7 prize categories via the _runBafJackpot function in DegenerusJackpots.

BAF Century Scatter (PAY-04)

Fires at levels 100, 200, 300, etc. (every prevMod100 == 0). Uses 20% of baseFuturePool. Century scatter uses a distinct sampling pattern: 4 rounds at lvl+1, 4 rounds at lvl+2, 4 rounds at lvl+3, then 38 rounds randomly selected from the past 99 levels. Both share the same 7-category distribution mechanics via _runBafJackpot.

Edge Cases

  • totalBurn == 0: No winners; full pool returned as decRefund. In normal gameplay, recycled to the next round. In GAMEOVER, returned to terminal jackpot.
  • lastDecClaimRound overwrite: By-design per v1.1 spec (EDGE-04). Previous round's unclaimed funds are recycled to the next round. Only one decimator round can be active at a time.
  • claimed[player] flag: Prevents double-claim within a round. Per-level per-player flag.
  • prizePoolFrozen guard: Blocks claims during jackpot phase days 1-4 to prevent interference with jackpot distribution.

3. Coinflip Economy

BURNIE Isolation: The coinflip economy operates entirely in BURNIE tokens via a burn-and-mint model. It is fully isolated from the ETH claimablePool -- no coinflip operation can affect protocol ETH solvency. Net house edge is ~1.575% per flip (structurally deflationary).

Edge Cases

  • Zero stake: coinflipBalance cleared to 0 on first processing; no double-payout path exists.
  • RNG locked: rngLockedFlag blocks claims during VRF resolution on all 6 external claim/toggle paths.
  • Claim window asymmetry: 30 days first-time, 90 days returning (by-design; PAY-07-I01 INFORMATIONAL). First-time players who wait 31+ days lose winnings.
  • Auto-rebuy carry zeroed on loss: carry = 0 on loss branch -- all accumulated carry is destroyed.
  • Fully isolated from ETH claimablePool: No coinflip path mutates ETH pool accounting.

6. GAMEOVER Terminal Distribution

GAMEOVER is the terminal state of the Degenerus Protocol. When the death clock expires (365 days at level 0, 120 days at level 1+), all remaining protocol funds are distributed in a single irreversible sequence. The GAMEOVER path is protected by idempotency latches (gameOver, gameOverFinalJackpotPaid, finalSwept) that prevent re-execution.

FINDING-MEDIUM (GO-05-F01): _sendToVault uses hard reverts on ETH/stETH transfer failures. If the vault or sDGNRS contract cannot receive funds (e.g., Lido stETH pause), handleGameOverDrain and handleFinalSweep revert permanently. Risk: 7 dangerous revert sites across the GAMEOVER path. Mitigation: Recipients are immutable, protocol-owned contracts. See audit/KNOWN-ISSUES.md for full analysis.

7. Cross-System: claimablePool Invariant

Invariant INV-01: claimablePool <= address(this).balance + steth.balanceOf(address(this)) — Verified PASS at all 15 mutation sites across GAMEOVER and normal gameplay paths.

The claimablePool tracks ETH owed to players. This invariant ensures the protocol always holds enough ETH+stETH to cover all pending claims. Every code path that increases claimablePool also receives corresponding ETH, and the only code path that decreases claimablePool is the claimWinnings() function which transfers the exact amount being deducted.
