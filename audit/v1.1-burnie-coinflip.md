## 1. Overview

BURNIE coinflip is the central wagering mechanic of the BurnieCoin ecosystem. Players deposit BURNIE into daily windows, a VRF-based 50/50 outcome determines win or loss, and winners receive their principal back plus a variable bonus multiplier (1.5x to 2.5x total payout).

Key characteristics:
- **One flip per day:** All deposits for a given day share the same outcome and multiplier
- **Burn-and-mint model:** Deposits burn BURNIE supply; claims mint new BURNIE supply
- **Variable multiplier:** Three-tier system with 90% of outcomes in the [1.78x, 2.15x] range
- **Bounty system:** Accumulating BURNIE pool rewarding the largest single depositor
- **Recycling bonuses:** Percentage bonus on re-wagered winnings (compound via auto-rebuy)
- **Coinflip boons:** Single-use percentage boosts from deity lootbox rewards

**Primary source:** `contracts/BurnieCoinflip.sol`

## 2. Core Mechanics

### 2a. Deposit

**Entry point:** `depositCoinflip(address player, uint256 amount)`
**Source:** `BurnieCoinflip.sol` lines 231--244

Players deposit BURNIE into the next day's coinflip window. The deposit is burned immediately via `burnForCoinflip`:

```solidity
// BurnieCoinflip.sol:255
if (amount < MIN) revert AmountLTMin();
// MIN = 100 ether                                  (BurnieCoinflip.sol:119)
```

```solidity
// BurnieCoinflip.sol:271
burnie.burnForCoinflip(caller, amount);
```

**Burn implementation** in `BurnieCoin.sol`:

```solidity
// BurnieCoin.sol:517-519
function burnForCoinflip(address from, uint256 amount) external {
    if (msg.sender != coinflipContract) revert OnlyGame();
    _burn(from, amount);
}
```

**Minimum deposit:** 100 BURNIE (100e18 wei).

### 2b. Window Resolution

**Entry point:** `processCoinflipPayouts(bool bonusFlip, uint256 rngWord, uint48 epoch)`
**Source:** `BurnieCoinflip.sol` lines 802--889

One flip resolves per day. The game contract calls `processCoinflipPayouts` with a VRF random word:

```solidity
// BurnieCoinflip.sol:841
bool win = (rngWord & 1) == 1;
```

The win/loss outcome is a pure 50/50 coin flip: the least significant bit of the VRF random word determines the result.

### 2c. Claim (Win)

**Entry point:** `_claimCoinflipsInternal(address player, bool deepAutoRebuy)`
**Source:** `BurnieCoinflip.sol` lines 430--627

On a win, the payout is calculated as:

```solidity
// BurnieCoinflip.sol:532-534
uint256 payout = stake + (stake * uint256(rewardPercent)) / 100;
```

Where `rewardPercent` is the day's bonus percentage (see Section 3). Winnings are minted as new BURNIE via `mintForCoinflip`:

```solidity
// BurnieCoin.sol:526-529
function mintForCoinflip(address to, uint256 amount) external {
    if (msg.sender != coinflipContract) revert OnlyGame();
    _mint(to, amount);
}
```

**Supply impact:** Wins create new BURNIE supply. The protocol is inflationary on winning days and deflationary on losing days.

### 2d. Loss

On a loss, the player's staked principal is permanently destroyed (it was burned on deposit and no mint occurs on loss). As consolation, the player receives 1 WWXRP per losing day:

```solidity
// BurnieCoinflip.sol:623
wwxrp.mintPrize(player, lossCount * COINFLIP_LOSS_WWXRP_REWARD);
// COINFLIP_LOSS_WWXRP_REWARD = 1 ether             (BurnieCoinflip.sol:120)
```

### 2e. Deposit Lock During BAF Resolution

Deposits are blocked during BAF (Big-Ass Flip) resolution at every 10th level transition to prevent front-running the BAF leaderboard:

```solidity
// BurnieCoinflip.sol:1026-1039
function _coinflipLockedDuringTransition() private view returns (bool locked) {
    (uint24 purchaseLevel_, bool inJackpotPhase, bool lastPurchaseDay_, bool rngLocked_,)
        = degenerusGame.purchaseInfo();
    locked = (!inJackpotPhase) && !degenerusGame.gameOver()
        && lastPurchaseDay_ && rngLocked_ && (purchaseLevel_ % 10 == 0);
}
```

## 3. Payout Multiplier Distribution

**Entry point:** `processCoinflipPayouts` (payout tier selection)
**Source:** `BurnieCoinflip.sol` lines 808--831

Each day's payout multiplier is determined by a separate seed derived from VRF:

```solidity
// BurnieCoinflip.sol:808
uint256 seedWord = uint256(keccak256(abi.encodePacked(rngWord, epoch)));
```

The multiplier tier is selected by `roll = seedWord % 20`:

```solidity
// BurnieCoinflip.sol:813-824
uint256 roll = seedWord % 20;
uint16 rewardPercent;
if (roll == 0) {
    rewardPercent = 50;     // 1.50x total
} else if (roll == 1) {
    rewardPercent = 150;    // 2.50x total
} else {
    rewardPercent = uint16(
        (seedWord % COINFLIP_EXTRA_RANGE) + COINFLIP_EXTRA_MIN_PERCENT
    );
    // COINFLIP_EXTRA_MIN_PERCENT = 78               (BurnieCoinflip.sol:121)
    // COINFLIP_EXTRA_RANGE = 38                     (BurnieCoinflip.sol:122)
    // Result: rewardPercent in [78, 115]
}
```

### 3a. Multiplier Tier Table

| Roll Value | Probability | rewardPercent | Total Payout | Description |
|-----------|-------------|---------------|--------------|-------------|
| 0 | 1/20 (5%) | 50% | 1.50x | Low bonus |
| 1 | 1/20 (5%) | 150% | 2.50x | High bonus |
| 2--19 | 18/20 (90%) | [78%, 115%] | [1.78x, 2.15x] | Normal range |

### 3b. Presale Bonus

During presale, an additional +6 percentage points is added to `rewardPercent`:

```solidity
// BurnieCoinflip.sol:826-831
bool presaleBonus = bonusFlip && game.lootboxPresaleActiveFlag();
if (presaleBonus) {
    unchecked {
        rewardPercent += 6;
    }
}
```

With presale bonus, the tiers become: 56% (1.56x), 156% (2.56x), [84%, 121%] ([1.84x, 2.21x]).

### 3c. Expected Value Calculation (Base, No Presale)

The normal-range rewardPercent follows a uniform distribution over 38 discrete values [78, 79, ..., 115]:

```
Average normal rewardPercent = (78 + 115) / 2 = 96.5%
Average normal payout = 1.965x
```

**Weighted average payout per flip (conditional on win):**

```
E[payout | win] = 0.05 * 1.50 + 0.05 * 2.50 + 0.90 * 1.965
                = 0.075 + 0.125 + 1.7685
                = 1.9685x
```

**EV per flip (accounting for 50% win rate):**

```
EV = 0.50 * 1.9685 + 0.50 * 0.00
   = 0.98425x per flip
```

This matches the contract constant `COINFLIP_REWARD_MEAN_BPS = 9685` (BurnieCoinflip.sol:128), representing 96.85% expected return per 1x stake, since `(rewardPercent_mean * 100) / 2 + 5000 = 96.5 * 50 + 5000 = 9685 bps` (half the mean bonus times probability of winning, plus the 50% chance of getting principal back).

**The house edge is approximately 1.575% per flip without recycling bonuses.**

## 4. Last Purchase Day EV Adjustment

**Entry point:** `_coinflipTargetEvBps(uint256 prevTotal, uint256 currentTotal)`
**Source:** `BurnieCoinflip.sol` lines 1086--1109

On non-presale bonus flip days (last purchase day of a level), the payout multiplier is adjusted based on the ratio of current day's total flip volume to the previous day's total:

```solidity
// BurnieCoinflip.sol:833-838
if (bonusFlip && !presaleBonus) {
    (uint256 prevTotal, uint256 currentTotal) = game.lastPurchaseDayFlipTotals();
    int256 evBps = _coinflipTargetEvBps(prevTotal, currentTotal);
    rewardPercent = _applyEvToRewardPercent(rewardPercent, evBps);
}
```

### 4a. Ratio to EV Mapping

```solidity
// BurnieCoinflip.sol:1086-1109
function _coinflipTargetEvBps(uint256 prevTotal, uint256 currentTotal) private pure returns (int256 evBps) {
    if (prevTotal == 0) return COINFLIP_EV_EQUAL_BPS;   // 0 bps
    uint256 ratioBps = (currentTotal * COINFLIP_RATIO_BPS_SCALE) / prevTotal;
    if (ratioBps <= COINFLIP_RATIO_BPS_EQUAL) return COINFLIP_EV_EQUAL_BPS;  // 0 bps
    if (ratioBps >= COINFLIP_RATIO_BPS_TRIPLE) return COINFLIP_EV_TRIPLE_BPS; // 300 bps
    return _lerpEvBps(COINFLIP_RATIO_BPS_EQUAL, COINFLIP_RATIO_BPS_TRIPLE,
                      COINFLIP_EV_EQUAL_BPS, COINFLIP_EV_TRIPLE_BPS, ratioBps);
}
```

| Condition | Ratio (current/prev) | evBps |
|-----------|----------------------|-------|
| ratio <= 1.0x | ratioBps <= 10,000 | 0 bps |
| 1.0x < ratio < 3.0x | 10,000 < ratioBps < 30,000 | Linear interpolation [0, 300] bps |
| ratio >= 3.0x | ratioBps >= 30,000 | 300 bps |

### 4b. Applying EV Adjustment to rewardPercent

```solidity
// BurnieCoinflip.sol:1128-1142
function _applyEvToRewardPercent(uint16 rewardPercent, int256 evBps) private pure returns (uint16) {
    int256 targetRewardBps = int256(uint256(BPS_DENOMINATOR)) + (evBps * 2);
    // BPS_DENOMINATOR = 10,000                       (BurnieCoinflip.sol:129)
    int256 deltaBps = targetRewardBps - int256(uint256(COINFLIP_REWARD_MEAN_BPS));
    // COINFLIP_REWARD_MEAN_BPS = 9685                (BurnieCoinflip.sol:128)
    int256 adjustedBps = int256(uint256(rewardPercent) * 100) + deltaBps;
    if (adjustedBps <= 0) return 0;
    uint256 rounded = (uint256(adjustedBps) + 50) / 100;
    adjustedPercent = uint16(rounded);
}
```

**Worked example:** If `evBps = 300` (3x ratio, maximum boost):

```
targetRewardBps = 10,000 + (300 * 2) = 10,600
deltaBps = 10,600 - 9,685 = 915
```

For a normal-range flip with `rewardPercent = 96` (mid-range):

```
adjustedBps = 96 * 100 + 915 = 10,515
rounded = (10,515 + 50) / 100 = 105
```

So `rewardPercent` shifts from 96% to 105%, adding ~9 percentage points on a maximum-boost day.

**At maximum boost (evBps=300), the overall EV shifts from ~0.984x to ~1.03x, making it slightly player-favorable.**

## 5. Bounty System

**Entry point:** `_addDailyFlip` (arming) and `processCoinflipPayouts` (resolution)
**Source:** `BurnieCoinflip.sol` lines 634--693 (arming), 849--879 (resolution)

The bounty is an accumulating BURNIE pool that rewards the player who sets the all-time largest single deposit.

### 5a. Accumulation

Each time a coinflip window resolves, the bounty pool grows by `PRICE_COIN_UNIT` (1,000 BURNIE):

```solidity
// BurnieCoinflip.sol:876-879
unchecked {
    currentBounty = currentBounty_ + uint128(PRICE_COIN_UNIT);
}
// PRICE_COIN_UNIT = 1000 ether                      (BurnieCoinflip.sol:135)
```

**Starting value:** `currentBounty` is initialized at 1,000 BURNIE (BurnieCoinflip.sol:167).

### 5b. Arming

A player "arms" the bounty by setting a new all-time `biggestFlipEver` record via a direct deposit:

```solidity
// BurnieCoinflip.sol:669-692
if (canArmBounty && bountyEligible && recordAmount != 0) {
    uint128 record = biggestFlipEver;
    if (recordAmount > record && !game.rngLocked()) {
        // ...
        biggestFlipEver = uint128(recordAmount);

        // If bounty already armed, must exceed current record by 1% (min +1 wei)
        uint256 threshold = record;
        if (currentBountyOwner != address(0)) {
            uint256 onePercent = uint256(record) / 100;
            threshold = uint256(record) + (onePercent == 0 ? 1 : onePercent);
        }
        if (recordAmount >= threshold) {
            bountyOwedTo = player;
        }
    }
}
```

**Arming restrictions:**
- `canArmBounty` = true only for direct deposits (not credited flips or bounty payouts) (BurnieCoinflip.sol:317)
- `recordAmount` is the raw deposit amount before bonuses (BurnieCoinflip.sol:317: `directDeposit ? amount : 0`)
- Cannot arm during RNG lock (`!game.rngLocked()`)
- If bounty is already armed by another player, must exceed the current record by at least 1% (minimum +1 wei if 1% rounds to zero)

### 5c. Resolution

When `processCoinflipPayouts` fires and a bounty is armed:

```solidity
// BurnieCoinflip.sol:855-870
if (bountyOwner != address(0) && currentBounty_ > 0) {
    slice = currentBounty_ >> 1;          // half of bounty pool removed
    unchecked {
        currentBounty_ -= uint128(slice);
    }
    if (win) {
        to = bountyOwner;
        _addDailyFlip(to, slice, 0, false, false);  // credited as flip stake
        game.payCoinflipBountyDgnrs(to);             // also awards DGNRS token
        bountyPaid = uint128(slice);
    }
    // Clear bounty owner regardless of win/loss
    bountyOwedTo = address(0);
}
```

**Resolution outcomes:**

| Outcome | Bounty Pool Effect | Bounty Owner Effect |
|---------|-------------------|-------------------|
| **Win** | Half removed | Half credited as flip stake + DGNRS reward |
| **Loss** | Half removed (destroyed) | Nothing -- bounty cleared |

After resolution, `bountyOwedTo` is cleared to `address(0)` regardless of outcome. The remaining half stays in the pool and continues accumulating.

**Important:** The bounty payout on win is credited as a flip stake for the next day (`_addDailyFlip` with `canArmBounty=false`), NOT minted directly. The bounty winner must win the next coinflip to realize the value.

### 5d. Bounty Lifecycle Diagram

```
  +-- resolve (no bounty armed) --> bounty += 1000 BURNIE --> pool grows
  |
  +-- arm: player sets new biggestFlipEver record
  |     bountyOwedTo = player
  |
  +-- resolve (bounty armed):
        half = currentBounty >> 1
        currentBounty -= half
        WIN:  credit half as flip stake to bountyOwedTo + DGNRS
        LOSS: half destroyed
        bountyOwedTo = address(0)
        currentBounty += 1000 BURNIE (accumulation continues)
```

## 6. Recycling Bonuses

**Source:** `BurnieCoinflip.sol` lines 1042--1066

Recycling bonuses reward players for re-wagering winnings instead of claiming. The bonus is added to the credited flip stake on deposit.

### 6a. Normal Recycling (Non-AfKing)

```solidity
// BurnieCoinflip.sol:1042-1049
function _recyclingBonus(uint256 amount) private pure returns (uint256 bonus) {
    if (amount == 0) return 0;
    bonus = amount / 100;                 // 1% of rebet amount
    uint256 bonusCap = 1000 ether;
    if (bonus > bonusCap) bonus = bonusCap;
}
```

**Formula:** `bonus = min(amount * 1%, 1000 BURNIE)`

The cap of 1,000 BURNIE is reached at a rebet amount of 100,000 BURNIE.

### 6b. AfKing Recycling (Enhanced)

```solidity
// BurnieCoinflip.sol:1053-1066
function _afKingRecyclingBonus(uint256 amount, uint16 deityBonusHalfBps) private pure returns (uint256 bonus) {
    if (amount == 0) return 0;
    uint256 baseHalfBps = uint256(AFKING_RECYCLE_BONUS_BPS) * 2;
    // AFKING_RECYCLE_BONUS_BPS = 160                 (BurnieCoinflip.sol:130)
    // baseHalfBps = 320

    if (deityBonusHalfBps == 0 || amount <= DEITY_RECYCLE_CAP) {
        uint256 totalHalfBps = baseHalfBps + uint256(deityBonusHalfBps);
        return (amount * totalHalfBps) / (uint256(BPS_DENOMINATOR) * 2);
        // denominator = 10,000 * 2 = 20,000
    }
    // Deity cap split: full rate on first 1M, base-only on remainder
    uint256 fullHalfBps = baseHalfBps + uint256(deityBonusHalfBps);
    return (DEITY_RECYCLE_CAP * fullHalfBps + (amount - DEITY_RECYCLE_CAP) * baseHalfBps)
        / (uint256(BPS_DENOMINATOR) * 2);
    // DEITY_RECYCLE_CAP = 1,000,000 ether            (BurnieCoinflip.sol:133)
}
```

**Base afKing rate:** `baseHalfBps = 320`, effective rate = `320 / 20,000 = 1.60%`

**Note on "half bps":** The contract uses "half bps" internally (each unit = 0.005%), so all half-bps values are divided by 20,000 to get the actual percentage.

### 6c. Deity Bonus

Players with deity passes earn an additional recycling bonus that scales with levels:

```solidity
// BurnieCoinflip.sol:1069-1083
function _afKingDeityBonusHalfBpsWithLevel(address player, uint24 currentLevel) private view returns (uint16) {
    uint24 activationLevel = degenerusGame.afKingActivatedLevelFor(player);
    if (activationLevel == 0) return 0;
    if (currentLevel <= activationLevel) return 0;

    uint24 levelsActive = currentLevel - activationLevel;
    uint24 bonus = levelsActive * uint24(AFKING_DEITY_BONUS_PER_LEVEL_HALF_BPS);
    // AFKING_DEITY_BONUS_PER_LEVEL_HALF_BPS = 2      (BurnieCoinflip.sol:131)
    if (bonus > AFKING_DEITY_BONUS_MAX_HALF_BPS) {
        return AFKING_DEITY_BONUS_MAX_HALF_BPS;
    }
    // AFKING_DEITY_BONUS_MAX_HALF_BPS = 300           (BurnieCoinflip.sol:132)
    return uint16(bonus);
}
```

**Deity bonus:** `+2 halfBps per level since activation`, capped at `300 halfBps`.

- At maximum (300 halfBps): `300 / 20,000 = 1.50%` additional bonus
- Total afKing rate at max deity: `(320 + 300) / 20,000 = 3.10%`
- Maximum levels to cap: `300 / 2 = 150 levels`

**Deity cap:** When `amount > DEITY_RECYCLE_CAP` (1,000,000 BURNIE), the deity bonus applies only to the first 1M BURNIE. The remainder receives base-only (1.60%).

### 6d. Worked Example: 50,000 BURNIE Rebet with 10 Levels of Deity

Given: `amount = 50,000 BURNIE`, `levelsActive = 10`, deity pass active

1. **Deity bonus halfBps:** `10 * 2 = 20 halfBps`
2. **Base halfBps:** `160 * 2 = 320 halfBps`
3. **Total halfBps:** `320 + 20 = 340 halfBps`
4. **Amount <= DEITY_RECYCLE_CAP (1M):** Yes, use simple formula
5. **Bonus:** `(50,000 * 340) / 20,000 = 850 BURNIE`
6. **Effective rate:** `850 / 50,000 = 1.70%`

For comparison:
- Normal recycling on 50,000: `min(50,000 / 100, 1,000) = 500 BURNIE` (1.00%)
- AfKing no deity on 50,000: `(50,000 * 320) / 20,000 = 800 BURNIE` (1.60%)
- AfKing max deity on 50,000: `(50,000 * 620) / 20,000 = 1,550 BURNIE` (3.10%)

### 6e. When Recycling Applies

Recycling bonuses apply on the **rebet portion** of a deposit -- the part funded by prior winnings rather than fresh BURNIE:

```solidity
// BurnieCoinflip.sol:293-312 (inside _depositCoinflip)
uint256 rollAmount = state.autoRebuyEnabled ? state.autoRebuyCarry : mintable;
uint256 rebetAmount = creditedFlip <= rollAmount ? creditedFlip : rollAmount;
if (rebetAmount != 0) {
    uint256 bonus;
    bool isAfKing = game.afKingModeFor(caller);
    if (isAfKing) {
        // ... afKing recycling
        bonus = _afKingRecyclingBonus(rebetAmount, deityBonusHalfBps);
    } else {
        bonus = _recyclingBonus(rebetAmount);
    }
    creditedFlip += bonus;
}
```

During auto-rebuy claim processing, recycling also applies to the carry amount:

```solidity
// BurnieCoinflip.sol:553-561 (inside _claimCoinflipsInternal, win with auto-rebuy)
if (carry != 0) {
    if (afKingActive) {
        carry += _afKingRecyclingBonus(carry, deityBonusHalfBps);
    } else {
        carry += _recyclingBonus(carry);
    }
}
```

## 7. Claim Windows and Expiry

**Source:** `BurnieCoinflip.sol` lines 430--627

### 7a. Claim Window Durations

```solidity
// BurnieCoinflip.sol:470
uint8 windowDays = start == 0 ? COIN_CLAIM_FIRST_DAYS : COIN_CLAIM_DAYS;
// COIN_CLAIM_FIRST_DAYS = 30                        (BurnieCoinflip.sol:137)
// COIN_CLAIM_DAYS = 90                              (BurnieCoinflip.sol:136)
```

| Condition | Window | Duration |
|-----------|--------|----------|
| First-time claim (`lastClaim == 0`) | 30 days | From `flipsClaimableDay - 30` to `flipsClaimableDay` |
| Subsequent claims | 90 days | From `flipsClaimableDay - 90` to `flipsClaimableDay` |

The claim window is measured backwards from `flipsClaimableDay` (the most recently resolved day). Days outside the window are permanently lost.

```solidity
// BurnieCoinflip.sol:477-481
unchecked {
    minClaimableDay = latest > windowDays ? latest - windowDays : 0;
}
```

If `lastClaim < minClaimableDay`, the pointer is advanced to `minClaimableDay`, and all days before that are forfeited.

### 7b. Auto-Rebuy Extension

When auto-rebuy is enabled, the claim window extends back to `autoRebuyStartDay` instead of the standard window:

```solidity
// BurnieCoinflip.sol:472-476
if (rebuyActive) {
    minClaimableDay = state.autoRebuyStartDay;
    if (minClaimableDay > latest) {
        minClaimableDay = latest;
    }
}
```

**Auto-rebuy prevents expiry** -- as long as auto-rebuy remains enabled, no days are forfeited. The system processes winnings and carries them forward automatically.

### 7c. Deep Auto-Rebuy Off Processing

When auto-rebuy is disabled after a long period, the system processes up to `AUTO_REBUY_OFF_CLAIM_DAYS_MAX` days:

```solidity
// BurnieCoinflip.sol:496-501
if (deep) {
    uint48 available = latest - start;
    uint48 cap = available > AUTO_REBUY_OFF_CLAIM_DAYS_MAX
        ? AUTO_REBUY_OFF_CLAIM_DAYS_MAX : available;
    remaining = uint32(cap);
}
// AUTO_REBUY_OFF_CLAIM_DAYS_MAX = 1095              (BurnieCoinflip.sol:138)
```

**Maximum processing:** 1,095 days (approximately 3 years). If a player had auto-rebuy enabled for longer, any days beyond the 1,095-day lookback are lost.

### 7d. Expiry as Supply Sink

Unclaimed coinflip winnings beyond the claim window are permanently lost. The BURNIE was burned on deposit, and if the player fails to claim within the window, no mint ever occurs. This acts as a supply sink:

- **First-time players:** Must claim within 30 days of their first resolved flip
- **Returning players:** 90-day rolling window
- **Inactive players:** If auto-rebuy is off and they miss the window, principal from winning days is permanently destroyed

## 8. Coinflip Boons

**Source:** `BurnieCoinflip.sol` lines 643--653 (consumption), `DegenerusGameLootboxModule.sol` lines 237--239 (BPS values), `DegenerusGameBoonModule.sol` line 23 (expiry)

Coinflip boons are single-use percentage boosts awarded from deity lootbox rolls. They add bonus BURNIE to the credited flip stake on manual deposit.

### 8a. Boon Tiers

| Tier | BPS Value | Max Deposit Cap | Max Bonus | Source Constant |
|------|----------|-----------------|-----------|-----------------|
| COINFLIP_5 | 500 bps (5%) | 100,000 BURNIE | 5,000 BURNIE | `LOOTBOX_BOON_BONUS_BPS = 500` (LootboxModule:218) |
| COINFLIP_10 | 1,000 bps (10%) | 100,000 BURNIE | 10,000 BURNIE | `LOOTBOX_COINFLIP_10_BONUS_BPS = 1000` (LootboxModule:237) |
| COINFLIP_25 | 2,500 bps (25%) | 100,000 BURNIE | 25,000 BURNIE | `LOOTBOX_COINFLIP_25_BONUS_BPS = 2500` (LootboxModule:239) |

### 8b. Application Logic

```solidity
// BurnieCoinflip.sol:643-653
if (recordAmount != 0) {  // manual deposits only
    uint16 boonBps = game.consumeCoinflipBoon(player);
    if (boonBps > 0) {
        uint256 maxDeposit = 100_000 ether;
        uint256 cappedDeposit = coinflipDeposit > maxDeposit ? maxDeposit : coinflipDeposit;
        uint256 boost = (cappedDeposit * boonBps) / 10_000;
        coinflipDeposit += boost;
    }
}
```

**Rules:**
- **Single-use:** Consumed on the next manual deposit after being awarded (BoonModule:54-58)
- **Manual deposits only:** `recordAmount != 0` is true only for direct deposits, not for credited flips or bounty payouts
- **Deposit cap:** Boost is calculated on `min(coinflipDeposit, 100,000 BURNIE)` regardless of actual deposit size
- **Expiry:** Boons expire after 2 days if unused (`COINFLIP_BOON_EXPIRY_DAYS = 2`, BoonModule:23)
- **Stacking:** Only one boon can be held at a time; a higher-tier boon overwrites a lower-tier one (LootboxModule:1365)

### 9b. Auto-Rebuy Compound Modeling

With auto-rebuy enabled, wins compound:

```
Day N win:   payout = stake * (1 + rewardPercent/100)
             carry = payout + recyclingBonus(payout)
Day N+1:     stake = carry (from previous win)
Day N+1 win: payout = carry * (1 + rewardPercent/100)
             carry = payout + recyclingBonus(payout)
Day N+1 loss: carry = 0 (all accumulated value lost)
```

With take-profit enabled (`autoRebuyStop`):

```
Day N win:   payout = stake * (1 + rewardPercent/100)
             reserved = floor(payout / takeProfit) * takeProfit
             carry = payout - reserved + recyclingBonus(payout - reserved)
             mintable += reserved
```

### 9c. Bounty as Probabilistic Income

For an agent modeling bounty income:

```
Bounty accumulation: +1,000 BURNIE per day
Expected armed bounty value (if arming every K days):
  E[bounty payout] = (bountyPool / 2) * P(win) * P(arming)
  = (bountyPool / 2) * 0.5 * P(arming)
```

The bounty payout is credited as flip stake (not direct mint), so its realized value depends on winning the following day's flip.

### 9d. Supply Sink Modeling

Track net supply impact per day:

```
Net mint = (sum of all winning stakes * (1 + rewardPercent/100)) - (sum of all stakes)
         = winning_stakes * rewardPercent/100 - losing_stakes
```

Additional sinks:
- Claim expiry (Section 7)
- Bounty loss destruction (Section 5c: half of pool destroyed on loss resolution)

## Common Pitfalls

> **Pitfall 1: Wins Mint New Supply; Losses Burn Permanently.**
> Deposits always burn BURNIE. Wins mint new BURNIE (potentially more than was burned). Losses result in net supply destruction. The system is not a pool -- there is no "house balance." Supply elasticity is the mechanism.

> **Pitfall 2: Bounty Payout Is Flip Stake, Not Direct Mint.**
> The bounty winner receives their half as a flip stake for the next day (`_addDailyFlip` with `canArmBounty=false`, BurnieCoinflip.sol:863). They must win the next flip to realize it as claimable BURNIE. The bounty payout also cannot arm a new bounty.

> **Pitfall 3: Recycling Bonus Applies Only to Rebet Portion.**
> When a player deposits fresh BURNIE on top of prior winnings, the recycling bonus applies only to the `rebetAmount` (the portion funded by prior claim/carry), not the full deposit (BurnieCoinflip.sol:296-298).

> **Pitfall 4: AfKing Deity Cap Splits at 1M BURNIE.**
> For afKing players with deity passes and rebets exceeding 1,000,000 BURNIE, the deity bonus rate applies only to the first 1M. The remainder receives the base 1.60% rate (BurnieCoinflip.sol:1063-1065).

> **Pitfall 5: Boons Are Manual-Deposit Only and Expire.**
> Coinflip boons are consumed only on manual deposits (`recordAmount != 0`), not on auto-rebuy carries or credited flips. Unused boons expire after 2 days (BoonModule:23, 48).

> **Pitfall 6: First-Time Claim Window Is 30 Days, Not 90.**
> A player's first claim window is 30 days (`COIN_CLAIM_FIRST_DAYS`). If they deposit and do not interact for 31 days, their winnings are permanently lost. Subsequent claims get the full 90-day window.
