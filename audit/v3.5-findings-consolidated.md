## Cross-Cutting Observations

1. **v3.3/v3.4 rewrites introduced 3 regressions.** The gambling burn rewrite (v3.3) and skim/lootbox changes (v3.4) overwrote 3 prior NatSpec fixes. This suggests a need for regression testing on NatSpec after code rewrites.

2. **Comment correctness scales with codebase size.** 26 findings across ~26,300 lines (1 per ~1,010 lines) is a reasonable density for a NatSpec audit of this thoroughness.

3. **Gas optimization is structurally bounded.** The protocol's existing Slot 0/1 packing is aggressive (32/32 and 27/32 bytes). Remaining opportunities require mapping-level packing (boon pairs) or full storage layout reorders across mapping boundaries -- high-effort changes.

4. **Gas ceiling concern is advanceGame-only.** All 6 purchase paths have >13M headroom. The 2 AT_RISK advanceGame stages (8 and 11) are tight only when ALL winners use auto-rebuy, which is economically implausible at scale.

5. **Dead code is minimal.** 5 items (1 error, 4 events) out of ~450 checked. The `ApprovalForAll` event in DegenerusDeityPass is retained for ERC-721 compliance despite the soulbound design.

6. **Prior finding cleanup is thorough.** 34 of 38 checked prior findings are confirmed FIXED. The 3 regressions are all documented with new v3.5 IDs.
