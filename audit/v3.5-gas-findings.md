## Storage Packing Opportunities (GAS-03)

Storage packing analysis based on `forge inspect <Contract> storageLayout` output cross-referenced with co-access patterns from liveness analysis. All packing opportunities require contract redeployment due to DegenerusGameStorage's slot stability policy.

**Important context:** DegenerusGameStorage already employs aggressive packing in its most critical slots. Slot 0 packs 15 variables into 32 bytes (zero padding). Slot 1 packs 7 variables into 27 bytes (5 bytes padding). The remaining slots are predominantly mappings (each occupying a full slot root by EVM design) with a few scalar outliers.

### Scalar Variable Slots with Wasted Bytes

| Slot | Variables | Bytes Used | Bytes Wasted | Notes |
|------|-----------|-----------|--------------|-------|
| 0 | 15 scalars (levelStartTime..dailyEthBucketCursor) | 32/32 | 0 | Fully packed |
| 1 | 7 scalars (dailyEthPhase..prizePoolFrozen) | 27/32 | 5 | After prizePoolFrozen |
| 17 | 3 scalars (ticketCursor + ticketLevel + dailyEthWinnerCursor) | 9/32 | 23 | |
| 19 | 1 scalar (dailyCarryoverWinnerCap uint16) | 2/32 | 30 | |
| 21 | 1 scalar (lootboxPresaleActive bool) | 1/32 | 31 | |
| 24 | 3 scalars (gameOverTime + gameOverFinalJackpotPaid + finalSwept) | 8/32 | 24 | |
| 43 | 3 scalars (lastDailyJackpotWinningTraits + lastDailyJackpotLevel + lastDailyJackpotDay) | 13/32 | 19 | |
| 58 | 1 scalar (vrfCoordinator address = 20 bytes) | 20/32 | 12 | |
| 61 | 1 scalar (lootboxRngIndex uint48) | 6/32 | 26 | |
| 74 | 1 scalar (midDayTicketRngPending bool) | 1/32 | 31 | |
| 104 | 1 scalar (centuryBonusLevel uint24) | 3/32 | 29 | |

However, most of these slots are **isolated between mappings** (which each consume a full slot root). Re-ordering variables to merge these slots would require breaking the storage layout ordering constraint. The analysis below identifies which combinations have co-access patterns that would actually benefit from packing.

### PACK-01: Slot 17 + Slot 19 (Jackpot Cursor State)

| Field | Value |
|-------|-------|
| **Contract** | DegenerusGameStorage.sol |
| **Current slots** | Slot 17 (9/32 bytes: ticketCursor uint32, ticketLevel uint24, dailyEthWinnerCursor uint16) + Slot 19 (2/32 bytes: dailyCarryoverWinnerCap uint16) |
| **Proposed** | Pack dailyCarryoverWinnerCap into Slot 17 at offset 9 (11/32 bytes total) |
| **Savings** | 2,100 gas per daily jackpot co-access (1 cold SLOAD saved) |
| **Co-access** | Yes -- DegenerusGameJackpotModule reads dailyEthWinnerCursor (Slot 17) and dailyCarryoverWinnerCap (Slot 19) in the same payDailyJackpot flow (lines 336, 471, 531-562) |
| **Requires redeployment** | Yes |
| **Severity** | GAS-INFO |

**Blocker:** Slot 18 (dailyCarryoverEthPool uint256) sits between Slots 17 and 19. Moving dailyCarryoverWinnerCap before dailyCarryoverEthPool would violate declaration order. Feasible only on full redeploy with new layout.

### PACK-05: Boon Mapping Packing Pattern (Active + Day pairs)

| Field | Value |
|-------|-------|
| **Contract** | DegenerusGameStorage.sol |
| **Current slots** | Each boon has 2 separate mappings: `{boon}Active` (mapping(address => bool)) + `{boon}Day` (mapping(address => uint48)). Each lookup requires 2 cold SLOADs. |
| **Proposed** | Merge into 1 mapping(address => uint256) with bool + uint48 packed. Example: `lootboxBoon5Active` (Slot 26) + `lootboxBoon5Day` (Slot 27) become a single mapping with `bool active` at bits[0] + `uint48 day` at bits[8:56]. |
| **Savings** | 2,100 gas per boon check (1 cold SLOAD saved per co-access) |
| **Co-access** | Yes -- every boon check reads both Active and Day in the same function call. For example, DegenerusGameBoonModule reads `lootboxBoon5Active` then `lootboxBoon5Day` for expiry check (lines 180-188). DegenerusGameMintModule reads both (lines 1125-1127). DegenerusGameWhaleModule reads both (lines 799-801). |
| **Requires redeployment** | Yes |
| **Severity** | GAS-INFO |

**Concrete example -- lootboxBoon5:**
- Current: `lootboxBoon5Active[player]` (Slot 26 SLOAD) + `lootboxBoon5Day[player]` (Slot 27 SLOAD) = 2 cold SLOADs = 4,200 gas
- Proposed: `lootboxBoon5Packed[player]` (1 SLOAD) = 2,100 gas
- Saving: 2,100 gas per boon check
