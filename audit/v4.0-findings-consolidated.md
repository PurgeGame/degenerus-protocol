### Withdrawn (2)

#### ~~DEC-01~~: decBucketOffsetPacked Collision Between Regular and Terminal Decimator [WITHDRAWN -- originally MEDIUM]

**Source:** Phase 87 Plan 03 (Decimator audit)
**Document:** `audit/v4.0-other-jackpots-decimator.md` Section 3
**Location:** DM:248 (regular write), DM:817 (terminal write), DM:281 (regular claim read)

**Original claim:** Terminal decimator overwrites regular decimator's packed subbucket offsets when both resolve at the same level, corrupting regular claim validation.

**Withdrawal reason:** FALSE POSITIVE -- the regular decimator never resolves at a stalled level. GAMEOVER prevents level completion, so the regular decimator never writes `decBucketOffsetPacked` at the GAMEOVER level. The terminal decimator writes to a level where regular decimator never wrote, so there is no overwrite. Regular claims for that level revert at the `decClaimRounds[lvl].poolWei == 0` guard (DM:275) before any packed offset read.

---

#### ~~DGN-01~~: _collectBetFunds Uses `<=` Instead of `<` for Claimable Balance Check [WITHDRAWN -- originally LOW]

**Source:** Phase 87 Plan 04 (Degenerette audit)
**Document:** `audit/v4.0-other-jackpots-degenerette.md` Section 10
**Location:** DDM:552

**Original claim:** `_collectBetFunds` uses `<=` instead of `<` for claimable balance check, reverting when player has exactly enough claimable ETH.

**Withdrawal reason:** FALSE POSITIVE -- the `<=` is intentional. `claimableWinnings` uses a 1-wei sentinel to keep the storage slot warm (DG:1367: `claimableWinnings[player] = 1; // Leave sentinel`). The `<=` check correctly ensures the sentinel is preserved. Using `<` would allow the slot to be zeroed, breaking the gas optimization.
