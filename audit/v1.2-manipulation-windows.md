## Section 1a: Daily Consumption Points (D1-D9)

### Shared Temporal Window

All 9 daily consumption points share a single temporal window:

1. **Block N:** `advanceGame()` calls `_requestRng()`, sets `rngLockedFlag=true`, fires VRF request to Chainlink.
2. **Block N+K:** `rawFulfillRandomWords()` stores raw VRF word in `rngWordCurrent` (AdvanceModule:1338). `rngLockedFlag` remains true.
3. **Block N+K+M:** `advanceGame()` calls `rngGate()`, which reads `rngWordCurrent`, calls `_applyDailyRng()` (applies nudge offset from `totalFlipReversals`), then executes all 9 daily consumers atomically within a single transaction.

**Critical property:** Between Block N (rngLockedFlag set true) and Block N+K+M (advanceGame consumes word), 12 entry points are blocked by rngLockedFlag revert guards:
- `requestLootboxRng` (AdvanceModule:674)
- `reverseFlip` (AdvanceModule:1299)
- `purchaseDeityPass` (WhaleModule:468)
- `setAutoRebuy` (DegenerusGame:1526)
- `setDecimatorAutoRebuy` (DegenerusGame:1505)
- `setAutoRebuyTakeProfit` (DegenerusGame:1541)
- `setAfKingMode` (DegenerusGame:1606)
- `claimCoinflips` (BurnieCoinflip:336)
- `claimCoinflipsTakeProfit` (BurnieCoinflip:347)
- `claimCoinflipsFromBurnie` (BurnieCoinflip:357)
- `consumeCoinflipsForBurn` (BurnieCoinflip:367)
- BAF credit gate (BurnieCoinflip:603, at x10 levels)

**Entry points still callable during lock (13):**
- `advanceGame`, `rawFulfillRandomWords` -- system functions
- `purchase`, `purchaseCoin`, `purchaseBurnieLootbox`, `purchaseWhaleBundle`, `purchaseLazyPass` -- ticket purchases (write to write-buffer, not read-buffer)
- `openLootBox`, `openBurnieLootBox` -- independent lootbox RNG
- `placeFullTicketBets` -- assigned to current lootbox index
- `resolveDegeneretteBets` -- uses lootbox RNG word
- `issueDeityBoon` -- read-only RNG access
- `setLootboxRngThreshold` -- admin config only

**Atomicity guarantee:** Once `advanceGame` enters `rngGate` and `_applyDailyRng` returns the finalized word, all 9 consumption points execute within the same transaction. No user transaction can interleave between D1-D9. The word is consumed and `_unlockRng` clears the lock before the transaction completes.

## Section 1b: Lootbox Consumption Points (L1-L8)

### Shared Temporal Window

Lootbox consumption points have a fundamentally different temporal model from daily points:

1. **Block N:** `requestLootboxRng()` fires VRF request. `rngLockedFlag=false` (mid-day path). `rngRequestTime!=0` blocks concurrent requests.
2. **Block N+K:** `rawFulfillRandomWords()` finalizes word directly to `lootboxRngWordByIndex[index]` (AdvanceModule:1342). This is a **direct finalize** -- the word is immediately available for consumption. `rngLockedFlag` remains false.
3. **Block N+K+M (variable, per-user):** Users call `openLootBox`/`openBurnieLootBox`/`resolveDegeneretteBets` in separate transactions, potentially many blocks later.

**Critical difference from daily:** The window between word finalization (Block N+K) and consumption (Block N+K+M) is OPEN -- `rngLockedFlag` is false, and all entry points are callable. However, lootbox outcomes use **per-player entropy derivation**: `keccak256(word, player, day, amount)` or `keccak256(word, index, player, spinIdx)`, making outcomes deterministic per-player once the word is set.

**Entry points callable during lootbox window (all 25+ external entry points):** Since `rngLockedFlag=false`, nothing is blocked by the lock. Since `prizePoolFrozen` may or may not be true (depends on whether jackpot phase is active), pool-dependent operations may be blocked by freeze but not by lootbox RNG state.

### Piggyback Pattern Analysis

When `_finalizeLootboxRng` is called during daily `rngGate` processing (AdvanceModule:766), the daily finalized word is written to the pending lootbox index:
- `lootboxRngWordByIndex[index] = rngWord` (AdvanceModule:788)
- `lastLootboxRngWord = rngWord` (AdvanceModule:789)

This piggyback write occurs **within the same transaction as rngGate**, meaning:
1. The word written to the lootbox index was committed in the VRF callback (Block N+K) and consumed by `_applyDailyRng` (Block N+K+M, same tx).
2. The lootbox word write happens atomically with the daily word write -- there is no window between the daily word being known and the lootbox word being set.
3. The same VRF word (with nudge applied) is used for both daily draws AND the piggybacked lootbox index. This is documented behavior -- the daily word also resolves the batch reserved during `_requestRng`.
4. **No cross-path window exists for the piggyback write** because both writes occur in the same atomic transaction.
