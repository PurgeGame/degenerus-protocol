## Slot 0 Assembly Audit

### Objective

Verify that no inline `assembly` block in production contracts performs raw `sstore` or `sload` on EVM Slot 0, where VRF state variables (`rngLockedFlag`, `rngRequestTime`, `dailyIdx`, `levelStartTime`, and other packed fields) reside. A raw assembly write to Slot 0 could corrupt the compiler-managed packed layout, silently breaking VRF state.

### Method

Searched all production contracts for `assembly` blocks using `grep -rn "assembly" contracts/`, then manually inspected each block for storage operations targeting Slot 0.

### Results: 8 assembly blocks found, 0 touch Slot 0

| # | Contract | Line | Operation | Storage Impact |
|---|----------|------|-----------|----------------|
| 1 | DegenerusGame.sol | 1025 | `revert(add(32, reason), mload(reason))` | Memory-only (revert bubble). No storage access. |
| 2 | DegenerusGameAdvanceModule.sol | 532 | Same `_revertDelegate` pattern | Memory-only. No storage access. |
| 3 | DegenerusGameDecimatorModule.sol | 82 | Same `_revertDelegate` pattern | Memory-only. No storage access. |
| 4 | DegenerusGameDegeneretteModule.sol | 143 | Same `_revertDelegate` pattern | Memory-only. No storage access. |
| 5 | DegenerusGameMintModule.sol | 493 | `keccak256(lvl, traitBurnTicket.slot)` | Computes deep mapping slot for `traitBurnTicket`. NOT Slot 0. |
| 6 | DegenerusGameMintModule.sol | 504 | `sload(elem)`, `sstore(elem, newLen)` on trait mapping | Reads/writes deep mapping elements (trait ticket counts). NOT Slot 0. |
| 7 | DegenerusGameJackpotModule.sol | 2188-2199 | Same trait ticket pattern as MintModule | Deep mapping operations. NOT Slot 0. |
| 8 | DegenerusJackpots.sol | 518 | `mstore(winners, n)`, `mstore(amounts, n)` | Memory-only (array length trimming). No storage access. |

### Verdict: SAFE

All 8 assembly blocks operate exclusively on memory (4 blocks -- revert bubbling, 1 block -- array trimming) or on deep mapping slots for `traitBurnTicket` (3 blocks). No assembly block reads or writes Slot 0. All modifications to `rngLockedFlag`, `rngRequestTime`, `dailyIdx`, and other Slot 0 fields go through normal Solidity assignments, ensuring the compiler manages the packed layout correctly.

## Cross-Cutting Observations

1. **VRF callback is deliberately minimal.** The `rawFulfillRandomWords` function performs at most 2 storage writes (mid-day path) and emits 1 event. All complex processing (jackpot resolution, ticket assignment, prize distribution) is deferred to subsequent `advanceGame` calls. This architecture ensures the callback never approaches the 300k gas budget.

2. **Retry detection relies on a three-variable conjunction.** `_finalizeRngRequest` determines `isRetry` via `vrfRequestId != 0 && rngRequestTime != 0 && rngWordCurrent == 0`. This is correct because: (a) vrfRequestId is nonzero only when a request is in-flight, (b) rngRequestTime is nonzero only after a request was sent, and (c) rngWordCurrent == 0 means the previous request was never fulfilled. All three conditions together uniquely identify the timeout-retry scenario.

3. **Mutual exclusion is enforced at the request layer, not the callback layer.** The callback does not need to check for collisions because `requestLootboxRng` guards against `rngLockedFlag == true` and `rngRequestTime != 0`. This means a mid-day request can only be made when no daily RNG is pending. The callback then simply branches on `rngLockedFlag` to determine which path to take.

4. **Slot 0 packing is entirely compiler-managed.** Despite 31 bytes of packed fields in Slot 0 (including the security-critical `rngLockedFlag` and `rngRequestTime`), no assembly block in the codebase touches this slot. All reads and writes go through normal Solidity variable access, eliminating the risk of manual packing errors.

5. **Test infrastructure established for future VRF phases.** The VRFCore.t.sol test file provides reusable helpers (`_completeDay`, `_readVrfRequestId`, `_readRngWordCurrent`, `_readRngRequestTime`, `_doCoordinatorSwap`, `_setupForMidDayRng`) that future phases (lootbox lifecycle, stall edge cases) can inherit or reference.
