# v4.0 Ticket Processing Mechanics Audit

**Date:** 2026-03-23
**Milestone:** v4.0 (Ticket Lifecycle & RNG-Dependent Variable Re-Audit)
**Phase:** 82 (Ticket Processing Mechanics)
**Methodology:** Exhaustive code trace with file:line citations; all prior audit prose treated as unverified

## Executive Summary

Exhaustive trace of both ticket processing functions -- `processTicketBatch` (JackpotModule) and `processFutureTicketBatch` (MintModule) -- documenting entry points, all callers, trigger conditions from advanceGame, processing loop structure, gas budgeting, and the complete RNG word derivation chain from VRF callback through trait generation. Two distinct entropy sources are identified and traced independently.

### Key Conclusions

1. `processTicketBatch` (JM:1889) uses `lastLootboxRngWord` (JM:1915) as entropy; triggered from 3 advanceGame paths via `_runProcessTicketBatch` (AM:1198)
2. `processFutureTicketBatch` (MM:298) uses `rngWordCurrent` (MM:301) as entropy; triggered from 2 call sites via `_processFutureTicketBatch` (AM:1134)
3. Both share identical `_raritySymbolBatch` LCG PRNG algorithm (duplicated for delegatecall isolation) with the same LCG constant
4. The mid-day path entropy source for `processTicketBatch` can differ from the daily word -- it reads from `lootboxRngWordByIndex[lootboxRngIndex - 1]` (AM:159) rather than the daily VRF word
5. Five total advanceGame trigger points for ticket processing are identified and verified

## 1. processTicketBatch Entry Point (TPROC-01)

### 1.1 Function Signature

```
function processTicketBatch(uint24 lvl) external returns (bool finished)
```

**Location:** JM:1889
**Module:** DegenerusGameJackpotModule.sol
**Return:** `bool finished` -- true if all tickets for this level have been fully processed

### 1.2 Delegatecall Entry

The single entry point into `processTicketBatch` is `_runProcessTicketBatch` at AM:1198:

```
function _runProcessTicketBatch(uint24 lvl) private returns (bool worked, bool finished)
```

**Mechanism (AM:1198-1215):**
1. Records pre-call cursor state: `prevCursor = ticketCursor` (AM:1201), `prevLevel = ticketLevel` (AM:1202)
2. Delegatecalls `GAME_JACKPOT_MODULE` with selector `IDegenerusGameJackpotModule.processTicketBatch.selector` (AM:1203-1210)
3. Reverts on failure via `_revertDelegate(data)` (AM:1211)
4. Reverts with `E()` if `data.length == 0` (AM:1212)
5. Decodes `finished` from return data (AM:1213)
6. Determines `worked` by comparing cursors: `worked = (ticketCursor != prevCursor) || (ticketLevel != prevLevel)` (AM:1214)

### 1.3 advanceGame Trigger Paths

**Path 1: Mid-day (same day) -- AM:154-181**

Conditions:
- `day == dailyIdx` (AM:154) -- caller purchases on the same day as the current daily index
- `!ticketsFullyProcessed` (AM:156) -- read slot not yet fully drained

Processing flow:
1. If `midDayTicketRngPending` is true (AM:158), reads `word = lootboxRngWordByIndex[lootboxRngIndex - 1]` (AM:159)
   - Reverts with `NotTimeYet()` if `word == 0` (AM:160) -- VRF not yet arrived
   - Sets `lastLootboxRngWord = word` (AM:162) -- **updates entropy source from mid-day lootbox VRF word**
2. Checks `_tqReadKey(purchaseLevel)` queue has entries (AM:165-166)
3. Calls `_runProcessTicketBatch(purchaseLevel)` (AM:170)
4. If `ticketWorked || !ticketsFinished` (AM:171):
   - If `ticketsFinished`: sets `ticketsFullyProcessed = true` (AM:173), clears `midDayTicketRngPending = false` (AM:174)
   - Emits `Advance(STAGE_TICKETS_WORKING, lvl)` (AM:176)
   - Credits caller bounty (AM:177)
   - Returns (AM:178) -- single batch per call

**Path 2: Daily drain gate -- AM:204-219**

Conditions:
- `day > dailyIdx` (implicit from reaching this code path after AM:184 `revert NotTimeYet()` for same-day)
- `!ticketsFullyProcessed` (AM:205) -- read slot not yet fully drained

Processing flow:
1. Checks `_tqReadKey(purchaseLevel)` queue has entries (AM:206-207)
2. Calls `_runProcessTicketBatch(purchaseLevel)` (AM:211)
3. If `ticketWorked || !ticketsFinished` (AM:212):
   - Emits `Advance(STAGE_TICKETS_WORKING, lvl)` (AM:213)
   - Credits caller bounty (AM:214)
   - Returns (AM:215) -- single batch per call
4. If queue empty or just finished: sets `ticketsFullyProcessed = true` (AM:218) -- **regardless of whether processing ran**

**Path 3: New-day post-RNG current-level -- AM:269-276**

Conditions:
- `day > dailyIdx` (new day)
- `ticketsFullyProcessed` is true (daily drain gate passed at AM:204-219)
- RNG word received (`rngGate` returned non-1 at AM:225-236)
- `!phaseTransitionActive` (AM:239)
- Near-future ticket processing complete (AM:255-266, `_prepareFutureTickets(lvl)` returned true)

Processing flow:
1. Calls `_runProcessTicketBatch(purchaseLevel)` (AM:269-271)
2. If `ticketWorked || !ticketsFinished` (AM:272):
   - Sets `stage = STAGE_TICKETS_WORKING` (AM:273)
   - Breaks to emit advance event and credit bounty
3. On completion: sets `ticketsFullyProcessed = true` at AM:276 (comment: "ADV-03: set before jackpot/phase logic")

### 1.4 Processing Loop Structure

**processTicketBatch (JM:1889-1950):**

1. **Queue selection:** `rk = _tqReadKey(lvl)` (JM:1890), `queue = ticketQueue[rk]` (JM:1891)
2. **Level switch:** If `ticketLevel != lvl`: resets `ticketLevel = lvl`, `ticketCursor = 0` (JM:1895-1897)
3. **Already done check:** If `idx >= total`: deletes queue, resets cursor to 0, resets level to 0, returns `true` (JM:1901-1907)
4. **Gas budget:** `writesBudget = WRITES_BUDGET_SAFE` (550) at JM:1909; if `idx == 0` (first batch): `writesBudget -= (writesBudget * 35) / 100` = 357 (JM:1910-1912) -- 65% scaling for cold storage
5. **Entropy source:** `entropy = lastLootboxRngWord` (JM:1915)
6. **Processing loop (JM:1918-1937):** While `idx < total && used < writesBudget`:
   - Calls `_processOneTicketEntry(queue[idx], lvl, rk, writesBudget - used, processed, entropy, idx)` (JM:1919-1927)
   - Returns `(writesUsed, advance)` (JM:1919)
   - If `writesUsed == 0 && !advance`: breaks (budget exhausted) (JM:1928)
   - If `advance`: increments `idx`, resets `processed = 0` (JM:1931-1933)
   - Else: `processed += writesUsed >> 1` (approximate tickets processed) (JM:1935)
7. **Save cursor:** `ticketCursor = uint32(idx)` (JM:1940)
8. **Completion check:** If `idx >= total`: deletes queue, resets `ticketCursor = 0`, `ticketLevel = 0`, returns `true` (JM:1942-1948)

### 1.5 Helper Chain (processTicketBatch only)

`processTicketBatch` delegates to a chain of four helper functions. `processFutureTicketBatch` has inline processing (no helpers).

**_processOneTicketEntry (JM:1984-2047):**
1. Reads `packed = ticketsOwedPacked[rk][player]` (JM:1993), extracts `owed = uint32(packed >> 8)` (JM:1994)
2. Constructs `rollSalt = (uint256(lvl) << 224) | (queueIdx << 192) | (uint256(uint160(player)) << 32)` (JM:1995-1997)
3. Zero-owed handling: calls `_resolveZeroOwedRemainder` (JM:2002-2009)
4. Budget calculation: `baseOv = (processed == 0 && owed <= 2) ? 4 : 2` (JM:2017), batch size limited by `room - baseOv` (JM:2021-2025)
5. Calls `_generateTicketBatch(player, lvl, processed, take, entropy, queueIdx)` (JM:2030)
6. Calculates actual writes used (JM:2033-2036)
7. Calls `_finalizeTicketEntry(rk, player, packed, owed, take, entropy, rollSalt)` (JM:2037-2045)

**_generateTicketBatch (JM:2050-2070):**
1. Constructs `baseKey = (uint256(lvl) << 224) | (queueIdx << 192) | (uint256(uint160(player)) << 32)` (JM:2058-2060)
2. Calls `_raritySymbolBatch(player, baseKey, processed, take, entropy)` (JM:2061)
3. Emits `TraitsGenerated(player, lvl, uint32(queueIdx), processed, take, entropy)` (JM:2062-2069)

**_finalizeTicketEntry (JM:2073-2098):**
1. Extracts `rem = uint8(packed)` (JM:2082)
2. Calculates `remainingOwed = owed - take` (JM:2085)
3. If `remainingOwed == 0 && rem != 0`: calls `_rollRemainder(entropy, rollSalt, rem)` (JM:2087-2088); if wins, `remainingOwed = 1` (JM:2089)
4. Updates `ticketsOwedPacked[rk][player]` if changed (JM:2093-2095)
5. Returns `done = (remainingOwed == 0)` (JM:2097)

### 1.6 Gas Budget Detail

| Constant | Value | Location |
|----------|-------|----------|
| `WRITES_BUDGET_SAFE` | 550 | JM:157 |
| Cold storage scaling | 35% reduction (first batch) | JM:1910-1911 |
| Effective first-batch budget | 357 | Computed: `550 - (550 * 35) / 100` |
| Per-ticket write cost | ~2 SSTOREs (trait array push + length) | JM:2198-2216 assembly |
| Per-player overhead | 2-4 (baseOv depends on processed == 0 && owed <= 2) | JM:2017 |

## 2. processFutureTicketBatch Entry Point (TPROC-02)

### 2.1 Function Signature

```
function processFutureTicketBatch(uint24 lvl) external returns (bool worked, bool finished, uint32 writesUsed)
```

**Location:** MM:298
**Module:** DegenerusGameMintModule.sol
**Return:** Three values: `worked` (any tickets minted), `finished` (queue fully drained), `writesUsed` (budget units consumed)

### 2.2 Delegatecall Entry

The single entry point into `processFutureTicketBatch` is `_processFutureTicketBatch` at AM:1134:

```
function _processFutureTicketBatch(uint24 lvl) private returns (bool worked, bool finished, uint32 writesUsed)
```

**Mechanism (AM:1134-1148):**
1. Delegatecalls `GAME_MINT_MODULE` with selector `IDegenerusGameMintModule.processFutureTicketBatch.selector` (AM:1137-1143)
2. Reverts on failure via `_revertDelegate(data)` (AM:1145)
3. Reverts with `E()` if `data.length == 0` (AM:1146)
4. Decodes `(bool, bool, uint32)` from return data (AM:1147)

### 2.3 Call Sites

**Call Site 1: _prepareFutureTickets -- AM:1156-1184**

Called at AM:262 in the new-day path: `_prepareFutureTickets(lvl)`

**_prepareFutureTickets (AM:1156-1184):**
1. Computes range: `startLevel = lvl + 2`, `endLevel = lvl + 6` (AM:1157-1158)
2. Reads resume state: `resumeLevel = ticketLevel` (AM:1159), `baseResume = resumeLevel & ~uint24(TICKET_FAR_FUTURE_BIT)` (AM:1162) -- strips FF bit for range comparison
3. **Resume in-flight level first (AM:1165-1170):** If `baseResume >= startLevel && baseResume <= endLevel`, calls `_processFutureTicketBatch(baseResume)`. If `worked || !levelFinished`: returns false (more work needed)
4. **Iterate remaining levels (AM:1173-1183):** For `target = startLevel` to `endLevel`, skipping `baseResume`, calls `_processFutureTicketBatch(target)`. If `worked || !levelFinished`: returns false
5. Returns true when all 5 target levels fully processed

**Call Site 2: Direct call at AM:305**

Location: AM:298-310, in the purchase-phase last-purchase-day path

Conditions:
- `!inJackpot` (AM:279) -- still in purchase phase
- `lastPurchaseDay` is true (AM:281-296 -- target prize pool reached)
- After daily jackpots and current-level ticket processing complete

Processing flow:
1. `nextLevel = purchaseLevel + 1` (AM:300)
2. Calls `_processFutureTicketBatch(nextLevel)` (AM:305)
3. If `futureWorked || !futureFinished` (AM:306): sets `stage = STAGE_FUTURE_TICKETS_WORKING` (AM:307), breaks

### 2.4 Dual-Queue Drain Logic

`processFutureTicketBatch` implements a two-phase queue drain (read-side first, then far-future):

**Phase detection (MM:302):**
`inFarFuture = (ticketLevel == (lvl | TICKET_FAR_FUTURE_BIT))`

**Queue selection (MM:303):**
`rk = inFarFuture ? _tqFarFutureKey(lvl) : _tqReadKey(lvl)`

**Empty queue handling (MM:306-318):**
If `total == 0`:
- a. If NOT in FF phase: checks `_tqFarFutureKey(lvl)` for entries (MM:308-309)
  - If FF queue has entries: sets `ticketLevel = lvl | TICKET_FAR_FUTURE_BIT`, resets `ticketCursor = 0` (MM:310-311), returns `(false, false, 0)` -- signals FF transition
  - If FF queue also empty: resets `ticketCursor = 0`, `ticketLevel = 0` (MM:315-316), returns `(false, true, 0)` -- done
- b. If in FF phase (and FF queue is empty): same reset (MM:315-316), returns done

**Level switch (MM:320-323):**
If `!inFarFuture && ticketLevel != lvl`: sets `ticketLevel = lvl`, `ticketCursor = 0`

**Read-side exhausted mid-processing (MM:326-338):**
If `idx >= total` (read-side fully iterated):
1. Deletes `ticketQueue[rk]` (MM:327)
2. If NOT in FF phase: checks FF queue for entries (MM:329-330)
   - If FF exists: transition to FF (MM:331-332), returns `(false, false, 0)`
   - If no FF: resets cursor, returns `(false, true, 0)` (MM:336-338)

**Completion on finish (MM:436-452):**
When inline processing loop completes (`idx >= total`):
1. Deletes queue (MM:437)
2. If NOT in FF: checks for FF transition (MM:438-443)
3. If in FF (or no FF entries): resets `ticketCursor = 0`, `ticketLevel = 0` (MM:445-446 or MM:449-450)

### 2.5 Inline Processing Loop

Unlike `processTicketBatch` which delegates to a helper chain, `processFutureTicketBatch` has the processing logic inline (MM:350-430):

1. **Load player data (MM:351-357):**
   - `player = queue[idx]` (MM:351)
   - `baseKey = (uint256(lvl) << 224) | (idx << 192) | (uint256(uint160(player)) << 32)` (MM:352-354)
   - `packed = ticketsOwedPacked[rk][player]` (MM:355)
   - `owed = uint32(packed >> 8)`, `rem = uint8(packed)` (MM:356-357)

2. **Zero-owed handling (MM:358-382):**
   - If `owed == 0 && rem == 0`: cleanup skip, charges 1 budget unit (MM:359-367)
   - If `owed == 0 && rem != 0`: calls `_rollRemainder(entropy, baseKey, rem)` (MM:369)
     - Roll fails: cleanup, advance (MM:370-373)
     - Roll wins: set `packed = uint40(1) << 8`, `owed = 1`, `rem = 0` (MM:375-381)

3. **Batch size calculation (MM:383-390):**
   - `room = writesBudget - used` (MM:383)
   - `baseOv = (processed == 0 && owed <= 2) ? 4 : 2` (MM:384)
   - `maxT = (room <= 256) ? (room / 2) : (room - 256)` (MM:388)
   - `take = owed > maxT ? maxT : owed` (MM:389)

4. **Trait generation (MM:392):**
   - `_raritySymbolBatch(player, baseKey, processed, take, entropy)` -- same function as JM copy

5. **Finalization (MM:407-429):**
   - Calculates `remainingOwed = owed - take` (MM:409)
   - Remainder roll if `remainingOwed == 0 && rem != 0` (MM:411-415)
   - Updates `ticketsOwedPacked[rk][player]` if changed (MM:418-420)
   - If `remainingOwed == 0`: advances `idx`, resets `processed = 0` (MM:426-428)

### 2.6 Structural Difference from processTicketBatch

| Aspect | processTicketBatch (JM) | processFutureTicketBatch (MM) |
|--------|------------------------|------------------------------|
| Loop body | Delegated to `_processOneTicketEntry` (JM:1984) | Inline (MM:350-430) |
| Helper chain | `_processOneTicketEntry` -> `_generateTicketBatch` -> `_raritySymbolBatch` -> `_finalizeTicketEntry` | Direct `_raritySymbolBatch` call |
| baseKey construction | In `_generateTicketBatch` (JM:2058-2060) | Inline (MM:352-354) |
| Emit pattern | In `_generateTicketBatch` (JM:2062-2069) | Inline (MM:393-400) |
| Queue drain | Single queue: `_tqReadKey` only | Dual queue: `_tqReadKey` then `_tqFarFutureKey` |
| Entropy source | `lastLootboxRngWord` (JM:1915) | `rngWordCurrent` (MM:301) |
| Gas budget | Identical: WRITES_BUDGET_SAFE=550, 65% cold scaling | Identical: WRITES_BUDGET_SAFE=550, 65% cold scaling |
