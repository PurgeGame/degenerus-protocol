## Section 4: Prize Pool Consolidation Mechanics (PPF-04)

### 4a. Pre-Consolidation Steps (AdvanceModule)

These run at `AM:313-316` inside the purchase-to-jackpot phase transition:

**Step 0a:** Record level target
```solidity

```
Stores the next pool value as the level's "target" for future overshoot calculations.

**Step 0b:** Time-based next-to-future skim
```solidity
// AM:315
_applyTimeBasedFutureTake(ts, purchaseLevel, rngWord);
```

`_applyTimeBasedFutureTake` at `AM:1029-1103`:
1. Computes base BPS from elapsed time (`_nextToFutureBps` with MIN=1300, FAST=3000)
2. Adds x9 bonus (+200 BPS) if `lvl % 10 == 9` (`AM:1038`)
3. Ratio-adjusts ±4% targeting 2:1 future/next ratio (`AM:1044-1053`)
4. Adds overshoot surcharge (hyperbolic) if nextPool > 1.25x previous level (`AM:1056-1065`)
5. Adds random 0-10% on BPS (`AM:1069`)
6. Applies ±25% triangular variance (`AM:1077-1093`)
7. Caps at 80% of nextPool (`AM:1097-1098`)
8. Deducts 1% insurance skim to yieldAccumulator (`AM:1100-1103`)
9. Final writes: `_setNextPrizePool(nextPoolBefore - take - insuranceSkim)` and `_setFuturePrizePool(futurePoolBefore + take)` (`AM:1101-1102`)

**CONFIRMED:** `_applyTimeBasedFutureTake` does NOT touch `currentPrizePool`. It only reads/writes `prizePoolsPacked` (next and future components), `levelPrizePool[lvl-1]`, `levelStartTime`, and `yieldAccumulator`.

### 4b. consolidatePrizePools (JM:879-908) -- 5 Steps

**Step 1: x00 Yield Dump** (`JM:881-886`)
```solidity
if ((lvl % 100) == 0) {
    uint256 acc = yieldAccumulator;
    uint256 half = acc >> 1;
    _setFuturePrizePool(_getFuturePrizePool() + half);
    yieldAccumulator = acc - half; // rounds in favor of retention

```
On x00 milestone levels only: moves 50% of `yieldAccumulator` into `futurePrizePool`. The retained half serves as terminal insurance.

**Step 2: Merge Next into Current** (`JM:889-890`)
```solidity

_setNextPrizePool(0);
```
Unconditional: merges the entire `nextPrizePool` into `currentPrizePool` and zeros the next component. This is the primary funding mechanism for `currentPrizePool`.

**Step 3: x00 Keep Roll** (`JM:892-902`)
```solidity
if ((lvl % 100) == 0) {
    uint256 keepBps = _futureKeepBps(rngWord);
    uint256 fp = _getFuturePrizePool();
    if (keepBps < 10_000 && fp != 0) {
        uint256 keepWei = (fp * keepBps) / 10_000;
        uint256 moveWei = fp - keepWei;
        if (moveWei != 0) {
            _setFuturePrizePool(keepWei);
            currentPrizePool += moveWei;
        }
    }
}
```
On x00 milestone levels only: rolls `_futureKeepBps(rngWord)` at `JM:1281-1296` which uses 5 dice with range [0,3] each, mapped to 3000 + (total * 3500) / 15, giving a range of **3000-6500 BPS (30-65%, avg ~47.5%)**. The kept portion stays in `futurePrizePool`; the remainder moves to `currentPrizePool`.

**Step 4: Credit Coinflip** (`JM:905`)
```solidity
_creditDgnrsCoinflip(currentPrizePool);
```
At `JM:2346-2351`: computes `(prizePoolWei * PRICE_COIN_UNIT) / (priceWei * 20)` BURNIE and credits to sDGNRS coinflip via `coin.creditFlip`. Reads `currentPrizePool` (now includes merged + x00 transferred amounts).

**Step 5: Yield Surplus Distribution** (`JM:907`)
```solidity
_distributeYieldSurplus(rngWord);
```
At `JM:912-943`: computes `obligations = currentPrizePool + _getNextPrizePool() + claimablePool + _getFuturePrizePool() + yieldAccumulator` (`JM:915-919`). If `totalBalance > obligations`, distributes surplus: 23% each to DGNRS/Vault claimable, 46% to `yieldAccumulator`, ~8% unextracted buffer.

function _drawDownFuturePrizePool(uint24 lvl) private {
    uint256 reserved;
