### 2.3 Sentinel-Index Correctness Verification

The guard at AdvanceModule:1174 reads `rngWordByDay[idx + 1]` where `idx = dailyIdx`. For the guard to correctly detect "backfill not yet run for this lock window," the sentinel index must satisfy the following four conditions:

1. **The sentinel key is in the gap range:** `idx + 1 ∈ [startDay, endDay)` where `startDay = idx + 1` and `endDay = day`. Confirmed: `startDay == idx + 1` by construction at L1176 (`_backfillGapDays(currentWord, idx + 1, day, bonusFlip)`). The gap range starts AT the sentinel key.

2. **The sentinel key is the FIRST gap day written by `_backfillGapDays`:** The loop at L1761 starts at `gapDay = startDay = idx + 1`, and the first iteration writes `rngWordByDay[idx + 1] = derivedWord` at L1766. The sentinel write happens on the first loop iteration, so the sentinel flips from 0 → non-zero atomically with the first state mutation in the guarded block. No intermediate state is observable where the sentinel reads 0 but `_backfillGapDays` already mutated `rngWordByDay`.

3. **No off-by-one against `idx`:** sentinel reads `idx + 1` (NOT `idx`). At entry, `dailyIdx == idx` already has `rngWordByDay[idx]` set (from the prior wall-clock day's normal `rngGate` cycle that produced that idx via `_unlockRng`). If the guard read `rngWordByDay[idx]` it would always evaluate FALSE (already-set) and skip backfill on the very first lock-window entry — defeating the purpose of the guard. The chosen index `idx + 1` is correct.

4. **No off-by-one against `day`:** sentinel reads `idx + 1` (NOT `day - 1`). `day - 1` is the LAST gap day; reading `rngWordByDay[day - 1]` would incorrectly evaluate as "not yet run" on the second invocation if a hypothetical partial loop got partway through and didn't finish (impossible under current single-call semantics, but a defensive-correctness check). The chosen index `idx + 1` (the FIRST gap day) is the safer choice because it commits atomically with the very first state mutation inside the loop.

**Conclusion:** Sentinel index `idx + 1` is correct. No off-by-one against either `idx` or `day`. BFL-02 sentinel-correctness invariant HOLDS.
