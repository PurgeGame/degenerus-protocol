### 1.2 Stale Word Routing (Cross-Day VRF Callback)

When a VRF word arrives from a previous day's request (`requestDay < day`):

```
rngGate() AdvanceModule:749-759
  |
  | currentWord != 0 && rngRequestTime != 0
  | requestDay < day (stale)
  v
_finalizeLootboxRng(currentWord)    -- routes stale word to lootbox index
  AdvanceModule:756
rngWordCurrent = 0                  -- clears daily slot
  AdvanceModule:757
_requestRng(...)                    -- fires fresh VRF request for today
  AdvanceModule:758
return 1                            -- signals "RNG requested"
```

The stale daily word is not wasted -- it is recycled to the pending lootbox index. A fresh VRF request is made for the current day's daily draws.

### 1.4 Gameover Fallback Entropy

When VRF stalls during gameover, `_gameOverEntropy` (AdvanceModule:797) provides fallback:

```
_gameOverEntropy()
  |
  | rngRequestTime != 0 && elapsed >= GAMEOVER_RNG_FALLBACK_DELAY
  v
_getHistoricalRngFallback(day) AdvanceModule:859
  |
  | Collects up to 5 early historical rngWordByDay[1..30]
  | combined = keccak256(combined, w) per found word
  | return keccak256(combined, currentDay, block.prevrandao)
  v
_applyDailyRng(day, fallbackWord)
  |
  | Same nudge application as normal path
  v
coinflip.processCoinflipPayouts(...)
_finalizeLootboxRng(fallbackWord)
return fallbackWord
```

**Security note:** Fallback uses committed VRF words (non-manipulable) plus `block.prevrandao` (1-bit validator bias). This is gameover-only and the trade-off is documented in source comments.

### 2.2 Daily Path Lootbox Finalization (Piggyback)

When the daily VRF callback routes through `rngGate`, it also finalizes any pending lootbox index:

```
rngGate() -> _finalizeLootboxRng(finalWord)
  AdvanceModule:785
  |
  | index = lootboxRngRequestIndexById[vrfRequestId]
  | if index == 0: return (no pending lootbox)
  |
  | lootboxRngWordByIndex[index] = rngWord  (:788)
  | lastLootboxRngWord = rngWord             (:789)
  | emit LootboxRngApplied
```

This means every daily VRF word also resolves the lootbox batch that was reserved during the daily `_requestRng` call. The same word is used for both daily draws and lootbox resolution.

### 2.4 Lootbox Index Lifecycle

```
lootboxRngIndex: 1 (initial)
    |
    | _reserveLootboxRngIndex(requestId)
    | AdvanceModule:1310
    |   lootboxRngRequestIndexById[requestId] = index
    |   lootboxRngIndex = index + 1
    |   lootboxRngPendingEth = 0
    |   lootboxRngPendingBurnie = 0

lootboxRngIndex: 2
    |
    | New purchases use index 2 (lootboxRngIndex)
    | MintModule:684, MintModule:1012, WhaleModule:752, DegeneretteModule:466
    |
    | VRF callback resolves index 1:
    |   lootboxRngWordByIndex[1] = word

```

**Key invariant:** Purchases are always assigned to the *current* `lootboxRngIndex`, but VRF resolves the *previous* index (mapped via `lootboxRngRequestIndexById`). This ensures a commit-reveal separation: purchases happen before the RNG word is known.

### 3.2 Interaction Between Daily and Lootbox Paths

```
+------------------------------------------------------------------+
|                        TIME AXIS (one day)                        |
+------------------------------------------------------------------+

  advanceGame()           requestLootboxRng()        advanceGame()
  [daily path]            [mid-day path]             [mid-day drain]
       |                        |                         |
       |  Daily VRF fires       |  Lootbox VRF fires      |  Drains tickets
       |  rngLockedFlag=true    |  rngLockedFlag=false     |  using lootbox word
       |                        |                         |
       v                        v                         v
  rngWordByDay[D]=W1      lootboxRngWordByIndex[N]=W2  lastLootboxRngWord=W2
       |                        |                         |
       |  _finalizeLootboxRng   |  Swaps ticket buffer    |  processTicketBatch
       |  piggybacks daily      |  midDayTicketRngPending  |  reads lastLootboxRngWord
       |  VRF to lootbox        |  = true                 |
       v                        v                         v
  [Daily draws use W1]    [Lootbox opens use W2]    [Trait assignment uses W2]
  [Lootbox index also     [Bets resolve with W2]
   gets W1 if pending]
```

**Key separation:** Daily draws use `rngWordByDay[day]` (= W1 with nudge applied). Lootbox opens/bets use `lootboxRngWordByIndex[index]` (= W2, standalone mid-day word OR W1 piggybacked). These are independent entropy streams after the VRF callback. The only cross-path is:

1. **Piggyback:** When `_finalizeLootboxRng` is called during daily processing, the daily finalized word is also written to the pending lootbox index.
2. **Mid-day ticket entropy:** `lastLootboxRngWord` is set either from the piggybacked daily word (via `_finalizeLootboxRng` at AdvanceModule:789) or from the mid-day drain path (AdvanceModule:166), then consumed by `processTicketBatch` (JackpotModule:1975).

### 3.3 Combined Entropy Source Table

| Entropy Variable | Source | Used By | Independent? |
|-----------------|--------|---------|-------------|
| `rngWordByDay[day]` | VRF word + nudge offset via `_applyDailyRng` | Daily jackpots, coinflip, winning traits, BAF, decimator | Yes -- unique per day, immutable once written |
| `rngWordCurrent` | Raw VRF word (pre-nudge) while in flight; finalized word after `_applyDailyRng` | Intermediate; also used by `processFutureTicketBatch`, `recordMintData`, `issueDeityBoon` fallback | Transient -- zeroed by `_unlockRng` |
| `lootboxRngWordByIndex[N]` | Direct VRF callback (mid-day) or piggybacked from daily finalized word | Lootbox opens, degenerette bet resolution, deity boon check | Yes per index -- immutable once written |
| `lastLootboxRngWord` | Copy of most recent `lootboxRngWordByIndex` value | `processTicketBatch` trait assignment | Overwritten on each lootbox resolution; never zeroed |
