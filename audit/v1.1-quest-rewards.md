## 1. Overview

The Degenerus quest system provides two daily quest slots incentivizing player engagement across all game mechanics. Quests reward BURNIE tokens credited as coinflip stakes.

Key characteristics:
- **Two daily slots:** Slot 0 is always MINT_ETH (fixed), slot 1 is randomly rolled from 8 eligible types
- **Fixed rewards:** Slot 0 pays 100 BURNIE, slot 1 pays 200 BURNIE (total 300 BURNIE/day if both completed)
- **Slot prerequisite:** Slot 1 cannot be completed until slot 0 is done for the day
- **Streak system:** Consecutive daily quest completions build a streak contributing up to 100% bonus to activity score
- **Shield protection:** Shields cover missed days to preserve streaks (1 shield consumed per missed day)
- **Version gating:** Progress resets when quest day or version changes, preventing stale carry-over

**Architecture:** DegenerusQuests.sol operates as a standalone external contract (not delegatecall) called by the BurnieCoin (COIN) contract. All player-action handlers are COIN-gated via the `onlyCoin` modifier. Quest rewards are routed through BurnieCoin's `_questApplyReward` and credited as coinflip stakes via `IBurnieCoinflip.creditFlip`.

**Primary source:** `contracts/DegenerusQuests.sol`

### 2b. Target Resolution

Targets are fixed per quest type with no difficulty variance:

```solidity
// DegenerusQuests.sol:1254-1283
function _questTargetValue(DailyQuest memory quest, uint8 slot, uint256 mintPrice)
    private pure returns (uint256)
{
    uint8 qType = quest.questType;
    if (qType == QUEST_TYPE_MINT_ETH) {
        uint256 mult = slot == 0
            ? QUEST_DEPOSIT_ETH_TARGET_MULTIPLIER   // 1x for slot 0
            : QUEST_LOOTBOX_TARGET_MULTIPLIER;       // 2x if somehow in slot 1
        uint256 target = mintPrice * mult;
        return target > QUEST_ETH_TARGET_CAP ? QUEST_ETH_TARGET_CAP : target;
    }
    if (qType == QUEST_TYPE_LOOTBOX || qType == QUEST_TYPE_DEGENERETTE_ETH) {
        uint256 target = mintPrice * QUEST_LOOTBOX_TARGET_MULTIPLIER; // 2x
        return target > QUEST_ETH_TARGET_CAP ? QUEST_ETH_TARGET_CAP : target;
    }
    if (qType == QUEST_TYPE_MINT_BURNIE) {
        return QUEST_MINT_TARGET; // 1 ticket
    }
    if (qType == QUEST_TYPE_FLIP || qType == QUEST_TYPE_DECIMATOR
        || qType == QUEST_TYPE_AFFILIATE || qType == QUEST_TYPE_DEGENERETTE_BURNIE) {
        return QUEST_BURNIE_TARGET; // 2000 BURNIE
    }
    return 0;
}
```

## 3. Quest Rewards (QRWD-01)

### 3a. BURNIE Payout Amounts

| Slot | Reward Constant | Amount |
|------|----------------|--------|
| Slot 0 (MINT_ETH) | QUEST_SLOT0_REWARD | 100 BURNIE |
| Slot 1 (random) | QUEST_RANDOM_REWARD | 200 BURNIE |

Both rewards are denominated in BURNIE base units (18 decimals): 100 BURNIE = `100 ether` in Solidity.

### 3b. Reward Crediting Path

Quest rewards are **not** minted directly to the player. They are credited as coinflip stakes:

```
Player action
  -> BurnieCoin.notifyQuest*() or BurnieCoin.affiliateQuestReward()
    -> DegenerusQuests.handle*() returns (reward, questType, streak, completed)
      -> BurnieCoin._questApplyReward() emits QuestCompleted event
        -> IBurnieCoinflip(coinflipContract).creditFlip(player, questReward)
```

```solidity
// DegenerusQuests.sol:1437
uint256 rewardShare = slot == 1 ? QUEST_RANDOM_REWARD : QUEST_SLOT0_REWARD;
```

```solidity
// BurnieCoin.sol:1007-1022
function _questApplyReward(
    address player, uint256 reward, uint8 questType,
    uint32 streak, bool completed
) private returns (uint256) {
    if (!completed) return 0;
    emit QuestCompleted(player, questType, streak, reward);
    return reward;
}
```

```solidity
// BurnieCoin.sol:794-796 (example from notifyQuestMint)
if (questReward != 0) {
    IBurnieCoinflip(coinflipContract).creditFlip(player, questReward);
}
```

> **AGENT NOTE:** Quest rewards enter the player's economy as coinflip deposit stakes, not as liquid BURNIE. The player must wait for the coinflip outcome to realize the value. This is consistent across all quest completion paths (mint, flip, lootbox, degenerette, affiliate, decimator).

## 4. Slot System and Daily Rolling (QRWD-02)

### 4a. Slot Assignment Rules

- **Slot 0:** Always MINT_ETH (fixed). Set unconditionally in `_rollDailyQuest`.
- **Slot 1:** Randomly selected from weighted pool using VRF entropy. The primary type (MINT_ETH) is excluded from the pool.

### 4b. Slot 1 Draw Weights

| Quest Type | Weight | Probability (with decimator) | Probability (without decimator) |
|-----------|--------|------------------------------|--------------------------------|
| MINT_BURNIE | 10 | 10/24 = 41.67% | 10/20 = 50.00% |
| FLIP | 4 | 4/24 = 16.67% | 4/20 = 20.00% |
| DECIMATOR | 4 | 4/24 = 16.67% | N/A (excluded) |
| LOOTBOX | 3 | 3/24 = 12.50% | 3/20 = 15.00% |
| AFFILIATE | 1 | 1/24 = 4.17% | 1/20 = 5.00% |
| DEGENERETTE_ETH | 1 | 1/24 = 4.17% | 1/20 = 5.00% |
| DEGENERETTE_BURNIE | 1 | 1/24 = 4.17% | 1/20 = 5.00% |
| **Total** | **24 / 20** | **100%** | **100%** |

RESERVED (ID 4) always has weight 0 and is skipped.

### 4c. Rolling Implementation

```solidity
// DegenerusQuests.sol:378-419
function _rollDailyQuest(uint48 day, uint256 entropy)
    private returns (bool rolled, uint8[2] memory questTypes, bool highDifficulty)
{
    DailyQuest[QUEST_SLOT_COUNT] storage quests = activeQuests;
    bool decAllowed = _canRollDecimatorQuest();

    // Swap 128-bit halves to derive independent entropy for slot 1
    uint256 bonusEntropy = (entropy >> 128) | (entropy << 128);

    uint8 primaryType = QUEST_TYPE_MINT_ETH;
    uint8 bonusType = _bonusQuestType(bonusEntropy, primaryType, decAllowed);

    _seedQuestType(quests[0], day, primaryType);
    _seedQuestType(quests[1], day, bonusType);
    // ...
}
```

The `_bonusQuestType` function (lines 1305-1375) builds a weight table excluding the primary type and RESERVED, then performs weighted random selection: `roll = entropy % total`.

> **CRITICAL PITFALL: Slot 0 Prerequisite.** Slot 1 cannot be completed unless slot 0 is already complete for that day. This check appears in every progress handler:
>
> ```solidity
> // DegenerusQuests.sol:587 (example from handleFlip)
> if (slotIndex == 1 && (state.completionMask & 1) == 0) {
>     return (0, quest.questType, state.streak, false);
> }
> ```
>
> **This forces daily ETH purchase activity (completing the MINT_ETH quest) before any bonus quest can be completed.** An agent cannot earn the 200 BURNIE slot 1 reward without first completing the 1x mintPrice ETH deposit in slot 0.

## 5. Decimator Quest Availability (QRWD-01)

Decimator quests are conditionally available in the slot 1 pool. Availability is checked at quest rolling time via `_canRollDecimatorQuest`:

```solidity
// DegenerusQuests.sol:1014-1023
function _canRollDecimatorQuest() private view returns (bool) {
    IDegenerusGame game_ = questGame;
    if (!game_.decWindowOpenFlag()) return false;
    uint24 lvl = game_.level();
    // Always available at 100-level milestones
    if (lvl != 0 && (lvl % DECIMATOR_SPECIAL_LEVEL) == 0) return true;
    // Available at X5 levels (5, 15, 25, 35...) except X95
    if (lvl < 5) return false;
    return (lvl % 10) == 5 && (lvl % 100) != 95;
}
```

**Availability conditions (all must hold):**

1. `decWindowOpenFlag` must be true (set by game during decimator windows)
2. Level satisfies one of:
   - Multiple of 100 (level 100, 200, 300, ...) AND level != 0
   - Ends in 5 (level 5, 15, 25, 35, ...) EXCEPT levels ending in 95 (95, 195, 295, ...)

**Effect on draw weights:** When decimator is unavailable, its weight 4 is removed and total weight drops from 24 to 20, increasing all other types' probabilities proportionally.

**Examples of eligible levels:** 5, 15, 25, 35, 45, 55, 65, 75, 85, 100, 105, 115, ..., 185, 200, 205, ...
**Examples of ineligible levels:** 0, 1-4, 6-14, 95, 195, 295, ...

## 6. Streak System (QRWD-02)

### 6a. Streak Increment

The streak increments by 1 on the **first quest completion of any slot per day**. The `QUEST_STATE_STREAK_CREDITED` flag (bit 7 of completionMask) prevents double-crediting within the same day.

```solidity
// DegenerusQuests.sol:1424-1435
uint32 newStreak = uint32(state.streak);

// Streak is credited on the first quest completion of the day.
if ((mask & QUEST_STATE_STREAK_CREDITED) == 0) {
    mask |= QUEST_STATE_STREAK_CREDITED;
    if (newStreak < type(uint24).max) {
        newStreak += 1;
    }
    state.streak = uint24(newStreak);
    state.lastCompletedDay = questDay24;
}
```

Completing slot 0 first (required by the prerequisite) will credit the streak. Completing slot 1 afterward does not increment the streak again.

### 6b. Streak Reset

Missing a day resets the streak to 0, unless shields cover all missed days.

```solidity
// DegenerusQuests.sol:1123-1156
function _questSyncState(PlayerQuestState storage state, address player, uint48 currentDay)
    private
{
    uint24 prevStreak = state.streak;
    uint24 anchorDay = state.lastActiveDay != 0 ? state.lastActiveDay : state.lastCompletedDay;
    if (anchorDay != 0 && currentDay > uint48(anchorDay + 1)) {
        uint32 missedDays = uint32(currentDay - uint48(anchorDay) - 1);
        uint16 shields = questStreakShieldCount[player];
        if (shields != 0) {
            uint32 used = missedDays > uint32(shields) ? uint32(shields) : missedDays;
            questStreakShieldCount[player] = shields - uint16(used);
            if (used != 0) {
                emit QuestStreakShieldUsed(player, uint16(used),
                    questStreakShieldCount[player], currentDay);
            }
            if (missedDays > uint32(shields)) {
                state.streak = 0; // Missed more days than shields available
            }
        } else {
            state.streak = 0; // Full miss (no quest completion) for at least one day
        }
    }
    // ...
    // On new day: reset completionMask and snapshot baseStreak
    if (state.lastSyncDay != currentDay24) {
        state.lastSyncDay = currentDay24;
        state.completionMask = 0;
        state.baseStreak = state.streak; // Snapshot for consistent rewards
    }
}
```

**Reset trigger:** `currentDay > anchorDay + 1` (gap of 2+ between last active day and current day, meaning at least 1 calendar day missed).

### 6c. Shield Protection

- **Storage:** `questStreakShieldCount` mapping per player (`mapping(address => uint16)`)
- **Consumption:** 1 shield per missed day, consumed automatically during `_questSyncState`
- **Partial coverage:** If shields < missedDays, all shields are consumed and streak resets to 0
- **Full coverage:** If shields >= missedDays, shields are decremented and streak is preserved

**Example:** Player has streak=15, shields=3, and misses 2 days. Two shields are consumed (3 -> 1 remaining), streak stays at 15. If they had missed 4 days instead, all 3 shields would be consumed and streak resets to 0.

### 6d. Version Gating

Progress for each slot resets when the quest day or quest version changes:

```solidity
// DegenerusQuests.sol:1168-1179
function _questSyncProgress(PlayerQuestState storage state, uint8 slot,
    uint48 currentDay, uint24 questVersion) private
{
    uint24 currentDay24 = uint24(currentDay);
    if (state.lastProgressDay[slot] != currentDay24
        || state.lastQuestVersion[slot] != questVersion)
    {
        state.lastProgressDay[slot] = currentDay24;
        state.lastQuestVersion[slot] = questVersion;
        state.progress[slot] = 0; // Reset stale progress
    }
}
```

This prevents exploits where progress accumulated on day N could be applied to day N+1's quest.

### 6e. Combo Completion

When a quest slot is completed, the system checks if the paired slot's progress already meets its target. If so, the paired slot is also completed in the same transaction, aggregating rewards.

```solidity
// DegenerusQuests.sol:1465-1504
function _questCompleteWithPair(/* ... */) private returns (/* ... */) {
    (reward, questType, streak, completed) = _questComplete(player, state, slot, quest);
    if (!completed) return (reward, questType, streak, false);

    uint8 otherSlot = slot ^ 1; // XOR to flip 0 <-> 1
    (uint256 extraReward, uint8 extraType, uint32 extraStreak, bool extraCompleted)
        = _maybeCompleteOther(player, state, quests, otherSlot, currentDay, mintPrice);

    if (extraCompleted) {
        reward += extraReward;
        questType = extraType;
        streak = extraStreak;
    }
    return (reward, questType, streak, true);
}
```

> **AGENT NOTE:** Combo completion means a single ETH mint that completes slot 0 can also auto-complete slot 1 if the player had already met the slot 1 target from an earlier action that day (e.g., flipped 2000 BURNIE before minting). The total reward would be 100 + 200 = 300 BURNIE in one transaction.

### 6f. External Streak Manipulation

One external function modifies streaks outside the normal quest flow:

1. **`awardQuestStreakBonus(address player, uint16 amount, uint48 currentDay)`** (line 343): Called by GAME contract. Adds `amount` to player's streak (clamped at uint24 max). Used for bonus streak awards.

## 7. Quest Streak in Activity Score (QRWD-02)

Quest streak is the largest single component of the player activity score, contributing up to 100% (10,000 BPS).

```solidity
// DegenerusGame.sol:2436-2442
// Quest streak: 1% per quest streak, max 100%
(uint32 questStreakRaw, , , ) = questView.playerQuestStates(player);
uint256 questStreak = questStreakRaw > 100 ? 100 : uint256(questStreakRaw);
bonusBps += questStreak * 100;
```

**Formula:**
- **Contribution:** `min(questStreak, 100) * 100 BPS`
- **Rate:** +1% (100 BPS) per streak day
- **Cap:** 100% (10,000 BPS) at streak >= 100 days
- **Read from:** `questView.playerQuestStates(player)` returns the raw streak value

**Comparison to other activity score components:**

| Component | Max Contribution | Cap |
|-----------|-----------------|-----|
| Quest streak | 100% (10,000 BPS) | 100 days |
| Mint streak | 50% (5,000 BPS) | 50 levels |
| Affiliate points | 50% (5,000 BPS) | 50 points |
| Mint count | 25% (2,500 BPS) | 100% participation |
| Deity pass bonus | 80% (8,000 BPS) | Binary (has/hasn't) |

Quest streak at 100% is **double** the next-largest streak component (mint streak at 50%). This makes consistent daily quest engagement the single most impactful player behavior for activity score optimization.

## 9. Agent Simulation Notes

### 9a. Daily Quest Expected Value

- **Guaranteed (slot 0):** 100 BURNIE/day from MINT_ETH quest (requires 1x mintPrice ETH deposit, capped at 0.5 ETH)
- **Bonus (slot 1):** 200 BURNIE/day from random quest (requires slot 0 completion first)
- **Total daily EV:** 300 BURNIE/day assuming both quests are completable
- **Rewards credited as:** Coinflip stakes (not liquid BURNIE)

### 9b. Streak Value Analysis

The quest streak's activity score contribution creates compounding value:

| Streak Days | Activity Score Bonus | Marginal Value |
|------------|---------------------|----------------|
| 1 | +1% | First percentage point |
| 10 | +10% | Meaningful decimator/coinflip boost |
| 25 | +25% | Quarter of maximum |
| 50 | +50% | Half of maximum; equals mint streak cap |
| 100 | +100% | Maximum; double all other streak components combined |

Breaking a 100-day streak costs 100 * 100 = 10,000 BPS of activity score -- the largest single-event activity score loss possible.

### 9c. Shield Optimization

Shields are consumed automatically. The optimal strategy depends on the cost of shields vs. the value of the activity score bonus from the current streak:

- **Shield value = streak * 100 BPS of activity score** (the bonus that would be lost on reset)
- At streak=100 (max), each shield protects 10,000 BPS of activity score
- Shields are consumed in order: first missed day uses first shield, etc.
- If `missedDays > shields`, ALL shields are consumed AND streak resets (shields are not refunded)

### 9d. Slot 1 Quest Probability Summary

For agent quest planning, the most common slot 1 quests:

1. **MINT_BURNIE (41.67% / 50.00%):** Requires 1 BURNIE ticket mint. Cheapest quest to complete.
2. **FLIP (16.67% / 20.00%):** Requires 2,000 BURNIE in coinflip stakes. Moderate cost.
3. **DECIMATOR (16.67% / N/A):** Requires 2,000 BURNIE in decimator burns. Only at specific levels.
4. **LOOTBOX (12.50% / 15.00%):** Requires 2x mintPrice ETH in lootbox purchases. Potentially expensive.
5. **AFFILIATE/DEGENERETTE_ETH/DEGENERETTE_BURNIE (4.17% / 5.00% each):** Low probability, varied cost.

### 9e. Simulation Pseudocode

```python
def simulate_daily_quest(player, day, entropy, dec_allowed):
    """Simulate quest assignment and completion for one day."""
    # Slot 0: always MINT_ETH
    slot0_target = min(mint_price * 1, 0.5e18)  # 1x mintPrice, cap 0.5 ETH

    # Slot 1: weighted random
    weights = {
        MINT_BURNIE: 10, FLIP: 4, LOOTBOX: 3,
        AFFILIATE: 1, DEGENERETTE_ETH: 1, DEGENERETTE_BURNIE: 1,
    }
    if dec_allowed:
        weights[DECIMATOR] = 4
    total = sum(weights.values())  # 24 or 20

    bonus_entropy = (entropy >> 128) | (entropy << 128)
    roll = bonus_entropy % total
    slot1_type = select_from_weights(weights, roll)

    # Completion: slot 0 MUST complete before slot 1
    slot0_reward = 100e18 if complete_slot0(player, slot0_target) else 0
    slot1_reward = 200e18 if (slot0_reward > 0 and complete_slot1(player, slot1_type)) else 0

    # Streak: +1 on first completion of day
    total_reward = slot0_reward + slot1_reward  # credited as coinflip stakes
    return total_reward
```
