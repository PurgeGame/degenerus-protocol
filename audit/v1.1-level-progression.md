## 1. Overview

The Degenerus game uses a 100-level repeating price cycle with intro-tier overrides for the first 10 levels. Levels 0--9 offer discounted ticket prices to bootstrap the game; levels 10--99 run the first full cycle; levels 100+ repeat the same tier structure indefinitely, with each x00 milestone level carrying a premium price and three special economic behaviors.

This document covers:
- **Price curve:** Exact ticket price at every level (Section 2)
- **Level transitions:** When the purchase phase ends and the jackpot phase begins (Section 3)
- **Future pool dynamics:** How time-based skimming transfers ETH from next pool to future pool (Section 4)
- **Whale bundle economics:** 100-level pass ticket-value-to-cost ratios across price tiers (Section 5)
- **Lazy pass economics:** 10-level pass cost tables and bonus ticket mechanics (Section 6)
- **x00 milestones:** Consolidated reference for the three special behaviors at century boundaries (Section 7)

## 2. Complete Price Curve (LEVL-01)

### 2a. Exact Solidity

**Source:** `PriceLookupLib.sol:21-46`

```solidity
function priceForLevel(uint24 targetLevel) internal pure returns (uint256) {
    // Intro tiers (levels 0-9)
    if (targetLevel < 5) return 0.01 ether;
    if (targetLevel < 10) return 0.02 ether;

    // Levels 10-99 (first full cycle without intro tiers)
    if (targetLevel < 30) return 0.04 ether;
    if (targetLevel < 60) return 0.08 ether;
    if (targetLevel < 90) return 0.12 ether;
    if (targetLevel < 100) return 0.16 ether;

    uint256 cycleOffset = targetLevel % 100;

    // Price tiers within the repeating 100-level cycle (levels 100+)
    if (cycleOffset == 0) {
        return 0.24 ether; // Milestone levels: 100, 200, 300...
    } else if (cycleOffset < 30) {
        return 0.04 ether; // Early cycle: x01-x29
    } else if (cycleOffset < 60) {
        return 0.08 ether; // Mid cycle: x30-x59
    } else if (cycleOffset < 90) {
        return 0.12 ether; // Late cycle: x60-x89
    } else {
        return 0.16 ether; // Final cycle: x90-x99
    }
}
```

## 3. Purchase Target and Level Advancement (LEVL-02)

### 3a. Purchase Target Ratchet

Each level has a purchase target stored in `levelPrizePool[level]`. The purchase phase ends when the next pool reaches or exceeds this target.

**Level 0 (bootstrap):**
- `levelPrizePool[0]` is effectively `BOOTSTRAP_PRIZE_POOL = 50 ether`
- **Source:** `DegenerusGameStorage.sol:137-138`

**Normal levels (N >= 1, non-x00):**
- At level transition, the current next pool is snapshot as the target for the next level:
  ```
  levelPrizePool[purchaseLevel] = _getNextPrizePool()
  ```
- **Source:** `DegenerusGameAdvanceModule.sol:269`

**x00 milestone levels:**
- Override: target is set to one-third of the future pool (not the next pool snapshot):
  ```
  levelPrizePool[lvl] = _getFuturePrizePool() / 3
  ```
- **Source:** `DegenerusGameAdvanceModule.sol:429-431` (inside `_endPhase()`)

### 3b. Level Advancement Trigger

**Source:** `DegenerusGameAdvanceModule.sol:243-246`

```solidity
if (_getNextPrizePool() >= levelPrizePool[purchaseLevel - 1]) {
    lastPurchaseDay = true;
    compressedJackpotFlag = (day - purchaseStartDay <= 2);
}
```

The level advances when the current next pool meets or exceeds the previous level's target. If this condition is met within the first 2 days of the purchase phase, the `compressedJackpotFlag` is set, which affects the jackpot phase timing.

> **Pitfall:** The comparison uses `purchaseLevel - 1`, NOT `purchaseLevel`. The target being compared is the PREVIOUS level's stored target. This is because `levelPrizePool[N]` is set at the END of level N's purchase phase (at transition), so during level N+1's purchase phase, the target to beat is `levelPrizePool[N]`.

### 3c. Jackpot Phase Entry

Once `lastPurchaseDay` is set, the next advance triggers the jackpot phase:
1. Pool consolidation: `levelPrizePool[purchaseLevel] = _getNextPrizePool()` (snapshot for next level's target)
2. Time-based future take applied (Section 4)
3. Prize pool consolidation via JackpotModule
4. `levelStartTime = ts` (reset for death clock)
5. Future pool drawdown (Section 4d)

**Source:** `DegenerusGameAdvanceModule.sol:268-294`

## 4. Time-Based Future Take Dynamics (LEVL-02)

### 4a. Base BPS Calculation

The function `_nextToFutureBps(elapsed, lvl)` determines what percentage of the next pool is skimmed into the future pool at each level transition.

**Source:** `DegenerusGameAdvanceModule.sol:834-860`

```solidity
function _nextToFutureBps(uint48 elapsed, uint24 lvl) private pure returns (uint16) {
    uint256 lvlBonus = (uint256(lvl % 100) / 10) * 100;
    uint256 bps;
    if (elapsed <= 1 days) {
        bps = NEXT_TO_FUTURE_BPS_FAST + lvlBonus;
    } else if (elapsed <= 14 days) {
        uint256 elapsedAfterDay = elapsed - 1 days;
        uint256 delta = NEXT_TO_FUTURE_BPS_FAST - NEXT_TO_FUTURE_BPS_MIN;
        bps = NEXT_TO_FUTURE_BPS_FAST - (delta * elapsedAfterDay) / 13 days;
    } else if (elapsed <= 28 days) {
        uint256 elapsedAfterMin = elapsed - 14 days;
        uint256 delta = NEXT_TO_FUTURE_BPS_FAST + lvlBonus - NEXT_TO_FUTURE_BPS_MIN;
        bps = NEXT_TO_FUTURE_BPS_MIN + (delta * elapsedAfterMin) / 14 days;
    } else {
        bps = NEXT_TO_FUTURE_BPS_FAST + lvlBonus +
            ((elapsed - 28 days) / 1 weeks) * NEXT_TO_FUTURE_BPS_WEEK_STEP;
    }
    return uint16(bps > 10_000 ? 10_000 : bps);
}
```

**Time brackets with concrete values:**

The `elapsed` parameter is computed as `reachedAt - (levelStartTime + 11 days)`, meaning the first 11 days of a level are "free" and do not contribute to the elapsed time. If the target is reached before day 11, elapsed is clamped to zero.

**Source for elapsed offset:** `DegenerusGameAdvanceModule.sol:867-868`

| Elapsed Time        | BPS Formula                                              | Range (level 0, lvlBonus=0) | Range (level 50, lvlBonus=500) |
|---------------------|----------------------------------------------------------|------------------------------|-------------------------------|
| <= 1 day            | `3000 + lvlBonus`                                        | 3000                         | 3500                          |
| 1--14 days          | Linear decay: `3000 - (1700 * elapsedAfterDay / 13d)`   | 3000 -> 1300                 | 3000 -> 1300                  |
| 14--28 days         | Linear rise: `1300 + ((3000+lvlBonus-1300) * t / 14d)`  | 1300 -> 3000                 | 1300 -> 3500                  |
| > 28 days           | `3000 + lvlBonus + weeksOver28 * 100`                    | 3000+ (uncapped to 10000)    | 3500+ (uncapped to 10000)     |

Where `lvlBonus = (lvl % 100 / 10) * 100` -- adds +100 BPS per 10 levels within the 100-level cycle:

| Level (mod 100) | lvlBonus (BPS) |
|-----------------|----------------|
| 0--9            | 0              |
| 10--19          | 100            |
| 20--29          | 200            |
| 30--39          | 300            |
| 40--49          | 400            |
| 50--59          | 500            |
| 60--69          | 600            |
| 70--79          | 700            |
| 80--89          | 800            |
| 90--99          | 900            |

### 4b. Additional Modifiers

After computing the base BPS from time brackets, three additional adjustments are applied:

**Source:** `DegenerusGameAdvanceModule.sol:862-927`

**1. x9 level bonus:**
```solidity
if (lvl % 10 == 9) bps += NEXT_TO_FUTURE_BPS_X9_BONUS;  // +200 BPS
```
Levels 9, 19, 29, 39, ... get an extra 200 BPS. Source: `AdvanceModule.sol:871`

**2. Ratio adjust (+/- 200 BPS max):**
Based on the ratio `futurePool / nextPool` compared to a 2:1 baseline:
```solidity
uint256 ratioPct = (futurePoolBefore * 100) / nextPoolBefore;
if (ratioPct < 200) {
    bps += 200 - ratioPct;       // Under-funded future: increase skim
} else {
    uint256 penalty = ratioPct - 200;
    bps = penalty >= bps ? 0 : bps - (penalty > 200 ? 200 : penalty);
}
```
- If future/next ratio < 2:1: BPS increases (up to +200)
- If future/next ratio > 2:1: BPS decreases (up to -200, capped)
- Source: `AdvanceModule.sol:878-884`

**3. Growth adjust (+/- 200 BPS max):**
Based on how much the next pool grew compared to the previous level's target:
```solidity
if (lastPool != 0) {
    if (nextPoolBefore <= lastPool) {
        bps += 200;                          // No growth: max boost
    } else {
        uint256 excessBps = ((nextPoolBefore - lastPool) * 10_000) / lastPool;
        if (excessBps >= 2000) bps -= 200;   // >=20% growth: max penalty
        else if (excessBps <= 1000) bps += 200 - (excessBps / 5);  // <=10%: partial boost
        else bps -= (excessBps - 1000) / 5;  // 10-20%: partial penalty
    }
}
```
- No growth or shrinkage: +200 BPS
- 0--10% growth: partial boost (200 -> 0 BPS)
- 10--20% growth: partial penalty (0 -> -200 BPS)
- 20%+ growth: -200 BPS
- Source: `AdvanceModule.sol:887-902`

### 4c. Variance (Random Adjustment)

After all modifiers, the final take amount has random variance applied:

```solidity
uint256 variance = (take * NEXT_SKIM_VARIANCE_BPS) / 10_000;       // 10% of take
uint256 minVariance = (nextPoolBefore * NEXT_SKIM_VARIANCE_MIN_BPS) / 10_000;  // 10% of pool
if (variance < minVariance) variance = minVariance;
// roll in range [0, 2*variance], center at variance
uint256 roll = rngWord % (variance * 2 + 1);
// Apply: take += (roll - variance), clamped to [0, nextPoolBefore]
```

Source: `AdvanceModule.sol:908-923`

### 4d. Future Pool Drawdown

At each jackpot phase entry, a portion of the future pool is moved back to the next pool to seed the upcoming level:

**Source:** `DegenerusGameAdvanceModule.sol:929-941`

```solidity
function _drawDownFuturePrizePool(uint24 lvl) private {
    uint256 reserved;
    if ((lvl % 100) == 0) {
        reserved = 0;                              // x00 levels: skip draw
    } else {
        reserved = (_getFuturePrizePool() * 15) / 100;  // Normal: 15%
    }
    if (reserved != 0) {
        _setFuturePrizePool(_getFuturePrizePool() - reserved);
        _setNextPrizePool(_getNextPrizePool() + reserved);
    }
}
```

| Level Type | Drawdown | Effect |
|------------|----------|--------|
| Normal     | 15% of futurePool -> nextPool | Seeds next level, reduces future pool |
| x00        | 0% (skip)                     | Future pool preserved for x00 milestone target |

### 6b. Cost Calculation

**Source:** `DegenerusGameWhaleModule.sol:614-621`

```solidity
function _lazyPassCost(uint24 startLevel) private pure returns (uint256 total) {
    for (uint24 i = 0; i < LAZY_PASS_LEVELS; ) {
        total += PriceLookupLib.priceForLevel(startLevel + i);
        unchecked { ++i; }
    }
}
```

The lazy pass cost is the sum of 10 consecutive level prices starting from `startLevel`. For levels 0--2, the payment is a flat 0.24 ETH regardless of this sum, and any excess between 0.24 ETH and the actual cost is converted into bonus tickets.

### 6d. Bonus Tickets at Early Levels

For levels 0--2, the flat 0.24 ETH payment typically exceeds the actual sum-of-prices cost. The excess is converted to bonus tickets:

```solidity
uint256 balance = benefitValue - baseCost;  // 0.24 - actual cost
if (balance != 0) {
    uint256 ticketPrice = PriceLookupLib.priceForLevel(startLevel);
    bonusTickets = uint32((balance * 4) / ticketPrice);
}
```

**Source:** `WhaleModule.sol:375-379`

**Example at level 0 (startLevel = 1):**
- baseCost = 0.18 ETH (sum of prices for levels 1--10)
- balance = 0.24 - 0.18 = 0.06 ETH
- ticketPrice at level 1 = 0.01 ETH
- bonusTickets = (0.06 * 4) / 0.01 = 24 bonus tickets (added to level 1 only)
- Total tickets: 4 x 10 = 40 standard + 24 bonus = 64 tickets across 10 levels

## 7. x00 Milestone Level Special Cases

Century boundary levels (100, 200, 300, ...) have three distinct economic behaviors that differ from all other levels. Agents must account for all three when modeling x00 levels.

### Special 1: Premium Price

**x00 ticket price = 0.24 ETH** (50% more than the next-highest tier of 0.16 ETH)

Source: `PriceLookupLib.sol:35-36`
```solidity
if (cycleOffset == 0) {
    return 0.24 ether; // Milestone levels: 100, 200, 300...
}
```

### Special 2: Purchase Target from Future Pool

**x00 purchase target = futurePool / 3** (not a next pool snapshot from the prior level)

Source: `DegenerusGameAdvanceModule.sol:429-431` (inside `_endPhase()`)
```solidity
if (lvl % 100 == 0) {
    levelPrizePool[lvl] = _getFuturePrizePool() / 3;
}
```

This means x00 level targets scale with the accumulated future pool, not the prior level's next pool. Since the future pool accumulates across all levels via time-based skimming (Section 4), x00 targets can be significantly larger than adjacent levels.

### Special 3: Zero Future Pool Drawdown

**x00 drawdown = 0%** (the standard 15% future -> next transfer is skipped)

Source: `DegenerusGameAdvanceModule.sol:931-932`
```solidity
if ((lvl % 100) == 0) {
    reserved = 0; // Skip extra future->next move on x00 levels
}
```

This preserves the future pool balance entering the x00 level, which matters because the x00 target is set to futurePool / 3. If the 15% draw occurred first, the target would be lower.

> **Why this matters for agents:** x00 levels have fundamentally different economics than all other levels. The purchase target is tied to the future pool (which grows monotonically via skimming), the ticket price is the highest in the cycle, and the future pool is not drawn down. These three factors combine to make x00 levels the longest and most expensive levels to complete, with the largest jackpot pools.

## 9. Agent Simulation Notes

### 9a. Price Lookup (Any Level)

```
function priceForLevel(level):
    if level < 5: return 0.01
    if level < 10: return 0.02
    if level < 30: return 0.04
    if level < 60: return 0.08
    if level < 90: return 0.12
    if level < 100: return 0.16
    offset = level % 100
    if offset == 0: return 0.24
    if offset < 30: return 0.04
    if offset < 60: return 0.08
    if offset < 90: return 0.12
    return 0.16
```

### 9c. Time-Based Future Take BPS

```
function nextToFutureBps(elapsedSeconds, level):
    lvlBonus = (level % 100) // 10 * 100
    if elapsedSeconds <= 86400:  # 1 day
        bps = 3000 + lvlBonus
    elif elapsedSeconds <= 1209600:  # 14 days
        after = elapsedSeconds - 86400
        bps = 3000 - (1700 * after) // (13 * 86400)
    elif elapsedSeconds <= 2419200:  # 28 days
        after = elapsedSeconds - 1209600
        delta = 3000 + lvlBonus - 1300
        bps = 1300 + (delta * after) // (14 * 86400)
    else:
        weeks_past = (elapsedSeconds - 2419200) // 604800
        bps = 3000 + lvlBonus + weeks_past * 100
    return min(bps, 10000)
```

Note: The actual `elapsed` parameter to `_nextToFutureBps` is `reachedAt - (levelStartTime + 11 days)`, clamped to zero if the target is reached within the first 11 days. This means the first 11 days of every level have no impact on the future take BPS.

### 9d. Lazy Pass Cost

```
function lazyPassCost(startLevel):
    total = 0
    for i in range(10):
        total += priceForLevel(startLevel + i)
    return total
```

### 9e. Whale Bundle Ticket Face Value

```
function whaleBundleTicketValue(purchaseLevel, quantity=1):
    startLevel = max(purchaseLevel, 1)
    total = 0
    for i in range(100):
        lvl = startLevel + i
        tickets = 40 * quantity if (lvl >= purchaseLevel and lvl <= 10) else 2 * quantity
        total += tickets * priceForLevel(lvl)
    return total
```
