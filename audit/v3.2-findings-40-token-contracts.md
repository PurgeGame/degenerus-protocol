### New Findings

#### CMT-059: _burn @dev CEI note lists only 2 of 4 callers

- **What:** The `_burn` function's `@dev` at line 442 says `SECURITY: Burns BEFORE any external calls (CEI pattern) in burnCoin/decimatorBurn.` The `_burn` function is called from 4 sites: `burnForCoinflip` (line 476), `burnCoin` (line 823), `decimatorBurn` (line 857), and `terminalDecimatorBurn` (line 947). The CEI note lists only 2 of 4 callers. While `burnForCoinflip` is a simple pass-through with no external calls after _burn (so CEI is trivially satisfied), `terminalDecimatorBurn` does call `degenerusGame.recordTerminalDecBurn` after `_burn` -- making it a relevant CEI site that should be documented.
- **Where:** BurnieCoin.sol:442
- **Why:** The v3.1 fix (CMT-051) replaced "depositCoinflip" with "burnCoin" but did not add the other two callers. A warden verifying the CEI claim would trace _burn call sites and find undocumented ones -- potentially leading to a QA finding about incomplete security annotations.
- **Suggestion:** Change to: `SECURITY: Burns BEFORE any external calls (CEI pattern) in burnForCoinflip/burnCoin/decimatorBurn/terminalDecimatorBurn.`
- **Category:** comment-inaccuracy
- **Severity:** INFO

#### CMT-060: VaultAllowanceSpent event @notice inaccurate for vaultMintTo emission path

- **What:** The `VaultAllowanceSpent` event NatSpec at line 105 says `@notice Emitted when the vault spends from its mint allowance without minting tokens.` and `@param spender The vault address.` The event is emitted in two locations: (1) `_burn` at line 454 as `VaultAllowanceSpent(from, amount)` where from=VAULT and no tokens are minted -- NatSpec accurate for this path. (2) `vaultMintTo` at line 664 as `VaultAllowanceSpent(address(this), amount)` -- here tokens ARE minted (line 665 emits Transfer from address(0)), and `address(this)` is BurnieCoin, not the vault. The @notice "without minting tokens" is false for path (2), and @param "The vault address" is false for path (2) since spender is BurnieCoin's own address.
- **Where:** BurnieCoin.sol:105-108
- **Why:** A warden or event indexer consuming VaultAllowanceSpent events would see inconsistent spender values (sometimes VAULT, sometimes BurnieCoin) and the "without minting tokens" claim contradicts the vaultMintTo path where tokens are actively minted. This creates confusion about the event's semantics -- does it mean "allowance reduced" (the common factor) or "allowance spent without minting" (only true for _burn)?
- **Suggestion:** Change @notice to: `Emitted when the vault's mint allowance is reduced.` Change @param spender to: `The address that triggered the allowance reduction (VAULT for burns, BurnieCoin for vaultMintTo).`
- **Category:** comment-inaccuracy
- **Severity:** INFO
