## 1. Overview

The affiliate system is a 3-tier referral reward mechanism that pays affiliates in BURNIE (credited as coinflip stakes by default). When a referred player makes a purchase, the affiliate earns a percentage of the purchase amount as BURNIE, with smaller shares flowing to up to two upline tiers.

Key characteristics:
- **3-tier chain:** Direct affiliate receives base share; upline tier 1 gets 20% of scaled amount; upline tier 2 gets 4% of scaled amount
- **ETH-type-dependent rates:** Fresh ETH pays 25% (levels 0--3) or 20% (levels 4+); recycled ETH pays 5% at all levels
- **Per-referrer commission cap:** Maximum 0.5 ETH BURNIE per sender-affiliate pair per level
- **Lootbox activity taper:** High-activity buyers see fresh ETH payouts reduced to as low as 25%
- **Weighted random lottery:** When multiple recipients exist, a single winner receives the combined total (EV-preserving, high variance)
- **Kickback:** Configurable 0--25% discount returned to the referred player
- **Leaderboard tracking:** Full untapered, pre-kickback amount always used for scoring
- **Endgame DGNRS claims:** Per-level proportional share of the affiliate DGNRS pool for qualified affiliates

**Primary source:** `contracts/DegenerusAffiliate.sol`

## 2. Reward Rates by ETH Type and Level (AFFL-01)

### Fresh vs Recycled ETH

**Fresh ETH** is direct ETH payment from a player's wallet for purchases (tickets, lootboxes, whale bundles, deity passes). **Recycled ETH** is BURNIE-converted value -- when a player spends BURNIE (which was previously minted from ETH), the underlying ETH value is "recycled" through the system. The `isFreshEth` flag is set by the calling contract (DegenerusGame or BurnieCoin).

### Rate Table

| ETH Type | Game Levels | Rate (BPS) | Effective % |
|----------|-------------|------------|-------------|
| Fresh ETH | 0--3 | 2500 | 25% |
| Fresh ETH | 4+ | 2000 | 20% |
| Recycled ETH | All levels | 500 | 5% |

### Solidity: scaledAmount Computation

**Source:** `DegenerusAffiliate.sol` lines 557--567

```solidity
uint256 rewardScaleBps;
if (isFreshEth) {
    // Fresh ETH: 25% for first 4 levels (0-3), 20% for levels 4+
    rewardScaleBps = lvl <= 3
        ? REWARD_SCALE_FRESH_L1_3_BPS   // 2500
        : REWARD_SCALE_FRESH_L4P_BPS;   // 2000
} else {
    // Recycled ETH: 5%
    rewardScaleBps = REWARD_SCALE_RECYCLED_BPS;  // 500
}
uint256 scaledAmount = (amount * rewardScaleBps) / BPS_DENOMINATOR;
```

**Example:** Player purchases 1 ETH worth at level 2 (fresh ETH):
- `scaledAmount = (1 ETH * 2500) / 10000 = 0.25 ETH BURNIE`

**Example:** Player purchases 1 ETH worth at level 5 (fresh ETH):
- `scaledAmount = (1 ETH * 2000) / 10000 = 0.20 ETH BURNIE`

**Example:** Player re-purchases with 1 ETH of recycled BURNIE at any level:
- `scaledAmount = (1 ETH * 500) / 10000 = 0.05 ETH BURNIE`

## 3. Per-Referrer Commission Cap (AFFL-01)

Each sender-affiliate pair is capped at **0.5 ETH BURNIE** of commission per level. This prevents a single whale from dominating an affiliate's leaderboard score.

**Constant:** `MAX_COMMISSION_PER_REFERRER_PER_LEVEL = 0.5 ether`
**Source:** `DegenerusAffiliate.sol` line 207

### Solidity: Cap Enforcement

**Source:** `DegenerusAffiliate.sol` lines 578--590

```solidity
{
    uint256 alreadyEarned = affiliateCommissionFromSender[lvl][affiliateAddr][sender];
    if (alreadyEarned >= MAX_COMMISSION_PER_REFERRER_PER_LEVEL) {
        // Cap fully reached - no more commission from this sender this level.
        emit Affiliate(amount, storedCode, sender);
        return 0;
    }
    uint256 remainingCap = MAX_COMMISSION_PER_REFERRER_PER_LEVEL - alreadyEarned;
    if (scaledAmount > remainingCap) {
        scaledAmount = remainingCap;
    }
    affiliateCommissionFromSender[lvl][affiliateAddr][sender] = alreadyEarned + scaledAmount;
}
```

### Worked Example

**Scenario:** Affiliate A has 3 referrals (B, C, D) each spending 1 ETH at level 5 (fresh ETH, 20% rate).

| Referral | Spend | scaledAmount | Cap Used | Cap Remaining | Actual Payout |
|----------|-------|-------------|----------|---------------|---------------|
| B (1st purchase) | 1.0 ETH | 0.20 ETH | 0.20 ETH | 0.30 ETH | 0.20 ETH |
| C (1st purchase) | 1.0 ETH | 0.20 ETH | 0.20 ETH | 0.30 ETH | 0.20 ETH |
| D (1st purchase) | 1.0 ETH | 0.20 ETH | 0.20 ETH | 0.30 ETH | 0.20 ETH |

Each sender-affiliate pair has its own cap. B, C, D each have 0.30 ETH cap remaining for affiliate A at this level. Total affiliate earnings: 0.60 ETH BURNIE.

**Cap hit scenario:** If referral B spends 2.5 ETH total at level 5:
- Purchase 1 (1.0 ETH): scaledAmount = 0.20, cap used = 0.20
- Purchase 2 (1.0 ETH): scaledAmount = 0.20, cap used = 0.40
- Purchase 3 (0.5 ETH): scaledAmount = 0.10, cap used = 0.50 (full)
- Purchase 4 (any): returns 0 (cap exhausted for B -> A at this level)

At levels 0--3 (25% rate), the cap is hit after 2.0 ETH spend per referral: `2.0 * 0.25 = 0.5 ETH`.

## 4. 3-Tier Distribution Chain (AFFL-01)

After computing `scaledAmount` and applying the commission cap, rewards flow through a 3-tier chain:

| Tier | Recipient | Base Share | Fraction of scaledAmount |
|------|-----------|------------|--------------------------|
| Direct affiliate | Code owner | (scaledAmount - kickback) + quest bonus | ~75--100% after kickback |
| Upline tier 1 | Affiliate's referrer | 20% of scaledAmount + quest bonus | 20% |
| Upline tier 2 | Upline 1's referrer | 4% of scaledAmount + quest bonus | 4% |

### Solidity: Tier Computation

**Source:** `DegenerusAffiliate.sol` lines 628--671

```solidity
// Direct affiliate: (scaledAmount - kickback) + quest bonus
uint256 questReward = coin.affiliateQuestReward(affiliateAddr, affiliateShareBase);
uint256 totalFlipAward = affiliateShareBase + questReward;
players[cursor] = affiliateAddr;
amounts[cursor] = totalFlipAward;

// Upline tier 1: 20% of scaledAmount + quest bonus
address upline = _referrerAddress(affiliateAddr);
if (upline != address(0)) {
    uint256 baseBonus = scaledAmount / 5; // 20%
    uint256 questRewardUpline = coin.affiliateQuestReward(upline, baseBonus);
    uint256 totalUpline = baseBonus + questRewardUpline;
    players[cursor] = upline;
    amounts[cursor] = totalUpline;

    // Upline tier 2: 4% of scaledAmount (20% of tier 1)
    address upline2 = _referrerAddress(upline);
    if (upline2 != address(0)) {
        uint256 bonus2 = scaledAmount / 25; // 20% of 20% = 4%
        uint256 questReward2 = coin.affiliateQuestReward(upline2, bonus2);
        uint256 totalUpline2 = bonus2 + questReward2;
        players[cursor] = upline2;
        amounts[cursor] = totalUpline2;
    }
}
```

> **CRITICAL PITFALL: Weighted Random Lottery.** When multiple recipients exist (2 or 3 tiers), rewards are **NOT** split proportionally among them. Instead, a **single winner** is selected via weighted random roll and receives the **combined total** of all tier amounts. The probability of each recipient winning is proportional to their share. This preserves expected value (EV) but introduces high variance on individual payouts.

### Solidity: Weighted Random Winner Selection

**Source:** `DegenerusAffiliate.sol` lines 898--930

```solidity
function _rollWeightedAffiliateWinner(
    address[3] memory players,
    uint256[3] memory amounts,
    uint256 count,
    uint256 totalAmount,
    address sender,
    bytes32 storedCode
) private view returns (address winner) {
    uint48 currentDay = GameTimeLib.currentDayIndex();

    uint256 entropy = uint256(
        keccak256(
            abi.encodePacked(
                AFFILIATE_ROLL_TAG,       // keccak256("affiliate-payout-roll-v1")
                currentDay,
                sender,
                storedCode
            )
        )
    );
    uint256 roll = entropy % totalAmount;

    uint256 running;
    for (uint256 i; i < count; ) {
        running += amounts[i];
        if (roll < running) return players[i];
        unchecked { ++i; }
    }
    return players[0]; // Unreachable fallback
}
```

**Determinism note:** The entropy uses `currentDay`, `sender`, and `storedCode` -- no block-level randomness. This means for a given day, sender, and affiliate code, the lottery outcome is deterministic. Multiple purchases from the same sender on the same day with the same code will always select the same winner.

### EV-Equivalence Proof

For a 1 ETH purchase at level 5 (fresh ETH, 20%), 0% kickback, all 3 tiers present:

| Tier | Base Amount | Quest Bonus (assume 0) | Total | Win Probability | EV |
|------|------------|------------------------|-------|-----------------|-----|
| Direct | 0.200 ETH | 0 | 0.200 ETH | 0.200 / 0.248 = 80.65% | 0.200 ETH |
| Upline 1 | 0.040 ETH | 0 | 0.040 ETH | 0.040 / 0.248 = 16.13% | 0.040 ETH |
| Upline 2 | 0.008 ETH | 0 | 0.008 ETH | 0.008 / 0.248 = 3.23% | 0.008 ETH |
| **Total** | | | **0.248 ETH** | **100%** | **0.248 ETH** |

The combined pot is 0.248 ETH BURNIE. Each tier's probability of winning equals their share divided by the total. Therefore:
- EV(direct) = P(win) * totalPot = (0.200/0.248) * 0.248 = **0.200 ETH**
- EV(upline1) = (0.040/0.248) * 0.248 = **0.040 ETH**
- EV(upline2) = (0.008/0.248) * 0.248 = **0.008 ETH**

Each tier's expected value exactly equals what a deterministic split would pay. QED.

**Variance impact:** On any single purchase, one tier receives 0.248 ETH and the other two receive 0. Over many referral purchases, payouts converge to the deterministic split values.

### Single-Recipient Bypass

When only 1 recipient exists (no uplines), the lottery is skipped entirely -- the direct affiliate receives their full reward directly via `_routeAffiliateReward`:

```solidity
if (cursor == 1) {
    _routeAffiliateReward(players[0], amounts[0]);
}
```

## 5. Lootbox Activity Taper (AFFL-02)

High-activity lootbox buyers see their fresh ETH affiliate payouts reduced. This prevents lootbox farming from generating excessive affiliate rewards.

**Applies to:** Fresh ETH payouts only. Recycled ETH is never tapered.
**Leaderboard tracking:** Always uses the **full untapered** `scaledAmount` (taper is applied after leaderboard update).

### Taper Breakpoints

| Activity Score | Taper Factor | Payout % |
|---------------|-------------|----------|
| 0 -- 9,999 | 10000 BPS (no taper) | 100% |
| 10,000 | 10000 BPS | 100% |
| 13,875 | 8125 BPS | 81.25% |
| 17,750 | 6250 BPS | 62.5% |
| 21,625 | 4375 BPS | 43.75% |
| 25,500+ | 2500 BPS (floor) | 25% |

### Taper Formula

```
taperFactor = max(25%, 100% - 75% * (score - 10000) / (25500 - 10000))
            = max(2500, 10000 - 7500 * (score - 10000) / 15500) [in BPS]
```

### Solidity: Taper Calculation

**Source:** `DegenerusAffiliate.sol` lines 887--895

```solidity
function _applyLootboxTaper(uint256 amt, uint16 score) private pure returns (uint256) {
    if (score >= LOOTBOX_TAPER_END_SCORE) {         // 25500
        return (amt * LOOTBOX_TAPER_MIN_BPS) / BPS_DENOMINATOR;  // 25%
    }
    uint256 excess = uint256(score) - LOOTBOX_TAPER_START_SCORE;  // score - 10000
    uint256 range = uint256(LOOTBOX_TAPER_END_SCORE) - LOOTBOX_TAPER_START_SCORE;  // 15500
    uint256 reductionBps = (BPS_DENOMINATOR - LOOTBOX_TAPER_MIN_BPS) * excess / range;
        // = 7500 * excess / 15500
    return (amt * (BPS_DENOMINATOR - reductionBps)) / BPS_DENOMINATOR;
}
```

**Taper is applied in `payAffiliate` at line 607--609:**

```solidity
// Taper payout for high-activity lootbox buyers
// (leaderboard already recorded full amount).
if (lootboxActivityScore >= LOOTBOX_TAPER_START_SCORE) {
    scaledAmount = _applyLootboxTaper(scaledAmount, lootboxActivityScore);
}
```

### Worked Example

A player with activity score 17,750 refers a purchase of 1 ETH at level 5 (fresh ETH):

1. `scaledAmount = 1 ETH * 2000 / 10000 = 0.20 ETH` (before taper)
2. Leaderboard records full 0.20 ETH
3. Taper: `excess = 17750 - 10000 = 7750`, `range = 15500`
4. `reductionBps = 7500 * 7750 / 15500 = 3750`
5. `tapered = 0.20 * (10000 - 3750) / 10000 = 0.20 * 0.625 = 0.125 ETH`

The affiliate's leaderboard tracks 0.20 ETH but the actual payout is 0.125 ETH BURNIE.

## 6. Reward Routing (AFFL-01)

All affiliate rewards are routed exclusively through `coin.creditFlip()`, crediting the full amount as BURNIE coinflip stakes. There is no configurable payout mode.

### Solidity: Reward Routing

**Source:** `DegenerusAffiliate.sol`

```solidity
function _routeAffiliateReward(address player, uint256 amount) private {
    if (player == address(0) || amount == 0) return;
    coin.creditFlip(player, amount);
}
```

All tiers (direct affiliate, upline 1, upline 2) and kickback returns flow through `creditFlip`. The credited BURNIE is locked into the recipient's next coinflip wager.

## 7. Kickback (AFFL-02)

Kickback is a buyer-facing discount set by the affiliate. A percentage of the `scaledAmount` (post-taper) is returned to the referred player by the calling contract.

**Range:** 0--25% (enforced by `MAX_KICKBACK_PCT = 25`)
**Source:** `DegenerusAffiliate.sol` line 197

### Solidity: Kickback Computation

**Source:** `DegenerusAffiliate.sol` lines 612--621

```solidity
uint256 affiliateShareBase;
uint256 kickbackShare;
if (kickbackPct == 0) {
    affiliateShareBase = scaledAmount;
} else {
    kickbackShare = (scaledAmount * uint256(kickbackPct)) / 100;
    affiliateShareBase = scaledAmount - kickbackShare;
}
playerKickback = kickbackShare;
```

The `playerKickback` is returned to the calling contract, which credits it to the buyer as FLIP.

### Key Properties

1. **Does NOT reduce leaderboard score.** Leaderboard tracking uses the full `scaledAmount` before kickback deduction (line 593--594: `newTotal = earned[affiliateAddr] + scaledAmount`).
2. **Reduces affiliate payout.** The direct affiliate receives `scaledAmount - kickback` (before quest bonus).
3. **Upline tiers are unaffected.** Upline shares are computed from `scaledAmount`, not `affiliateShareBase`. Kickback only reduces the direct affiliate's share.

### Worked Example

Affiliate sets 10% kickback. Player purchases 1 ETH at level 2 (fresh ETH, 25%):

| Step | Value |
|------|-------|
| scaledAmount | 0.25 ETH |
| Leaderboard records | 0.25 ETH (full amount) |
| kickbackShare | 0.25 * 10 / 100 = 0.025 ETH |
| affiliateShareBase | 0.25 - 0.025 = 0.225 ETH |
| playerKickback (returned to buyer) | 0.025 ETH |
| Upline 1 share (from scaledAmount) | 0.25 / 5 = 0.05 ETH |

## 8. Affiliate Bonus Points (AFFL-02)

Affiliate bonus points contribute to the player's lootbox mint trait roll. Points are based on the affiliate's earnings over the previous 5 levels.

### Formula

```
sum = affiliateCoinEarned[currLevel-1][player]
    + affiliateCoinEarned[currLevel-2][player]
    + ... + affiliateCoinEarned[currLevel-5][player]

points = min(sum / 1 ether, 50)
```

- **1 point per 1 ETH** of summed affiliate earnings across previous 5 levels
- **Cap:** 50 points (AFFILIATE_BONUS_MAX = 50)
- **Levels below offset are skipped** (e.g., at level 3, only levels 2 and 1 are summed)

### Solidity: Bonus Calculation

**Source:** `DegenerusAffiliate.sol` lines 743--759

```solidity
function affiliateBonusPointsBest(uint24 currLevel, address player)
    external view returns (uint256 points)
{
    if (player == address(0) || currLevel == 0) return 0;
    uint256 sum;
    unchecked {
        for (uint8 offset = 1; offset <= 5; ) {
            if (currLevel <= offset) break;
            uint24 lvl = currLevel - offset;
            sum += affiliateCoinEarned[lvl][player];
            ++offset;
        }
    }
    if (sum == 0) return 0;
    uint256 ethUnit = 1 ether;
    points = sum / ethUnit;
    return points > AFFILIATE_BONUS_MAX ? AFFILIATE_BONUS_MAX : points;
}
```

### Worked Example

At level 7, affiliate earned: level 6 = 12 ETH, level 5 = 8 ETH, level 4 = 5 ETH, level 3 = 3 ETH, level 2 = 2 ETH.

- Sum = 12 + 8 + 5 + 3 + 2 = 30 ETH
- Points = min(30, 50) = **30 bonus points**

At 50+ ETH summed, points cap at 50.

## 9. Top Affiliate Endgame DGNRS Rewards (AFFL-03)

Qualified affiliates can claim DGNRS tokens proportional to their affiliate score each level. This is a per-level reward from the affiliate DGNRS pool.

### Eligibility

1. Current level must be > 1 (claims are for `prevLevel = currLevel - 1`)
2. Player has not already claimed for this level (`affiliateDgnrsClaimedBy[prevLevel][player]` must be false)
3. **Either:** `affiliateScore(prevLevel, player) >= 10 ETH` (AFFILIATE_DGNRS_MIN_SCORE), **or** player holds a deity pass

### Claim Formula

```
denominator = levelPrizePool[prevLevel]
if denominator == 0:
    denominator = BOOTSTRAP_PRIZE_POOL    // 50 ETH fallback

poolBalance = dgnrs.poolBalance(Pool.Affiliate)
levelShare  = (poolBalance * 500) / 10000    // 5% of current pool
reward      = (levelShare * score) / denominator
```

**Pool decay:** Each claim transfers DGNRS from the affiliate pool, reducing `poolBalance`. Later claimants in the same level get proportionally less because the pool shrinks. However, since `levelShare` is 5% of the current pool, early claimants within a level have an advantage.

### Deity Pass Bonus

If the claimant holds a deity pass and has a non-zero score:

```
bonus = (score * 2000) / 10000                      // 20% of affiliate score as BURNIE
cap   = (5 ETH * PRICE_COIN_UNIT) / price           // 5 ETH worth of BURNIE at current price
                                                     // PRICE_COIN_UNIT = 1000 ether
if bonus > cap:
    bonus = cap
creditFlip(player, bonus)                            // Credited as FLIP stakes
```

### Solidity: claimAffiliateDgnrs

**Source:** `DegenerusGame.sol` lines 1410--1461

```solidity
function claimAffiliateDgnrs(address player) external {
    player = _resolvePlayer(player);

    uint24 currLevel = level;
    if (currLevel <= 1) revert E();
    uint24 prevLevel = currLevel - 1;

    if (affiliateDgnrsClaimedBy[prevLevel][player]) revert E();

    uint256 score = affiliate.affiliateScore(prevLevel, player);
    bool hasDeityPass = deityPassCount[player] != 0;
    if (!hasDeityPass && score < AFFILIATE_DGNRS_MIN_SCORE) revert E();

    uint256 denominator = levelPrizePool[prevLevel];
    if (denominator == 0) {
        denominator = BOOTSTRAP_PRIZE_POOL;    // 50 ETH
    }

    uint256 poolBalance = dgnrs.poolBalance(IDegenerusStonk.Pool.Affiliate);
    uint256 levelShare = (poolBalance * AFFILIATE_DGNRS_LEVEL_BPS) / 10_000;
    if (levelShare == 0) revert E();
    uint256 reward = (levelShare * score) / denominator;
    if (reward == 0) revert E();

    uint256 paid = dgnrs.transferFromPool(
        IDegenerusStonk.Pool.Affiliate,
        player,
        reward
    );
    if (paid == 0) revert E();

    if (hasDeityPass && score != 0) {
        uint256 bonus = (score * AFFILIATE_DGNRS_DEITY_BONUS_BPS) / 10_000;
        uint256 cap = (AFFILIATE_DGNRS_DEITY_BONUS_CAP_ETH * PRICE_COIN_UNIT) / price;
        if (bonus > cap) {
            bonus = cap;
        }
        if (bonus != 0) {
            coin.creditFlip(player, bonus);
        }
    }

    affiliateDgnrsClaimedBy[prevLevel][player] = true;
    emit AffiliateDgnrsClaimed(player, prevLevel, msg.sender, score, paid);
}
```

### Worked Example

**Setup:** Level just transitioned to 6. Affiliate has score of 15 ETH at level 5. `levelPrizePool[5] = 200 ETH`. Affiliate DGNRS pool has 300B DGNRS. No deity pass.

1. `denominator = 200 ETH`
2. `levelShare = (300B * 500) / 10000 = 15B DGNRS` (5% of pool)
3. `reward = (15B * 15 ETH) / 200 ETH = 1.125B DGNRS`

**With deity pass bonus (same scenario):** price = 500 (0.5 ETH per BURNIE equivalent):

1. `bonus = (15 ETH * 2000) / 10000 = 3.0 ETH BURNIE`
2. `cap = (5 ETH * 1000 ETH) / 500 = 10 ETH BURNIE`
3. `bonus (3.0) < cap (10.0)`, so bonus = 3.0 ETH BURNIE credited as FLIP

## 11. Agent Simulation Notes

### Affiliate EV Pseudocode

```python
def affiliate_ev_per_eth(level, is_fresh, kickback_pct, activity_score, has_upline1, has_upline2):
    """Compute expected BURNIE reward per 1 ETH referred."""

    # Step 1: Base rate
    if is_fresh:
        rate = 0.25 if level <= 3 else 0.20
    else:
        rate = 0.05

    scaled = 1.0 * rate  # scaledAmount per 1 ETH

    # Step 2: Commission cap check
    # scaled is clamped to (0.5 - alreadyEarned) per sender per level
    # For first purchase from a sender: no cap hit unless scaled > 0.5

    # Step 3: Taper (fresh ETH only)
    if is_fresh and activity_score >= 10000:
        if activity_score >= 25500:
            taper = 0.25
        else:
            taper = 1.0 - 0.75 * (activity_score - 10000) / 15500
        payout_scaled = scaled * taper
    else:
        payout_scaled = scaled

    # Step 4: Kickback deduction
    kickback = payout_scaled * kickback_pct / 100
    affiliate_share = payout_scaled - kickback

    # Step 5: Upline EV (lottery preserves these)
    upline1_ev = scaled * 0.20 if has_upline1 else 0
    upline2_ev = scaled * 0.04 if has_upline2 else 0

    # Step 6: Total BURNIE created = affiliate + uplines + kickback to buyer
    total_burnie_minted = affiliate_share + upline1_ev + upline2_ev + kickback

    return {
        "direct_affiliate_ev": affiliate_share,
        "upline1_ev": upline1_ev,
        "upline2_ev": upline2_ev,
        "buyer_kickback": kickback,
        "total_burnie_created": total_burnie_minted,
        "leaderboard_score": scaled  # always full, untapered
    }
```

### DGNRS Claim Projections

```python
def dgnrs_claim(pool_balance, score, level_prize_pool, has_deity, price):
    """Compute DGNRS reward for affiliate claim."""
    denom = level_prize_pool if level_prize_pool > 0 else 50e18  # BOOTSTRAP
    level_share = pool_balance * 500 // 10000  # 5% of pool
    reward = level_share * score // denom

    deity_bonus = 0
    if has_deity and score > 0:
        bonus = score * 2000 // 10000  # 20% of score
        cap = 5e18 * 1000e18 // price  # 5 ETH in BURNIE
        deity_bonus = min(bonus, cap)

    return reward, deity_bonus
```

### Lottery Variance Note

For agents modeling affiliate income: over N referral events with 3-tier chains, the standard deviation of affiliate earnings is approximately:

```
sigma = sqrt(N * p * (1-p)) * totalPot_per_event
```

where `p = direct_share / total_pot` (typically ~80%). After ~25 events, the 95% confidence interval narrows to within +/-20% of the deterministic EV. For low-volume affiliates, the lottery introduces meaningful income volatility.
