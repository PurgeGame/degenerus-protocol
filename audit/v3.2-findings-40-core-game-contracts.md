#### NEW-001: IDegenerusGameAdmin interface @dev still mentions "liveness"

- **What:** The `IDegenerusGameAdmin` interface `@dev` comment at line 66 reads `"Game contract admin interface (VRF + liquidity + liveness)"`. The "liveness" concept referred to the removed death clock / jackpotPhase interaction. No remaining interface member implements or relates to liveness checking. The only liveness-adjacent function is `lastVrfProcessed()` which supports governance stall detection, but that falls under "VRF" not "liveness" in the original terminology.
- **Where:** DegenerusAdmin.sol:66
- **Why:** A warden reading the interface would expect to find a liveness-related function (such as the removed `jackpotPhase()`). When none is found, they may file a finding about incomplete interface design or missing functionality. The v3.1 DRIFT-001 fix removed `jackpotPhase()` but did not update this comment.
- **Suggestion:** Change line 66 to: `/// @dev Game contract admin interface (VRF + liquidity management).`
- **Category:** comment-inaccuracy
- **Severity:** INFO
