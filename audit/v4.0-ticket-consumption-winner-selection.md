## Section 2: ticketQueue Reads for Winner Selection (TCON-01)

### Methodology

Full grep of all `ticketQueue[` and `_tqFarFutureKey|_tqReadKey|_tqWriteKey` references across `contracts/` directory. Each result classified by purpose and documented below.

### 2.1 Winner Selection Reads

| # | Function | File:Line | Key Used | Purpose | Jackpot Type |
|---|----------|-----------|----------|---------|--------------|
| 1 | `_awardFarFutureCoinJackpot` | JM:2543 | `_tqFarFutureKey(candidate)` | Select winner from FF ticket queue for far-future BURNIE coin jackpot | Far-Future Coin |
| 2 | `sampleFarFutureTickets` | DG:2681 | `_tqWriteKey(candidate)` | Sample ticket holders for BAF scatter far-future draws (view function, indirect) | BAF (external contract) |

#### Detail: _awardFarFutureCoinJackpot (JM:2543)

```solidity
// JM:2543 -- direct winner selection from FF key
address[] storage queue = ticketQueue[_tqFarFutureKey(candidate)];
uint256 len = queue.length;
if (len != 0) {
    address winner = queue[(entropy >> 32) % len];  // JM:2547
```

- **Key space:** `_tqFarFutureKey(candidate)` = `candidate | TICKET_FAR_FUTURE_BIT` (bit 22 set)
- **Correctness:** CORRECT -- reads from the far-future key space where lootbox-origin far-future tickets are stored
- **Winner index formula:** `(entropy >> 32) % queue.length` -- no deity virtual entries, direct array index
- **Jackpot type:** Far-future BURNIE coin jackpot (25% of daily coin budget via FAR_FUTURE_COIN_BPS = 2500)
- **Called from:** `payDailyJackpotCoinAndTickets` at JM:707, `payDailyCoinJackpot` at JM:2369

#### Detail: sampleFarFutureTickets (DG:2681)

```solidity
// DG:2681 -- view function used by BAF jackpot in DegenerusJackpots.sol
address[] storage queue = ticketQueue[_tqWriteKey(candidate)];
uint256 len = queue.length;
if (len != 0) {
    uint256 idx = (word >> 32) % len;
    address winner = queue[idx];  // DG:2685
```

- **Key space:** `_tqWriteKey(candidate)` -- reads from WRITE buffer, not far-future key
- **Correctness:** KNOWN ISSUE (DSC-02 from Phase 81) -- should read `_tqFarFutureKey(candidate)` for far-future tickets. Currently reads from the active write buffer which may be empty or contain different entries than the FF key.
- **Winner index formula:** `(word >> 32) % queue.length` -- similar to FF coin jackpot
- **Jackpot type:** BAF (Big-Ass-Fucking) jackpot -- external contract DegenerusJackpots.sol calls this view function
- **Impact:** INFO severity -- BAF jackpot is a separate contract that uses the view function as a sampling heuristic for scatter draws, not a definitive winner selection. The BAF contract selects from the returned sample, so an empty/wrong pool just means no far-future BAF winners (graceful degradation).

**Total ticketQueue read sites found:** 20 across 5 contracts
- **Winner selection:** 2 (JM:2543 direct, DG:2681 indirect view)
- **Processing:** 9 (JM:1891/1903/1944, MM:304/309/327/330/437/440)

**Key finding:** Only `_awardFarFutureCoinJackpot` directly selects winners from ticketQueue. All other jackpot types use traitBurnTicket for winner selection. The BAF jackpot uses ticketQueue indirectly via the `sampleFarFutureTickets` view function (DSC-02 applies).
