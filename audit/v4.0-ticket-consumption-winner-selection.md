### 3.1 Core Winner Selection Helpers

All trait-based winner selection routes through two private helper functions in DegenerusGameJackpotModule.sol. Every caller documented in Section 3.2 delegates to one of these.

#### _randTraitTicket (JM:2237-2280)

```solidity
function _randTraitTicket(
    address[][256] storage traitBurnTicket_,
    uint256 randomWord,
    uint8 trait,
    uint8 numWinners,
    uint8 salt
) private view returns (address[] memory winners) {
    address[] storage holders = traitBurnTicket_[trait];  // JM:2244
    uint256 len = holders.length;

    // Deity virtual entries: floor(2% of bucket), min 2, if deity exists
    uint8 fullSymId = (trait >> 6) * 8 + (trait & 0x07);  // JM:2250
    address deity;
    uint256 virtualCount;
    if (fullSymId < 32) {
        deity = deityBySymbol[fullSymId];
        if (deity != address(0)) {
            virtualCount = len / 50;
            if (virtualCount < 2) virtualCount = 2;
        }
    }

    uint256 effectiveLen = len + virtualCount;  // JM:2261
    // ...
    uint256 slice = randomWord ^ (uint256(trait) << 128) ^ (uint256(salt) << 192);  // JM:2268-2270
    for (uint256 i; i < numWinners; ) {
        uint256 idx = slice % effectiveLen;  // JM:2272
        winners[i] = idx < len ? holders[idx] : deity;  // JM:2273
        slice = (slice >> 16) | (slice << 240);  // bit rotation for next winner
    }
}
```

- **Storage read:** `traitBurnTicket_[trait]` at JM:2244 -- reads the address array for a specific trait at the caller's level
- **Deity virtual entries:** `virtualCount = max(holders.length / 50, 2)` if a deity exists for the trait's symbol (fullSymId < 32). Indices >= `len` map to deity address. No deity entries are stored -- they are logical additions at selection time.
- **Winner index formula:** `idx = slice % effectiveLen` where `effectiveLen = holders.length + virtualCount`
- **Entropy derivation:** Initial `slice = randomWord XOR (trait << 128) XOR (salt << 192)`. Each subsequent winner uses `slice = (slice >> 16) | (slice << 240)` (16-bit rotation).
- **Duplicates:** Intentionally allowed -- more tickets = more chances to win multiple times
- **Used by:** `awardFinalDayDgnrsReward` (JM:785), `_runEarlyBirdLootboxJackpot` (JM:833), `_distributeTicketsToBucket` (JM:1191)

#### _randTraitTicketWithIndices (JM:2283-2337)

```solidity
function _randTraitTicketWithIndices(
    address[][256] storage traitBurnTicket_,
    uint256 randomWord,
    uint8 trait,
    uint8 numWinners,
    uint8 salt
) private view returns (address[] memory winners, uint256[] memory ticketIndexes) {
    address[] storage holders = traitBurnTicket_[trait];  // JM:2294
    uint256 len = holders.length;

    // Identical deity virtual entry logic to _randTraitTicket (JM:2297-2309)
    // ...
    uint256 effectiveLen = len + virtualCount;  // JM:2311
    // ...
    uint256 slice = randomWord ^ (uint256(trait) << 128) ^ (uint256(salt) << 192);  // JM:2319-2321
    for (uint256 i; i < numWinners; ) {
        uint256 idx = slice % effectiveLen;  // JM:2323
        if (idx < len) {
            winners[i] = holders[idx];  // JM:2325
            ticketIndexes[i] = idx;  // JM:2326
        } else {
            winners[i] = deity;  // JM:2328
            ticketIndexes[i] = type(uint256).max;  // JM:2329 -- sentinel for deity virtual entry
        }
        slice = (slice >> 16) | (slice << 240);
    }
}
```

- **Storage read:** `traitBurnTicket_[trait]` at JM:2294 -- identical to `_randTraitTicket`
- **Key difference:** Returns `ticketIndexes` array alongside `winners`. Real tickets get their array index; deity virtual entries get `type(uint256).max` as sentinel.
- **Winner index formula:** Identical to `_randTraitTicket`: `idx = slice % effectiveLen`
- **Used by:** `_processDailyEthChunk` (JM:1448), `_resolveTraitWinners` (JM:1641), `_awardDailyCoinToTraitWinners` (JM:2459)
